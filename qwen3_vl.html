Qwen3-VL Architecture

Dense Transformer with GQA, mRoPE 3-way positional encoding, Deepstack visual injection, dynamic-resolution ViT

1. Multimodal RoPE (mRoPE)

Qwen3-VL uses a 3-way split rotary positional embedding: [temporal=24, height=20, width=20] dimensions interleaved across the head_dim. For text tokens all 3 channels share the same position; for vision tokens each channel encodes spatial/temporal grid coordinates independently.

class Qwen3VLMultimodalRotaryEmbedding(nn.Module):
    # mrope_section = [24, 20, 20]  # T, H, W

    def apply_interleaved_mrope(self, freqs, section):
        """Interleave [T,H,W] freqs into head_dim."""
        freqs_t = freqs[0]  # start with temporal
        for dim, offset in enumerate((1,2), start=1):
            length = section[dim] * 3
            idx = slice(offset, length, 3)
            freqs_t[..., idx] = freqs[dim, ..., idx]
        return freqs_t

    def forward(self, position_ids, mrope_section):
        # position_ids: [3, batch, seq_len]
        seq = position_ids.float()
        inv_freq = self.inv_freq  # [dim//2]
        freqs = (inv_freq @ seq)  # [3, bs, seq, dim]
        freqs = self.apply_interleaved_mrope(freqs, section)
        emb = torch.cat((freqs, freqs), dim=-1)
        return emb  # [seq, bs, 1, 2*dim]

Why mRoPE?

Standard RoPE encodes 1D position. Vision tokens exist in a 3D grid (time, height, width). mRoPE assigns independent position IDs per axis.

$\text{head\_dim} = 128$ $\text{section} = [24, 20, 20] \; (\text{sum}=64)$ $\text{interleaved: } [T,H,W,T,H,W\ldots]$

Qwen3-VL uses timestamps for video frames (not absolute time positions), allowing variable frame rates.

The interleaved layout ensures each frequency band mixes all 3 spatial dimensions for richer representations.

2. Vision Encoder (ViT with 2D RoPE)

Conv3d patch embedding (kernel=[2,14,14]) handles both images and video. Learnable 2D positional embedding on a 48x48 grid with bilinear interpolation for dynamic resolution. 2D RoPE inside ViT attention blocks. Window attention via packed sequences (per-frame).

class Qwen3VLVisionModel(VisionModule):
    # patch_size=14, temporal_patch_size=2
    # spatial_merge_size=2, num_pos_embed=2304

    def __init__(self, config):
        self.patch_embed = Conv3d(
            3, hidden, kernel=[2,14,14])
        self.pos_embed = Embedding(2304, hidden)
        self.rotary_pos_emb = VisionRotaryEmb(
            head_dim // 2)  # 2D RoPE
        self.decoder = VisionTransformerBlock(...)
        self.merger = PatchMerger(...)

    def forward(self, pixels, grid_thw):
        x = self.patch_embed(pixels)
        x += self.fast_pos_embed_interpolate(grid_thw)
        rope = self.rot_pos_emb(grid_thw)  # 2D
        x, deepstack = self.decoder(x, rope)
        x = self.merger(x)  # spatial merge 2x2
        return x, deepstack

Dynamic Resolution

No fixed image size. The 48x48 learned position grid is bilinearly interpolated to any resolution at runtime.

$\text{patch\_size} = 14,\; \text{temporal\_patch\_size} = 2$ $\text{spatial\_merge} = 2 \times 2 \rightarrow 4 \times \text{ reduction}$ $\text{num\_pos\_embeddings} = 2304 \; (48 \times 48)$

2D RoPE in vision (not mRoPE): each attention head uses row/col coordinates for rotary embeddings within each frame.

Window attention: each video frame is treated as a separate sequence via packed_seq_params (cu_seqlens per frame).

3. Deepstack Visual Injection

Instead of injecting vision features only at the input, Qwen3-VL extracts intermediate ViT features at specific layers (e.g. [8, 16, 24]) and adds them into corresponding LLM decoder layers. Each injection point has its own PatchMerger projector.

class Qwen3VLVisionTransformerBlock:
    # deepstack_visual_indexes = [8, 16, 24]

    def __init__(self, config):
        self.deepstack_merger_list = nn.ModuleList([
            PatchMerger(config, use_postshuffle_norm=True)
            for _ in config.deepstack_visual_indexes
        ])

    def forward(self, hidden_states, ...):
        deepstack_features = []
        for l_no, layer in enumerate(self.layers):
            hidden_states = layer(hidden_states, ...)
            if l_no in self.deepstack_visual_indexes:
                idx = self.deepstack_visual_indexes.index(l_no)
                feat = self.deepstack_merger_list[idx](
                    hidden_states)
                deepstack_features.append(feat)
        return hidden_states, deepstack_features

# In LLM decoder:
def _deepstack_process(self, h, mask, emb):
    h[mask] = h[mask].clone() + emb
    return h

Multi-Scale Features

Deepstack extracts features at multiple ViT depths, providing the LLM with both low-level (edges, textures) and high-level (semantics) visual information.

$\text{deepstack\_visual\_indexes} = [8, 16, 24]$ $\text{Each has its own PatchMerger}$ $\text{Injection: } h[\text{mask}] \mathrel{+}= \text{projected\_feat}$

Each PatchMerger uses postshuffle normalization: first reshape to merged spatial dims, then LayerNorm, then 2-layer MLP projection.

This is a key architectural difference from Qwen2.5-VL which only injects at the input layer.

4. Patch Merger (MLP Projector)

The PatchMerger spatially merges 2x2 adjacent vision patches into one token, then projects to the LLM hidden dimension via a 2-layer MLP with GELU activation. This reduces vision token count by 4x while projecting to language space.

class Qwen3VLVisionPatchMerger(MegatronModule):
    # spatial_merge_size = 2 -> merge 2x2 patches

    def __init__(self, config):
        merge_dim = hidden * (spatial_merge**2)
        # merge_dim = hidden * 4

        self.patch_norm = LayerNorm(merge_dim)
        self.linear_fc1 = ColumnParallelLinear(
            merge_dim, merge_dim)  # TP split
        self.activation = gelu_tanh
        self.linear_fc2 = RowParallelLinear(
            merge_dim, out_hidden_size)  # -> D_llm

    def forward(self, hidden_states):
        # Reshape: [N, H] -> [N/4, H*4]
        x = hidden_states.view(-1, self.merge_dim)
        x = self.patch_norm(x)
        x = self.linear_fc1(x)
        x = self.activation(x)
        x = self.linear_fc2(x)  # [N/4, D_llm]
        return x

Token Compression

A 448x448 image produces 32x32=1024 patches at patch_size=14. After 2x2 merge: 16x16=256 tokens fed to the LLM.

$N_{\text{patches}} = \frac{H}{14} \times \frac{W}{14}$ $N_{\text{tokens}} = \frac{N_{\text{patches}}}{4}$ $\text{merge\_dim} = \text{hidden} \times 4$ $\text{output\_dim} = D_{\text{llm}}$

The merger uses Tensor Parallelism (ColumnParallel + RowParallel) for efficient multi-GPU training.

GELU with tanh approximation matches the vision encoder's activation function (not SwiGLU used in the LLM).

5. Full Model Pipeline (Qwen3VLModel)

The complete forward pass: vision encoder produces embeddings + deepstack features, which are scattered into the LLM token sequence at vision token positions. The LLM uses GQA with mRoPE and receives deepstack injections at early layers. Supports context parallelism, sequence packing (THD), and distributed training.

class Qwen3VLModel(MegatronModule):
    def forward(self, input_ids, pixel_values,
              image_grid_thw, ...):
        # 1. Vision encoding
        vis_data, grid, mask = reorganize_inputs(...)
        vis_emb, deepstack = self.vision_model(
            vis_data, grid)

        # 2. Text embedding + vision scatter
        combined = self.language_model.embedding(
            input_ids)
        combined[mask] = vis_emb  # scatter

        # 3. Compute mRoPE position_ids
        pos_ids, _ = get_rope_index(
            spatial_merge_size,
            image_token_id, video_token_id,
            input_ids, image_grid_thw, ...)

        # 4. LLM forward with deepstack
        output = self.language_model(
            decoder_input=combined,
            position_ids=pos_ids,
            deepstack_visual_embeds=deepstack,
            visual_pos_masks=mask)
        return output

Key Design Choices

GQA in the LLM (fewer KV heads) reduces memory for long multimodal sequences.

$\text{position\_embedding\_type} = \text{"mrope"}$ $\text{CP: zigzag split for load balance}$ $\text{vision\_dp\_when\_cp} = \text{True}$

Vision tokens are processed with data parallelism across CP ranks (each rank processes a subset of images), then all-gathered before scattering into the LLM sequence.

Qwen3-VL uses timestamps (not absolute frame indices) for video temporal encoding, enabling variable frame-rate support.
