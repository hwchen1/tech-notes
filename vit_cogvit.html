# 1. 动态切片策略
# 类似 LLaVA-UHD 的方法
def split_image(img, max_tiles=6):
    # 选择最优切片方案
    # 约束: 每个 tile 为 448×448
    # 总 tile 数 ≤ max_tiles
    best = find_best_grid(img.size)
    tiles = crop_to_grid(img, best)
    return tiles  # list of 448² tiles

# 2. 每个 tile 独立编码
for tile in tiles:
    tokens = cogvit(tile)   # 32×32=1024

# 3. 拼接所有 tile tokens
visual_tokens = concat(all_tokens)
# + 全局缩略图 tokens

动态切片

将高分辨率图像切成多个 448×448 tiles，每个 tile 独立编码。保留一个全局缩略图提供整体语义。

与 Qwen-VL 的 dynamic resolution 类似，但切片粒度更大（448 vs 28×n）。

$\text{tokens}_{\text{tile}} = (448/14)^2 = 1024$ $\text{total} = N_{\text{tiles}} \times 1024 + 1024_{\text{global}}$

优势

高分辨率输入不会丢失细节（OCR、文档理解），同时通过限制 tile 数控制计算量。

极深架构

63 层是目前 VLM 中最深的视觉编码器之一。深层网络能学习更抽象的视觉表征。

EVA2 继承

基于 EVA2-CLIP-E 初始化，继承了 MIM+CLIP 联合预训练的优势，在此基础上针对 VLM 场景微调。

$\text{Params} \approx 4.4\text{B}$ $\text{Depth: 63 layers (vs ViT-L: 24)}$ $\text{FLOPs/tile} \approx 6.5 \text{T}$
