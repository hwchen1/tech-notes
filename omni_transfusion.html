核心创新

关键洞察: Transfusion 不将图像离散化为 token (避免 VQ 量化损失)，而是直接保留连续向量表示，用 diffusion loss 训练图像部分。

$\mathcal{L} = \lambda_t \sum_{i \in \text{text}} -\log P(x_i \mid x_{\lt i}) + \lambda_I \sum_{j \in \text{img}} \|\epsilon - \epsilon_\theta(z_j^t, t)\|^2$

vs Chameleon / Emu3

Chameleon/Emu3: 图像 → 离散 token (VQ) → NTP loss。量化是有损的，图像质量有上限。

Transfusion: 图像保持连续 latent → diffusion loss。保留完整信息，图像质量无上限。

Best of Both Worlds

1. 文本 — LLM 的 AR 生成能力完整保留

2. 图像 — Diffusion 的高质量生成完整保留

3. 统一 — 共享一个 Transformer 的表示空间

2. 混合序列 — 离散 + 连续

# Sequence construction: discrete + continuous

# Text: standard BPE tokenization
# x ∈ {0, 1, ..., V-1}, discrete integer IDs
text_tokens = bpe_tokenize(text)
# → [234, 1089, 42, 7731, ...]
# Embedded via nn.Embedding lookup

# Image: VAE encoder → continuous latents
# z ∈ R^{h x w x c}, continuous float vectors
image_latent = vae_encoder(image)
# → shape: (32, 32, 4) for 256x256 img
# Patchified into sequence of vectors
image_patches = patchify(image_latent)
# → (N_patches, patch_dim)
# Each patch is a continuous vector, NOT a token

# Sequence assembly
sequence = concat([
    embed(text_tokens),  # (T, d_model)
    embed(BOI),          # (1, d_model)
    linear(image_patches), # (N, d_model)
    embed(EOI),          # (1, d_model)
    embed(more_text),    # (T2, d_model)
])

# Attention pattern (CRITICAL):
# - Text: CAUSAL (standard AR)
# - Image patches: BIDIRECTIONAL within image
# - Cross: causal (text after img sees img)

def attention_mask(seq):
    mask = causal_mask(len(seq))
    # Override: image patches see each other
    img_start = find(BOI) + 1
    img_end = find(EOI)
    mask[img_start:img_end, img_start:img_end] = 1
    return mask

离散 vs 连续

Text token: 离散整数 ID，通过 Embedding 查找表变为向量。

$x \in \{0, 1, \ldots, V-1\}$ (BPE discrete)

Image latent: 连续浮点向量，通过线性投影映射到模型维度。

$z \in \mathbb{R}^{h \times w \times c}$ (VAE continuous)

混合注意力

关键设计: 整体序列是 causal (AR)，但图像 patch 之间是 bidirectional。

这模拟了 diffusion model 中 U-Net 的全局空间注意力 — 图像的每个 patch 需要看到其他所有 patch 才能协调生成。

为什么不全用 causal?

如果图像 patch 也用 causal attention，则等于强制光栅扫描顺序生成图像 — 这远不如 diffusion 的全局去噪高效。Bidirectional 让图像部分获得 diffusion 的全部表达力。

# Training: joint optimization
# Data: mixed text corpora + image-text pairs

# Loss function
def transfusion_loss(model, batch):
    hidden = model(batch.sequence)

    # Text: cross-entropy NTP
    text_h = hidden[batch.text_mask]
    logits = model.lm_head(text_h)
    ce_loss = F.cross_entropy(
        logits, batch.text_targets
    )

    # Image: diffusion (epsilon prediction)
    img_h = hidden[batch.img_mask]
    # Noise schedule: t ~ U(0, T)
    t = torch.randint(0, T, (n_img,))
    eps = torch.randn_like(batch.img_latents)
    z_t = sqrt_alpha[t] * batch.img_latents \
        + sqrt_one_minus_alpha[t] * eps
    eps_pred = model.noise_head(img_h, t)
    diff_loss = F.mse_loss(eps_pred, eps)

    # Combined
    loss = 1.0 * ce_loss + 1.0 * diff_loss
    return loss

# Inference: dual mode
def generate(model, prompt):
    # Text: standard autoregressive
    tokens = ar_sample(model, prompt)

    # When BOI is generated, switch to diffusion
    if tokens[-1] == BOI:
        # Image: iterative denoising
        z = torch.randn(n_patches, d)  # noise
        for t in reversed(range(T)):
            # Denoise step (DDPM)
            eps = model.predict_noise(
                context + z, t
            )
            z = denoise_step(z, eps, t)
        image = vae_decode(z)
        # Continue AR after EOI
    return tokens, image

训练策略

数据混合: 纯文本语料 + 图文对 + 交错多模态文档。Joint training，两个 loss 同时优化。

$\mathcal{L} = \lambda_t \cdot \mathcal{L}_{\text{CE}} + \lambda_I \cdot \mathcal{L}_{\text{diff}}$

实验中 lambda_t = lambda_I = 1.0，两个 loss 量级自然平衡。

推理: 双模式

文本: 标准 AR 采样 (token-by-token)。

图像: 当模型生成 BOI token 时，切换到 diffusion 模式 — 从纯噪声 z_T 开始，迭代去噪 250 步得到 z_0，再用 VAE decoder 解码为像素。
