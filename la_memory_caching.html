Memory Caching — 弹性记忆

隐状态 Checkpoint 缓存 — 在 O(1) RNN 和 O(L²) Transformer 之间灵活插值 | Behrouz, Li et al. | 2026.02 (arxiv 2602.24281)

1. 核心理念 — 固定 vs 增长记忆的折中

# RNN/Linear Attention 的根本限制:
# 固定大小状态 → 信息必须被压缩/遗忘
# 序列越长, 信息损失越大
#
# Transformer 的优势和代价:
# KV Cache 随序列增长 → 无信息丢失
# 但 O(L²) 计算, O(L) 内存
#
# Memory Caching (MC) 的方案:
# 在 RNN 运行过程中, 周期性缓存 hidden state
# → 记忆容量随序列增长 (像 Transformer)
# → 但增长速度可控 (不是 O(L))
#
# 类比: 阅读一本书
# - RNN: 只记住"当前理解" (1个状态)
# - Transformer: 记住"每一页" (L个KV)
# - MC: 记住"每章总结" (L/C个checkpoint)
#
# GRM (Gated Recurrent Memory) 变体:
# - 缓存 hidden states 作为 memory bank
# - 用 gated aggregation 读取:
#   o_t = Gate(h_t, attend(q_t, cached_states))
# - 类似 "Memory Soup": 混合多个历史状态
#
# 复杂度: O(L/C × d) — 介于 O(d) 和 O(L×d)

Memory Caching 在 O(1) 固定记忆和 O(L) 全量缓存之间提供连续可调的中间地带

可调的记忆-效率 Trade-off

MC 的核心洞察：不必在"完全固定"和"完全增长"之间二选一。通过调节缓存间隔 C，可以在 O(1) 到 O(L) 之间的任何点运行。大 C = 接近 RNN（快但有损），小 C = 接近 Transformer（慢但精确）。

$\text{Memory Size} = \frac{L}{C} \times d$ $C = 1 \Rightarrow O(L \cdot d) \text{ (Transformer-like)}$ $C = L \Rightarrow O(d) \text{ (RNN-like)}$
模型无关的增强技术：Memory Caching 可以应用到任何 RNN/Linear Attention 模型上。论文将 MC 加在 Titans 和 GDN ('DLA') 上，两者都获得提升。

# Titans (LMM)        : 15.60
# Titans + GRM        : 15.37  ← 最优
#

# Transformer++       : 53.19
# Gated DeltaNet      : 53.72
# DLA + GRM           : 55.96  ← +2.24!
# Titans (LMM)        : 56.82

#
# NIAH S-NIAH-2 (16K)↑:
# Titans base         : 75.4%

#
# 关键发现:
# 1. GRM 对 GDN (DLA): Acc +2.24, PPL -0.23
# 2. GRM 对 Titans: NIAH +12.8%
# 3. 缓存使得短距离精确 recall 大幅提升
#    (cached state 可以 "回忆" 之前的信息)

GRM 对 GDN 提升更大 (+2.24 Acc)，因为 GDN 固定状态容量有限；对 Titans 的 NIAH recall 提升最显著

即插即用的提升

Base Model+GRM PPL改善+GRM Acc改善
GDN (DLA)-0.23+2.24

Memory Caching 对 GDN 的提升更显著（+2.24 Acc），因为 GDN 的固定状态 \(d \times d\) 容量有限，缓存能直接补上这个短板。Titans 已经有更好的记忆管理机制（Long-term Memory Module），所以边际收益较小。

NIAH 关键结论：Titans + GRM 在 S-NIAH-2 (16K) 上从 75.4% 跃升至 88.2%，提升 12.8 个百分点。缓存的 hidden states 充当了精确检索锚点，弥补了纯 RNN 在长程 recall 上的固有弱点。

为什么 GDN 受益更大？

GDN 的状态矩阵 \(S \in \mathbb{R}^{d \times d}\) 是固定大小的，每个新 token 都必须压缩写入同一空间。当序列很长时，早期信息被不断覆盖。MC 通过缓存历史 checkpoint，让模型可以直接"翻阅"历史状态，绕过压缩瓶颈。

3. 实践意义

# MC 的四种变体:
#
# 1. Log-Linear++ (LL++):
#    每 2^k tokens 缓存一次
#    近期密, 远期稀 (指数衰减)
#    内存: O(log L × d)
#
# 2. GRM (Gated Recurrent Memory):
#    固定间隔 C 缓存
#    gated attention 读取
#    内存: O(L/C × d)
#
# 3. SSC (Sparse Selective Caching):
#    根据 surprise 选择性缓存
#    只缓存"重要"的状态
#    内存: 自适应
#
# 4. Memory Soup:
#    加权混合多个 cached states
#    简单平均或 learned mixing
#
# 推理时配置:
# 短序列 (<4K): 不需要 MC, 纯 RNN
# 中序列 (4-32K): LL++ (log 缓存)
# 长序列 (32K+): GRM (固定间隔)
# 超长 (128K+): SSC (选择性)

四种变体适用于不同场景：LL++ 对话、GRM 检索、SSC 流式、Memory Soup 简单聚合

部署场景选择

场景推荐变体内存开销
RAG/检索GRMO(L/C)
对话LL++O(log L)
流式处理SSC自适应
端侧无 MCO(1)
最大实践价值：你可以先训练一个纯 GDN/Titans 模型，然后在推理时根据序列长度动态决定是否启用 MC 以及缓存密度。训练时无需改变，只影响推理策略。

推理时动态配置

MC 的一大优势是 推理时可配置：

- 短序列 (\(\lt\)4K): 纯 RNN，不启用 MC

- 中序列 (4-32K): LL++ log 缓存

- 长序列 (32K+): GRM 固定间隔

- 超长 (128K+): SSC 选择性缓存

与 KV Cache 的本质区别：Transformer 的 KV Cache 缓存的是原始 token 表示，MC 缓存的是 RNN 的 compressed hidden state。同样的内存预算下，MC 的每个 checkpoint 包含了对应区间的全部压缩信息，信息密度更高。
