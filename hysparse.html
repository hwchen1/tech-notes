设计思想

HySparse 将 Transformer 层分为两类交替排列：

Full 层 (25%): 做完整 attention，额外计算每个 Q 对所有 KV block 的最大 attention score，选出 TopK 个最相关 block。结果缓存到 KVBlockManager。
Sparse 层 (75%): 双分支并行 — ①SWA 用独立 K/V 做局部窗口 attention（精确近距离），经 linear_proj 走主残差；②Block Sparse 复用 Full 层缓存的 K/V，只对选中的 block 做 sparse FA（高效远距离），经 sparse_linear_proj 作为 bypass 输出走独立 bypass_mlp 旁路再求和。

核心优势: Full 层只占 1/4，但为后续 3 个 Sparse 层提供全局 "哪些 KV block 重要" 的信息。Sparse 层复用这些信息，避免重复计算全局 attention。

↓ 它与 DSA (DeepSeek Sparse Attention) 在选择机制上的根本差异，以及完整对比，见 Section 2「机制图解」。

2. 机制图解 — DSA Indexer vs HySparse Block TopK

两种稀疏注意力都解决同一个问题：长序列下，让每个 Query 只关注最相关的一部分 KV，而不是全部 N 个 token。核心差异在于 "如何决定哪些 KV 相关"（选择机制）以及 "这个决策的成本与精度"。下面分别图解。

2.1 DSA — 轻量 Indexer 网络预测相关位置

# DeepseekV32Indexer.forward (简化)
# 一个独立的"打分小网络", 与主
# attention 并行, 维度极小 (head_dim≈128)

# ① Query: 复用主 MLA 的压缩表示
q = self.wq_b(q_compressed)   # q_lora_rank→n_heads·d
q = q.view(b, s, n_heads, head_dim)
q_pe, q_nope = split(q, [rope_d, nope_d])
q_pe = apply_rope(q_pe, cos, sin)
q = rotate_activation(cat(q_pe, q_nope))  # Hadamard

# ② Key: 从 hidden 单独投影 (单 head!)
k = self.wk(hidden_states)    # hidden→head_dim
k = self.k_norm(k)            # fp32 norm
k_pe, k_nope = split(k, [rope_d, nope_d])
k_pe = apply_rope(k_pe, cos, sin)
k = rotate_activation(cat(k_pe, k_nope))

# ③ 每 head 权重 (fp32 投影)
weights = self.weights_proj(hidden.float())
weights *= n_heads**-0.5 * softmax_scale

# ④ 打分 → ReLU → 加权聚合 head
scores = einsum('bshd,btd->bsht', q, k)
scores = relu(scores)
index_score = (scores * weights).sum(2)
# index_score: (b, s, kv_len)

# ⑤ 选 top-k 位置 (index_topk≈2048)
topk_indices = index_score.topk(
    index_topk, dim=-1)[1]
# 主 attention 只对这些位置算

DSA 的核心:学习式索引

DSA 不直接用主 attention 的分数选 KV，而是额外训练一个超轻量的 Indexer 网络专门负责"打分排序"。

为什么轻: Indexer 的 Key 是单 head 投影 (\(d_{index}\approx128\))，Query 复用主 MLA 已压缩的 q_compressed。打分是 \(O(N^2 \cdot d_{index})\)，但 \(d_{index}\) 远小于 \(d_{model}\)，且无需 V、无需输出投影。
为什么需要 KL loss: Indexer 是代理模型，它的 \(\text{softmax}(\text{index\_score})\) 必须逼近真实主 attention 的概率分布。训练时用 \(\text{KL}(\text{真分布}\,\|\,\text{indexer分布})\) 蒸馏。若学不好 → 漏选高分位置 → 主 attention 看错地方。
数值细节 (AMFP patch): RoPE 顺序 PE-first/NOPE-last；k_norm 用 fp32；rotate_activation 前 cast bf16；SFT 时 KL loss 只统计 response token (DSALossMaskHolder)。

选择粒度: token 级 (每个 KV token 单独打分)，topk_indices 直接是 token 下标，不分块。

2.2 HySparse — 用真实 Attention Score 做 Block-Max TopK

# Full 层: 真 attention + 额外算 block 分
# (无代理网络, 直接复用真实 QKᵀ)

# ① 正常 full attention 输出
out = flash_attn(Q, K, V)        # O(N²)

# ② block_attn_scores_group (Triton)
#    把 K 按 block_size=64 分块
#    每个 Q 对每个 block 取 max 分
block_scores = block_attn_scores_group(
    Q, K, block_size=64)
# block_scores: (b, Sq, Hkv, n_blocks)
# GQA group 内所有 Q head 取 max

# ③ 选 TopK 个 block
#    topk_tokens=1024 / block_size=64 = 16
topk_blk = block_scores.topk(16, -1)[1]

# ④ 缓存给后续 3 个 Sparse 层
kv_manager.store(layer, K, V, topk_blk)

# ───────── Sparse 层 (复用) ─────────
K, V, topk_blk = kv_manager.get(layer)
# 双分支并行:
swa  = swa_attn(Q, K_swa, V_swa)   # 局部窗口(自有K/V)
spar = sparse_flash_attention(      # 全局稀疏
    Q, K, V, topk_blk)              # 只算选中 block
# 各自独立输出投影 (非门控!)
swa_out    = linear_proj(swa)        # → 主残差路径
sparse_out = sparse_linear_proj(spar)# → bypass 旁路
# 返回 3 元组 → BypassMLPLayer 接管:
#   swa_out 走标准 attn_bda 残差
#   sparse_out 走独立 bypass_mlp
#   out = mlp(x) + scale·bypass_mlp(sparse_out)
#   scale = bypass_mlp_scale = 0.0 (当前关闭)

HySparse 的核心:真分数选块

HySparse 不训练任何代理网络。它让 Full 层在做正常 attention 的同时，"顺手" 用真实的 \(\text{softmax}(QK^T)\) 分数对 KV block 排序。

为什么精确: 选择依据是真实 attention 分数本身，不是学出来的近似。高分 block 一定被选中，不存在"代理网络学不好导致漏选"的问题，也无需 KL loss。
为什么用 block: 以 64 token 为一块，对 block 取 max 分再 TopK (选 16 块 = 1024 token)。块粒度让 Triton kernel 能连续读 K/V，对显存带宽友好；token 级反而难高效实现。
双分支互补: Block Sparse 覆盖全局远距离的重要块；但块粒度可能漏掉窗口内的精确近邻，于是再并联一条 SWA 滑动窗口分支专管局部。这是 DSA(单分支) 没有的设计。注意两路不是简单加权门控：SWA 经 linear_proj 走主残差路径，Sparse 经 sparse_linear_proj 作为第 3 个返回值（bypass）走独立的 bypass_mlp 旁路，最后才求和（详见 Section 4 BypassMLPLayer）。
代价: Full 层必须真的算完 \(O(N^2)\) attention 才能拿到分数（虽然只占 25% 层）；且 Sparse 层有数据依赖，必须等同组 Full 层完成 → PP 流水并行受限。

选择粒度: block 级 (64 token/块)，缓存的是 block_indices。

2.3 完整对比 — DSA vs HySparse

两者都属于 "1 个决策层 : 3 个复用层" 的共享稀疏范式，都把 "全局 attention 该看哪里" 的信息传给后续层复用。根本差异在于如何产生这个决策，由此分化出训练方式、精度、并行性等一系列不同。

维度DSA (DeepSeek Sparse Attention)HySparse (Block TopK)
选择机制独立轻量 Indexer 网络，用压缩特征打分预测哪些 KV 相关（学习式代理）直接用 Full 层真实 attention score 做 block-max → TopK（无代理网络）

选择精度近似 — 代理网络可能漏选高分位置，依赖 KL 蒸馏学得好坏精确 — 真实 \(QK^T\) 值排序，高分 block 必被选中，不漏选
决策层复杂度仍是 \(O(N^2)\)，但常数极小: 单 head + FP8 + ReLU(非 softmax) + 无 V/无输出投影 → 实测远低于主 attention\(O(N^2)\) full attention（必须真算完才有分数）+ 额外 block-max。只占 1/4 层

DSA 优势: ① indexer 每层独立决策 → PP 友好；② 决策网络极轻量（单 head + FP8）；③ 大规模工程验证成熟。代价: indexer 是代理模型，需两阶段训练 + KL 蒸馏对齐，学不好会漏选。
HySparse 优势: ① TopK 零训练成本、不漏选（真分数）；② 双分支 SWA+Sparse 互补覆盖局部+全局；③ Sparse 层省参数 + 推理 KV cache 降至 ~1/4。代价: Full 层是 \(O(N^2)\) 瓶颈（虽只占 25%）；层间数据依赖 → PP 受限；未大规模验证。
一句话: DSA 用 "便宜的学习式近似"（小网络打分，PP 友好但需训练）；HySparse 用 "昂贵的精确真分数"（真 attention 排序，不漏选但 PP 受限）。本质是 "索引开销 vs 索引精度" 和 "并行灵活性 vs 训练简单性" 两组权衡的不同取舍。
⚠ 常见误解: DSA 的 Indexer 并非线性复杂度 —— 它仍要对所有 KV 位置打分，是 \(O(N^2)\)。它 "轻" 是因为常数小（单 head、低维 \(d_{index}\)、FP8、ReLU 代替 softmax、无 value 计算），而非阶数低。真正决定长序列上限的仍是这个 \(O(N^2)\) 打分（DeepSeek 用 FP8 kernel 压成本）。

3. 稀疏 Kernel — Block Score 计算 + Sparse Flash Attention (Triton / TileLang)

// ① block_attn_scores_group (Triton)
// 输入:
//   Q: [B, Sq, Hq, D]
//   K: [B, Sk, Hkv, D]
// 输出:
//   scores: [B, Sq, Hkv, num_blocks]
// 逻辑:
//   对每个 Q token, 将 K 按 block_size
//   分块, 计算 Q×K^T 的 block-max score
//   GQA group 内所有 Q head 取 max

// ② sparse_flash_attention (Triton)
// Forward kernel: _sparse_flash_fwd_kernel
// Grid: (batch, num_heads_kv, seq_len_q)
// 每个 thread block: 1 token × GROUP_SIZE heads
//
// 关键设计:
//   - K/V/topk 只加载一次
//   - GROUP_SIZE 个 head 共享 K/V load
//   - 遍历 NUM_SELECTED 个 topk block
//   - online softmax (FlashAttention style)
//     m_i, l_i, acc 累加

for sel_idx in range(NUM_SELECTED):
    kv_block_idx = tl.load(topk + sel_idx)
    start_n = kv_block_idx * BLOCK_N
    // load K[start_n:start_n+BLOCK_N]
    // QK = Q @ K^T  [GROUP_SIZE, BLOCK_N]
    // causal mask
    // online softmax update
    // acc += P @ V[start_n:start_n+BLOCK_N]

// ③ build_reverse_mask (Triton)
// 构建 KV block → Q tokens 的反向索引
// 用于 backward dK/dV 计算
// reverse_mask[b,h,kv_blk,q] = True
//   表示 Q token q 选择了 kv_blk

三个 Triton Kernel

block_attn_scores_group: Full 层调用。计算每个 Q token 对所有 KV block 的 max attention score。GQA group 内取 max，输出 [B, Sq, Hkv, num_blocks]。用于后续 TopK 选择。(另有 block_attn_scores_group_pytorch 纯 PyTorch 参考实现，由 enable_triton 切换。)
sparse_flash_attention (forward): Sparse 层调用。只对 TopK 选中的 block 做 FlashAttention。Group-fused: 同一 GQA group 的所有 Q head 共享 K/V load，减少显存带宽。Online softmax 保证数值稳定。
build_reverse_mask: Backward 用。构建反向索引: 哪些 Q token 选了某个 KV block → 用于 dK/dV 计算时只遍历相关 Q。

TileLang 后端 (新增)

block_sparse_attn_tilelang: Triton kernel 的 drop-in 替代，接口与 sparse_flash_attention 完全一致 (BSHD layout)。封装为 torch.autograd.Function，fwd/bwd 分别由 tilelang_block_sparse_fwd/bwd 实现。优势: 单次 kernel launch 内原生 GQA、block 级连续 K/V load，显存访问模式更优。

Autotune 配置

Forward kernel 用 triton.autotune 搜索最优 (num_warps, num_stages) 组合，key 包含 HEAD_DIM, V_HEAD_DIM, GROUP_SIZE, NUM_SELECTED, BLOCK_N。

复杂度

Full Attention: \(O(N^2)\)。Sparse Attention: \(O(N \times topk\_tokens)\) = \(O(N \times 1024)\)。当 seq_len=32K 时稀疏比例仅 3.1%。

4. 工程实现 — KVBlockManager + 双路 BypassMLP + Embedding Gate

// KVBlockManager — Full→Sparse KV 传递
class KVBlockManager:
    // max_layer_diff = num_sparse_per_full
    // _kv_cache: dict {layer_no: (K,V,idx)}
    // Full 层 store(layer_no, K, V, indices,
    //                need_clear=True)
    // Sparse 层 get(layer_no): 从 layer_no
    //   往前 i∈[1, max_layer_diff] 回溯,
    //   命中最近 Full 层的 (K,V,idx)

// BypassMLPLayer — 双路 MLP (非 warmup!)
class BypassMLPLayer(TransformerLayer):
  // self_attention 返回 3 值时,
  // 第 3 值 = bypass_attn (=sparse_output)
  def _forward_mlp(h, bypass_attn):
    // ① 标准路: pre_mlp_ln→mlp→mlp_bda
    mlp_h = mlp_bda(mlp(ln(h)), residual)
    // ② 旁路: 独立 norm + bypass_mlp
    //    bypass_mlp 用 moe_ffn_hidden_size
    byp = bypass_mlp(pre_byp_ln(bypass_attn))
    // ③ 求和 (scale 全局常量)
    return mlp_h + bypass_mlp_scale * byp
  // bypass_mlp_scale = 0.0  ← 当前禁用旁路

// EmbeddingGatedMLPLayer (Full 层可选)
// input_ids→Embedding→RMSNorm→emb_gate
// mlp_out *= (1 + emb_gate) 后再残差
// 由 enable_embedding_gated_mlp 开关

// 训练配置
// model: MiMo-7B (36 layers)
// MoE: DeepEP, flex dispatcher
// MTP: 1 layer, loss_scale=0.1
// recompute_num_layers: 1 (FULL)
// kernel: Triton / TileLang 可切

KVBlockManager

核心数据结构，解决 Full→Sparse 的 KV 传递问题：

store(): Full 层计算完 Q/K/V 后，将 K, V, block_indices 存入 _kv_cache 字典（key=layer_number）；need_clear=True 时先清空旧缓存。
get(): Sparse 层 forward 时，从自身 layer_number 向前回溯 \(i\in[1, \text{max\_layer\_diff}]\)，命中最近的 Full 层缓存。max_layer_diff = num_sparse_per_full，保证只看到同组 Full 层。

这允许 gradient checkpointing 时避免重算 Full 层的 K/V（最重的部分）——K/V/block_indices 作为 checkpoint 的显式输入张量传入 _checkpointed_forward，只需重算 Sparse 层自己的 SWA K/V。

BypassMLPLayer (双路 MLP)

Sparse 层的输出处理。当 self_attention 返回 3 个值时，第 3 值（Block Sparse 分支经 sparse_linear_proj 的输出）被识别为 bypass attention，走一条独立的 norm + bypass_mlp 旁路（bypass_mlp 用 moe_ffn_hidden_size 维度），再与标准 MLP 输出求和：out = mlp_h + bypass_mlp_scale × bypass_out。

⚠ 修正旧版描述: 当前实现中 bypass_mlp_scale = 0.0（全局常量，非渐进式 warmup scaler）。即旁路 MLP 当前被完全关闭，Sparse 层实际只走 SWA 主路 + 标准 MLP；bypass 路径是为后续实验预留的结构。

EmbeddingGatedMLPLayer (Full 层可选)

由 enable_embedding_gated_mlp 开关。Full 层把 input_ids 过 LanguageModelEmbedding + RMSNorm 得到 emb_gate，在残差前对 MLP 输出做逐元素门控：mlp_out *= (1 + emb_gate)。
