设计思想

HySparse 将 Transformer 层分为两类交替排列：

Full 层 (25%): 做完整 attention，额外计算每个 Q 对所有 KV block 的最大 attention score，选出 TopK 个最相关 block。结果缓存到 KVBlockManager。
Sparse 层 (75%): 双分支并行 — ①SWA 用独立 K/V 做局部窗口 attention（精确近距离）；②Block Sparse 复用 Full 层缓存的 K/V，只对选中的 block 做 sparse FA（高效远距离）。两分支通过独立 linear_proj 融合。

核心优势: Full 层只占 1/4，但为后续 3 个 Sparse 层提供全局 "哪些 KV block 重要" 的信息。Sparse 层复用这些信息，避免重复计算全局 attention。

HySparse 优势: ① TopK 选择零训练成本（无需学 indexer）；② 选择精度更高（真实 attention score vs 代理网络）；③ 双分支 SWA+Sparse 互补覆盖局部+全局；④ Sparse 层参数更少（复用 Full 层 KV）。
HySparse 劣势: ① Full 层是 O(N²) 瓶颈（虽然只占 25%）；② 层间有数据依赖（Sparse 层必须等 Full 层完成）→ PP 并行受限；③ 尚未在大规模模型验证。

三个 Triton Kernel

block_attn_scores_group: Full 层调用。计算每个 Q token 对所有 KV block 的 max attention score。GQA group 内取 max，输出 [B, Sq, Hkv, num_blocks]。用于后续 TopK 选择。
sparse_flash_attention (forward): Sparse 层调用。只对 TopK 选中的 block 做 FlashAttention。Group-fused: 同一 GQA group 的所有 Q head 共享 K/V load，减少显存带宽。Online softmax 保证数值稳定。
build_reverse_mask: Backward 用。构建反向索引: 哪些 Q token 选了某个 KV block → 用于 dK/dV 计算时只遍历相关 Q。

Autotune 配置

Forward kernel 用 triton.autotune 搜索最优 (num_warps, num_stages) 组合，key 包含 HEAD_DIM, V_HEAD_DIM, GROUP_SIZE, NUM_SELECTED, BLOCK_N。

复杂度

Full Attention: \(O(N^2)\)。Sparse Attention: \(O(N \times topk\_tokens)\) = \(O(N \times 1024)\)。当 seq_len=32K 时稀疏比例仅 3.1%。

KVBlockManager

核心数据结构，解决 Full→Sparse 的 KV 传递问题：

store(): Full 层计算完 attention 后，将 K, V, block_indices 存入 manager。key 是 layer_number。
retrieve(): Sparse 层 forward 时，根据 layer_number 找到最近的 Full 层缓存。max_layer_diff 保证 Sparse 层只看到同组的 Full 层。

这允许 gradient checkpointing 时避免重算 Full 层的 K/V（最重的部分），只需重算 Sparse 层自己的 SWA K/V。

BypassMLPLayer

Sparse 层的 MLP 用渐进式 warmup：前 2500 步 scaler 从 0 线性增到 1。这是因为 HySparse 可能在已有 pretrained 模型上做 continued training，新增的 Sparse 层需要平滑引入。
