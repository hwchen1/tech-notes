# 架构: T5-style Encoder-Decoder
class UnifiedIO2(nn.Module):
    def __init__(self, d_model, n_layers):
        # 模态特定 tokenizer (frozen)
        self.tokenizers = {
            "image": VQGANTokenizer(codebook=16384),
            "audio": AudioVQGAN(codebook=16384),
            "video": PerFrameVQGAN(codebook=16384),
            "3d":    PointCloudQuantizer(codebook=16384),
        }
        # 单一 Encoder-Decoder Transformer
        self.encoder = TransformerEncoder(
            layers=n_layers, d_model=d_model
        )
        self.decoder = TransformerDecoder(
            layers=n_layers, d_model=d_model
        )

    def forward(self, input_tokens, target_tokens):
        # input: 任意模态组合的 tokens
        enc_out = self.encoder(input_tokens)
        # decoder: 生成任意模态的 tokens
        logits = self.decoder(target_tokens, enc_out)
        return logits
        # seq2seq loss 统一所有任务!

Any-to-Any 统一架构

核心洞察: 大多数"全能"模型实际只处理 2-3 种模态 (通常是 text + image)。Unified-IO 2 处理 7+ 种模态，包括非常规的 3D 网格、机器人动作、深度图、表面法线等。

设计原则

1. 单一权重 — 所有任务共享同一个 Encoder-Decoder，无模态特定 head

2. 离散化统一 — 所有非文本模态通过 VQGAN 类 tokenizer 转为离散 token

3. Seq2Seq 范式 — 输入 = prompted multimodal tokens，输出 = target multimodal tokens

$\mathcal{L} = -\sum_{t=1}^{T} \log P(y_t \mid y_{\lt t}, \text{enc}(x))$

标准 encoder-decoder seq2seq loss 统一覆盖所有模态的所有任务。

2. 模态 Tokenizer 设计

# Unified-IO 2: Modality Tokenizers
# 关键: 所有模态共享同一个 codebook (16384 entries)

# === Image Tokenizer ===
class ImageTokenizer:
    # VQGAN with 16384 codebook entries
    # Input: [B, 3, 256, 256]
    # Output: [B, 1024] discrete tokens (32x32 grid)
    codebook_size = 16384
    spatial_grid = 32 * 32  # = 1024 tokens

# === Audio Tokenizer ===
class AudioTokenizer:
    # Step 1: Audio → Mel Spectrogram
    # Step 2: Spectrogram → VQGAN → tokens
    # Treats spectrogram as a 2D "image"
    codebook_size = 16384  # shared!
    # Output: variable-length tokens

# === Video Tokenizer ===
class VideoTokenizer:
    # Per-frame VQGAN tokenization
    # Each frame → 1024 tokens
    # 8 frames → 8192 tokens total
    codebook_size = 16384  # shared!
    tokens_per_frame = 1024
    num_frames = 8

# === Depth/Normal Map Tokenizer ===
class DepthTokenizer:
    # Normalize depth to [0, 1] range
    # Treat as single-channel "image"
    # Apply same VQGAN pipeline
    codebook_size = 16384  # shared!

# === 3D Point Cloud Tokenizer ===
class PointCloudTokenizer:
    # Voxelize 3D space into grid
    # Quantize occupancy → discrete tokens
    # Map to shared codebook range
    codebook_size = 16384  # shared!

# === Action Tokenizer ===
class ActionTokenizer:
    # Robot actions: continuous → discrete
    # Each DOF discretized into bins
    # [x, y, z, rx, ry, rz, grip]
    # → 7 discrete token ids in [0, 16384]
    bins_per_dim = 256
    codebook_size = 16384  # shared!

# === 统一 Vocabulary ===
# Total vocab = text_vocab + shared_codebook
text_vocab = 32128   # SentencePiece (T5)
modality_vocab = 16384  # shared across ALL
# 文本和非文本共用输出 softmax
total_vocab = text_vocab + modality_vocab
# = 48512 total tokens

Shared Codebook 设计

关键创新: 所有非文本模态共享同一个 16384-entry codebook。这意味着模型的词表仅为:

$V = V_{\text{text}} + V_{\text{modality}} = 32128 + 16384 = 48512$

Decoder 可以在输出中自由混合文本和非文本 token。

VQGAN Codebook 选择

为什么 16384? 比 Chameleon (8192) 更大的 codebook 允许更高保真度的重建。更大的 codebook 意味着更细粒度的量化，减少信息损失。

共享 vs 独立 codebook: 共享 codebook 的好处是 decoder 不需要知道输出是哪种模态 — 它只生成 token ids，由下游 detokenizer 解释。

为什么 Encoder-Decoder?

T5 架构优势: Encoder-Decoder 天然适合 "input → output" 的任务框架。Encoder 理解多模态输入，Decoder 生成多模态输出。

$\mathcal{L} = -\sum_{t=1}^{T} \log P(y_t \mid y_{\lt t}, \text{Enc}(x_1, \ldots, x_n))$

训练技巧

1. Task mixing ratio: 精心调配 120+ 数据集的采样比例，防止大数据集主导

2. Prompt engineering: 通过文本 prompt 指定期望输出 ("segment", "describe", "generate")

3. No task heads: 没有任何任务特定的输出头 — 全部通过 sequence generation 实现

与 Chameleon 对比

Chameleon: Decoder-only, 2 modalities (text + image)

UIO-2: Encoder-Decoder, 7+ modalities, 120+ tasks, 更全面但生成质量相对较低 (codebook 重建瓶颈)
