Pretraining 革新

9 个月重建整个 stack。核心衡量：fit scaling law on small models，对比达到目标 performance 的 FLOPs。结果：相同能力所需计算量 比 Llama 4 Maverick 少 10 倍以上。

RL 的关键发现

pass@1 和 pass@16 同步增长意味着模型变强的同时没有丧失推理多样性。这很重要：如果只有 pass@1 涨，说明模型 collapse 到一种解法。

Thought Compression

RL 中加入 thinking time penalty 后出现相变：模型先学会长思考，然后被迫压缩推理链（用更少 token 解决同难度问题），之后在压缩基础上继续提升。这是效率与能力的双赢。

3. Contemplating Mode 与 Multi-Agent 推理

// Contemplating Mode 设计思路
// 问题: 单 Agent 长思考 → 延迟线性增长
// 解法: N 个 Agent 并行推理 → 延迟不变

传统 test-time scaling:
  latency = O(thinking_tokens)
  performance = f(thinking_tokens)
  // 线性 latency-performance tradeoff

Multi-Agent scaling:
  latency = O(max(agent_i_tokens))
  performance = g(N_agents, aggregation)

为什么是 Multi-Agent？

传统 test-time scaling 的瓶颈：思考越长 → 延迟越高。对于面向消费者的产品（meta.ai），延迟是硬约束。

Multi-Agent 方案把 "深度" 转化为 "宽度"：N 个 agent 各想一个方向，最后聚合。延迟 = max(单 agent 时间)，而非 sum。

Thought Compression 相变

RL 加入 thinking time penalty 后的三阶段：

1. 扩展：模型学会长推理

2. 压缩：penalty 迫使用更少 token 达到同效果（类似 "蒸馏自己"）

3. 再扩展：在压缩后的高效表示上继续发展更强能力

Evaluation Awareness

Apollo Research 发现 Muse Spark 会在推理中主动识别 "alignment test"，选择诚实行为。Meta 判定不影响发布，但值得持续研究。
