SigLIP

Google 2023 — 用 Sigmoid Loss 替代 Softmax，解除 batch size 对对比学习的限制

1. Sigmoid vs Softmax Loss

# CLIP (Softmax) loss
logits = I @ T.T * exp(τ)
loss = CE(logits, arange(N))  # N×N softmax
# 问题: softmax 归一化需要全局 N×N
# → 多机 all-gather 通信瓶颈

# SigLIP (Sigmoid) loss
logits = I @ T.T * exp(τ) + bias
labels = 2 * eye(N) - 1  # +1/-1
loss = -log_sigmoid(labels * logits)
loss = loss.sum() / N
# 每对独立计算，无需全局归一化!

核心改进

CLIP 用 softmax 做 N 分类：一张图在 N 个文本中找匹配。需要全局归一化 → 通信量大。

SigLIP 改为二分类：每对独立判断是否匹配。可分片计算，突破 batch size 限制。

$\text{CLIP: } -\log\frac{e^{s_{ii}/\tau}}{\sum_j e^{s_{ij}/\tau}}$ $\text{SigLIP: } -\frac{1}{N}\sum_{i,j}\log\sigma(y_{ij}(s_{ij} \cdot e^\tau + b))$ $y_{ij} = \begin{cases} +1 & i=j \\ -1 & i \neq j \end{cases}$

Shape Optimization

SoViT 通过 NAS 搜索，在固定 FLOPs 预算下找最优超参组合。结果打破了传统整数比例约束。

MAP Pooling

去掉 [CLS]，用 cross-attention 聚合所有 patch 特征。相比 [CLS] token 能更好地整合空间信息。

$\text{MAP: } o = \text{MHA}(q_{\text{probe}}, K_{\text{patches}}, V_{\text{patches}})$ $\text{SoViT-400M: } 27 \times 1152 + \text{MLP}(4304)$
