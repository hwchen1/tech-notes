1. LongAlign — 长上下文指令对齐 (清华, EMNLP 2024)

# ====== LongAlign 核心思路 ======
# Self-Instruct 生成长上下文指令数据

# ====== 三大训练策略 ======

# 策略 1: Packing (多条短样本拼接)
def packing_collate(samples, max_len=64000):
    packed = []
    current_seq, current_len = [], 0
    for s in sorted(samples, key=len):
        if current_len + len(s) <= max_len:
            current_seq.append(s)
            current_len += len(s)
        else:
            packed.append(concat(current_seq))
            current_seq, current_len = [s], len(s)
    return packed

# 策略 2: Sorted Batching (按长度排序)
# 同一 batch 内样本长度接近 → 减少 padding
# GPU 利用率从 ~40% 提升到 ~85%

# 策略 3: Loss Weighting (长序列加权)
def weighted_loss(tokens, labels, seq_lens):
    # 防止短样本稀释长样本梯度
    losses = cross_entropy(tokens, labels)
    weights = seq_lens / seq_lens.sum()
    return (losses * weights).sum()

# 9 个 task 类型:
#   summarization, QA, coding, math,
#   translation, rewriting, brainstorming,
#   open_qa, closed_qa
# 总量: 10K 样本
# 长度分布: 均匀覆盖 10K-64K

LongAlign 三大训练策略: Packing + Sorted Batching + Loss Weighting

核心贡献

Self-Instruct for Long Context: 利用 GPT-4 + 长文档生成覆盖 10K-64K 的指令数据, 9 种任务类型确保多样性
Packing: 将多条短样本拼接为一个长序列, 避免 batch 内长度不均匀导致的计算浪费
Loss Weighting: 按序列长度比例给 token 加权, 防止短样本的 loss 稀释长样本的学习信号
关键发现: 仅靠 10K 条合成数据 + 上述训练策略, 即可将 4K context 模型对齐到 64K, 无需修改模型架构

2. LongLoRA / LongAlpaca — 高效长上下文扩展 (CUHK, ICLR 2024)

# ====== LongLoRA 核心: S2-Attn ======
# Shifted Sparse Attention
# 训练时: shifted group attention (低成本)
# 推理时: 恢复 full attention (无损)

def shifted_sparse_attn(x, group_size=2048):
    B, L, D = x.shape
    n_groups = L // group_size

    # 一半 head 正常分组
    x_normal = x.view(B, n_groups, group_size, D)
    attn_normal = group_attention(x_normal)

    # 另一半 head 偏移半个 group
    x_shift = x.roll(-group_size // 2, dim=1)
    x_shift = x_shift.view(B, n_groups, group_size, D)
    attn_shift = group_attention(x_shift)
    attn_shift = attn_shift.roll(group_size // 2, dim=1)

    # 合并: 交错的组覆盖全局信息
    return concat_heads(attn_normal, attn_shift)

S2-Attn: 训练时用 shifted group attention 降低成本, 推理时恢复 full attention 无损

关键洞察

S2-Attn 核心思想: 将序列分成固定大小的组, 一半 attention head 正常计算组内注意力, 另一半偏移半个组大小 -- 交错覆盖确保信息全局流动
训练-推理解耦: S2-Attn 仅在训练时使用以降低成本 (接近短上下文 LoRA), 推理时直接恢复标准 full attention, 效果等价
RoPE 扩展: 将 RoPE base frequency 从 10000 扩大到 500000+, 使位置编码能表达更长位置, 配合 LoRA 微调即可适应
效率: 4K context 扩展到 100K context, 仅需 LoRA 参数量的训练成本, 无需全量微调

3. 对比与选型 — 长上下文扩展方法全景

# ====== 方法定位 ======
#
# LongAlign: 数据侧
#   - 合成长上下文指令数据
#   - 训练策略 (packing, weighting)
#   - 解决: "能遵循长上下文指令"
#
# LongLoRA: 模型侧
#   - 高效注意力 (S2-Attn)
#   - LoRA 参数高效微调
#   - 解决: "能处理长输入"
#
# ====== 最优组合路径 ======
#
# Step 1: LongLoRA 扩展 context window
#   4K → 100K (S2-Attn + RoPE 扩展)
#   成本: 仅 LoRA 微调
#
# Step 2: LongAlign 策略做 SFT 对齐
#   合成 10K-64K 指令数据
#   packing + loss weighting 训练
#
# Result: 模型既能处理长输入,
#         又能遵循长上下文中的指令
#
# ====== 位置编码方法对比 ======
#
# PI (Position Interpolation):
#   直接线性插值位置 → 简单但损失精度
#
# YaRN (Yet another RoPE extensioN):
#   NTK-aware 插值 + 注意力温度缩放
#   → 更好保持短距离精度
#
# LongLoRA (RoPE base 扩展):
#   直接增大 base → 延长周期
#   配合 S2-Attn 训练 → 推理无修改

LongLoRA 解决"扩展", LongAlign 解决"对齐" -- 二者互补构成最优路径

选型建议

LongAlign 适用场景: 已有长上下文基础模型, 需要做长上下文 SFT 对齐; 或短上下文模型需要对齐到 64K (不修改架构)
LongLoRA 适用场景: 需要低成本扩展 context window (4K->100K), LoRA 预算下的长上下文适配
互补关系: LongLoRA 扩展窗口 + LongAlign 数据策略 = 完整的长上下文对齐 pipeline. 前者解决模型"能力", 后者解决"对齐"
vs YaRN/PI: YaRN 和 PI 仅修改位置编码 (无需训练或少量微调), 但推理时需要保持修改; LongLoRA 推理时完全恢复标准 attention, 部署更简洁
