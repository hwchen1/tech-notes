1. 四轴设计空间

# Miras 框架: 4 个正交设计轴
# 任何序列模型都可以定位到这个空间
#
# 轴 1: Memory Architecture (记忆结构)
#   - 向量 (RNN hidden state)
#   - 矩阵 (Linear Attention S ∈ R^{d×d})
#   - 网络 (Titans θ, 参数即记忆)
#
# 轴 2: Attentional Bias Objective (偏置目标)
#   - L2 loss: ||S·k - v||² → DeltaNet/GDN
#   - Dot-product: softmax(QK^T) → Transformer
#   - Surprise: L(M(x)) → Titans
#   - KL divergence → Moneta
#
# 轴 3: Retention Gate (遗忘门)
#   - 无门控 → Vanilla Linear Attention
#   - 固定标量 → RetNet
#   - 数据依赖标量 → GLA/GDN
#   - Channel-wise → KDA/GDN-2
#
# 轴 4: Learning Algorithm (学习算法)
#   - 无学习 → Transformer (lookup)
#   - SGD → DeltaNet
#   - SGD + weight decay → GDN
#   - Momentum → MDN
#   - Adam-like → Miras (Moneta)

# 实例化已有模型:
# Transformer = {矩阵KV, dot-product, 无gate, 无学习}
# GLA = {矩阵, L2, data-dep scalar, SGD}
# GDN = {矩阵, L2, data-dep scalar, SGD+decay}
# Titans = {网络, surprise, η-gate, gradient}
# Moneta = {矩阵, KL-div, adaptive, Adam}

四个正交设计轴定义了所有序列模型的位置 — Moneta 占据每个轴的最先进选择

为什么是四个轴？

Miras 的核心洞察：从 Transformer 到 Titans，所有架构本质都在做同一件事——读写 associative memory。区别仅在于四个设计选择。

这个框架的价值在于：不是 benchmark paper，而是 principled recipe for designing new architectures。

当你看到一个新模型时，只需要问：它在四个轴上的选择是什么？答案立刻告诉你它的优势和局限。

关键洞察：Miras 证明了 GDN 的主要瓶颈不是 gate 设计（GDN-2 已经解决了 erase/write 解耦），而是学习算法太简单。GDN 用的是 SGD+weight decay，而 Moneta 用 Adam-like 优化器，recall 直接跳升 17.7 点。这解释了为什么 MDN（加动量）也能显著提升。

消融实验

逐步替换 GDN 的设计选择：

1. L2 bias → KL bias: PPL 从 16.42 降到 ~15.5

2. SGD+decay → Adam: recall 从 75.8 跳到 93.5

3. 两者都换 (Moneta): 同时获得两个提升

结论：learning algorithm 的提升远大于 gate 设计的提升。

3. 实践意义与开放问题

# Miras 的实践指导:
#
# 如果你需要:
#   更好的 recall → 换 optimizer (Adam > SGD)
#   更好的 PPL → 换 bias (KL > L2)
#   更低训练成本 → 保持 SGD (GDN)
#   端侧部署 → 保持矩阵 memory (不用网络)
#
# 架构选择清单:
# ┌────────────────┬───────────────┐
# │ 需求           │ 推荐          │
# ├────────────────┼───────────────┤
# │ 最强 recall    │ Moneta/ATLAS  │
# │ 最好 PPL       │ Atlas++       │
# │ 训练效率优先   │ GDN + TNT     │
# │ 推理速度优先   │ Mamba-3 MIMO  │
# │ 端侧/固定mem   │ GDN / Mamba-3 │
# └────────────────┴───────────────┘
#
# 开放问题:
# 1. Adam-like optimizer 训练成本高
#    (需要维护 momentum + 二阶矩)
# 2. 网络型记忆 (Titans) 训练不易并行
# 3. 实际部署中 recall 差距能否通过
#    少量 attention 层补上？

没有免费午餐 — 更强的学习算法需要更多训练开销，选择取决于部署约束

没有银弹——trade-off 仍在

Miras/ATLAS 在 PPL 和 recall 上大幅超越 GDN，但训练成本也更高（Adam 需要更多状态，网络型记忆需要 BPTT）。实际选择取决于部署约束。TNT（17x 加速）部分缓解了训练问题。

重要警示：在真实 long-context QA（SCROLLS）上，Transformer（53.55）仍然显著优于 ATLAS（43.70）和 GDN（40.28）。合成 recall 任务和真实任务之间仍有 gap。
