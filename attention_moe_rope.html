Part 1: Attention 机制演进

从 MHA 到 MLA：KV Cache 优化之路

推理时，每个 token 生成都需要缓存之前所有 token 的 Key 和 Value（KV Cache）。随着序列变长、batch 变大，KV Cache 成为内存瓶颈。MQA → GQA → MLA 的演进核心目标：在保持表达能力的同时压缩 KV Cache。

图 1: 四种 Attention 机制的 Q/K/V head 关系。从左到右：MHA(全独立) → MQA(全共享) → GQA(分组共享) → MLA(低秩压缩缓存)。

MLA 的核心创新：低秩 KV 压缩

图 2: MLA 通过低秩压缩将 KV Cache 从 num_heads × head_dim 降到单个 d_c 维度（如 512）。推理时将 Up-Proj 矩阵"吸收"进 Q 的投影，避免真正恢复完整 KV。

机制KV Cache / token参数效率质量代表模型
MHA 2 × n_heads × d_head 基准 (100%) 最高 (完全独立 KV) GPT-3, Llama-1
MQA 2 × 1 × d_head ↓ 87.5% (8 heads) 有退化 (共享太激进) PaLM, Falcon
GQA 2 × n_kv_groups × d_head ↓ 75% (typical) 接近 MHA Llama-2/3, Qwen2/3
MLA 1 × d_c (d_c ≪ n_h × d_h) ↓ 90%+ = MHA (理论无损) DeepSeek-V2/V3, Kimi-K2

Part 2: MoE vs MoT

MoE — Mixture of Experts (混合专家)

将 FFN 层替换为多个并行的"专家" FFN，每个 token 只激活 Top-K 个专家。总参数量大但激活参数少 → 参数高效扩展。

图 3: MoE 层结构。Router 为每个 token 选择 Top-K 专家，只激活被选中的专家 FFN。DeepSeek 系列额外加入 Shared Expert 保证基础能力。

MoT — Mixture of Transformers

MoT (Mixture of Transformers) 将 MoE 的"混合"思想从 FFN 层扩展到整个 Transformer Block（包含 Attention）。不同 modality 或 task 使用不同的 Transformer 专家。

图 4: MoE 只在 FFN 层做稀疏路由；MoT 将整个 Transformer Block 视为"专家"，不同 modality 使用不同的 Attention+FFN 组合。

MoE 的变体演进

Fine-grained MoE (DeepSeek-V3): 256 个小专家 + Top-8 routing, 比 8 大专家 + Top-2 更灵活
Shared + Routed Expert (DeepSeek-V2/V3): 1-2 个共享专家保证基础能力 + N 个路由专家提供特化能力
Expert Choice (Switch-C): 不是 token 选 expert, 而是 expert 选 token (更均衡)
MoA (Mixture of Attention): 将 MoE 思想应用于 Attention head (不同 head 处理不同子集)

Part 3: RoPE — Rotary Position Embedding

核心直觉

RoPE 的核心思想：用旋转矩阵编码位置信息。将 token 的 embedding 视为复数平面上的向量，不同位置施加不同角度的旋转。两个 token 的点积只取决于它们的相对位置（旋转角度差），而非绝对位置。

图 5: RoPE 将 head_dim 的维度两两配对，每对施加不同频率的旋转。低维对高频旋转编码近距离关系，高维对低频旋转编码远距离关系。

数学公式

$$\text{RoPE}(\mathbf{x}_m, m) = R(m \cdot \theta) \cdot \mathbf{x}_m \quad \text{where} \quad R(\alpha) = \begin{bmatrix} \cos\alpha & -\sin\alpha \\ \sin\alpha & \cos\alpha \end{bmatrix}$$
$$\langle \text{RoPE}(\mathbf{q}, m),\ \text{RoPE}(\mathbf{k}, n) \rangle = \langle R((m-n)\cdot\theta) \cdot \mathbf{q},\ \mathbf{k} \rangle \quad \leftarrow \text{只依赖相对位置 } (m-n)$$
$$\theta_i = \text{base}^{-2i/d} \quad \text{where base} = 10000,\quad i = 0, 1, \ldots, d/2-1$$

简洁实现

import torch

def precompute_freqs_cis(dim: int, seq_len: int, base: float = 10000.0):
    """预计算 RoPE 的复数旋转因子 (cos + i*sin)"""
    # θ_i = base^(-2i/dim), i = 0..dim/2-1
    freqs = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
    # 位置 m 的旋转角: m * θ_i
    t = torch.arange(seq_len)
    angles = torch.outer(t, freqs)  # [seq_len, dim/2]
    # 复数形式: e^(i·angle) = cos(angle) + i·sin(angle)
    return torch.polar(torch.ones_like(angles), angles)  # [seq_len, dim/2]


def apply_rope(x: torch.Tensor, freqs_cis: torch.Tensor):
    """对 Q 或 K 应用 RoPE 旋转"""
    # x: [batch, seq_len, n_heads, head_dim]
    # 将相邻两个维度视为一个复数: (x0 + i*x1)
    x_complex = torch.view_as_complex(x.float().reshape(*x.shape[:-1], -1, 2))
    # 复数乘法 = 旋转: (x0+i*x1) * (cos θ + i*sin θ)
    x_rotated = x_complex * freqs_cis.unsqueeze(0).unsqueeze(2)
    # 转回实数
    return torch.view_as_real(x_rotated).flatten(-2).type_as(x)


# === 等价的 sin/cos 实现 (不用复数, 更直观) ===

def apply_rope_sincos(x: torch.Tensor, cos: torch.Tensor, sin: torch.Tensor):
    """RoPE 的 sin/cos 形式: 旋转公式展开"""
    # x: [batch, seq, heads, dim]
    # cos, sin: [seq, dim/2] (预计算)
    d = x.shape[-1] // 2
    x1, x2 = x[..., :d], x[..., d:]  # 前半/后半维度
    # 旋转: [cos θ, -sin θ; sin θ, cos θ] · [x1; x2]
    out1 = x1 * cos - x2 * sin
    out2 = x1 * sin + x2 * cos
    return torch.cat([out1, out2], dim=-1)

RoPE 长度外推: NTK / YaRN

图 6: RoPE 长度扩展方法。YaRN 的核心创新是分频段处理：保持高频维度不变（近距离精度），只缩放低频维度（远距离扩展），加温度修正补偿 softmax 分布变化。

YaRN 实现要点

# YaRN RoPE 配置 (我们的 Qwen3-4B 128K 训练)
cfg.model.position_embedding_type = "yarn"
cfg.model.yarn_original_max_position_embeddings = 40960   # 原始训练长度
cfg.model.yarn_rotary_scaling_factor = 131072 / 40960    # ≈ 3.2x 扩展
cfg.model.yarn_beta_fast = 32.0    # 高频截止 (dim_i < beta_fast → 不缩放)
cfg.model.yarn_beta_slow = 1.0     # 低频截止 (dim_i > beta_slow → 全缩放)
cfg.model.yarn_mscale = 1.0       # attention temperature 修正系数


# YaRN 频率计算核心逻辑
def yarn_freqs(dim, base, scale_factor, beta_fast, beta_slow, original_max_pos):
    """YaRN: 分频段差异化缩放 + 温度修正"""
    freqs = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))

    # 波长 = 2π/freq, 原始能覆盖的位置数 = wavelength / (2π)
    wavelengths = 2 * math.pi / freqs
    low_freq_wavelen = original_max_pos / beta_slow
    high_freq_wavelen = original_max_pos / beta_fast

    # 三段: 高频(不缩放) / 过渡(插值) / 低频(全缩放)
    scaled_freqs = []
    for freq, wl in zip(freqs, wavelengths):
        if wl < high_freq_wavelen:
            scaled_freqs.append(freq)           # 高频: 保持原样
        elif wl > low_freq_wavelen:
            scaled_freqs.append(freq / scale_factor)  # 低频: 线性缩放
        else:
            # 过渡区: 线性插值
            ratio = (wl - high_freq_wavelen) / (low_freq_wavelen - high_freq_wavelen)
            scaled_freqs.append(freq * (1 - ratio) + freq / scale_factor * ratio)

    return torch.tensor(scaled_freqs)
方法原理优点缺点代表模型
Position Interpolation 所有位置线性压缩: pos' = pos / scale 简单 近距离精度大幅下降 CodeLlama
NTK-Aware 增大 base → 所有频率均匀降低 免训练即可外推 高频维度被不必要压缩 -
YaRN 分频段: 高频不动 + 低频缩放 + temperature 最优的精度-长度平衡 需要少量继续训练 Qwen2.5/3, 我们的 4B 128K
LongRoPE 进化搜索每个维度的最优缩放因子 极长外推 (2M+) 搜索成本高 Phi-3
为什么 RoPE 取代了 Learned / Sinusoidal PE？

1. 相对位置编码：Attention score 自动只依赖 (m-n)，无需额外设计
2. 无额外参数：不像 Learned PE 需要可训练的 embedding table
3. 长度外推：通过调整 base / 频率可以扩展到训练时未见过的长度
4. 计算高效：只需要逐元素乘法 (cos/sin)，可融合进 GEMM kernel
5. 与 FlashAttention 兼容：可以在 kernel 内部 fused 应用
