1. 核心理念 — AR + Discrete Diffusion 统一

# Show-o: One Transformer, Two Generation Paradigms
# Text → Autoregressive (causal attention)
# Image → Discrete Diffusion (full attention)

# === Text Understanding/Generation (AR mode) ===
# Standard next-token prediction with causal mask
text_input = tokenize("Describe this image:")
img_tokens = magvit_encode(image)  # 256 tokens

# Causal attention: each token sees only past
output = transformer(
    [img_tokens, text_input],
    attention="causal"  # AR mode
)
text_out = output.generate_text()

# === Image Generation (Diffusion mode) ===
# Discrete diffusion: mask → unmask iteratively
prompt_tokens = tokenize("A cat on the moon")
img_tokens = [MASK] * 256  # all masked

for t in range(T):  # T denoising steps
    # Text portion: causal attention (frozen)
    # Image portion: FULL bidirectional attention
    logits = transformer(
        [prompt_tokens, img_tokens],
        attention="causal_text + full_image"
    )
    # Unmask highest-confidence positions
    img_tokens = unmask_topk(logits, schedule(t))

image = magvit_decode(img_tokens)

Dual Paradigm, Single Model

Show-o recognizes that autoregressive (left-to-right) is optimal for sequential text, but suboptimal for images where spatial structure matters. Discrete diffusion (parallel mask-and-denoise) is better for images.

$$\text{Text: } P(x_t | x_{\lt t}) \quad \text{(AR, causal)}$$ $$\text{Image: } P(x_m | x_{\text{unmasked}}) \quad \text{(Diffusion, full)}$$

The key insight: by switching attention patterns within the same Transformer, Show-o gets the best of both worlds without needing a separate diffusion model (like Stable Diffusion) or a separate language model.

This is fundamentally different from Emu3 (pure AR for everything) and Chameleon (pure AR with VQ tokens). Show-o uses the right inductive bias for each modality.

2. 离散 Diffusion 图像生成

# Discrete Diffusion Process in Show-o
# Unlike continuous diffusion (Gaussian noise),
# discrete diffusion: mask → unmask tokens

# MAGVIT-v2 tokenizer
# Codebook size: 262144 (2^18)
# Image: 256x256 → 16x16 = 256 tokens

class DiscreteDiffusion:
    def __init__(self, T=25):
        self.T = T  # total denoising steps

    def mask_schedule(self, t):
        # Cosine schedule: how many remain masked
        # At t=0: all masked; at t=T: none masked
        ratio = math.cos(t/self.T * pi/2)
        return int(ratio * 256)

    def denoise_step(self, x, t, text_cond):
        # x: current tokens (some [MASK])
        # text_cond: text prompt tokens

        # Forward pass with mixed attention:
        # - text_cond: causal attention
        # - x (image): FULL bidirectional attention
        logits = self.transformer(
            text=text_cond,  # causal
            image=x,         # full attention
        )

        # Predict tokens at masked positions
        n_unmask = self.mask_schedule(t-1) \
                 - self.mask_schedule(t)

        # Select top-k confident predictions
        confidence = logits[masked_pos].max(-1)
        top_k = confidence.topk(n_unmask)
        x[top_k.indices] = logits[top_k.indices].argmax(-1)

        return x

    def generate(self, text_prompt):
        x = torch.full((256,), MASK_TOKEN)
        for t in range(1, self.T + 1):
            x = self.denoise_step(x, t, text_prompt)
        return magvit_decode(x)

Discrete Diffusion Advantage

vs AR image generation (Emu3): No left-to-right bias. AR generates image tokens sequentially, which introduces directional artifacts. Discrete diffusion denoises all positions in parallel, achieving better spatial coherence.

$$\text{AR: } P(x_i | x_1, ..., x_{i-1}) \quad \text{sequential}$$ $$\text{Diffusion: } P(x_m | x_{\text{unmask}}) \quad \text{parallel}$$

vs Continuous diffusion (Stable Diffusion): Shares the same discrete token space as the language model. No need for a separate U-Net or VAE latent space. The Transformer itself IS the denoiser.

MAGVIT-v2 provides a massive 262,144-entry codebook, enabling high-fidelity reconstruction while keeping the token count manageable (256 tokens per image).

Only 25 denoising steps needed (vs DDPM's 1000), because discrete space has finite states and top-k selection is highly efficient.

3. 统一训练与 Omni 能力

# Show-o Training: Unified Multi-Task
# Same model trained on all tasks simultaneously

class ShowOTraining:
    def compute_loss(self, batch):
        total_loss = 0

        # === Task A: Text-only (standard NTP) ===
        if batch.has_text_only:
            # Causal attention, standard LM loss
            logits = self.forward(
                batch.text_tokens,
                mask="causal"
            )
            loss_text = F.cross_entropy(
                logits[:, :-1], batch.text_tokens[:, 1:]
            )
            total_loss += loss_text

        # === Task B: Image Understanding ===
        if batch.has_image_text:
            # [img_tokens | text_prompt | text_answer]
            # All causal attention
            seq = concat(batch.img_tokens,
                        batch.question, batch.answer)
            logits = self.forward(seq, mask="causal")
            # Loss only on answer tokens
            loss_und = F.cross_entropy(
                logits[:, -len(batch.answer)-1:-1],
                batch.answer
            )
            total_loss += loss_und

        # === Task C: Image Generation ===
        if batch.has_generation:
            # Random masking ratio r ~ U(0, 1)
            r = random.uniform(0, 1)
            n_mask = int(r * 256)
            masked_img = random_mask(
                batch.img_tokens, n_mask
            )
            # text: causal | image: full attention
            logits = self.forward(
                text=batch.prompt,      # causal
                image=masked_img,       # full attn
            )
            # Loss: predict original at masked pos
            loss_gen = F.cross_entropy(
                logits[masked_positions],
                batch.img_tokens[masked_positions]
            )
            total_loss += loss_gen

        return total_loss

# Training data mix:
# - Text corpus (for language ability)
# - Image-text pairs (for understanding)
# - Text-image pairs (for generation)
# All in the same batch!

Unified Training Objective

The training loss combines two objectives applied to different parts of the sequence within the same forward pass:

$$\mathcal{L} = \mathcal{L}_{\text{NTP}} + \mathcal{L}_{\text{mask}}$$ $$\mathcal{L}_{\text{NTP}} = -\sum_{t} \log P(x_t | x_{\lt t})$$ $$\mathcal{L}_{\text{mask}} = -\sum_{m \in \mathcal{M}} \log P(x_m | x_{\backslash \mathcal{M}})$$

Show-o achieves competitive results across ALL tasks simultaneously in a single 1.3B model:

- Image generation: comparable to DALL-E 2 and SD v1.5

- Image understanding: comparable to LLaVA-Phi and InstructBLIP

- Text generation: inherits Phi-1.5 quality

The key training insight: random masking ratio during training (uniform sampling of mask percentage) makes the model robust at any denoising step during inference.
