Nemotron-3-Ultra: Mamba+Transformer 交替 + LatentMoE + MTP

架构设计哲学

Layer Pattern: 非均匀交替 — Mamba-2 层占主导（降 KV cache），少量 Attention 层穿插（保精确 recall）。每层配 LatentMoE（512 experts, top-22）。

LatentMoE: 在 2048 维潜在空间做路由决策，比标准 Granular MoE 更好的 accuracy/param ratio。

MTP: 2 个共享权重的预测头，每个由 1 Attention + 1 MoE 层组成。训练时 aux loss 0.1；推理时做 native speculative decoding。

NVFP4 预训练: E2M1 格式 + 2D block quantization + Random Hadamard + Stochastic Rounding。后 15% 网络 (16 层) + 嵌入层保高精度。BF16 vs NVFP4 loss gap \lt 0.4%。

预训练: 20T tokens (Phase 1: 15T 多样性 → Phase 2: 5T 高质量)。WSD 学习率调度。

MOPD: Student 生成 → Teachers 评分 → RL 优化 → 迭代进化

MOPD 核心机制

目标函数: Minimize reverse-KL: \(\text{KL}(\pi_\theta(\cdot|y_{\lt t}) \| \pi_{T_d}(\cdot|y_{\lt t}))\)。Teacher 对 student 生成的轨迹做 dense token-level scoring，不是 sparse reward。

异步解耦: 行为策略 π_behav (生成 rollout) 与近端策略 π_prox (trust region 中心) 分离。PPO-clip 应用在 π_prox 上。使用 IcePop token-level masking。

MOPD Warmup: 发现分布不匹配时 teacher 信号质量差。先对 student 做轻量 SFT 对齐 teacher 分布，显著提升 agentic 域效果（GDPVal: 35.3→46.7）。

Co-Evolution: Iter 2 从 MOPD1 的 student 初始化新 teachers，同时复用 Iter 1 teachers。

Teacher 训练细节

SWE Teacher: 3 阶段 — Agentic SFT → PivotRL (单步环境) → 端到端 SWE-RL (192K gen, 200 turns)。防作弊：删除 future commits + 封锁 git remote。

Search Teacher: 训练 context-management — discard-all resets + summary compression，超越官方 context 限制。

Terminal Teacher: 长超时设置 (up to 1 hour)，PivotRL 迭代 + re-profiling。

STEM Teacher: 40B token SFT (58.75% science + 23.63% math + 10.13% coding + 7.5% general) → RL。超越 DeepSeek V4 Pro。

vs GLM-5.1 (744B-A40B): 参数量小 26% 但吞吐 5.9× 快。Agentic 多数平手，Terminal 稍弱 (56.4 vs 59.3)。

vs Qwen3.5 (397B-17B): 吞吐 1.6× 快。Agent 全面超越，Ruler@1M 领先 4.6 分。

Reasoning Budget Control

支持推理时动态调节 thinking token 预算 — 简单子任务快速完成，复杂规划分配更多算力。SFT 阶段已训练 medium-effort 模式。

定位总结: Nemotron-3-Ultra 核心优势是 accuracy-per-FLOP — 在同等质量下推理吞吐远超竞品。设计哲学是"agent 任务不需要每个 token 都全力推理"，通过 Mamba 降 attention 成本 + MoE 稀疏激活 + NVFP4 + MTP，把单位 GPU 产出最大化。弱点在纯 coding 领域。

5 层设计: 架构 → 预训练 → SFT → RLVR 环境 → MOPD dense signal

为什么能 Long-Horizon

问题: 传统 RL 在长 session 中面临：(1) KV cache 内存爆炸 (2) sparse reward credit assignment (3) goal drift (4) error accumulation。

逐层解决

架构层: Mamba-2 O(N) + GQA 2KV → 物理上支撑 200+ turn 不 OOM。比纯 Transformer (O(N²) + full KV) 节省 10×+ 内存。

RLVR 环境设计: 不是短 prompt→response，而是真实的多轮 agent loop (最多 200 turns)。模型在训练时就经历长 session 的全部挑战。

Context Management: Search teacher 专门训练 discard-all + summary compression 策略，让模型在 context 快满时主动管理信息，而不是被动截断。

Anti-Shortcut: Conv tool-use 数据专门解决"过早终止"问题 — 训练强调必须完成 sequential dependent actions 不能半途放弃。

MOPD 解决 Credit Assignment

核心洞察: 长 session 的 sparse final reward（任务成功/失败）无法告诉模型哪一步出了问题。MOPD 的 teacher 提供 每个 token 的 dense signal（reverse-KL），相当于对整个轨迹做精细纠错。

配合措施: (1) Mask unfinished trajectories (timeout/max turns 不给 reward) (2) Penalize malformed reasoning/tool calls 的 specific tokens with negative advantage (3) 防作弊机制确保 reward 信号可信。

三条数据 Pipeline: Terminal (合成+转化) / SWE (3阶段) / Search (context管理)

Terminal 370K 数据如何构建

双轨生产: (1) Dataset Adaptation — 将现有 benchmark (DevOps 运维题、shell 编程题) 改写成真实 terminal 环境任务; (2) Synthetic Task Generation — 用 Skill Taxonomy 驱动 LLM 合成全新任务。两路合并后经 verifier 过滤，保留可执行、有意义的 370K 条轨迹。

Skill Taxonomy: 预定义技能分类树 (文件操作/网络/Git/Docker/调试...)，按组合采样确保 skill 覆盖率。不是随机生成，而是系统性覆盖终端操作的各个象限。

LiteCoder 扩展 (长程): 5 阶段 pipeline 专门生成需要长时间交互的复杂任务 — Instruction Refinement → Environment Init → Solution Synthesis → Verifier Gen → Resource Config。每个环境都是独立 Docker 容器，可设 1hr+ timeout。

SWE-RL 数据: 3 阶段渐进

数据来源: GitHub 真实 Issues（筛选有 test patch 的）→ 回退到 fix 前状态 → 模型在容器中尝试修复 → hidden test 判成败。

Agentic SFT: 先用强模型 (或人工) 收集成功修复轨迹作为 SFT 数据（Nemotron-SFT-SWE-v3）。

PivotRL: 在简化环境做单步 RL — 每步给 instant feedback，成本低、迭代快。训练模型基本的工具调用和错误恢复能力。

端到端 SWE-RL: 完整 200 turn agent loop，gen 192K tokens，只在任务结束给 binary reward。此时模型已有 PivotRL 打底，能有效利用 sparse signal。

Search Agent 数据: Context Management

独特需求: Search 任务可能需要 50-100 次查询才能找到答案。context window 会被搜索结果撑满。

关键训练信号: 教模型两个策略 — (1) discard-all reset: 发现搜索方向错误时清空 context 重来; (2) summary compression: 将已收集信息压缩为摘要释放空间。

效果: 模型能"突破"官方 context 限制 — 通过主动管理信息而非被动等 context 满。
