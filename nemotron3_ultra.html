Nemotron-3-Ultra: Mamba+Transformer 交替 + LatentMoE + MTP

架构设计哲学

Hybrid 动机: 纯 Transformer 在 1M context 下计算爆炸；纯 Mamba 精确 recall 不足。交替使用两者互补。

LatentMoE: 与标准 Top-K MoE 不同，在潜在空间中做路由决策，支持推理/代码/工具调用/领域逻辑间灵活切换。

MTP: 训练时预测多个 future tokens（类似 DeepSeek-V3），推理时做 speculative decoding，对多轮 agent 对话特别有效。

NVFP4: 同一 checkpoint 跨 Ampere/Hopper/Blackwell 运行，BF16 → NVFP4 在 Blackwell 上 5× 吞吐。

2. 训练方法 — Multi-Teacher On-Policy Distillation (MOPD)

# MOPD: Multi-Teacher On-Policy Distillation
# 核心: 10+ teacher 模型各司其职, 联合蒸馏到 student

# Step 1: 训练多个领域专用 Teacher
teachers = {
    "math": train_teacher(domain="math"),
    "code": train_teacher(domain="code"),
    "legal": train_teacher(domain="legal"),
    "science": train_teacher(domain="science"),
    "agent": train_teacher(domain="agent_tool_use"),
    # ... 10+ specialized teachers
}

# Step 2: Student 生成 On-Policy Rollout
for task in multi_domain_tasks:
    rollout = student.generate(task.prompt)

    # Step 3: 对应 Teacher 打分
    domain = classify_domain(task)
    reward = teachers[domain].score(rollout)

    # Step 4: RL 优化 (GRPO-style)
    student.update(rollout, reward)

# 关键: 异步 Pipeline
# Student Rollout | Teacher Scoring | Student Update
# 三者全流水线并行, 最大化 GPU 利用

# Step 5: 迭代 Co-Evolution
# 每轮 MOPD 后:
# - Teacher 从更新后的 Student 初始化
# - Teacher 在各自领域继续训练
# - 新一轮 MOPD → 更强的 Student
# 学生和老师共同进化!

MOPD: Student 生成 → Teachers 评分 → RL 优化 → 迭代进化

MOPD 关键创新

vs 标准蒸馏: 传统蒸馏用单一 teacher 的 logits；MOPD 用 10+ teachers 的 reward signal，在 student 自己的 on-policy 轨迹上学习。

异步流水线: Rollout 生成、Teacher 打分、Student 更新三阶段完全 pipeline 并行，最大化训练效率。

Co-Evolution: 每轮结束后 teacher 从新 student 初始化再特化训练，两者共同进步，避免 teacher 水平成为上限。
