2. Kimi Delta Attention (KDA) — 混合线性注意力

KDA 是 Kimi Linear（arXiv 2510.26692）路线的产品化落地，可看作 Gated DeltaNet 的细粒度门控改良：用固定大小的循环状态 $S_t$ 取代随长度增长的 KV-cache，每步用 delta 规则做“擦除-写入”，并引入 per-channel（细粒度）遗忘门控。KDA 层与全注意力（Gated MLA）以 3:1 交错，兼顾局部记忆效率与全局精确检索。

# KDA 递推（简化）: 固定状态 S ∈ R^{d_k×d_v}
def kda_step(S, k, v, q, alpha, beta):
    # alpha: per-channel 遗忘门 (细粒度)
    # beta : delta 写入强度
    S = S * alpha[:, None]          # 逐通道衰减
    S = S + beta * outer(k, v - S.T @ k) # delta 擦除-写入
    o = S.T @ q                     # 读出
    return S, o

# 层堆叠：3 KDA : 1 full-attn
for i in range(n_layers):
    if (i + 1) % 4 == 0:
        x = gated_mla(x)   # 全局精确检索
    else:
        x = kda(x)         # O(1) 状态，长序列友好

为什么省显存

标准注意力 KV-cache 随序列线性增长，1M token 直接爆显存。KDA 用固定大小状态 $S$ 递推，显存与序列长度解耦。

细粒度门控：per-channel 的遗忘门 $\alpha$ 让每个特征维度独立决定“记多久”，比 Gated DeltaNet 的标量/头级门控表达力更强。

3:1 混合：纯线性注意力在精确检索（如大海捞针）上有损失，插入 1/4 的 Gated MLA 全注意力层补回全局精度。

渊源：见 la_kimi_linear.html 与 kimi_k25_vl.html，KDA 是 Kimi Linear 从研究到 3T 级产品的规模化验证。

3. Attention Residuals 与架构全景

Attention Residuals（AttnRes）改造残差流：不再让每层均匀累加，而是让当前块从更浅的若干块（n−1, n−2, n−3）做选择性检索，缓解深层网络中注意力信号的稀释与同质化。配合 Gated MLA（全注意力层）、SiTU 激活与 Per-Head Muon 优化器，构成 K3 在 3T 规模下仍稳定收敛的关键。

# Attention Residuals: 跨深度选择性检索
def attn_residual(x_n, hist):
    # hist = [h_{n-1}, h_{n-2}, h_{n-3}]
    g = softmax(gate(x_n))      # 选择权重
    r = sum(g[k] * hist[k] for k in ...)
    return x_n + r             # 非均匀残差

# SiTU: Sigmoid-Tanh Unit 激活
def situ(x):
    return tanh(x) * sigmoid(gate(x))

# Per-Head Muon: 每个注意力头独立
# 做 Newton-Schulz 正交化更新
for head in heads:
    G = head.grad
    O = newton_schulz(G)   # 正交化
    head.W -= lr * O

四个稳定器

AttnRes：跨块选择性检索，深层不再被均匀残差冲淡关键信号。

Gated MLA：3:1 混合里的全注意力层，带门控的多头潜在注意力，在大规模下保持检索选择性。

SiTU：Sigmoid-Tanh Unit，门控式激活，控制激活幅度、利于低精度训练。

Per-Head Muon：把 Muon 的正交化更新细化到每个注意力头，是 2.8T 规模稳定训练的核心优化器改动。

4. Stable LatentMoE 与 MXFP4/8 QAT

Stable LatentMoE 是 K3 的稀疏放大层：896 个路由专家，每 token 激活 16 个，用 Quantile Balancing（分位数均衡）替代传统 aux-loss 做负载均衡，让专家利用率更平稳。从 SFT 阶段起，权重采用 MXFP4、激活采用 MXFP8 做量化感知训练（QAT），使得 3T 级模型在推理端仍可负担。

# Stable LatentMoE: 16 / 896
n_experts   = 896
top_k       = 16
balance     = "quantile"  # 非 aux-loss

def route(x):
    logits = router(x)              # [896]
    # Quantile Balancing: 按分位阈值路由，
    # 强制各专家吞吐落在均衡区间
    thr = quantile(logits, q=1-top_k/n_experts)
    idx = topk(logits, top_k)
    return idx, softmax(logits[idx])

# MXFP4 / MXFP8 QAT (SFT 起)
w_q = mxfp4(expert.W)     # 权重 4-bit
a_q = mxfp8(act)          # 激活 8-bit
y   = grouped_gemm(a_q, w_q) # 低精度稀疏计算

稀疏 + 低精度双杠杆

16/896 极稀疏：激活比 ~1.8%，2.8T 总容量但单 token 计算量接近中型 dense 模型。

Quantile Balancing：用分位阈值直接约束各专家吞吐区间，比 aux-loss 更少干扰主任务 loss，专家坍塌风险更低。

QAT from SFT：不是训练后 PTQ，而是从 SFT 起就在 MXFP4/8 下前反传，模型学会适配低精度，推理无明显掉点。

成本：官方定价 $0.30 / $3 / $15（缓存命中输入 / 输入 / 输出，每百万 token），得益于稀疏 + 低精度组合。
