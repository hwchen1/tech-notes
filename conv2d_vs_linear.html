Conv2d vs Linear

CLIP ViT Patch Embedding -- Conv2d 如何将图像切成 token 序列

1. Conv2d 工作原理

# Conv2d(in_ch, out_ch, kernel, stride)
# 卷积核在图像上滑动
# stride=kernel 时 = 不重叠分块

conv = nn.Conv2d(
    in_channels=3,      # RGB
    out_channels=1024,  # embed_dim
    kernel_size=14,     # patch大小
    stride=14           # 步幅=核大小
)

# 权重形状:
# W: [1024, 3, 14, 14]
# 参数量: 1024 * 3 * 14 * 14
#        = 602,112
Conv2d 做 patch embedding：

1. 卷积核大小 = patch 大小 (14x14)

2. stride = kernel_size，所以 patch 之间不重叠

3. 每个 14x14x3 的 patch 与 1024 个滤波器做点积，得到 1024 维向量

4. 336/14 = 24，所以横竖各 24 个 patch，共 576 个

5. 输出 reshape 成 [576, 1024] 即 token 序列

本质：一次卷积 = 分块 + 线性投影

2. Conv2d(k=14, s=14) 等价于 Linear

# 方法 A: Conv2d
conv = nn.Conv2d(3, 1024,
    kernel_size=14, stride=14)
out = conv(img)  # [B,1024,24,24]
tokens = out.flatten(2).transpose(1,2)
# -> [B, 576, 1024]

# 方法 B: 手动分 patch + Linear
patches = img.unfold(2,14,14).unfold(3,14,14)
# [B, 3, 24, 24, 14, 14]
patches = patches.reshape(B, 576, 588)
# 588 = 3 * 14 * 14

linear = nn.Linear(588, 1024)
tokens = linear(patches)
# -> [B, 576, 1024]

# 数学完全等价!
# conv.weight.reshape(1024, 588)
# == linear.weight
关键洞察：

当 kernel_size == stride 时，卷积核不重叠地覆盖输入。

这时每个输出位置看到的是一个独立 patch，做的就是一个 线性变换。

区别只在于：
- Conv2d: 隐式地 patchify
- Linear: 需要先手动切 patch

为什么 ViT 用 Conv2d?
1. 一步完成切块+投影
2. cuDNN 卷积核高度优化
3. 代码更简洁
4. 保持 2D 结构信息（可选）

3. Conv2d vs Linear 真正的差异

# === 标准 Conv2d (有重叠时) ===
# kernel=3, stride=1, padding=1
conv = nn.Conv2d(64, 128,
    kernel_size=3, stride=1, padding=1)

# 每个输出位置看 3x3 邻域
# -> 局部感受野 + 权重共享
# -> 不等价于 Linear!

# === Linear ===
linear = nn.Linear(588, 1024)
# 每个输入位置独立变换
# 没有邻域概念
# 没有权重共享

# === 总结 ===
# Conv2d(k=s): 等价 Linear
# Conv2d(k>1,s=1): 有重叠
#   -> 局部感受野
#   -> 平移等变性
#   -> 权重共享 (参数少)
三大区别：

1. 局部感受野
Conv2d 每个输出只看 k x k 邻域；Linear 看全部输入

2. 权重共享
Conv2d 所有空间位置共享同一组参数；Linear 每个位置有独立权重

3. 平移等变性
Conv2d: 猫移到哪，特征跟着移。Linear 没有这个性质

*ViT patch embed 的特殊性：
kernel=stride=14 时，patch 之间没有重叠，局部感受野退化为"整个 patch"，数学上等价于先切 patch 再做 Linear。

用 Conv2d 纯粹是工程上更方便高效。

4. CLIP ViT-L/14 完整 Patch Embedding

class CLIPVisionEncoder(nn.Module):
    """CLIP ViT-L/14
    Input: 336x336 image
    Patches: 14x14 -> 24x24 = 576 patches
    Output: 576 tokens x 1024-dim
    """
    def __init__(self):
        self.patch_embed = Conv2d(
            3, 1024,
            kernel_size=14, stride=14)

        # CLS token (可选)
        self.cls_token = nn.Parameter(
            torch.zeros(1, 1, 1024))

        # Position embedding
        self.pos_embed = nn.Parameter(
            torch.zeros(1, 577, 1024))
        # 577 = 576 patches + 1 cls

    def forward(self, x):
        # x: [B, 3, 336, 336]
        x = self.patch_embed(x)
        # [B, 1024, 24, 24]
        x = x.flatten(2).transpose(1,2)
        # [B, 576, 1024]
        cls = self.cls_token.expand(B,-1,-1)
        x = torch.cat([cls, x], dim=1)
        # [B, 577, 1024]
        x = x + self.pos_embed
        return x
完整流程：

Step 1: Conv2d 将 336x336 图像切成 24x24=576 个 patch，每个投影到 1024 维

Step 2: reshape 成 token 序列 [576, 1024]

Step 3: 拼接一个可学习的 [CLS] token

Step 4: 加上 position embedding 注入空间位置信息

输出：577 个 1024 维的 vision token，可以直接送入 Transformer encoder 或 LLM

关键数字：
- 336 / 14 = 24 (每边 patch 数)
- 24 x 24 = 576 (总 patch 数)
- 3 x 14 x 14 = 588 (每 patch 像素数)
- Conv2d 参数: 1024 x 588 = 602K
