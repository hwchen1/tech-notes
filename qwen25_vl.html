Qwen2.5-VL Architecture

Dense Transformer, GQA, mRoPE (3-axis multimodal positional encoding), ViT with 2D RoPE + Window Attention, Dynamic Resolution

1. Dense Transformer with GQA

Qwen2.5-VL uses a standard dense transformer (no MoE). Grouped Query Attention (GQA) reduces KV cache memory while maintaining quality. RMSNorm + SwiGLU MLP with QKV bias.

class Qwen25VLBlock(nn.Module):
    """Dense transformer block with GQA"""
    def __init__(self, hidden=3584, n_heads=28, n_kv=4):
        self.norm1 = RMSNorm(hidden)
        self.attn = GroupedQueryAttention(
            hidden_size=hidden,
            num_heads=n_heads,       # 28 query heads
            num_kv_heads=n_kv,       # 4 KV heads
            qkv_bias=True,          # Qwen2 specific
        )
        self.norm2 = RMSNorm(hidden)
        self.mlp = SwiGLU(
            hidden, intermediate=18944)

    def forward(self, x, pos_ids):
        # Pre-norm architecture
        h = x + self.attn(self.norm1(x), pos_ids)
        h = h + self.mlp(self.norm2(h))
        return h

# 72B: hidden=8192, heads=64, kv_heads=8
# 7B:  hidden=3584, heads=28, kv_heads=4
# 3B:  hidden=2048, heads=16, kv_heads=2

Grouped Query Attention

GQA groups multiple query heads to share a single KV head. For 7B: 28 query heads share 4 KV heads (ratio 7:1).

$\text{KV cache} = 2 \times n_{kv} \times d_{head} \times \text{seq\_len}$ $7 \times \text{ reduction vs full MHA}$

SwiGLU MLP

Gated linear unit with SiLU activation. Three weight matrices: gate_proj, up_proj, down_proj.

$\text{FFN}(x) = W_{down}(\text{silu}(W_{gate} x) \odot W_{up} x)$ $\text{intermediate\_size} = 18944 \text{ (for 7B)}$

QKV Bias is a Qwen2-family specific choice — most LLMs omit bias in attention projections.

2. Vision Encoder (ViT + Window Attention)

ViT-based encoder with Conv3d patch embedding (temporal_patch_size=2 for video). Uses windowed attention for efficiency on high-resolution images, with 2D RoPE for spatial position encoding.

class Qwen25VisionEncoder(nn.Module):
    """ViT with Conv3d + Window Attention + 2D RoPE"""
    def __init__(self):
        self.patch_embed = nn.Conv3d(
            in_channels=3,
            out_channels=1280,  # vision hidden
            kernel_size=(2, 14, 14),
            stride=(2, 14, 14)
        )
        # 32 ViT blocks with window attention
        self.blocks = nn.ModuleList([
            VisionBlock(
                hidden=1280,
                heads=16,
                window_size=112,  # 8x14 patches
                use_global=(i % 4 == 3)
            ) for i in range(32)
        ])
        # Merger: pool 2x2 patches then project
        self.merger = VisionMerger(
            in_dim=1280 * 4,
            out_dim=3584  # LLM hidden
        )

    def forward(self, pixels, grid_thw):
        x = self.patch_embed(pixels)
        x = x + self.rope_2d(x, grid_thw)
        for blk in self.blocks:
            x = blk(x)
        return self.merger(x)

Conv3d Patch Embedding

patch_size=14 (spatial), temporal_patch_size=2. A 448x448 image yields 32x32=1024 patches. Video merges 2 frames temporally.

Window Attention

Full global attention on all patches is O(N^2). Window attention restricts each token to attend within a local window of 112 tokens.

Every 4th block uses global attention to propagate information across windows.

$3 \times \text{Window} + 1 \times \text{Global} = 1 \text{ group}$ $8 \text{ groups} = 32 \text{ ViT blocks}$

Merger

Pools 2x2 adjacent patches into one token (4x reduction), then projects from 1280*4=5120 to LLM hidden dim (3584).

3. mRoPE (Multimodal RoPE)

Head dimension split into 3 sections [16, 24, 24] for temporal/height/width positional encoding. Language tokens use only temporal axis; vision tokens encode full 3D spatial-temporal position.

class MultimodalRoPE(nn.Module):
    """mRoPE: 3-way split [16, 24, 24] on head_dim=128"""
    # mrope_section = [16, 24, 24]
    # Each section gets independent rotary freqs

    def forward(self, q, k, position_ids):
        # position_ids: [B, 3, seq_len]
        #   dim 0: temporal positions
        #   dim 1: height positions
        #   dim 2: width positions

        # Split q,k along head_dim (128 total)
        q_t, q_h, q_w = q.split(
            [16*2, 24*2, 24*2], dim=-1)
        k_t, k_h, k_w = k.split(
            [16*2, 24*2, 24*2], dim=-1)

        # Apply rotary per axis
        q_t = apply_rotary(q_t, freqs_t)
        q_h = apply_rotary(q_h, freqs_h)
        q_w = apply_rotary(q_w, freqs_w)

        return cat([q_t, q_h, q_w], dim=-1)

    # Language: t=seq_pos, h=seq_pos, w=seq_pos
    # Image:   t=T, h=row_id, w=col_id
    # Video:   t=frame_id, h=row, w=col

3-Axis Position Encoding

Standard RoPE uses 1 position ID. mRoPE splits head_dim=128 into [16, 24, 24] sections and applies independent rotary embeddings per axis.

$\text{mrope\_section} = [16, 24, 24]$ $\text{axes: temporal, height, width}$

Unified Multimodal Positions

Language: All 3 axes share the same sequential position ID — equivalent to standard 1D RoPE.

Image: Temporal axis is constant (insertion point). Height/width encode the 2D patch grid coordinates.

Video: Temporal axis encodes frame timing (seconds_per_grid). Height/width encode spatial position within each frame.

This lets the LLM natively understand spatial relationships in images and temporal+spatial relationships in video.

4. Dynamic Resolution (NaViT-style)

No fixed image resolution. Images are processed at their native aspect ratio by dividing into variable-size grids of patches. This avoids distortion and preserves fine-grained details.

def smart_resize(image, min_pixels, max_pixels,
                  patch_size=14, merge_size=2):
    """Resize to nearest valid grid size"""
    h, w = image.size
    factor = patch_size * merge_size  # 28

    # Find optimal (h', w') such that:
    # 1. h' % 28 == 0 and w' % 28 == 0
    # 2. min_pixels <= h'*w' <= max_pixels
    # 3. Aspect ratio preserved

    h_bar = round(h / factor) * factor
    w_bar = round(w / factor) * factor

    # Clamp total pixels
    total = h_bar * w_bar
    if total > max_pixels:
        scale = sqrt(max_pixels / total)
        h_bar = round(h_bar*scale/factor)*factor
        w_bar = round(w_bar*scale/factor)*factor
    elif total < min_pixels:
        scale = sqrt(min_pixels / total)
        h_bar = round(h_bar*scale/factor)*factor
        w_bar = round(w_bar*scale/factor)*factor

    return resize(image, (h_bar, w_bar))

# Result: variable grid_thw per image
# e.g. (1, 32, 24) for a 448x336 image

No Fixed Resolution

Traditional ViTs resize all images to 224x224 or 448x448. Qwen2.5-VL processes images at their native aspect ratio.

Grid Constraints

Image dimensions must be multiples of 28 (patch_size=14 * merge_size=2). The smart_resize function finds the closest valid size.

$\text{min\_pixels} = 256 \times 28 \times 28$ $\text{max\_pixels} = 1280 \times 28 \times 28$ $\text{factor} = 14 \times 2 = 28$

Variable Sequence Length

Each image produces a different number of visual tokens based on its resolution. The model handles this via the grid_thw tensor that tracks (temporal, height, width) grid dimensions per image.

After merger (2x2 pooling), final token count is further reduced by 4x.

5. MLP Projector (Vision-Language Connector)

A simple 2-layer MLP projects vision features from the ViT hidden dimension to the LLM hidden dimension. The merger module combines spatial pooling with projection.

class VisionMerger(nn.Module):
    """2x2 spatial pooling + 2-layer MLP projection"""
    def __init__(self, vis_dim=1280, llm_dim=3584):
        # After 2x2 pooling: 4 patches concatenated
        self.mlp = nn.Sequential(
            nn.LayerNorm(vis_dim * 4),
            nn.Linear(vis_dim * 4, vis_dim * 4),
            nn.GELU(),
            nn.Linear(vis_dim * 4, llm_dim),
        )

    def forward(self, vis_tokens, grid_thw):
        # vis_tokens: [B, N, 1280]
        # Reshape to 2D grid per image
        # Pool 2x2 adjacent patches
        x = rearrange(vis_tokens,
            'b (h w) d -> b h w d',
            h=grid_h, w=grid_w)
        x = pool_2x2(x)  # [B, h/2, w/2, 4*d]
        x = x.flatten(1, 2)  # [B, N/4, 5120]

        # Project to LLM dimension
        return self.mlp(x)  # [B, N/4, 3584]

Why 2x2 Pooling?

Raw ViT output has too many tokens for the LLM to process efficiently. A 448x448 image produces 1024 patches; after 2x2 pooling, only 256 tokens enter the LLM.

$1024 \text{ patches} \rightarrow 256 \text{ tokens (4x reduction)}$ $\text{Each token} = \text{concat of 4 neighbors}$

Simple MLP Design

Unlike complex cross-attention connectors (e.g., Perceiver Resampler), Qwen2.5-VL uses a straightforward 2-layer MLP. This preserves spatial information without lossy compression.

LayerNorm before projection stabilizes the input distribution from the ViT encoder.

Freeze Strategy

During fine-tuning, the merger can be frozen independently from the ViT backbone and LLM, enabling flexible training configurations.
