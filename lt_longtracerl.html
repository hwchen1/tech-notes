1. LongTraceRL — 问题定义与动机

# ====== 核心论点 ======
# 现有长文本 RL 方法的三大缺陷:
#
# 缺陷 1: 无干扰 / 原文即上下文
#   LongAlign, LongRLVR, Crafting-Effective
#   → 模型不学"从噪声中定位"能力
#   → 测试时遇到干扰文档就崩溃
#
# 缺陷 2: 随机干扰 ≠ 真实干扰
#   LoongRL, EAPO, Self-Improve, CoT-Path
#   → 随机文档与 query 无语义关联
#   → 模型用简单关键词匹配就能区分
#   → 不训练"区分相似但错误信息"
#
# 缺陷 3: 上下文太短 (≤ 16K)
#   真实 search agent 单轮可产生 128K+
#   在 16K 上训练 → 128K 上泛化失败
#
# ====== LongTraceRL 的解法 ======
# 1. KG 游走: 可控生成 k-hop 问题
# 2. 分层干扰: Tier-1(KG邻居) + Tier-2(BM25)
# 3. 128K 填充: 匹配真实 agent 场景
# 4. GRPO RL: 用 search agent 轨迹训练
# 5. Rubric Reward: 分步骤给分

# ====== 关键数据 ======
# 基座: Qwen2.5-7B-Instruct
# 数据量: 4K 训练样本 (KG 游走生成)
# 上下文: 128K tokens
# 跳数: k=8
# 训练: GRPO, batch=128, 3 epochs
# 效果: RULER 128K +18.4%, HELMET +12.1%

LongTraceRL 5阶段流水线: KG游走 → 问题生成 → 分层干扰填充 → Agent轨迹 → GRPO

核心 Insight

真实 search agent 遇到的干扰不是随机的——搜索引擎返回的是语义相关但答案错误的文档。训练数据必须模拟这种分布。

核心创新：用 KG 拓扑结构生成分层干扰——同 entity 的邻居页面天然是"看起来相关但不是答案"的最强干扰。

为什么选 KILT Wikipedia?

KILT 提供 Wikipedia 的结构化知识图谱索引，每个实体有明确的关系边。随机游走天然产生多跳推理链，且邻居实体提供高质量干扰。

2. LongTraceRL — 分层干扰与 Reward 设计

# ====== 分层干扰构造细节 ======

def build_tiered_context(path, kg, question):
    gold_docs = [kg.get_doc(e) for e in path]

    # Tier-1: KG 邻居 (最强干扰)
    # 规则: 取 gold entity 的 1-hop 邻居
    # 特征: 同主题/同类型/共享属性
    # 例: gold="Albert Einstein"
    #     tier1=["Niels Bohr","Max Planck"]
    tier1 = []
    for entity in path:
        neighbors = kg.one_hop(entity)
        neighbors = [n for n in neighbors
                     if n not in path]
        tier1.extend(kg.get_doc(n)
                     for n in neighbors[:3])

    # Tier-2: BM25 检索 (中等干扰)
    # 规则: 用 question 做 BM25 检索
    # 特征: 词汇重叠但主题偏移
    tier2 = bm25.search(question, topk=100)
    tier2 = [d for d in tier2
             if d not in gold_docs + tier1]

    # 交错排列 + 填充至 128K
    context = interleave_and_pad(
        gold=gold_docs,
        tier1=tier1,
        tier2=tier2,
        target=128_000
    )
    return context

# ====== Rubric Reward 设计 ======
# 不只看最终答案对错，分步骤评分:

def rubric_reward(trace, gold_docs, answer):
    score = 0.0
    # 1. 证据召回: 轨迹中检索到几篇 gold?
    recalled = count_recalled(trace, gold_docs)
    score += recalled / len(gold_docs) * 0.4

    # 2. 推理链完整性: 是否逐步跳转?
    chain_complete = check_chain(trace)
    score += chain_complete * 0.3

    # 3. 最终答案 F1
    score += f1(trace.final_answer, answer) * 0.3
    return score

分层干扰 vs 随机干扰的本质区别 + Rubric Reward 多维评分

Tier-1 为什么有效?

KG 邻居是同一知识领域的实体。例如问"爱因斯坦的质能方程发表年份"，Tier-1 干扰包含"玻尔的原子模型"——同属物理学史，词汇高度重叠，但答案不同。

模型必须精确定位具体事实而非靠主题匹配蒙混。

Rubric Reward 的优势

过程奖励 vs 结果奖励：Binary reward 在 128K 上下文中极度稀疏（大部分轨迹答错）。Rubric 让模型从部分正确的轨迹中也能学习——"找到了 6/8 篇证据但推理出错"仍获得 0.3 分。

Ablation 结果

配置RULER 128K
Random distractor62.3
+ Tier-1 only68.7
+ Tier-1 + Tier-272.1
+ Rubric reward74.8
Binary reward69.4

分层干扰 + Rubric reward 组合贡献了 +12.5 的提升。

ACC: 录制强 agent 轨迹 → 过滤正确的 → 编译为 SFT 数据 → 训练弱模型

核心优势

教"怎么搜"而非"答案是什么"：模型学到的是完整的检索-推理 workflow (search→read→think→refine)，而非死记答案。

轨迹编译格式

输出格式让模型学习结构化行为：

1. 何时搜索（遇到未知信息）

2. 搜什么（query 精炼策略）

3. 如何判断（找到证据 vs 需继续）

4. 何时停止（信息足够回答）

5. LONGBLOCKS — 多任务证据散布框架

# ====== LONGBLOCKS 核心思想 ======
# 问题: 现有方法只覆盖 multi-hop QA
# 但长文本任务不只是"找答案":
#   - 需要定位 (grounding)
#   - 需要生成 (long-form output)
#   - 需要整合 (multi-source synthesis)
#
# 方案: 三类任务统一框架

# ====== 三大任务类型 ======

# Type 1: Multi-Hop Reasoning
# 和 LongTraceRL 类似但更结构化
task_1 = {
    "type": "multi_hop",
    "hops": 3-7,
    "evidence": "scattered in blocks",
    "query": "需整合多处信息回答",
}

# Type 2: Contextual Grounding
# 定位+引用: 找到信息并标注来源
task_2 = {
    "type": "grounding",
    "requirement": "cite exact position",
    "output": "answer + [Block X, Para Y]",
    "skill": "精确引用不hallucinate",
}

# Type 3: Long-Form Generation
# 基于分散材料生成长文本
task_3 = {
    "type": "generation",
    "input": "分散的素材/论据/数据",
    "output": "结构化长文 (1K-3K tokens)",
    "skill": "信息整合+组织+生成",
}

# ====== Block 构造 ======
def create_longblocks_sample(task_type):
    # 1. 选择/生成 evidence blocks
    evidence = generate_evidence(task_type)
    # 2. 生成 distractor blocks
    distractors = generate_distractors(
        evidence, strategy="topical")
    # 3. 随机排列
    blocks = shuffle(evidence + distractors)
    # 4. 生成 task instruction
    instruction = generate_instruction(
        task_type, evidence)
    return {
        "blocks": blocks,
        "instruction": instruction,
        "gold": evidence,
    }

LONGBLOCKS: 三种任务类型共享统一 Block 散布框架，难度可精细控制

为什么需要三类任务?

只训练 QA 是不够的：真实长文本场景包含定位引用（RAG）、长文生成（报告撰写）、信息整合（多源分析）。单一 multi-hop QA 训练会导致其他能力退化。

三类任务详解

Multi-Hop Reasoning: 经典多跳问答，需要整合 3-7 个分散 block 中的信息才能得到答案。类似 LongTraceRL。

Contextual Grounding: 模型必须给出答案并引用来源 block。训练精确引用能力，对抗 hallucination。

Long-Form Generation: 给定分散的素材/论据/数据点，生成结构化长文。训练信息整合和组织能力。

与 LongTraceRL 的区别

维度LONGBLOCKSLongTraceRL
任务类型3 种混合纯 multi-hop QA
训练方式SFT 为主RL (GRPO)
干扰策略Topical分层 Tier
输出格式多样Search 轨迹
侧重全面能力推理深度

组合策略

LONGBLOCKS (全面 SFT) → LongTraceRL (推理 RL) 是互补的：先用 LONGBLOCKS 教会基础长文本能力，再用 LongTraceRL 强化极限推理。
