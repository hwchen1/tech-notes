核心 Insight

真实 agent 检索时遇到的干扰不是随机的——是语义相关但答案错误的文档。

关键突破：用 KG 游走生成可控多跳问题 + 分层干扰模拟真实检索噪声 + 128K 长上下文 + RL 训练。

模型必须在语义高度相似的干扰中准确追踪推理链，而非简单的"大海捞针"。

2. 数据构造 — KG 游走 + 分层干扰

# LongTraceRL 数据合成 Pipeline

# Step 1: KG 随机游走生成多跳路径
kg = KILT_Wikipedia_KG()
path = kg.random_walk(
    start=random_entity(),
    hops=8,   # k=8 跳
)
# path = [e1→e2→...→e8], 每跳对应一个支撑文档

# Step 2: 生成多跳问答
question = llm.generate_question(path)
answer = path[-1].entity

# Step 3: 分层干扰构造
tier1 = []  # 语义近邻 (同 entity 相关页面)
for doc in path.gold_docs:
    tier1 += kg.neighbors(doc, exclude=path)

tier2 = bm25.search(question, topk=50)
tier2 = [d for d in tier2
         if d not in path.gold + tier1]

# Step 4: 填充至 128K
context = interleave(
    gold=path.gold_docs,
    tier1=tier1,
    tier2=tier2,
    target_tokens=128_000
)

数据构造: KG游走 → Gold + 分层干扰 → 128K 上下文 → RL 训练

分层干扰设计

层级来源难度
GoldKG 路径上的 k 篇—
Tier-1Gold 的 KG 邻居高 (同主题)
Tier-2BM25 query 检索中 (词汇重叠)
为什么分层？真实检索 agent 遇到的干扰是分层的：search engine 返回的结果既有主题高度相关的（Tier-1），也有仅关键词匹配的（Tier-2）。模型需要在两种干扰中都保持准确。

核心收益总结

128K 真实长度：首个在 128K tokens 上做 RL 的长文本方法，匹配实际 agent 工作场景。
分层干扰 > 随机干扰：Tier-1 (KG邻居) 模拟"主题相关但答案错误"，Tier-2 (BM25) 模拟"关键词匹配但无关"——两层组合逼近真实搜索引擎返回分布。
真实 Agent 轨迹 RL：不用合成 CoT，直接用 search agent 的检索-阅读-推理轨迹做 GRPO reward，训练信号更接近下游任务。
可控推理深度：KG 游走控制 k=8 跳，确保训练样本的推理链足够长（vs 现有方法 2-4 跳）。
