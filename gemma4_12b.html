1. 核心创新：Encoder-Free 统一多模态

# ====== 关键突破 ======
# Gemma 4 E2B/E4B/31B 都使用
# 独立的 Vision Encoder (SigLIP)
# 12B 完全去掉了外部编码器

# 图像: ~35M 参数的轻量嵌入器
#   48×48 pixel patches
#   一次 matmul 投影
#   + 分解式 X/Y 位置编码
#   → 直接产出 LLM token

# 音频: 波形直接投影到嵌入空间
#   无独立 audio encoder
#   第一个支持原生音频的中型模型

# 视频: 逐帧视觉 token 序列
#   统一在 256K context 内处理

# ====== 为什么这很重要 ======
# 1. 延迟降低: 无需等 encoder 处理
#    LLM 立即开始处理所有 token
# 2. 架构简化: 1 个 transformer = 全部
# 3. 端侧友好: 16GB VRAM 即可运行
# 4. 跨模态对齐: 共享同一表征空间

Encoder-Free: 所有模态通过轻量投影直接进入统一 Transformer

设计哲学

传统多模态模型 (如 LLaVA, Gemma 4 31B) 使用独立 Vision Encoder → Projector → LLM 流水线。12B 版本彻底去掉外部编码器。

Vision Embedder (~35M 参数): 将图像切成 48×48 patches，单次矩阵乘法投影 + 分解式 X/Y 位置查表，生成 soft tokens。

Audio Embedder: 波形直接投影到嵌入空间，无独立 ASR/audio encoder。

统一处理: text/image/audio tokens 交织排列进入同一 transformer，LLM 自身承担所有跨模态推理。

# ====== Per-Layer Embeddings (PLE) ======
# 传统: token → 1 个 embedding → 所有层
# PLE: 每层有独立 embedding 注入
# 允许模型在不同深度引入/细化
# 不同模态的表征

# ====== Shared KV Cache ======
# num_kv_shared_layers 参数:
# 相邻层共享 KV cache
# → 大幅减少推理显存
# → 适配 16GB VRAM 约束

PLE 允许每层独立注入/修改 token 表征; Shared KV Cache 压缩推理显存

Per-Layer Embeddings (PLE)

传统 Transformer: 每个 token 在输入层获得一个 embedding，然后不变地传递到所有层。

PLE: 每层有独立的 embedding 参数，可以在网络不同深度引入新的 token 级信息。对于多模态 token 尤其重要 — 允许视觉/音频 token 在深层获得更精细的表征。

Shared KV Cache

引入 num_kv_shared_layers 超参: 多个相邻层共享同一组 KV 投影结果。推理时不需要为每层独立计算和存储 KV，大幅降低内存占用。

这是 12B 能在 16GB VRAM 上运行的关键技术之一。

3. 多模态处理：Vision + Audio 如何进入 Transformer

# ====== Vision 处理流程 ======
# 1. Pan-and-Scan 动态裁剪
#    输入图像 → 多个 crop 区域
#    适配不同分辨率/纵横比
#
# 2. Patch 化
#    每个 crop → 48×48 pixel patches
#    生成 patch 序列
#
# 3. 投影 (~35M params)
#    patch → matmul → hidden_size(3840)
#    + 分解式位置编码:
#      pos = embed_x(col) + embed_y(row)
#    → 产出视觉 soft tokens
#
# 4. 归一化
#    不使用 ImageNet mean/std
#    patch embedding 层内部处理缩放

# ====== Audio 处理流程 ======
# 波形 → 投影到嵌入空间
# 无需 Whisper/USM 等外部 encoder
# 首个支持原生音频的中型开放模型

# ====== Video ======
# 逐帧 → 视觉 token
# 帧间 token 按时序排列
# 256K context 容纳长视频

Vision 通过 Pan-and-Scan + 48×48 patch + matmul 投影; Audio 直接线性投影; 统一 token 流进入 Transformer

Pan-and-Scan

动态裁剪策略: 将输入图像按不同纵横比裁剪为多个区域，每个区域独立 patch 化。适配各种分辨率，避免 resize 导致的信息损失。

~35M Vision Embedder

对比: SigLIP ViT-SO400M 约 400M 参数。12B 的视觉嵌入器仅 ~35M (减少 10x)，因为重度视觉推理交给 LLM backbone 处理。

原生音频

首个支持原生音频的中型开放模型。波形直接投影，不依赖外部 ASR。支持 speech understanding、音频分类等任务。

vs 其他 Gemma 4

E2B/E4B/31B 使用独立 SigLIP Vision Encoder。12B 是 Gemma 4 家族中唯一的 encoder-free 设计。

12B 在 Pareto 前沿 — 用一半参数超越前代 27B 和同代 26B MoE

为什么超越更大模型

Gemma 4 12B 是 dense 模型 (所有参数都活跃), 而 26B-A4B 是 MoE (仅 4B 参数活跃)。Dense 12B 的每个 token 计算量远大于 26B-A4B。

Agentic 能力

原生支持 Function Calling, 适合构建自主 agent。计划、导航应用、完成多步任务。

256K Context

超越多数端侧模型，适合处理长文档 (财报、代码库)。配合 Shared KV Cache, 长上下文推理不会爆显存。

部署友好

16GB VRAM (FP16) 即可运行，4-bit 量化后仅需 ~8GB。支持 GGUF、MLX、TensorRT-LLM 等推理框架。

为什么 Encoder-Free 只在 12B

可能原因: 12B backbone 足够大，能承担视觉/音频推理而不依赖外部编码器。E2B/E4B 太小，需要 SigLIP 提供视觉特征。31B 发布时间更早，沿用经典架构。
