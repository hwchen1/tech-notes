2024-2026 年长上下文训练数据合成方法综述 — 从文档拼接到多跳证据散布 + RL

1. 方法全景：四大合成范式

# ====== 长文本合成数据的核心挑战 ======
# 真实长文本数据稀缺且标注昂贵
# 需要合成数据来训练长上下文能力
#
# ====== 四大范式 ======
#
# 范式 1: 文档拼接/聚合
#   多篇短文档 → 按相关性拼接成长文档
#   代表: Quest, LongAlign
#
# 范式 2: Self-Synthesis / Agent 生成
#   LLM 自动生成长上下文指令数据
#   代表: BYOCL, LongMagpie, WildLong
#
# 范式 3: 证据散布 + 多跳推理
#   关键证据分散插入 + 复杂 query
#   代表: IN2, KeyChain, LongTraceRL
#
# 范式 4: 层次化 / 模块化框架
#   可组合模块生成超长序列
#   代表: Hierarchical SDG, Modular
#
# ====== 趋势 ======
# 2024 初: 简单拼接为主
# 2024 下: 多跳 + 证据散布兴起
# 2025:    RL + Agent 轨迹成为主流
# 2026:    模块化框架 + 百万 token

四大范式: 从简单拼接到证据散布+RL，合成难度和训练效果递增

核心问题

长上下文能力训练需要大量高质量长文本数据，但真实标注极为昂贵。合成数据是唯一可扩展路径。

趋势演进

2024 初: 以文档拼接为主 (LongAlign), 简单有效但难度有限。

2024 下半年: 多跳证据散布兴起 (IN2), 开始关注"定位多处分散信息"能力。

2025: RL 驱动 (LoongRL, LongTraceRL) + Agent 轨迹编译 (ACC) 成为主流，训练信号更强。

2026: 模块化框架支持百万 token 级合成。

文档拼接局限: 虽然简单有效, 但拼接的文档间缺乏真正的跨文档推理需求, 模型可能只学到"长输入容忍"而非"长程推理"。

拼接类适合冷启动; Self-Synthesis 适合规模化生成多样数据

LongAlign (清华, EMNLP 2024)

首个系统化长上下文对齐方案: Self-Instruct 生成 10K 长指令样本, packing 训练 + sorted batching + loss 加权避免长序列被短序列稀释。

Quest (2024.05)

核心洞察: 相似 query 检索的文档天然相关但低冗余。按 query 相似度聚合文档, 生成需要整合多文档信息的 QA。

ProLong (ACL 2025)

关键发现: 长上下文训练的数据质量 >> 数据量。精心构造的合成数据只需真实数据 5% 的 token 量就能超越 Llama-3.1-8B。

BYOCL (2024.12)

Agent workflow: 多步骤自动构造长上下文场景 + 对应任务, 无需人工标注。

3. 范式 C：证据散布 + 多跳推理（面向长程任务）

# ====== 核心目标 ======
# 给定长上下文, 构造需要定位
# 多处分散证据才能回答的 query
# → 训练模型的"主动搜索+整合"能力

# ====== IN2 (NeurIPS 2024) ======
# "Make Your LLM Fully Utilize Context"
# 合成方式:
#   1. 选取关键证据段 (~128 tok)
#   2. 随机插入长文档不同位置
#   3. 生成需 多处证据 才能回答的 Q
#   4. 文档长度: 4K → 32K tokens
# 效果: 显著提升 lost-in-the-middle

# ====== KeyChain / LoongRL (2025.10) ======
# 合成方式:
#   1. 短 multi-hop QA 作为种子
#   2. 为每个 hop 生成 UUID chain
#      fact_1 → UUID_A → fact_2 → UUID_B
#   3. 将 facts+UUID 分散嵌入干扰文档
#   4. 用 RL 训练: 奖励 = 正确回答
# 难度可控: hop 数 / 干扰量 / 距离

# ====== LongTraceRL (2025.06) ======
# Tiered Distractor Construction:
#   Level 1: 主题高度相关干扰
#   Level 2: 部分相关干扰
#   Level 3: 完全无关干扰
# + Rubric Reward: 分步骤给分
# + Search Agent 轨迹作为训练信号

KeyChain: 短多跳 QA → UUID 链 → 散布干扰文档 → RL 训练长程推理

IN2 (NeurIPS 2024)

"Make Your LLM Fully Utilize the Context" — 首个系统性解决 lost-in-the-middle 问题的合成方法。证据随机插入，强迫模型全文搜索。

KeyChain / LoongRL

UUID chain 是巧妙设计: 模型必须先找到 Fact₁ 中的 UUID_A，再用 UUID_A 定位 Fact₂，逐步跳转。难度可精确控制。

LongTraceRL

Tiered Distractors 是关键创新: 高相关干扰比随机干扰难得多。分层构造让模型学会在"看起来相关但不是答案"的噪声中精准定位。

LONGBLOCKS

覆盖三类任务: multi-hop reasoning + contextual grounding + long-form generation。证据分散在不同 blocks。

ACC (Agent Trajectory)

将搜索 agent 的多步定位轨迹编译为直接监督信号。模型学到的不只是答案，还有"如何搜索"。

最佳实践 (2024.09 论文)

关键发现: 3-5 hop 最优, 证据位置要均匀分散, hard negatives (主题相关但无答案) 远比 random negatives 有效。

选型: 基础容忍用拼接, 多样化用 Self-Synthesis, 长程推理用证据散布+RL

选型建议

如果目标是"让模型接受长输入": 文档拼接足够，成本低，适合冷启动。

如果目标是"长文本上的多样任务": Self-Synthesis (LongMagpie, BYOCL)，需要一个强种子模型。

如果目标是"跨位置定位多证据推理": 证据散布 + RL (KeyChain, LongTraceRL, IN2)，效果最好但构造成本高。

组合策略 (推荐)

1. Phase 1: 大量拼接数据做 continual pretraining 扩展 ctx window

2. Phase 2: Self-Synthesis 做 SFT 教会模型处理多样任务

3. Phase 3: 证据散布+RL 做精细化训练, 强化长程推理
