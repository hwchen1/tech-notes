Activation Functions

常用激活函数对比 — 从 ReLU 到 SwiGLU 的演进

1. ReLU

# ReLU: Rectified Linear Unit
# f(x) = max(0, x)

def relu(x):
    return torch.max(x, 0)

# 导数:
# f'(x) = 1  if x > 0
# f'(x) = 0  if x < 0
# x=0 处不可导
最经典的激活函数

+ 计算极快：一次比较操作
+ 缓解梯度消失（正区间梯度恒=1）
+ 稀疏激活，正则化效果

− Dead neuron：x<0 梯度=0，永远不更新
− 输出非零中心
− x=0 不可导（硬拐角）

用于：CNN（ResNet, VGG）

2. Leaky ReLU

# Leaky ReLU
# f(x) = x       if x > 0
# f(x) = α·x     if x ≤ 0
# 通常 α = 0.01

def leaky_relu(x, alpha=0.01):
    return torch.where(x > 0, x, alpha * x)

# 变体: PReLU — α 可学习
# 变体: ELU — 负区间用指数曲线
解决 Dead Neuron

+ 负区间仍有梯度 (α)，不会"死亡"
+ 计算同样简单
+ PReLU 让 α 可学习，更灵活

− α 是超参数需要调
− 仍然非光滑 (x=0 有拐角)
− 实践中改进通常很小

用于：GAN (DCGAN), 目标检测

3. GELU

# GELU: Gaussian Error Linear Unit
# f(x) = x · Φ(x)
# Φ(x) = 标准正态 CDF

# Tanh 近似 (PyTorch 默认):
def gelu(x):
    return 0.5 * x * (
        1 + torch.tanh(
            math.sqrt(2/math.pi) *
            (x + 0.044715 * x**3)
        )
    )

# 直觉: x 越大 → Φ(x)≈1 → 保留
#       x 越小 → Φ(x)≈0 → 抑制
概率门控：x 自己决定保留多少

+ 处处光滑可导，梯度流稳定
+ 负值区域保留微弱信号
+ 非单调性增强表达能力
+ 自门控，无额外参数

− 计算比 ReLU 贵 (~4x)
− tanh 近似有微小精度差

用于：BERT, GPT-2/3, ViT

4. SiLU / Swish

# SiLU = Swish (Google Brain, 2017)
# f(x) = x · σ(x)
# σ = sigmoid

def silu(x):
    return x * torch.sigmoid(x)

# 对比 GELU:
# GELU = x · Φ(x)   Φ=正态CDF
# SiLU = x · σ(x)   σ=sigmoid
# 形状几乎一样！
# 两者差异 < 1% (在常见范围内)

# 通用形式: Swish-β
# f(x) = x · σ(βx)
# β→∞ 退化为 ReLU
# β=1 即 SiLU
Sigmoid 门控版本

+ 和 GELU 几乎等价，更简单的公式
+ 自门控：σ(x) 作为门控权重
+ Google NAS 搜索出的最优激活
+ 光滑、非单调

− 需要计算 sigmoid（一次 exp）
− 负区间最低 ≈ -0.28 (可能不稳定)

用于：EfficientNet, LLaMA (作为 SwiGLU 的一部分)

5. SwiGLU

# SwiGLU (Shazeer, 2020)
# GLU: Gated Linear Unit
# f(x) = (xW₁) ⊙ act(xV)
# SwiGLU 用 SiLU 作为 act

class SwiGLU(nn.Module):
    def __init__(self, d, d_ff):
        # 注意: 3 个投影矩阵!
        self.W1 = nn.Linear(d, d_ff)  # gate
        self.V  = nn.Linear(d, d_ff)  # value
        self.W2 = nn.Linear(d_ff, d)  # down

    def forward(self, x):
        return self.W2(
            F.silu(self.W1(x)) * self.V(x)
        )

# 参数量: 3·d·d_ff (vs 2·d·d_ff)
# 所以 d_ff 通常设为 2/3 × 4d
#   = 8d/3 ≈ 2.67d
显式门控：分离 gate 和 value

+ 比 GELU FFN 效果一致性更好
+ 门控路径和值路径解耦，更强表达力
+ PaLM 论文验证：同参数量下 loss 更低

− 多一个投影矩阵 (3 vs 2)
− 为保持参数量需缩小 d_ff

演进：
GELU FFN: W₂·GELU(W₁·x)
→ 隐式门控 (x 自己门控自己)

SwiGLU: W₂·[SiLU(xW₁)⊙xV]
→ 显式门控 (两个独立路径)

用于：LLaMA, Qwen, Mistral, Gemma

6. 总结对比

激活函数 公式 光滑 门控 计算成本 典型用途
ReLU max(0, x) ❌ ❌ 最低 CNN (ResNet, VGG)
Leaky ReLU max(αx, x) ❌ ❌ 最低 GAN, 目标检测
GELU x·Φ(x) ✅ 隐式 中 BERT, GPT, ViT
SiLU/Swish x·σ(x) ✅ 隐式 中 EfficientNet
SwiGLU SiLU(xW₁)⊙xV ✅ 显式 高 (3矩阵) LLaMA, Qwen, Mistral
演进主线：ReLU(硬截断) → GELU/SiLU(平滑自门控) → SwiGLU(显式双路径门控)。核心趋势：从"硬开关"走向"软概率门控"，从"隐式"走向"显式"分离 gate 和 value。

7. 选择指南

# 2024+ LLM 标配
# FFN with SwiGLU:
d_model = 4096
d_ff = int(8 * d_model / 3)  # 10922
# 对齐到 128 (tensor core)
d_ff = (d_ff + 127) // 128 * 128  # 11008

# LLaMA-7B: d=4096, d_ff=11008
# 参数: 3 × 4096 × 11008 = 135M per layer

# 对比 GELU FFN:
# d_ff = 4 × d = 16384
# 参数: 2 × 4096 × 16384 = 134M per layer
# → 参数量几乎一样，SwiGLU 效果更好
实用选择：

• 新 LLM → SwiGLU (无脑选)
• CNN / 轻量模型 → ReLU
• BERT-style 编码器 → GELU
• GAN → LeakyReLU

d_ff 调整公式：
SwiGLU 需要 3 个矩阵，为保持参数量不变：
d_ff = ⌈8d/3⌉ 对齐到 128

这样 3×d×d_ff ≈ 2×d×4d
