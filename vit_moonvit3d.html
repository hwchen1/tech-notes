1. 3D Patch Embedding

# MoonViT3d 核心: Conv3d patch embedding
# 输入: video frames [T, H, W, 3]

patch_embed = Conv3d(
    in_channels  = 3,
    out_channels = 1536,   # hidden_dim
    kernel_size  = (2, 14, 14),  # t,h,w
    stride       = (2, 14, 14),
)

# 时序维度: 每 2 帧合为 1 token
# 空间维度: 14×14 patch
# 输出 tokens: T/2 × H/14 × W/14

# 位置编码: 3D RoPE
# 分别对 t, h, w 三维编码
rope_t = RoPE(dim // 4)   # 时间
rope_h = RoPE(dim // 4)   # 高度
rope_w = RoPE(dim // 2)   # 宽度(更多)

3D 时空分词

与 2D ViT 不同，MoonViT3d 用 3D 卷积同时处理时间和空间维度，每 2 帧 × 14×14 patch 生成一个 token。

图片作为单帧视频处理（T=1），无需额外分支。

$\text{tokens} = \frac{T}{2} \times \frac{H}{14} \times \frac{W}{14}$ $\text{RoPE dims: } [t: d/4,\; h: d/4,\; w: d/2]$

为什么 3D

原生建模帧间运动关系，而非 2D ViT + 后融合的两阶段方案。时序信息从第一层就参与计算。

2. Transformer 结构

MoonViT3d 采用现代 LLM 组件（SwiGLU + RMSNorm + RoPE）构建视觉编码器，与 LLM 架构统一

$\text{Config: } d{=}1536,\; h{=}24,\; L{=}28,\; \text{head\_dim}{=}64$ $\text{SwiGLU: } \text{FFN}(x) = (\text{Linear}_1(x) \odot \text{SiLU}(\text{Linear}_g(x))) \cdot \text{Linear}_2$ $\text{Total Params} \approx 600\text{M}$
