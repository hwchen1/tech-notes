Chameleon — Early Fusion Multimodal

Meta | Early Fusion | Unified Token Space | VQGAN Image Tokenizer | 7B & 34B | Mixed-Modal Generation

1. 核心理念 — Early Fusion 统一 Token 空间

# Chameleon: Early Fusion 统一多模态
# 核心: 所有模态 → 离散 token → 单一 Transformer

# Text tokenization (BPE)
text_tokens = bpe_tokenizer(text)
# vocab size: 65536 (text)

# Image tokenization (VQGAN)
image_tokens = vqgan_encoder(image)
# vocab size: 8192 (image codebook)

# 统一词表
# V = V_text ∪ V_image = 65536 + 8192
unified_vocab_size = 65536 + 8192
# 加上特殊 token: BOI, EOI 等

# 序列构建: 直接拼接
# 没有模态标记的本质区别!
sequence = [
    *text_tokens,       # BPE ids
    BOI_TOKEN,          # image 开始
    *image_tokens,      # VQGAN codes
    EOI_TOKEN,          # image 结束
    *text_tokens_2,     # 继续 text
]

# 单一 Transformer Decoder
# 没有 ViT, 没有 adapter, 没有 cross-attn
class Chameleon(nn.Module):
    def __init__(self):
        self.embed = nn.Embedding(
            unified_vocab_size, d_model
        )
        self.transformer = TransformerDecoder(
            layers=32,  # 7B config
            d_model=4096,
            heads=32,
        )
        self.lm_head = nn.Linear(
            d_model, unified_vocab_size
        )

    def forward(self, tokens):
        # tokens 混合了 text 和 image ids
        x = self.embed(tokens)
        x = self.transformer(x)
        logits = self.lm_head(x)
        return logits
        # NTP loss 统一覆盖两种模态!

Early Fusion 核心

关键洞察: 推理时不需要独立的视觉 encoder。所有信息都是统一词表中的离散 token，共享同一个 Transformer。

$V = V_{\text{text}} \cup V_{\text{image}} = 65536 + 8192$

对比 Late Fusion (LLaVA)

Late Fusion: ViT 产出连续特征 → projector → LLM。视觉 encoder 始终存在。

Early Fusion (Chameleon): Image → 离散 token → 直接进入同一个 Transformer。没有额外 encoder。

优势

1. 真正的混合模态生成 — 可以交替输出 text 和 image tokens

2. 架构极简 — 只有一个模型

3. 训练统一 — NTP loss 覆盖所有模态

代价

1. Image tokenizer 质量成为瓶颈

2. 图像需要 1024 tokens (32x32)，序列很长

3. 混合模态训练不稳定

2. Image Tokenizer (Make-A-Scene VQGAN)

# VQGAN Image Tokenizer
# 基于 Make-A-Scene 的 VQGAN
# 将连续图像转为离散 token

# 编码过程
class VQGANEncoder(nn.Module):
    def __init__(self):
        self.encoder = CNNEncoder(
            in_channels=3,
            hidden_dims=[128, 256, 512],
            downsample_factor=16,
            # 512/16 = 32 → spatial 32x32
        )
        self.codebook = nn.Embedding(
            num_embeddings=8192,  # codebook size
            embedding_dim=256,
        )

    def encode(self, image):
        # image: [B, 3, 512, 512]
        z = self.encoder(image)
        # z: [B, 256, 32, 32]

        # 量化: 找最近邻 codebook entry
        z_flat = z.permute(0,2,3,1).reshape(-1, 256)
        # [B*1024, 256]

        distances = torch.cdist(z_flat, self.codebook.weight)
        indices = distances.argmin(dim=-1)
        # indices: [B*1024] — 离散 token ids!

        return indices.reshape(-1, 1024)
        # [B, 1024] = 32x32 spatial grid

# 解码过程 (生成图像时)
class VQGANDecoder(nn.Module):
    def decode(self, indices):
        # indices: [B, 1024]
        z_q = self.codebook(indices)
        # [B, 1024, 256]
        z_q = z_q.reshape(-1, 32, 32, 256)
        z_q = z_q.permute(0, 3, 1, 2)
        # [B, 256, 32, 32]

        image = self.decoder_cnn(z_q)
        # [B, 3, 512, 512]
        return image

# 训练: 重建 loss + GAN loss + 感知 loss
# loss = L_recon + L_GAN + L_perceptual
# + codebook commitment loss

VQGAN 关键参数

参数值
Codebook size8192
Tokens per image1024 (32x32)
Input resolution512x512
Downsample factor16x
Embedding dim256

设计权衡

Codebook 越大: 重建质量越高，但 vocabulary 膨胀，LLM 建模难度增加

Spatial grid 越大: 细节保留越多，但序列越长 (1024 tokens/image 已经很长)

$\text{tokens} = \left(\frac{512}{16}\right)^2 = 32^2 = 1024$

与其他方案对比

DALL-E: dVAE, 8192 codebook, 256 tokens (16x16)

Chameleon: VQGAN, 8192 codebook, 1024 tokens (32x32) — 更高分辨率

关键: Tokenizer 在 Chameleon 训练前 freeze，不参与端到端优化。图像生成质量的上限由 tokenizer 重建能力决定。

# === 稳定性 Tricks ===

# 1. QK-Norm: 防止 attention logit 爆炸
def qk_norm_attention(Q, K, V, tau=1.0):
    # 归一化 Q 和 K
    Q_hat = Q / Q.norm(dim=-1, keepdim=True)
    K_hat = K / K.norm(dim=-1, keepdim=True)
    # tau 是可学习的温度参数
    attn = softmax(Q_hat @ K_hat.T / tau)
    return attn @ V

# 2. Z-loss: 防止 softmax 坍缩
def z_loss(logits, weight=1e-4):
    # 惩罚 log(sum(exp(logits)))^2
    # 防止某些 logit 过大导致坍缩
    log_z = torch.logsumexp(logits, dim=-1)
    return weight * (log_z ** 2).mean()

# 3. Dropout on attention (仅训练时)
# 帮助正则化跨模态交互
attn_dropout = 0.1  # in multimodal stages

# 4. 为什么需要这些?
# 混合模态训练时:
# - image tokens 和 text tokens 分布差异大
# - attention logits 容易发散
# - softmax 容易 collapse 到单一 token
# QK-norm + z-loss 联合解决这些问题

训练不稳定性根源

问题: 当同一个模型同时建模 text 和 image tokens 时，两种模态的梯度分布差异很大，容易导致:

1. Attention logit 爆炸 (因为跨模态 QK 内积不受控)

2. Softmax 坍缩到极少数 token (输出概率分布退化)

QK-Norm

$\text{Attn}(Q,K) = \text{softmax}\left(\frac{\hat{Q}\hat{K}^T}{\tau}\right)$ $\text{where } \hat{Q} = \frac{Q}{\|Q\|}, \quad \hat{K} = \frac{K}{\|K\|}$

归一化后 QK 内积有上界 (余弦相似度 [-1, 1])，防止极端值。$\tau$ 是可学习温度。

Z-Loss

$\mathcal{L}_z = \alpha \cdot \mathbb{E}\left[(\log \sum_i e^{z_i})^2\right]$

惩罚 logits 的 partition function，防止某些 logits 过大导致 softmax 退化。$\alpha \approx 10^{-4}$。
