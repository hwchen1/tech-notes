1. 核心架构：双池异构 MoE (Dual-Pool Heterogeneous MoE)

# ERNIE 4.5 VL 最核心设计:
# 文本 token 和视觉 token 走不同的专家池

class ErnieMultiTypeMoE(nn.Module):
    def __init__(self, config):
        # 三组专家并行运行:
        self.text_moe = MoELayer(
            num_experts=64,
            ffn_hidden=1536,   # 较大
            top_k=6,
        )
        self.vision_moe = MoELayer(
            num_experts=64,
            ffn_hidden=512,    # 较小
            top_k=6,
        )
        self.shared_experts = SharedMLP(
            num_experts=2,
            ffn_hidden=3072,   # 最大
        )

    def forward(self, x, token_type_ids):
        # 按模态分流
        text_mask = (token_type_ids == 0)
        vis_mask = (token_type_ids >= 1)

        # 文本 → 文本专家池
        text_out = self.text_moe(x[text_mask])
        # 视觉 → 视觉专家池
        vis_out = self.vision_moe(x[vis_mask])
        # 所有 token → 共享专家
        shared_out = self.shared_experts(x)

        # 合并输出
        out = zeros_like(x)
        out[text_mask] = text_out
        out[vis_mask] = vis_out
        out += shared_out  # 叠加共享
        return out

# 路由: sigmoid scoring (同 DeepSeek V3)
# + Expert Bias 做 aux-free load balancing
# score = sigmoid(W·x) + bias (仅用于选择)
# weight = sigmoid(W·x) (不含 bias)

# 整体参数量:
# 28B total, 3B active per forward
# Layer 0: Dense MLP (无 MoE)
# Layer 1-28: ErnieMultiTypeMoE

为什么要双池？

文本和视觉 token 的信息密度、特征分布、FFN 需求截然不同。ERNIE 用模态分治策略：

组件参数服务对象
Text Pool64 experts × ffn 1536仅文本 token
Vision Pool64 experts × ffn 512仅视觉 token
Shared2 experts × ffn 3072所有 token
$\text{output} = \text{MoE}_{\text{pool}}(x) + \text{Shared}(x)$ $\text{pool} = \begin{cases} \text{text\_moe} & \text{if type\_id} = 0 \\ \text{vision\_moe} & \text{if type\_id} \geq 1 \end{cases}$

设计洞察：视觉专家 ffn 只有 512 (文本的 1/3)，因为视觉 token 经过 ViT + Resampler 后已高度压缩，不需要同等规模的 FFN 进一步变换。共享专家 ffn=3072 则做跨模态融合。

路由策略 (同 DeepSeek V3)：Sigmoid 独立打分 + Expert Bias 做无 aux-loss 的负载均衡。Bias 只在 top-k 选择时加入 (影响哪些专家被选中)，计算权重时不加 (保持梯度信号纯净)。Top-6 选 64 个专家 → 激活率 9.4%。

2. Interleaved 3D M-RoPE 位置编码

# ERNIE 使用自定义的 interleaved 3D M-RoPE
# 与 Qwen2-VL 的 contiguous 布局不同!

# 配置: head_dim=128, 64 frequency bands
# mrope_section = [22, 22, 20]  # H, W, T

# === Qwen2-VL: contiguous block 布局 ===
# bands[0:22]  → T (temporal)
# bands[22:44] → H (height)
# bands[44:64] → W (width)
# 每个维度连续占据一段 bands

# === ERNIE: interleaved 布局 ===
# even bands [0,2,4,...,42] → H (22 bands)
# odd  bands [1,3,5,...,43] → W (22 bands)
# bands [44:64]            → T (20 bands)
# H 和 W 交错排列!

class ErnieMultimodalRotaryEmbedding:
    def __init__(self, config):
        self.head_dim = 128
        self.mrope_section = [22, 22, 20]
        self.rope_base = 500000.0
        # rotary_interleaved = True

    def get_position_ids(self, input_ids, images):
        # 文本 token: T = H = W (三维相同)
        # 图像 token: T=frame_idx, H/W=grid坐标
        # 边界 token (IMAGE_START/END):
        #   RoPE: 当作文本 (T=H=W)
        #   MoE routing: 当作视觉!
        pos_ids = zeros(3, seq_len)
        for tok in input_ids:
            if tok.is_text:
                pos_ids[:, i] = [t, t, t]
            elif tok.is_image:
                pos_ids[:, i] = [h, w, t]
        return pos_ids

# 为什么 interleaved 更好？
# 直觉: H 和 W 交错 → 空间信息在频率域
# 更均匀分布，避免某些 band 只编码空间
# 某些 band 只编码时间的割裂

# 数学等价性:
# 对纯文本 (T=H=W), 两种布局结果完全相同
# 差异只在多模态 token (H≠W≠T) 时体现

ERNIE 和 Qwen2-VL 都使用 3D Multi-modal RoPE，但频率分配策略不同：

$\text{RoPE}(x, m) = x \cdot \cos(m\theta) + \text{rotate}(x) \cdot \sin(m\theta)$ $\theta_i = \text{base}^{-2i/d}, \quad \text{base} = 500000$ $\text{ERNIE band } i: \begin{cases} m = h_{\text{pos}} & i \text{ even}, i \lt 44 \\ m = w_{\text{pos}} & i \text{ odd}, i \lt 44 \\ m = t_{\text{pos}} & i \geq 44 \end{cases}$
LM backbone 参数：GQA 20 query heads / 4 KV heads, head_dim=128, RMSNorm, SiLU-gated MLP, 无线性 bias, hidden_dropout=0。与 DeepSeek V3 的 attention 配置接近，差异在 MoE 和 RoPE。

3. 视觉管线：DFN ViT + Variable-Resolution Resampler

# ERNIE ViT: DFN-style (不是 CLIP/SigLIP)
# 32 层, embed_dim=1280, 16 heads

class DFNRopeViT(nn.Module):
    config = {
        'depth': 32,
        'embed_dim': 1280,
        'num_heads': 16,
        'head_dim': 80,       # 非标准!
        'mlp_ratio': 4,       # ffn=5120
        'patch_size': 14,
        'activation': 'quick_gelu',
        'norm': 'LayerNorm',  # 非 RMSNorm!
        'bias': True,        # QKV+MLP 都有 bias
    }

# 特殊: Patch Embed 用 Linear 而非 Conv2d
# 输入已被 processor 预压平:
# [B, N_patches, C*P*P] = [B, N, 588]
class PatchEmbed(nn.Module):
    def __init__(self):
        self.proj = nn.Linear(588, 1280, bias=False)

# ViT 内部: Absolute 2D RoPE (非 interleaved)
# 注意: 与 LM 的 interleaved style 不同!
# ViT 用 rotate_half, LM 用 interleaved

# === Resampler: 变分辨率空间+时间池化 ===
class VariableResolutionResampler(nn.Module):
    def __init__(self):
        # 空间: 2×2 merge → 4 patches 合 1
        self.spatial_linear = MLP(
            1280 * 4, 1280 * 4, 2560
        )  # in=5120 → hidden=5120 → out=2560
        # 时间: 帧间合并 (视频)
        self.temporal_linear = MLP(...)
        # 最终投影到 LM hidden_size
        self.mlp = MLP(2560, 2560, 2560)

    def forward(self, vit_out, grid_h, grid_w):
        # 2×2 spatial merge
        x = rearrange(vit_out,
            '(h w) d -> (h/2 w/2) (4*d)')
        x = self.spatial_linear(x)
        # 时间 merge (如果是视频)
        x = self.temporal_linear(x)
        return self.mlp(x)

ViT + Resampler 设计

ERNIE 视觉管线的三个非标准选择：

选择ERNIE常见做法
Patch EmbedLinear(588→1280)Conv2d(3,d,14,14)
ViT 激活quick_geluGELU / SiLU
ViT NormLayerNorm + biasRMSNorm no-bias
head_dim8064 / 128
像素归一化模型 forward 中Processor 中
$\text{quick\_gelu}(x) = x \cdot \sigma(1.702 \cdot x)$ $\neq \text{gelu\_tanh}(x) = x \cdot \Phi_{\text{tanh}}(x)$

Resampler 空间压缩：2×2 merge 将 4 个相邻 patch 拼接为 1 个 token (5120d → 2560d)，视觉 token 数量降为原来的 1/4。对视频还有额外的帧间 temporal merge。

EP 下的 dummy token 技巧：当 Expert Parallel > 1 时，alltoall 要求所有 rank 参与。如果某个 rank 没有视觉 token，会注入一个 dummy token 到 vision pool，确保集合通信不死锁。文本池同理。这是分布式训练中双池 MoE 的额外工程代价。
