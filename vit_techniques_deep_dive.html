012D-RoPE & mRoPE

位置编码从 1D 到多维的演进。2D RoPE 解决了 ViT 的空间位置建模，mRoPE 进一步统一了文本、图像和视频的时空位置表示。

1.1 背景：为什么需要 2D 位置编码

传统 ViT 使用 1D learnable position embeddings（如原始 ViT、CLIP），将 2D 图像 patch 展平后赋予顺序位置。这种方案的核心问题：

  • 固定分辨率：训练时固定 224×224 或 336×336，推理时不同分辨率需要插值，精度损失
  • 丢失空间结构：相邻 patch 的行列关系被压缩成线性距离
  • 无法泛化：不同长宽比的图像强制 resize 或 padding，破坏原始几何信息

1.3 2D RoPE 数学形式

Standard RoPE (1D):
f(q, m) = q · e^{i·m·θ} where θ = 10000^{-2k/d}

2D RoPE (Seed-ViT):
For patch at grid position (h, w), embedding dim d = 1280, head_dim = 64:

q'[0:32] = q[0:32] · e^{i·h·θ₁} ← height component
q'[32:64] = q[32:64] · e^{i·w·θ₂} ← width component

mRoPE (Qwen2.5-VL), head_dim = 64:
q'[0:16] = q[0:16] · e^{i·t·θ_t} ← temporal (16 dims)
q'[16:40] = q[16:40] · e^{i·h·θ_h} ← height (24 dims)
q'[40:64] = q[40:64] · e^{i·w·θ_w} ← width (24 dims)

Attention score naturally encodes 3D distance:
Attn(i,j) ∝ exp[ q_t·k_t·cos(Δt·θ) + q_h·k_h·cos(Δh·θ) + q_w·k_w·cos(Δw·θ) ]

1.4 关键设计差异

2D RoPE (Seed-ViT)

  • Head dim 64，均分 32+32（height + width）
  • 从 MIM 阶段即开始使用，学生用 2D RoPE、教师用 learnable PE，不影响蒸馏效果
  • 天然支持任意分辨率：图像 resize 到 28 的倍数 → patch 14×14 → 不同 H/W grid
  • 无需插值：新分辨率直接用对应整数位置
  • 视频处理：通过 timestamp token 补充时间信息

mRoPE (Qwen2.5-VL)

  • Head dim 64，分为 16+24+24（T + H + W）
  • 时间维 ID 与真实时间戳对齐（非帧序号）
  • 文本时 T 递增、H/W 不变 → 退化为 1D RoPE
  • 图像时 T 不变、H/W 按 patch 位置 → 退化为 2D RoPE
  • 视频时三维全激活 → 同时编码"什么时候、在哪里"
  • 支持 YaRN 扩展到超长上下文（32K → 更长）
Seed-ViT 的发现：MIM 蒸馏中教师（EVA02-CLIP-E）使用 learnable position embedding，学生使用 2D RoPE，两者位置编码不一致并不影响训练效果。这证明 RoPE 的相对位置编码特性可以很好地学习空间结构，无需与教师的绝对编码对齐。
MoonViT 的混合方案：同时保留 SigLIP-SO-400M 原始的 learnable fixed-size absolute positional embeddings（通过插值适配新分辨率），并额外引入 2D RoPE。两种位置编码共同工作，前者保留预训练知识，后者提供高分辨率下的精细位置信息。

3.2 QK-Norm：训练稳定性保障

CogViT 引入 QK-Norm：在计算 attention 前对 Query 和 Key 向量做归一化。

Standard Attention:
Attn = softmax(Q·K^T / √d)

With QK-Norm:
Q' = LayerNorm(Q) or Q' = Q / ||Q||
K' = LayerNorm(K) or K' = K / ||K||
Attn = softmax(Q'·K'^T / √d)

Why needed:
- 大规模训练中 Q/K 向量范数会逐渐增大（logit explosion）
- 导致 attention 权重过于尖锐（near one-hot），梯度消失
- QK-Norm 将内积约束在有界范围内，确保训练稳定
实验结果（GLM-5V-Turbo 论文 Figure 1）：CogViT-L (403M) 在 ImageNet-1K zero-shot 上达到 83.5%，超过参数更多的 SigLIP2-SO (427M, 83.3%)、DFN-H (632M, 83.4%) 和 MetaCLIP2-H (632M, 79.8%)。双教师蒸馏 + QK-Norm 的组合在更少参数下实现了更优性能。

NaFlex 实现原生可变分辨率输入，SigLIP 提供高效的对比学习目标。两者结合是 CogViT Stage 2 和 Qwen2.5-VL 训练的关键。

4.2 NaFlex 的实现机制

NaFlex 的核心思想是让 ViT 直接处理原始尺寸的图像（对齐到 patch size 的倍数），不同图像产生不同数量的 token：

Token count calculation:
For image of size H×W with patch_size=14:
n_h = ceil(H / 14), n_w = ceil(W / 14)
num_tokens = n_h × n_w

Batch handling (sequence packing):
Multiple images with different token counts are packed into a single sequence:
[img1_tokens | img2_tokens | img3_tokens | ...]
Attention masks ensure cross-image isolation

CogViT Stage 2 specific:
- Stage 1 used fixed 224×224 → Stage 2 switches to NaFlex
- Preserves original aspect ratios
- Combined with SigLIP loss at batch_size = 64K

4.5 MoonViT 的 CoCa Loss 组合

MoonViT 使用 CoCa (Contrastive Captioners) 的联合训练目标：

MoonViT Total Loss:
L = L_siglip + λ · L_caption (λ = 2)

L_siglip (contrastive branch):
- Image encoder + text encoder compute embeddings
- Sigmoid-based pairwise contrastive loss
- Forces global alignment between image and text

L_caption (generative branch):
- Text decoder performs NTP conditioned on image features
- Cross-entropy loss on next-token prediction
- Forces detailed understanding (not just matching)

Why λ=2 (caption weighted higher)?
- Caption loss drives fine-grained understanding (OCR, counting, spatial relations)
- Contrastive loss alone only captures coarse semantics
- During training, OCR capability emerged when scaling up caption loss
- The text decoder is initialized from a tiny decoder-only LM
MoonViT 训练中的涌现现象：在 CoCa-style 训练中，随着 OCR 数据规模增大，caption loss 分支自发产生了 OCR 能力——文本解码器学会了从图像特征中"读出"文字内容。这种能力并非显式训练目标，而是 contrastive + generative 联合训练的涌现效应。
