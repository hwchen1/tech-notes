为什么能超越 Full Attention？

三个技术突破叠加：

1. Channel-wise decay：把 GDN 的 scalar α 升级为 per-channel vector。每个特征维度有独立的衰减速率 — 高频信息快速遗忘，低频信息长期保持。
2. DPLR 高效 chunk-wise：Diagonal-Plus-Low-Rank 转移矩阵的特化实现，比通用 DPLR 计算量大幅减少，同时保持与经典 delta rule 的一致性。
3. KDA + MLA 混合：大多数层用 KDA (高效，长距离)，少量关键层用 MLA (精确 recall)。最终整体效果 > 纯 MLA。

2. KDA 机制详解

# KDA 状态更新 (每个 head)
# 输入: x_t → q_t, k_t, v_t (线性投影)

# Channel-wise decay:
# α_t = sigmoid(W_α · x_t) ∈ (0,1)^d
# 每个维度 m 有独立的遗忘率 α_{t,m}

# Delta rule erase/write:
# β_t = sigmoid(W_β · x_t) ∈ (0,1)
# scalar, 控制擦写强度

# 完整更新:
# S_t = diag(α_t) · S_{t-1}
#     - β_t · k_t · (k_t^T · S_{t-1})^T
#     + k_t ⊗ v_t

# 读取:
# o_t = q_t^T · S_t

# ====== 关键: DPLR 转移矩阵 ======
# 状态转移: A_t = diag(α_t) - β_t · k_t · k_t^T
# 这是 "对角 + 低秩" (DPLR) 结构!
# diag(α_t) = D (对角部分)
# -β_t · k_t · k_t^T = -L·R^T (秩1)

# 利用 DPLR 结构的 chunkwise 算法:
# 1. chunk 内: inter-token 只需 O(C·d²)
# 2. chunk 间: 状态传递 O(d²)
# 总复杂度 O(N·d²) — 与 GDN 相同
# 但实际更快 (利用了对角结构)

KDA 的 channel-wise α 让不同频率的信息有不同的"记忆时长"

DPLR 的计算优势

$A_t = \underbrace{\text{diag}(\alpha_t)}_{D \text{ (对角)}} - \underbrace{\beta_t \cdot k_t k_t^\top}_{\text{秩-1 (低秩)}}$

DPLR 结构允许：

• 对角部分: 逐元素乘法 O(d)

• 低秩部分: 外积更新 O(d²) 但系数只有 1 项

• 比通用矩阵转移 O(d³) 快得多

与通用 DPLR 的区别：一般 DPLR (如 S5/DSS) 转移矩阵是固定参数。KDA 的 DPLR 是 input-dependent — α_t, β_t, k_t 全部由输入决定。这需要特化的 chunkwise 算法。

为什么 hybrid KDA+MLA？

纯 KDA 在大多数任务已超越 MLA，但极端精确 recall (如 exact copy) 场景下 softmax attention 仍更强。

解决方案：少量 MLA 层保证 recall 下限，大量 KDA 层提供效率和长距离能力。

工业影响

Kimi Linear 的意义不仅是学术突破，更是工业级验证：

Drop-in 替换：同训练 recipe、同数据，只换注意力机制 → 质量更高 + 效率更好。证明 Linear Attention 可以直接替换已有系统。
RL 友好：RL scaling 需要大量长输出 rollouts。恒定推理显存让 Kimi Linear 可以跑更多并发 rollouts → 更好的 RL 训练信号。这是 Full Attention 模型做不到的。
开源生态：KDA kernel (Triton) + vLLM 集成 + 预训练权重全部开源。社区可以直接在 vLLM 中部署 KDA 模型。
