1. 核心理念 — 去掉 ViT，图像 patch 直入 LLM

# Fuyu: 最简多模态架构
# 核心: Image → patches → Linear → LLM
# 没有 ViT, 没有 CLIP, 没有 Q-Former

# Step 1: 图像分割为 30x30 patches
patch_size = 30
H, W = image.shape[1], image.shape[2]
patches = split_into_patches(image, patch_size)
# shape: (H//30, W//30, 30*30*3)

# Step 2: 线性投影到 LLM hidden dim
# 唯一的"视觉处理" — 一个线性层!
class PatchProjection(nn.Module):
    def __init__(self, d_model=4096):
        self.proj = nn.Linear(
            30 * 30 * 3,  # 2700
            d_model          # 4096
        )

    def forward(self, patches):
        return self.proj(patches)

# Step 3: 构建序列 (带换行 token!)
# \n token 标记每行结束 → 保留 2D 空间
sequence = []
for row in patch_rows:
    sequence.extend(row)  # 该行的 patches
    sequence.append(NEWLINE_TOKEN)  # \n

# 拼接文本 token
sequence.extend(text_tokens)

# Step 4: 直接输入 decoder-only Transformer
output = transformer(sequence)
# 标准 causal attention, 标准 NTP loss

极简多模态架构

关键洞察: 完全移除视觉编码器 (ViT)。图像被切分为 30x30 patches，每个 patch 经过一个线性层投影到 LLM 隐藏维度，然后直接与文本 token 拼接输入 decoder。

$x_{\text{patch}} = W_p \cdot \text{flatten}(\text{patch}_{i,j}) + b_p \in \mathbb{R}^d$

"\n" Token 的作用

特殊的换行 token 插在每行 patches 末尾，让模型理解 2D 空间结构。序列为:

$\text{seq} = [p_1, p_2, \ldots, p_{W}, \texttt{\textbackslash n}, p_{W+1}, \ldots, \texttt{\textbackslash n}, \ldots, t_1, t_2, \ldots]$

为什么可行?

1. LLM 足够强大 — 大模型可以从 raw pixels 学到视觉特征

2. 线性层保留信息 — 不像 tokenizer 会丢失细节

3. 任意分辨率 — 无需 resize 到固定尺寸

4. 训练简单 — 只需标准 NTP loss

2. 原生可变分辨率

# Fuyu: 原生可变分辨率处理
# 不需要 resize, 不需要 padding, 不需要固定 grid

patch_size = 30

def compute_patch_count(H, W, s=30):
    """计算给定分辨率的 patch 数量"""
    rows = math.ceil(H / s)
    cols = math.ceil(W / s)
    # 总 token 数 = patches + newline tokens
    n_patches = rows * cols
    n_newlines = rows  # 每行一个 \n
    total = n_patches + n_newlines
    return total

# 不同分辨率 → 不同 token 数
# 640x480:
compute_patch_count(480, 640)
# → ceil(480/30) * ceil(640/30) + 16
# → 16 * 22 + 16 = 368 tokens

# 1920x1080 (Full HD):
compute_patch_count(1080, 1920)
# → 36 * 64 + 36 = 2340 tokens

# 3840x2160 (4K):
compute_patch_count(2160, 3840)
# → 72 * 128 + 72 = 9288 tokens

# 对比 ViT 方式 (固定分辨率):
# 总是 resize to 224x224
# → 14x14 = 196 tokens (固定!)
# 所有分辨率都压缩到同样的 token 数
# 大量信息丢失!

# Fuyu 的处理:
def process_image(image):
    # 直接在原始分辨率上操作
    # 无 resize, 无 crop, 无 padding
    H, W = image.shape[1:3]
    patches = unfold(image, patch_size)
    embeddings = linear_proj(patches)
    # 插入换行 token
    seq = insert_newlines(embeddings, cols=W//patch_size)
    return seq

可变分辨率的优势

零信息损失: 图像不经过任何 resize 或 crop，原始像素完整保留。这对文档解析、UI 截图等细粒度任务至关重要。

$N_{\text{patches}} = \lceil H/s \rceil \times \lceil W/s \rceil$, where $s=30$

Token 数计算

加上换行 token:

$N_{\text{total}} = \lceil H/s \rceil \times \lceil W/s \rceil + \lceil H/s \rceil$

天然适合的场景

1. 文档/PDF — 需要完整分辨率才能看清文字

2. UI 截图 — 按钮、菜单需要精确位置

3. 图表/表格 — 数据密集，不能丢失细节

代价

Token 数线性增长: 大图产生大量 token，计算代价为 $O(N^2)$ (self-attention)。实际部署中需要对最大分辨率做限制。

# Fuyu: 训练极其简单
# 标准自回归语言建模 loss
# 没有对比 loss, 没有重建 loss
# 没有特殊的视觉预训练阶段!

class FuyuTraining:
    def __init__(self):
        # 骨干: Persimmon-8B (Adept 自研 LLM)
        self.backbone = Persimmon8B()
        # 唯一的视觉组件: 线性投影
        self.patch_proj = nn.Linear(2700, 4096)

    def train_step(self, image, text):
        # 1. Image → patches → linear embed
        patches = split_patches(image)
        img_embeds = self.patch_proj(patches)

        # 2. Text → token embeddings
        txt_embeds = self.backbone.embed(text)

        # 3. 拼接 (with \n tokens)
        sequence = concat_with_newlines(
            img_embeds, txt_embeds
        )

        # 4. 标准 NTP loss (只在文本部分)
        logits = self.backbone(sequence)
        loss = cross_entropy(
            logits[img_len:],   # 只在文本 token 上算 loss
            text_targets
        )
        return loss

训练极简性

单阶段训练: 不需要 CLIP 预训练，不需要 alignment stage，不需要单独的视觉预训练。直接端到端学习多模态。

Fuyu-8B: 基于 Persimmon-8B 骨干 (Adept 自研 LLM)，加上一个线性投影层，就是完整的多模态模型。

Loss 设计

只在文本 token 上计算 NTP loss。图像 patches 作为 context 输入但不参与 loss 计算 (与 LLaVA 相同)。

优势领域

1. 文档理解 — 全分辨率保留文字细节

2. UI 截图 — 精确定位界面元素

3. 图表解析 — 数据密集内容

4. OCR — 不受 224px 瓶颈限制

局限性

1. 无预训练视觉特征 — 必须从零学习视觉 (数据需求大)

2. 大图推理慢 — token 数多，attention $O(N^2)$

3. 自然图像理解 — 不如有 CLIP 预训练的方案
