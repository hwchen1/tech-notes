1. Layer Structure: 1 Dense + 60 MoE

class KimiK25VL(nn.Module):
    def __init__(self):
        self.layers = nn.ModuleList()
        # Layer 0: Dense FFN
        self.layers.append(
            TransformerLayer(ffn=DenseMLP())
        )
        # Layers 1-60: MoE FFN
        for _ in range(60):
            self.layers.append(
                TransformerLayer(ffn=MoELayer(
                    num_experts=384,
                    top_k=8
                ))
            )
Hybrid Dense+MoE: The first layer uses a standard dense MLP, while the remaining 60 layers use Mixture-of-Experts with 384 experts and top-8 routing. All layers share the same MLA (Multi-Latent Attention) mechanism. This design gives the model a dense "warm-up" layer before routing to specialized experts.

2. MLA (Multi-Latent Attention)

class MLA(nn.Module):
    # q_lora_rank=1536, kv_lora_rank=512
    # qk_nope_head_dim=128, qk_rope_head_dim=64

    def forward(self, x):
        # Query path
        q_latent = self.q_down(x)  # → 1536
        q = self.q_up(q_latent)
        q_nope, q_rope = q.split(
            [128, 64], dim=-1)

        # KV path (compressed)
        kv_latent = self.kv_down(x) # → 512
        kv = self.kv_up(kv_latent)
        k_nope, k_rope, v = kv.split(
            [128, 64, v_dim], dim=-1)

        # Only rope parts get RoPE
        q_rope = apply_rope(q_rope)
        k_rope = apply_rope(k_rope)

        q = cat([q_nope, q_rope])
        k = cat([k_nope, k_rope])
        return attention(q, k, v)
Multi-Latent Attention compresses KV into a 512-d latent vector, dramatically reducing KV cache size. The query is similarly compressed to 1536-d before expansion.

Key insight: heads are split into nope (position-independent, 128-d) and rope (position-encoded, 64-d) parts. Only rope parts receive rotary embeddings. This separation allows the model to learn both positional and content-based attention patterns independently.

Cache benefit: Instead of storing full K,V per head, only the 512-d latent is cached — a massive reduction for long sequences.

3. MoE: 384 Experts, Top-8 Routing

class MoELayer(nn.Module):
    num_experts = 384
    top_k = 8
    scaling_factor = 2.827
    norm_topk_prob = True

    def forward(self, x):
        # Sigmoid routing with bias
        logits = self.gate(x)  # [B*S, 384]
        scores = torch.sigmoid(logits)
        scores += self.expert_bias

        # Select top-8 experts
        topk_scores, topk_ids = \
            scores.topk(8, dim=-1)

        # Normalize selected weights
        topk_scores = topk_scores / \
            topk_scores.sum(dim=-1, keepdim=True)

        # Scale by factor
        topk_scores *= 2.827

        # Dispatch to experts
        out = moe_dispatch(
            x, topk_ids, topk_scores,
            self.experts)
384 experts with top-8 routing provides extreme fine-grained specialization. Each token activates only 8/384 experts (~2%).

Sigmoid routing (not softmax): allows multiple experts to have high scores independently, unlike softmax which forces competition.

Expert bias: learnable per-expert bias added to routing scores for load balancing.

Scaling factor 2.827: compensates for the normalized top-k weights being smaller than 1, ensuring proper gradient magnitude.

4. YaRN RoPE (Extended Context)

class YaRNRoPE(nn.Module):
    # factor=32, base=50000
    # "Yet another RoPE extensioN"

    def __init__(self, dim, max_pos=131072):
        self.base = 50000
        self.factor = 32
        self.dim = dim  # qk_rope_head_dim=64

        # Compute frequency bands
        freqs = 1.0 / (self.base ** (
            torch.arange(0, dim, 2) / dim))

        # YaRN: interpolate + scale
        # Low-freq: interpolate (÷factor)
        # High-freq: keep original
        # Mid-freq: smooth blend
        freqs = yarn_scale(freqs,
            factor=32,
            beta_fast=32,
            beta_slow=1)

    def forward(self, x, pos):
        # Apply scaled rotary embeddings
        cos, sin = self.get_cos_sin(pos)
        return rotate_half(x, cos, sin)
YaRN (Yet another RoPE extensioN) extends context length without retraining by frequency-dependent scaling.

Low frequencies (capture long-range): interpolated by dividing by factor=32, extending effective context 32x.

High frequencies (capture local patterns): kept unchanged to preserve short-range precision.

Mid frequencies: smoothly blended between the two regimes. With base=50000 and factor=32, the model handles sequences up to 131K tokens.

5. Vision: MoonViT3d + PatchMergerMLP

class MoonViT3d(nn.Module):
    hidden_size = 1152

    def forward(self, image):
        # Patch embedding
        patches = self.patch_embed(image)
        # [B, H*W, 1152]

        # Vision transformer layers
        x = self.encoder(patches)

        # 2×2 spatial merge: 4 patches → 1
        x = self.patch_merger(x)
        # [B, H*W/4, hidden]
        return x

class PatchMergerMLP(nn.Module):
    """Merge 2×2 spatial patches into 1"""
    def forward(self, x):
        B, N, C = x.shape
        H = W = int(N ** 0.5)
        x = x.view(B, H, W, C)
        # Group 2×2 patches
        x = x.view(B, H//2, 2, W//2, 2, C)
        x = x.permute(0,1,3,2,4,5)
        # Concat: [B, H/2, W/2, 4*C]
        x = x.reshape(B, -1, 4*C)
        # Project back
        x = self.mlp(x)
        return x
MoonViT3d is Kimi's custom vision encoder with hidden_size=1152.

PatchMergerMLP: Groups 2x2 spatial patches, concatenates them (4*C channels), then projects back via MLP. This reduces visual token count by 4x, critical for efficiency with high-res images.

6. INT4 Quantized Experts

class QuantizedExpert(nn.Module):
    # INT4 with group_size=32
    # Offset binary format

    def __init__(self, in_feat, out_feat):
        # Packed INT4: 2 weights per byte
        self.weight = Parameter(
            torch.zeros(out_feat, in_feat//2,
                        dtype=torch.uint8))
        # Per-group scale (group_size=32)
        n_groups = in_feat // 32
        self.scale = Parameter(
            torch.ones(out_feat, n_groups))
        # Per-group zero point
        self.zero_point = Parameter(
            torch.zeros(out_feat, n_groups))

    def forward(self, x):
        # Dequantize on-the-fly
        w = unpack_int4(self.weight)
        # offset binary: val = int4 - 8
        w = (w - 8) * self.scale + \
            self.zero_point
        return F.linear(x, w)

# Memory: 384 experts × INT4
# vs FP16: ~48x reduction per expert
INT4 quantization makes 384 experts feasible in memory. Each weight is stored as 4 bits with group-wise dequantization.

Group size 32: Every 32 consecutive weights share one scale and one zero-point value, balancing accuracy vs overhead.

Offset binary: INT4 values [0,15] are mapped to [-8,7] by subtracting 8, then scaled and shifted per group.

Impact: Without INT4, 384 experts in FP16 would require prohibitive memory. Quantization enables the massive expert count that gives K2.5-VL its fine-grained specialization.
