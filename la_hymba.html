1. 核心理念：同层内并行混合头

# Hymba vs Griffin: 混合粒度不同
# Griffin: 层级混合 (交替层)
#   Layer 0: Attn, Layer 1: RG-LRU, ...
# Hymba: 头级混合 (同一层内)
#   Layer i: head0-3=Attn, head4-7=Mamba

class HymbaLayer(nn.Module):
    def __init__(self, d_model, n_attn_heads, n_ssm_heads):
        self.attn_heads = MultiHeadAttn(
            d_model, n_attn_heads
        )
        self.ssm_heads = MambaHeads(
            d_model, n_ssm_heads
        )
        self.meta_tokens = nn.Parameter(
            torch.randn(128, d_model)
        )
        self.out_proj = nn.Linear(d_model, d_model)

    def forward(self, x):
        # 1. 前置 meta tokens
        x_with_meta = torch.cat([
            self.meta_tokens.expand(B, -1, -1),
            x
        ], dim=1)

        # 2. 并行: Attn heads + SSM heads
        attn_out = self.attn_heads(x_with_meta)
        ssm_out = self.ssm_heads(x_with_meta)

        # 3. 拼接所有 head 的输出
        out = torch.cat([attn_out, ssm_out], dim=-1)

        # 4. 移除 meta tokens, 投影输出
        out = out[:, 128:]  # remove meta
        return self.out_proj(out)

头级 vs 层级混合

Griffin 用交替层（奇数 RG-LRU、偶数 Attn）混合。Hymba 更激进：同一层内部分 head 用 attention，部分用 SSM。

方面Griffin (层级)Hymba (头级)
粒度整层切换head 级别
信息融合隔层才融合每层都融合
KV cache50% 层需要仅 Attn heads
Meta Tokens：Hymba 独有的创新 — 128 个可学习 token 前置于每层输入。它们参与 attention 计算，作为"全局信息路由器"——让 SSM heads 也能通过 meta tokens 间接访问全局信息。

2. Meta Tokens 深入

# Meta Tokens 工作原理:
# 1. 可学习参数 (non-data, shared across inputs)
# 2. 前置于序列: [M1,M2,...,M_k, t1,t2,...,t_N]
# 3. 参与 attention: 所有 token 都能 attend to meta
# 4. 跨层传递: meta tokens 也有残差连接

class MetaTokens(nn.Module):
    def __init__(self, n_meta, d_model):
        # 128 个可学习 token
        self.tokens = nn.Parameter(
            torch.randn(n_meta, d_model) * 0.02
        )

    def prepend(self, x):
        # x: [B, N, d] → [B, N+n_meta, d]
        B = x.shape[0]
        meta = self.tokens.unsqueeze(0).expand(B,-1,-1)
        return torch.cat([meta, x], dim=1)

# Meta Tokens 的作用:
# 1. 全局信息池 (summary of all tokens)
# 2. 跨层通信管道 (不受 window 限制)
# 3. 让 SSM heads 间接做 "global attention"
#    SSM 看到 meta tokens → meta 已经聚合了全局信息
# 4. 类似 CLS token 的泛化版本 (多个)

为什么有效？

SSM/Linear Attention 的根本限制：只能通过有限状态 (d²) 传递全局信息。Meta Tokens 提供了一条绕过状态瓶颈的信息高速公路：

$\text{Meta Tokens} \approx \text{Learnable Global Memory}$ $\text{Cost: } O(128 \cdot N) \ll O(N^2)$

每层的 attention heads 会自然地把全局信息"写入"meta tokens，SSM heads 再从 meta tokens "读取"全局信息。

ablation 结果：去掉 meta tokens，Hymba-1.5B 在 recall 任务上下降 8%+，证明 meta tokens 对补偿 SSM 的 recall 弱点至关重要。
