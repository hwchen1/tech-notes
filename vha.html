VHA — Virtual Head Attention

自研 Virtual Head Attention | KV 头压到 2 + Share-KV 极致省 KVCache | Q-Premix(差异化旋转造虚拟头) + Linear-Postmix(低秩头间交互) 训练期增强 | 推理"吸收"塌成纯 GQA-Share-KV-2 | 相比 MLA: Loss↓0.005, 推理吞吐+12.5%, 解码+15%

1. 核心思路 — 在 GQA 谱系上"压到底再补回来"

VHA 的出发点是 attention 的经典权衡：KV 头越少 → KVCache 越小、解码越快，但表达力越弱。MHA / GQA / MQA 是这条轴上的三个点。VHA 把 KV 头直接压到 2（比常见 GQA-8 还狠），并用 Share-KV（K/V 共用投影）把 cache 再砍一半，然后用两个纯线性轻量模块（Premix / Postmix）在训练期把丢掉的表达力补回来。关键在于：这两个模块推理时可被"吸收"进 $W_Q$、$W_O$，部署态退化成一个没有任何额外算子的 GQA-Share-KV-2。

方案KV 头数KVCache表达力来源推理额外算子
MHA$n_q$ (=Q头)最大每 Q 头独立 KV无
GQA$n_{kv}$ (如 8)中组共享，组数决定上限无

# VHA 训练态 forward (概念伪代码)
# X: [s, b, d]   d = hidden

# --- Q 路径: 投影后 Premix ---
Q = W_Q(X)              # [s,b, H'q*d']  半数真实Q头
# Premix: 差异化旋转 → 虚拟全数Q头
for g in groups:          # 每组一个 W_g^premix
    Q_hat[g] = Q @ W_premix[g]  # [s,b, Hq*d]
Q_hat = RoPE(Q_hat)       # 注意: 在 Premix 之后

# --- KV 路径: Share-KV (K=V 投影) ---
KV = W_KV(X)             # [s,b, Hkv*d]  Hkv=2
K = V = RoPE(KV)         # 共享, cache 减半

# --- 注意力 (就是 GQA: 2 KV组) ---
Y = Attention(Q_hat, K, V) # [s,b, Hq*d]

# --- 输出: Postmix 低秩头间交互 ---
# Y' = Y + Y·A·Bᵀ = Y·(I + ABᵀ)
Y_p = Y + (Y @ A) @ B.T   # A:[Hq,r] B:[Hq,r]
out = W_O(Y_p)           # [s,b, d]

三件套各补什么

Share-KV (省): K 和 V 共用一份投影，KVCache 在"KV=2"基础上再减半。省下的参数预算用来增大 head_dim，把单头容量补回来。
Premix (补组间差异): 只算半数真实 Q 头，对每组施加不同的旋转矩阵 $W_g^{premix}$ 扩充出"虚拟"全数 Q 头。解决 GQA "多个 Q 头被迫看同一份 KV、组内没区分度"的痛点。
Postmix (补头间交互): 残差分支把头维信息低秩压缩($r$)再扩张，做 head 之间的重组融合：$Y'=Y(I+AB^\top)$。GQA 各头输出本来相互独立，这一步让它们互相利用。
关键约束: Premix/Postmix 都必须是纯线性（无激活），这是后面"推理吸收"成立的前提，也是表达力的天花板。

2. "推理吸收" — VHA 的命门，到底能不能吸收？

VHA "推理零开销" 的全部依据是吸收 (absorb / fold)：把训练时的额外线性层，推理前用矩阵乘法预先合并进已有的 $W_Q$、$W_O$，让它推理时不再作为独立算子存在。原理只有一句话——线性 × 线性 = 线性，两个矩阵可提前乘成一个，靠的是结合律，前向输出完全不变。

Premix 吸收（放在 RoPE 之前才成立）：$\;\hat{Q}_{i,g} = (X W_Q)\,W_g^{premix} = X\,\big(\underbrace{W_Q W_g^{premix}}_{\text{离线预乘}}\big)$

Postmix 吸收（作用在 $W_O$ 之前）：$\;\text{out} = Y'\,W_O = Y\,(I + A B^\top)\,W_O = Y\,\big[\underbrace{(I+AB^\top)W_O}_{\text{离线预乘}}\big]$

# 推理前的一次性 fold (离线)
# ---- Premix 吸收进 W_Q ----
for g in groups:
    # 静态权重，可提前乘
    W_Q_fold[g] = W_Q @ W_premix[g]
# 推理时一步出虚拟Q头，Premix 消失

# ---- Postmix 吸收进 W_O ----
# (I + A Bᵀ) 是 [Hq,Hq] 静态矩阵
W_O_fold = (eye(Hq) + A @ B.T) @ W_O
# 推理时 out = Y @ W_O_fold，Postmix 消失

# ===== 推理态 forward (吸收后) =====
Q = W_Q_fold(X)         # 直接出虚拟Q头
Q = RoPE(Q)
K = V = RoPE(W_KV(X))  # Share-KV
Y = Attention(Q, K, V)  # 纯 GQA
out = W_O_fold(Y)       # 无 Postmix 算子
# → 完全是 GQA-Share-KV-2，零额外算子

吸收是真的，但有前提

能吸收，因为被刻意设计成可吸收。 不是魔法——它是一个等价变换，要满足三条：①模块纯线性 ②权重静态 ③不被 RoPE/非线性隔断。
RoPE 的位置是关键设计： Premix 放在 $W_Q$ 之后、RoPE 之前，所以 $W_Q W_g^{premix}$ 先 fold 再过 RoPE，没问题。若 Premix 跨在 RoPE 两侧，RoPE 卡中间就吸收失败。
换个设计就吸收不了： 给 Premix/Postmix 加激活函数、或用数据依赖的动态权重、或放到 RoPE 后面——任一出现，"推理零开销" 立刻破功。
本质交易： 为了可吸收，两个模块被锁死为线性，表达力上限受限。这正是 VHA 用"训练期表达力"换"推理期零开销"的成交点。

3. VHA vs GQA — 优劣势对比

把 VHA 放回 GQA 的坐标系：推理态它就是 GQA（KV=2 + Share-KV），所以推理侧的好处全是 GQA 极致压缩带来的；训练态它比 GQA 多两个线性模块，代价也都压在训练侧。下表逐项对照。

对比结论

VHA 赢在推理： 推理态等价 GQA-Share-KV-2，是这条轴上 cache 最低的点，吞吐 +12.5% / 解码 +15%，且靠训练期补偿做到"同 cache 下质量比普通 GQA 高"。
GQA 赢在简单与生态： 标准实现、成熟 kernel、无训练额外负担。VHA 的 Premix 旋转 / Postmix 低秩需要自己写融合 kernel，否则训练反而可能变慢。
共同的天花板，但来源不同： GQA 表达力受"组数"限；VHA 受"模块必须线性"限——一旦想加非线性增强，吸收就失效，零开销卖点没了。真实 KV=2 也意味着信息瓶颈仍在。
一句话： VHA 把复杂度从推理挪到训练——推理端赚得干净，训练端付得起就值。长上下文、大 batch 高频部署场景收益最大；KV 头本就不多时，再压到 2 的边际收益与质量风险要重新验证。
