1. LongPack + NExtLong — 语义拼接与硬负样本

# ===== LongPack (2024) =====
# 核心: 按语义相关性将短文档智能拼接为长文档
# 不是随机拼接, 而是用 embedding similarity 排序

import numpy as np
from sentence_transformers import SentenceTransformer

# Step 1: Encode all short documents
model = SentenceTransformer("all-MiniLM-L6-v2")
embeddings = model.encode(short_docs)  # [N, dim]

# Step 2: Compute pairwise similarity matrix
sim_matrix = embeddings @ embeddings.T  # [N, N]

# Step 3: Greedy chain — 每次选最相似的下一个
def greedy_chain(sim_matrix, target_len):
    chain = [np.random.randint(len(sim_matrix))]
    used = set(chain)
    while total_tokens(chain) < target_len:
        last = chain[-1]
        scores = sim_matrix[last].copy()
        scores[list(used)] = -np.inf
        nxt = np.argmax(scores)
        chain.append(nxt)
        used.add(nxt)
    return chain

# ===== NExtLong (2025) =====
# 核心: 用 hard negatives 作为干扰文档

# Step 4: Mine hard negatives via dense retrieval
def mine_hard_negatives(query, corpus, retriever, k=5):
    # 检索最相似但不包含答案的文档
    candidates = retriever.retrieve(query, corpus, top_k=k*3)
    hard_negs = [c for c in candidates
                 if not contains_answer(c, query)][:k]
    return hard_negs

# Step 5: Assemble long context
def build_nextlong_sample(query, gold_doc, hard_negs):
    # 将金标文档插入硬负样本中间
    docs = hard_negs[:]
    insert_pos = random.randint(0, len(docs))
    docs.insert(insert_pos, gold_doc)
    return {
        "context": "\n\n".join(docs),
        "query": query,
        "answer_pos": insert_pos
    }

LongPack 通过 embedding 相似度构建语义连贯的长文档; NExtLong 用检索硬负样本训练模型区分"看起来对但实际错"的能力

LongPack (2024)

Core: 按语义相关性将短文档智能拼接为长文档, 不同于随机拼接

用 embedding similarity 排序, 相关文档相邻放置. 目标: 创造有内在关联的长上下文 (不只是长度, 而是有意义的长度).

训练时 packing 策略确保 GPU 利用率, 减少 padding waste.

NExtLong (2025)

Core: Dense retrieval 硬负样本作为干扰文档

干扰不是随机的, 而是检索模型认为"最像答案但不是"的文档. 这训练模型区分"看起来对但实际错"的能力.

类似对比学习中的 hard negatives, 短文档拼装 + BM25/dense retrieval 负样本 = 高质量长上下文训练数据.

\( \text{sim}(q, d^-) \gt \tau \) 且 \( d^- \notin \text{Answer}(q) \)
\( \text{Context} = [d_1^-, d_2^-, \ldots, d^+, \ldots, d_k^-] \)

2. LiteLong + Hierarchical SDG — 轻量合成与层次化生成

# ===== LiteLong (2025) =====
# 核心: 轻量级长上下文合成, 不需要大模型
# 三大技巧: entity replacement + position shuffling
#           + difficulty curriculum

def litelite_synthesize(doc, entities):
    # 技巧1: Entity Replacement
    # 替换实体让文档"不重复"但保持结构
    new_doc = doc
    for ent in entities:
        replacement = sample_same_type(ent)
        new_doc = new_doc.replace(ent, replacement)

    # 技巧2: Position Shuffling
    # 打乱段落顺序增加定位难度
    paragraphs = split_paragraphs(new_doc)
    random.shuffle(paragraphs)

    # 技巧3: Difficulty Curriculum
    # 从短到长, 逐渐增加难度
    return "\n".join(paragraphs)

# ===== Hierarchical SDG (2025) =====
# 核心: 三层结构支持百万 token 合成

class HierarchicalSDG:
    def __init__(self, target_length=1_000_000):
        self.target = target_length

    def generate_atomic_facts(self, topic, n=50):
        # Layer 0: 原子事实 (最小信息单元)
        return [llm.gen(f"Fact about {topic}")
                for _ in range(n)]

    def compose_paragraphs(self, facts, refs_per_para=3):
        # Layer 1: 段落 (组合多个原子事实)
        paras = []
        for chunk in batch(facts, refs_per_para):
            para = llm.gen(f"Write paragraph using: {chunk}")
            paras.append(para)
        return paras

    def compose_document(self, paragraphs):
        # Layer 2: 文档 (段落 + 跨层引用)
        doc = "\n\n".join(paragraphs)
        # 插入跨层引用 (上层引用下层内容)
        doc = self.inject_cross_refs(doc, paragraphs)
        return doc

    def compose_mega_document(self, documents):
        # Layer 3: 百万级 Mega-Document
        # 多文档 + 跨文档引用 → 1M+ tokens
        mega = "\n\n---\n\n".join(documents)
        mega = self.inject_cross_doc_refs(mega)
        return mega  # > 1M tokens

Hierarchical SDG 逐层合成从原子事实到百万级文档; LiteLong 用规则化方法 (实体替换 + 位置打乱) 零 LLM 成本合成

LiteLong (2025)

Core: 轻量级长上下文合成 — 不需要大模型生成, 用规则化方法

三大技巧: entity replacement (实体替换让文档不重复), position shuffling (打乱段落增加定位难度), difficulty curriculum (从易到难).

适合算力有限场景, 合成成本极低, 可大规模生成 pretrain 数据.

Hierarchical SDG (2025)

Core: 层次化合成支持百万 token 级文档

三层结构: Atomic facts → Paragraphs → Documents → Mega-documents. 每层独立合成, 逐级组合, 可控制任意长度.

关键: 跨层 reference (上层引用下层内容), 保证长距离依赖. 支持 1M+ tokens 合成.

\( \text{Length}(\text{Mega}) = \sum_{i=1}^{N} \text{len}(D_i) + \text{CrossRef}(D_i, D_j) \)
\( \text{where } N \cdot \overline{\text{len}(D)} \gt 10^6 \text{ tokens} \)

按训练阶段选择合成方法: Pretrain 重量, SFT 重质, 百万级唯一选 Hierarchical SDG

推荐路径

Pretrain 阶段: LongPack (大量语义拼接) + LiteLong (低成本补充多样性)
SFT 阶段: NExtLong (质量优先, 硬负样本训练判别力)
百万 Token: Hierarchical SDG (唯一能到 1M+ 的方案)

关键 Trade-offs

成本 vs 质量: LiteLong 零 LLM 成本但干扰质量一般; NExtLong 需要检索模型但干扰质量极高.

长度 vs 可控性: Hierarchical SDG 能到百万级但需要 LLM 生成原子事实; LongPack 受限于源文档长度.

通用建议: 先用 LongPack + LiteLong 大量合成 pretrain 数据, 再用 NExtLong 精炼 SFT 数据. 百万级需求直接上 Hierarchical SDG.

\( \text{Pipeline} = \underbrace{\text{LongPack} + \text{LiteLong}}_{\text{Pretrain: volume}} \to \underbrace{\text{NExtLong}}_{\text{SFT: quality}} \)
