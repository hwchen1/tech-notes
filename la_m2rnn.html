M²RNN

非线性 RNN + 矩阵值隐状态 — 突破 TC⁰ 表达力限制 | Mishra, Tan, Dao et al. | 2026.03 (arxiv 2603.14360)

1. 核心理念 — 为什么矩阵值状态 + 非线性？

# 表达力限制:
# Transformers, Linear RNN → TC⁰ 复杂类
# 无法解决: entity tracking, code execution
# 这些任务需要 NC¹ (超越 TC⁰)

# 传统 RNN: 向量状态 h ∈ R^d
# 表达力够，但状态太小 → 性能差

# Linear Attention: 矩阵状态 S ∈ R^(K×V)
# 状态大，但只有线性转移 → 仍在 TC⁰

# M²RNN: 两全其美
# 矩阵状态 H ∈ R^(K×V) + 非线性转移

# 核心递推:
# Z_t = tanh(H_{t-1} · W + k_t ⊗ v_t)
# H_t = f_t · H_{t-1} + (1-f_t) · Z_t
# y_t = H_t^T · q_t + w_r ⊙ v_t

# 关键参数:
# H: [K×V] = [128×672] = 86,016 (vs 向量 1,360)
# W: [V×V] 转移矩阵 (初始化为 I)
# f_t: forget gate (标量/每头)

# 结果: S₅ permutation tracking
# GDN: 完全失败
# M²RNN: 99.5%+ (泛化到 512 长度)

M²RNN = 大矩阵状态 (来自 Linear Attention) + 非线性转移 (来自传统 RNN) → NC¹ 表达力

状态大小 > 非线性

Paper 的核心发现是状态大小比非线性更重要。传统 RNN 弱不是因为非线性，而是因为向量状态太小（1,360 维 vs 矩阵 86,016 元素）。

M²RNN = 大矩阵状态 + 非线性 tanh 转移 + forget gate。表达力突破 TC⁰，可以解决 entity tracking 等 Transformer 理论上无法解决的任务。

2. Hybrid 架构 — 只需 1 层即可显著提升

# Hybrid 策略: 大部分层用 GDN，少量层用 M²RNN
# 为什么? M²RNN 不能并行化 (BPTT)
# 但即使 1 层也能带来巨大提升

# 7B MoE / 1.1B active, 100B tokens:
# Hybrid GDN         : Wiki PPL 13.51
# Hybrid M²RNN       : Wiki PPL 13.00 (-0.51)
# Hybrid GDN+M²RNN-1 : Wiki PPL 13.07 (-0.44)
# Hybrid GDN+M²RNN-5 : Wiki PPL 12.85 (-0.66)

# LongBench (7B MoE):
# Hybrid GDN         : 23.1
# Hybrid GDN+M²RNN-1 : 30.4 (+7.3!)
# Hybrid GDN+M²RNN-5 : 31.2 (+8.1!)

# 训练吞吐量:
# 替换 1 层 GDN → M²RNN: 仅损失 ~6%
# 收益远超成本

# 状态大小对比 (同等性能):
# Hybrid GDN: 需要 3x 状态大小
# Hybrid M²RNN: 1/3 状态即可匹敌

Hybrid 策略: 绝大部分层保持可并行的 GDN，仅 1-5 层替换为 M²RNN 即可获得巨大收益

1 层 = 7.3 分提升

非线性 RNN 层计算昂贵（无法对序列并行化，必须 BPTT），但即使单层 M²RNN 在 hybrid 架构中也能带来不成比例的巨大收益，尤其在长上下文任务上。

3. 与 Linear Attention 家族的关系

# 统一视角: 所有模型都是 associative memory
# S_t = A_t · S_{t-1} + B_t
#
# Linear Attention: A=I, B=k⊗v
# GLA:             A=diag(G), B=k⊗v
# DeltaNet:        A=I-βkk^T, B=k⊗v
# GDN:             A=αI-βkk^T, B=k⊗v
# M²RNN:           A=f·I, B=(1-f)·tanh(H·W+k⊗v)
#                  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
#                  B 依赖 H_{t-1} → 非线性!
#
# 关键区别:
# Linear 系列: B_t 只依赖当前输入
# M²RNN: B_t 依赖 H_{t-1} (上一状态)
# → 这个依赖使得 M²RNN 突破 TC⁰

# 训练代价:
# Linear: chunk-wise parallel O(N)
# M²RNN: BPTT O(N) 但不能并行
# 解法: Hybrid (大部分 linear + 少量 M²RNN)

# 代码: github.com/open-lm-engine/lm-engine
# 模型: huggingface.co/open-lm-engine/m2rnn

TC⁰ 包含大多数 LLM 任务，但 entity tracking / code execution 需要 NC¹ 表达力

TC⁰ vs NC¹: 为什么重要？

TC⁰（常数深度阈值电路）：Transformer 和所有 Linear RNN 的理论上限。大多数语言建模任务在此类内。

NC¹（对数深度电路）：严格超越 TC⁰。包含排列组合、实体追踪、代码执行等需要"真正记忆"的任务。

实践意义：大多数 LLM 任务在 TC⁰ 内（所以 linear attention 够用），但 entity tracking 和 code execution 需要 NC¹。M²RNN 是目前最实用的获取 NC¹ 表达力的方式。
