1. Infinite Attention 总览 — 为什么要双路

A3B 原生注意力是全局因果 GQA，序列越长 KV cache 与算力越爆炸（$O(L^2)$）。Infinite Attention 的思路：把"近处"和"远处"拆成两条路，再用 online-softmax 无损融合：

  • SWA 路（精确近场）：滑动窗口注意力，每个 query 只看最近 swa_window_size（默认 4096）个原始 token，$O(L\cdot W)$。
  • Memory 路（压缩远场）：把窗口之外的历史压缩成稀疏的 memory KV（压缩比 $R$，默认 4），query 再去 attend 这批压缩 KV，把远距离信息以低成本带回来。

实现上 Ernie5InfiniteAttention(Ernie5SelfAttention) 继承了 A3B 的 linear_qkv / q_k_layernorm / linear_proj / linear_gate，只是把"算 attention"那一步替换成内部模块 self.infinite_attn = InfiniteAttention(config, layer_number)。门控与投影完全复用 §A3B。

# Ernie5InfiniteAttention.forward (BSHD)
q,k,v = _get_qkv(h)      # 复用 A3B GQA
                          # QK-Norm + RoPE 已做

if yc is None:           # 无压缩 memory
    out = _fa_forward(q,k,v)  # 纯 SWA fallback
else:
    # yc = 压缩后的历史表示
    out = self.infinite_attn(
        q, k, v,
        yc=yc,            # memory 源
        rotary_pos_emb=rope,
        ...)              # 双路+融合

# === 复用 A3B 门控输出 ===
g,_ = self.linear_gate(h)
out,bias = self.linear_proj(
    out * sigmoid(g))  # [s,b,1536]

双路的核心直觉

分而治之： 近场要精确（语法、局部依赖）→ 用原始 token 的 SWA；远场可粗糙（主题、长程指代）→ 用压缩 memory。两路各算各的 attention，最后融合。
无损融合： 关键不是简单相加，而是用 online softmax 按各自 log-sum-exp (LSE) 加权——等价于把两路的 KV 拼在一起做一次完整 softmax，数学上无近似（§3）。
即插即用： InfiniteAttention 是独立 MegatronModule，A3B 注意力外壳（GQA / QK-Norm / RoPE / 门控 / proj）原样保留，只替换中间 attention 计算，因此能直接复用现成 ckpt 与门控权重。
退化路径： 当 yc is None（未提供压缩 memory）或 infinite_swa_only=True 时，直接走 _fa_forward 纯 SWA，行为等同普通滑窗注意力。

2. Memory 路径 — 压缩、投影与 head-fold shape 流

Memory 路要回答两个问题：历史怎么压缩成稀疏 KV，以及 query 怎么对齐这批压缩 KV。压缩比 $R$（compress_ratio，默认 4）意味着每 $R$ 个原始 token 折叠成 1 个 memory slot，memory 长度只有原始的 $1/R$。

压缩源 yc 经 memory_extract（可选 staircase / block_diag / kv_hca / cross_layer）抽取后，过独立的 mem_k_proj / mem_v_proj（ColumnParallelLinear, bias=False）生成 memory 的 K/V，再 mem_k_layernorm（对 head_dim 做 RMSNorm）稳定量纲，最后在位置 R-1::R 上打完整 RoPE（每个 memory slot 代表它覆盖的 $R$ 个原始位置里的最后一个）。

# InfiniteAttention.forward — memory 侧
# yc: 压缩源 [B, sq, d]
m = self.memory_extract(yc)  # 抽取历史

# 独立投影 (bias=False, ColumnParallel)
mk = self.mem_k_proj(m)  # [B,Lm,nkv*hd]
mv = self.mem_v_proj(m)
mk = mk.view(B, Lm, nkv, hd)
mv = mv.view(B, Lm, nkv, hd)

# mem K 单独 RMSNorm (over head_dim)
mk = self.mem_k_layernorm(mk)

# memory 位置 = R-1, 2R-1, 3R-1, ...
# (每 slot 取其覆盖段的最后位置)
pos = q_pos_emb[R-1::R]   # 步长 R
mk = apply_rotary(mk, pos) # 完整 RoPE

# --- head-fold: 把 R 折进 head 维 ---
# [B, sq, nh, hd]
#   → [B, sq//R, nh*R, hd]
sq_fold = sq // R
qf = q.reshape(B, sq_fold, nh*R, hd)
# 每个折叠 query-block 一次性
# 看齐它对应的 memory slot

shape 流要点

压缩比 R： memory 长度 $L_m = sq/R$。$R=4$ 时远场 KV 数量只有原始 $1/4$，attention 成本随之下降。压缩越狠（coarse 层 coarse_compress_ratio=128）远场越省，但分辨率越低。
独立 mem 投影： memory K/V 不复用 linear_qkv，而是单独的 mem_k_proj / mem_v_proj（无 bias），让压缩表示学习自己的 K/V 子空间。_init_mem_kv_from_swa 可从 GQA-交错的 linear_qkv 拷贝 K/V 权重做恒等初始化。
RoPE 位置 R-1::R： 第 $j$ 个 memory slot 覆盖原始位置 $[jR, (j{+}1)R)$，取段末 $((j{+}1)R-1)$ 作为它的 RoPE 位置，使压缩 KV 与 query 的相对位置语义对齐。
head-fold： 把相邻 $R$ 个 query 折进 head 维 [B,sq,nh,hd]→[B,sq//R,nh·R,hd]，让每个折叠块在一次 FA 调用里对齐到同一组 memory slot，避免 ragged 计算，GPU 友好。

3. Online Softmax 融合数学 + seg0 处理

两路各自跑一次 FlashAttention，除了输出 out 还会吐出每个 query 的 log-sum-exp（LSE，即该 query 那一行 softmax 分母的对数）。融合时不能简单平均，要按 LSE 把两路重新归一——这等价于把 SWA 与 memory 的 KV 拼接后做一次完整 softmax，数学上无近似。

记 SWA 路输出/LSE 为 $(\mathbf{o}_{\text{swa}},\,\ell_{\text{swa}})$，memory 路为 $(\mathbf{o}_{\text{mem}},\,\ell_{\text{mem}})$。融合的全局 LSE：

$\ell_m = \operatorname{logaddexp}(\ell_{\text{swa}},\,\ell_{\text{mem}}) = \log\!\big(e^{\ell_{\text{swa}}} + e^{\ell_{\text{mem}}}\big)$

两路权重（softmax 分母占比）与最终输出：

$\sigma_{\text{swa}} = e^{\ell_{\text{swa}} - \ell_m},\quad \sigma_{\text{mem}} = e^{\ell_{\text{mem}} - \ell_m},\quad \sigma_{\text{swa}}+\sigma_{\text{mem}}=1$

$\mathbf{o}_m = \sigma_{\text{swa}}\cdot\mathbf{o}_{\text{swa}} + \sigma_{\text{mem}}\cdot\mathbf{o}_{\text{mem}}$

# InfiniteAttentionFunc.forward
# Step1: SWA 路 (causal 窗口 FA)
o_swa, lse_swa = flash_attn(
    q, k_swa, v_swa,
    causal=True, window=W)

# Step2: memory 路 (head-fold + 因果 FA)
qf = fold(q, R)        # [B,sq/R,nh*R,hd]
o_mem, lse_mem = flash_attn(
    qf, mem_k, mem_v, causal=True)
o_mem = unfold(o_mem, R)  # 还原 [B,sq,..]

# Step2f: seg0 — 前 seg0_tokens 个
# 关掉 memory (纯 SWA 已足够)
o_mem[:, :seg0]  = 0
lse_mem[:, :seg0] = -inf  # σ_mem→0

# Step3: online softmax 融合 (fp32)
lse_m = logaddexp(lse_swa, lse_mem)
s_swa = exp(lse_swa - lse_m)
s_mem = exp(lse_mem - lse_m)
o_m = s_swa*o_swa + s_mem*o_mem
# seg0 处: s_mem=0 → o_m = o_swa

为什么 LSE 加权是无损的

softmax 可分块： 把全部 KV 分成 SWA 块和 memory 块，全局 softmax 分母 $=\sum_{\text{swa}}e^{\cdot}+\sum_{\text{mem}}e^{\cdot}=e^{\ell_{\text{swa}}}+e^{\ell_{\text{mem}}}$。$\sigma$ 就是各块对总分母的贡献占比，加权和恢复出完整 softmax 输出——这正是 FlashAttention 跨块合并的同一套数学。
fp32 融合： logaddexp 与 $\exp$ 在 float32 下算，避免 bf16 下 LSE 相减溢出/精度损失。
seg0 边界： 序列最前面 seg0_tokens = seg0_fold_size · R（其中 seg0_fold_size = memory_trunk_size // R）个 token 还没有"窗口之外的历史"，memory 无意义。这段直接把 o_mem=0, lse_mem=-∞，融合后 $\sigma_{\text{mem}}=0$，结果退化为纯 SWA。
正确性断言： 代码 assert swa_window_size ≥ seg0_tokens，保证 seg0 区段内 SWA 窗口覆盖全部前文，纯 SWA 恰好 = 完整因果注意力，无信息丢失。

4. 上下文并行 (Contiguous CP) + Cross-layer Memory + 重计算注意事项

长序列要用 Context Parallel (CP) 切分。普通全局 attention 常用 zigzag 切分来均衡因果负载，但 Infinite Attention 的 SWA 只需看最近 $W$ 个 token，于是改用更省通信的 连续切分 (contiguous)：每个 CP rank 持有一段连续的 token，只需向相邻 rank 借窗口跨界的那 W = swa_window_size - 1 个 KV token。

# 连续 CP: 只跟相邻 rank 借 W 个 KV
# W = swa_window_size - 1

# BSHD: _P2PSendRecvKVFunc
k_halo, v_halo = P2PSendRecvKV(
    k[:, :W], v[:, :W],   # 段首 W 个
    src=rank-1, dst=rank+1)
k_swa = cat([k_halo, k], dim=1)
v_swa = cat([v_halo, v], dim=1)

# THD (变长打包): 固定 buffer 防 NCCL hang
buf = zeros(W, nkv, hd)   # 定长
P2PSendRecvKVTHD(buf, ...,
    is_first_seq_continuation=flag)
# flag: 本 rank 段首是否延续上一序列
#  → 决定是否真的拼接 halo

# --- cross-layer memory ---
# memory_extract 在 checkpoint 区外预算
mem_out = memory_extract(yc) # 不重算
cross_layer_memory[L+1] = mem_out
# 下一层直接复用, 省 recompute OOM

CP 与工程注意

为什么连续而非 zigzag： SWA 的依赖是局部的（只看最近 $W$ 个），连续切分下跨界依赖只发生在段边界。每 rank 只需向上游邻居借 $W=\text{window}-1$ 个 KV，通信量 $O(W)$ 而非整圈 ring，远小于全局 attention 的 CP 开销。
BSHD vs THD： 定长 batch 用 _P2PSendRecvKVFunc 直接切段首 $W$ 个 KV；变长打包 (THD) 用 _P2PSendRecvKVFuncTHD + 固定尺寸 [W,nkv,hd] buffer，所有 rank 收发同样大小，避免不同长度导致 NCCL 集合通信挂死。
is_first_seq_continuation： THD 里多个序列拼在一个 buffer，该 flag 标记本 rank 段首是否是上一序列的延续——只有延续时才真正拼接 halo KV，否则段首是新序列起点不跨界。
cross-layer memory： 可选地把上一层算好的 memory 输出经 cross_layer_memory dict 传给下一层复用；memory_extract 在 checkpoint 重计算区之外预先算好，避免反向时重算 memory 抽取撑爆激活显存。
