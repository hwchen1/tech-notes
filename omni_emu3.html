Unified Next-Token Prediction

Emu3 proves a radical hypothesis: next-token prediction alone can match or exceed diffusion models for image generation AND CLIP-based models for image understanding.

Single Transformer, single autoregressive loss. No separate models for different modalities.

NTP Loss:
$\mathcal{L} = -\sum_t \log P(x_t \mid x_{\lt t})$

where $x$ includes both visual and text tokens in a unified sequence.

The only difference between tasks is what you condition on and what you predict.

2. Visual Tokenizer (SBER-MoVQGAN)

# SBER-MoVQGAN: Image → Discrete Tokens
# Key: Large codebook + variable resolution

class MoVQGAN:
    # Architecture
    codebook_size = 32768   # vs Chameleon 8192
    spatial_downsample = 8  # 8x reduction
    embed_dim = 256        # codebook dim

    # Encoder: Image → Latent
    # Conv layers with downsampling
    encoder = [
        Conv2d(3, 128, 3),
        ResBlock(128, 128),
        Downsample(128),    # ↓2x
        ResBlock(128, 256),
        Downsample(256),    # ↓4x
        ResBlock(256, 256),
        Downsample(256),    # ↓8x total
        ResBlock(256, 256),
        Attention(256),
        Conv2d(256, 256, 1),
    ]

    # Vector Quantization
    def quantize(self, z):
        # z: (B, 256, H/8, W/8)
        # Find nearest codebook entry
        indices = nearest(z, self.codebook)
        # indices: (B, H/8, W/8)
        return indices  # discrete tokens!

    # Decoder: Tokens → Image
    decoder = [
        Conv2d(256, 256, 1),
        ResBlock(256, 256),
        Attention(256),
        Upsample(256),     # ↑2x
        ResBlock(256, 256),
        Upsample(256),     # ↑4x
        ResBlock(256, 128),
        Upsample(128),     # ↑8x total
        ResBlock(128, 128),
        Conv2d(128, 3, 3),
    ]

# Variable resolution support
# 512×512 → 64×64 = 4096 tokens
# 768×512 → 96×64 = 6144 tokens
# 1024×1024 → 128×128 = 16384 tokens

SBER-MoVQGAN Tokenizer

The visual tokenizer converts images to discrete tokens that the LLM can predict, and decodes tokens back to pixels.

Key design choices:

32768 codebook (vs Chameleon's 8192) — larger vocabulary enables higher quality generation.

8x spatial downsampling — each token represents an 8x8 pixel patch.

Variable resolution — not fixed to 512x512, supports arbitrary aspect ratios.

Token count:
$N_{\text{visual}} = \frac{H}{8} \times \frac{W}{8}$

512x512 image = 4096 tokens
1024x1024 image = 16384 tokens

Training: encoder + decoder trained with reconstruction loss + GAN loss + perceptual loss. Frozen during LLM training.

3. 统一训练 + DPO 对齐

# Emu3 Training Pipeline
# Stage 1: Pretraining (mixed modality NTP)

class Emu3Pretrain:
    # Mixed data composition
    data_mix = {
        "text": 0.4,       # pure text
        "image_text": 0.35, # image-text pairs
        "video_text": 0.25, # video-text pairs
    }

    # Unified loss over all modalities
    def compute_loss(self, batch):
        # All tokens (text + visual) in one seq
        logits = self.model(batch.tokens)
        # NTP loss on ALL tokens equally
        loss = cross_entropy(
            logits[:, :-1],
            batch.tokens[:, 1:]
        )
        return loss

# Stage 2: Supervised Fine-Tuning (SFT)
class Emu3SFT:
    # Instruction-following format
    # [system] + [user query] + [response]
    # Loss only on response tokens
    loss_mask = "response_only"

# Stage 3: DPO (Direct Preference Optimization)
class Emu3DPO:
    beta = 0.1

    def dpo_loss(self, prompt, y_w, y_l):
        # y_w: preferred generation (image)
        # y_l: rejected generation (image)
        # Both are visual token sequences!

        log_pi_w = self.model.log_prob(y_w | prompt)
        log_pi_l = self.model.log_prob(y_l | prompt)
        log_ref_w = self.ref_model.log_prob(y_w | prompt)
        log_ref_l = self.ref_model.log_prob(y_l | prompt)

        # DPO objective
        loss = -log_sigmoid(
            self.beta * (log_pi_w - log_ref_w)
            - self.beta * (log_pi_l - log_ref_l)
        )
        return loss

    # Reward: human preference on generated images
    # "Which image better matches the prompt?"
    # Winning image → y_w, losing → y_l

Training Recipe

Stage 1 — Pretrain: Mixed text/image/video data, all under NTP loss. Generation and understanding emerge from the same training signal.

Stage 2 — SFT: Instruction tuning. Loss only on response tokens. Teaches the model to follow instructions for both generation and understanding.

Stage 3 — DPO: Key innovation. Treats generated images as "responses" and uses human preference to align generation quality.

DPO Loss:
$\mathcal{L}_{\text{DPO}} = -\log \sigma\!\left(\beta \log \frac{\pi(y_w)}{\pi_{\text{ref}}(y_w)} - \beta \log \frac{\pi(y_l)}{\pi_{\text{ref}}(y_l)}\right)$

Key insight: DPO replaces RLHF for generative models. No reward model needed — directly optimize the policy from pairwise preferences on generated images.

This enables the autoregressive model to close the quality gap with diffusion models.
