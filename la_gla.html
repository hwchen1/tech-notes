GLA — Gated Linear Attention

2024 | Tsinghua / Shanghai AI Lab | 数据依赖门控衰减 + Chunk-wise 并行

1. 核心理念：数据依赖的门控衰减

# GLA recurrent form
# G_t: data-dependent gate (per-head)
# 替代 RetNet 的固定 γ 衰减

def gla_recurrent(q, k, v, gate):
    B, H, N, d = q.shape
    S = torch.zeros(B, H, d, d)
    z = torch.zeros(B, H, d)
    outputs = []
    for t in range(N):
        # 门控衰减: 每步动态遗忘
        g_t = torch.sigmoid(gate[:,:,t])
        S = g_t.unsqueeze(-1) * S \
            + k[:,:,t].unsqueeze(-1) \
            * v[:,:,t].unsqueeze(-2)
        z = g_t * z + k[:,:,t]
        o_t = (q[:,:,t].unsqueeze(-2) @ S).squeeze(-2)
        o_t = o_t / (q[:,:,t] * z).sum(-1, keepdim=True)
        outputs.append(o_t)
    return torch.stack(outputs, dim=2)

为什么需要门控？

Vanilla linear attention 的状态 S 只做累加，永远不遗忘 — 随序列增长信息被噪声淹没。

RetNet 用固定 γ 做指数衰减，但对所有信息"一视同仁"，关键 fact 也会被衰减掉。

GLA 的核心突破：门控 G_t 由当前输入决定，模型学会了"什么时候该忘、忘多少"。

$S_t = \underbrace{G_t \odot S_{t-1}}_{\text{选择性遗忘}} + \underbrace{k_t \otimes v_t}_{\text{新信息写入}}$ $G_t = \sigma(W_g \cdot x_t + b_g) \in (0,1)^{d \times d}$

2. Chunk-wise 并行训练

# Chunk-wise GLA (simplified)
# chunk 内: quadratic attention
# chunk 间: 状态递推

def gla_chunk_wise(q, k, v, g, C=64):
    B, H, N, d = q.shape
    n_chunks = N // C
    S = torch.zeros(B, H, d, d)
    outputs = []

    for i in range(n_chunks):
        # 取出当前 chunk
        qc = q[:,:, i*C:(i+1)*C]
        kc = k[:,:, i*C:(i+1)*C]
        vc = v[:,:, i*C:(i+1)*C]
        gc = g[:,:, i*C:(i+1)*C]

        # Intra-chunk: 标准 attention
        # (小矩阵 C×C, GPU 高效)
        attn = qc @ kc.transpose(-1,-2)
        intra = attn @ vc

        # Inter-chunk: 从全局状态读取
        inter = qc @ S

        # 合并输出
        outputs.append(intra + inter)

        # 更新全局状态 (gated)
        G_chunk = gc.prod(dim=2)
        S = G_chunk.unsqueeze(-1) * S \
            + kc.transpose(-1,-2) @ vc

    return torch.cat(outputs, dim=2)

为什么 Chunk-wise？

纯递推 (for loop) 在 GPU 上极慢 — 无法并行。纯并行 (cumsum) 对 causal mask 不友好。

Chunk-wise 是黄金折中：

组件复杂度GPU 利用
Intra-chunkO(C²·d)高 (matmul)
Inter-chunkO(d²)中 (状态更新)
TotalO(N·d²)接近 FlashAttn

GLA 论文用 Triton 实现了硬件感知的 chunk-wise kernel，训练速度接近 FlashAttention-2。

3. 性能与对比

# GLA 与其他方法的关系
# 统一框架: S_t = A_t · S_{t-1} + B_t

# Linear Attention:
#   A = I (identity), B = k⊗v
#   → 无遗忘

# RetNet:
#   A = γ·I (固定标量), B = k⊗v
#   → 固定速率遗忘

# Mamba (S6):
#   A = diag(exp(-Δ·a)), B = Δ·b⊗v
#   → 对角 + 输入依赖 Δ

# GLA:
#   A = diag(G_t), B = k⊗v
#   → 完全数据依赖门控

# GLA 优势:
#   1. chunk-wise 训练高效
#   2. G_t 表达力 > 固定 γ
#   3. 比 Mamba scan 更简单
#   4. 可插入任何 Transformer

统一视角

所有 linear attention 变体都是同一个递推框架的不同实例化：

$S_t = A_t \cdot S_{t-1} + B_t$

差异仅在于 A_t 的设计：

模型A_t
VanillaI (恒等)
RetNetγ·I (固定)
Mambadiag(exp(-Δ·a))
GLAdiag(σ(W·x))
GLA 的位置：在表达力与效率之间取得最佳平衡。比 RetNet 强（数据依赖），比 Mamba 易训练（chunk-wise matmul 而非 scan），质量几乎追平 Transformer。
