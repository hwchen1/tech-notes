0. 为什么需要位置编码？

Transformer 的 Attention 机制本身不知道 token 的顺序。

"猫 吃 鱼" 和 "鱼 吃 猫" 在没有位置信息时，对模型来说完全一样。

比喻：盲人摸象

想象你收到一堆卡片，每张上面写了一个字。但卡片被打乱了，你不知道原来的顺序。
位置编码 = 在每张卡片上标注序号，这样你就能还原原始句子。

1. RoPE：旋转位置编码

核心idea：用旋转角度表示位置

把向量的每两个维度看作一个二维平面上的点，然后根据位置旋转它。

关键性质：两个 token 的注意力分数只取决于相对距离。
位置 3 和位置 7 的 attention = 位置 100 和位置 104 的 attention（都间隔 4）。

2. 时钟比喻：直觉理解 RoPE

想象一排时钟

你有 64 个时钟（对应 128 维向量的 64 个维度对）。
每个时钟的转速不同：第 1 个转得最快，第 64 个转得最慢。

每过一个 token 位置，所有时钟同时走一步。
快时钟几步就转一圈 → 区分相邻 token（"猫"和"吃"）
慢时钟要走很多步才转一圈 → 区分远距离 token（句首和句尾）

互动：拖动滑块看不同频率的时钟

base 越大 → 所有时钟转得越慢 → 能区分更远的位置

3. RoPE 的外推极限

模型只在 0 ~ max_position_embeddings (例如 40K) 范围内训练过。超过这个范围会怎样？

比喻：钢琴键盘

想象你的钢琴只有 88 个键（对应 40K 位置）。你学会了在这 88 个键上弹曲子。
现在有人给你一个 264 键的钢琴（128K 位置）。
低音区（前 88 键）你弹得很好。但高音区（88 键之后）你从没弹过，声音开始走调。

互动：外推时注意力质量如何衰减

绿色区域=训练范围内(安全), 黄色=轻微退化, 红色=严重退化

4. YaRN：智能缩放

核心 idea：把远方的位置"压缩"回来

比喻：地图缩放

你有一张只画了 40 公里范围的详细地图（40K 训练窗口）。
现在要导航 128 公里的路线。两种方法：

RoPE 外推 = 开出地图边界，靠猜继续走。近处还行，越远越迷路。
YaRN = 把 128 公里的路线等比缩小到地图上。 近处的街道细节保持不变（高频不缩放），远处的高速公路路线被压缩显示（低频缩放）。

具体如何压缩？— 从 128K 回到 40K

核心机制：降低旋转速度

YaRN 不改变位置编号（你还是在位置 128000）。
它改变的是旋转速度：让时钟转得更慢，这样位置 128K 产生的旋转角度 = 原来位置 40K 的角度。
对模型来说，它"看到"的角度永远在训练范围内。

互动：拖动位置看 YaRN 如何把角度压回训练范围

总结：YaRN 的压缩本质

位置 131072 (128K) 的低频旋转角度 = 位置 40960 (40K) 的原始角度

公式：angle_yarn = pos × (θ / 3.2) = (pos / 3.2) × θ

对模型来说，128K 位置的"感觉" ≈ 40K 位置 → 永远不会超出训练经验

5. YaRN 三频段策略

YaRN 的精妙之处：不是统一缩放所有维度，而是把维度按频率分成三组，差异化处理。

互动：YaRN 如何处理不同频率的维度

$$\textbf{YaRN 缩放规则:}$$ $$\text{高频 } (\lambda < \beta_{\text{fast}} \times L): \quad \theta' = \theta \quad \leftarrow \text{不动}$$ $$\text{低频 } (\lambda > \beta_{\text{slow}} \times L): \quad \theta' = \theta / s \quad \leftarrow \text{完全压缩}$$ $$\text{中间: } \quad \theta' = \frac{\theta}{1 + \text{ramp} \times (s-1)} \quad \leftarrow \text{平滑过渡}$$ $$s = \frac{\text{目标长度}}{\text{训练长度}} = \frac{131072}{40960} \approx 3.2$$

6. 对比总结

RoPE (外推)

优势 零改动，最简单

优势 短文完全不受影响

优势 推理无额外开销

适用 1~2x 外推 (如 40K→64K)

劣势 大幅外推质量快速下降

劣势 数学上无保证

YaRN (缩放)

优势 数学上保证位置在训练范围内

优势 高频不变 → 短文不退化

优势 3x+ 扩展仍稳定

适用 大幅扩展 (如 40K→128K)

劣势 需要设置额外参数

劣势 最好配合少量长文 SFT
