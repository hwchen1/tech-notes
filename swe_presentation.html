1. 第一幕 · Horizon 主轴与判据(swe_evolution)

开场用 3 页立住主轴。补丁阶段只作背景快速带过:世界是静态的,测试(oracle)由外部给,agent 只做一次条件采样 $p(\text{patch}\mid\text{issue},\text{repo})$。真正的戏在 horizon 拉长之后——用两个判据把三阶段区分开。

# 阶段判据: 两个问题定位任意一篇工作
def stage(work):
    # ① 谁判定"做完了没"?
    # ② 世界模型在哪?
    if work.judge == "外部 oracle 测试" \
       and work.horizon == "单步":
        return "补丁生成"       # SWE-bench
    if work.judge == "外部测试" \
       and work.horizon == "多步闭环":
        return "长程工程"       # DeepSWE
    if work.judge == "自判/自出题" \
       or work.world == "内部模型":
        return "自主构建"       # SSR / CWM

开场怎么讲

一句话钩子:"过去我们让模型写一个补丁;现在我们让它自己开一个项目。中间发生了什么?"——直接把 horizon 摆上台。

两个判据先抛出:后面每引一篇论文,都用这两问快速定位,听众不会迷路。

落差数据当悬念:CWM 在 SWE-bench Verified 拿 65.8,但 Terminal-Bench 类长程只有 26.3——用这个落差引出"长程 ≠ 补丁的放大",顺势进第二幕。

2. 第二幕 · 三堵墙 × 三个答案

主体。每篇论文讲成一个"瓶颈 → 方法 → 证据"的三段小故事,各 2-3 页,节奏统一。关键是先讲墙(问题),再讲论文(答案)——听众先痛,才记得住解药。

DeepSWE = {

}
# 墙③: 执行昂贵 & 静态文本≠动态语义
CWM = {
  "药": "代码世界模型, 脑内 rollout 预测执行",
  "钩": "少跑真环境也能想清后果",
  "证据": "65.8 (tts) / HaltEval 0.94",
}

三段的讲法要点

DeepSWE重点讲 Compact Filtering 的反直觉:只奖励主动 finish、mask 掉超长/超步/超时——防止"歪打正着"被强化,并让思考跨步摊薄。这是长程 RL 最出彩的一招。

SSR重点讲 自出题的闭环:题目难度随策略能力自动上移,不再依赖固定环境池,回答"数据从哪来"。

CWM重点讲 世界模型 vs 静态文本:mid-training 灌执行轨迹让模型学"代码运行会发生什么",从而能脑内推演、预测终止(HaltEval)。

顺序建议:DeepSWE(怎么稳定训长程)→ SSR(题从哪来)→ CWM(能否内化世界)。这是"先能训 → 再有料 → 再变聪明"的递进,也从"外部反馈"平滑过渡到"内部模型",为第三幕合流铺垫。

3. 第三幕 · 横切综合:三条腿合流

这是全场分析高潮(把框架 B 的三维对比放这里)。前面三篇是并列的,这一幕要把它们拧成一个 agent:三者恰好补齐"自主自演化"缺的三块——DeepSWE 给稳定的训练算法,SSR 给自生的数据/课程,CWM 给内部世界模型。缺任一条腿,长程自主 agent 都站不稳。

# 合流: 一个理想 agent 需要三者同时在场
class AutonomousSWEAgent:
    trainer     = DeepSWE   # 长程 RL 稳定收敛
    curriculum  = SSR       # 自出题, 难度随能力涨
    world_model = CWM       # 脑内 rollout 少跑真环境

    def improve(self):
        task = self.curriculum.propose()   # SSR
        plan = self.world_model.rollout(task)# CWM
        traj = self.act(task, plan)
        self.trainer.update(traj)          # DeepSWE
        # 数据自生 + 内部模拟 + 稳定RL → 飞轮

# 缺腿的后果
without_SSR  = "数据天花板"
without_CWM  = "每步都得真跑, 慢且贵"
without_RL   = "训不稳 / reward collapse"

合流一页怎么说

把并列变递进再变闭环:SSR 出题 → CWM 脑内推演可行方案 → agent 执行 → DeepSWE 用轨迹稳定更新 → 能力涨 → SSR 出更难的题。这就是自演化飞轮。

用"缺腿"反证互补:缺 SSR 撞数据天花板;缺 CWM 每步真跑又慢又贵;缺 DeepSWE 式稳定 RL 直接 reward collapse。三者不可互相替代。

高潮句"这三篇不是让你三选一,而是在告诉你——一个能自己找活干、想清后果、还越训越强的软件智能体,需要哪三块拼图。"呼应开场总纲,完成闭环。

三条 takeaway(结尾板)

① horizon 是主线:能力边界由"能自主推进多长的任务"定义,不是单点准确率。

② 三块拼图缺一不可:稳定长程 RL(DeepSWE)+ 数据自生(SSR)+ 内部世界模型(CWM)。

③ 下一步是合流:把三者拧成自演化飞轮,同时逼近"可验证性 / human-in-loop 边界 / 对齐"这些真正的天花板。
