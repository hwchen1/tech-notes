Gated DeltaNet-2

解耦 Erase 和 Write — Channel-wise 独立控制擦除/写入强度 | Hatamizadeh, Choi, Kautz | 2025.05 (arxiv 2605.22791)

1. 核心问题：为什么要解耦 Erase 和 Write？

# Gated DeltaNet (v1) 的更新规则:
# S_t = α·S_{t-1} - β·k(S·k)^T + k⊗v
#                   ^^^^^^^^^^^^^^^^
# β 同时控制:
#   1. 擦除旧值的强度 (erase)
#   2. 写入新值的承诺 (write)
# 但这两件事在语义上是不同的!

# 例子: "纠正拼写错误"
# - 需要大力擦除旧值 (强 erase)
# - 同时大力写入新值 (强 write)
# → β 大, OK

# 例子: "删除无关信息"
# - 需要大力擦除 (强 erase)
# - 不需要写入新东西 (弱 write)
# → β 大 = 擦除强...但也强制写入! ✗

# 例子: "追加新信息 (不影响旧值)"
# - 不需要擦除 (弱 erase)
# - 需要写入 (强 write)
# → β 小 = 写入弱...矛盾! ✗

# ====== GDN-2 的解法 ======
# 拆分为两个独立的 channel-wise 门:
# b_t ∈ R^d — erase gate (逐维度)
# w_t ∈ R^d — write gate (逐维度)
# 两者独立参数化，完全解耦!

GDN-2 是 GDN/KDA 的严格泛化 — 两个门独立后可以灵活组合各种记忆策略

动机总结

Gated DeltaNet v1 和 KDA 都用 单一标量 β 同时控制 erase 和 write 两个语义不同的操作。这限制了模型在以下场景的表现：

场景需求v1 能力
纠错强擦 + 强写✓
删除强擦 + 弱写✗
追加弱擦 + 强写✗
保持弱擦 + 弱写✓
核心创新：拆分 β 为独立的 b_t (erase) 和 w_t (write)，两者都是 channel-wise 向量 ∈ R^d，实现四象限全覆盖。

2. 更新公式与训练算法

# GDN-2 更新规则 (per head)
# α_t ∈ R^d — channel-wise decay (同 KDA)
# b_t ∈ R^d — channel-wise erase gate
# w_t ∈ R^d — channel-wise write gate

# 擦除矩阵 (逐维度缩放 key):
# k_erase = diag(b_t) · k_t
# k_write = diag(w_t) · k_t

# 更新公式:
# S_t = diag(α_t) · S_{t-1}
#     - k_erase · (S_{t-1}^T · k_erase)^T
#     + k_write ⊗ v_t

# 三项含义:
# 1. diag(α_t)·S: 全局 channel-wise 衰减
# 2. -k_erase·(...): 精确擦除旧关联 (强度=b_t)
# 3. +k_write⊗v_t: 写入新关联 (强度=w_t)

# 门控参数化:
# α_t = sigmoid(W_α · x_t)  ∈ (0,1)^d
# b_t = sigmoid(W_b · x_t)  ∈ (0,1)^d
# w_t = sigmoid(W_w · x_t)  ∈ (0,1)^d

# Chunk-wise 训练:
# 使用 WY decomposition 变体
# α decay 吸收进不对称 erase factor
# gate-aware backward pass
# → 保持高效并行训练

# 框架: flash-linear-attention
from fla.models import GatedDeltaNet2Config
from fla.models import GatedDeltaNet2ForCausalLM

三种门独立参数化：衰减 (α) + 擦除 (b) + 写入 (w) = 最灵活的记忆管理

公式对比

$\text{GDN v1: } S_t = \alpha S_{t-1} - \beta k(S \cdot k)^\top + kv^\top$ $\text{GDN-2: } S_t = \text{diag}(\alpha_t) S_{t-1} - \tilde{k}_b (\tilde{k}_b^\top S)^\top + \tilde{k}_w v_t^\top$ $\text{where } \tilde{k}_b = \text{diag}(b_t) k_t, \quad \tilde{k}_w = \text{diag}(w_t) k_t$

工程亮点：通过将 channel-wise decay 吸收进 WY 分解的不对称 erase factor，GDN-2 保持了与 GDN v1 相同的 chunk-wise 并行训练效率。gate-aware backward pass 确保梯度正确传播。

为什么 Multi-Key Recall 最受益？

当需要同时维护多个 key-value 关联时：

• v1：擦除 key_A 时 β 增大 → 写入也增强 → 可能覆盖 key_B

• GDN-2：b 控制对 key_A 的擦除，w 控制 key_C 的写入 → 互不干扰

RULER 基准：在 multi-key needle-in-haystack 设置下，GDN-2 改进最大。这验证了解耦 erase/write 的直觉——多路精确关联管理是单一 β 的瓶颈所在。

与 Kimi Linear 的关系

GDN-2 泛化了 KDA（Kimi Delta Attention）：

• KDA = channel-wise α + 单一 scalar β (erase=write)

• GDN-2 = channel-wise α + channel-wise b + channel-wise w

当 b=w=同一标量时，GDN-2 退化为 KDA。

实践意义：GDN-2 代码已开源 (flash-linear-attention)，是当前学术界最强的纯 linear attention。对于不需要混合 MLA 的场景，GDN-2 是首选。
