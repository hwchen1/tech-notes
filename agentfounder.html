1. 病根与解法 — 为什么开源 agent 打不过 OpenAI Deep Research

开源 deep research agent 在 BrowseComp 上长期落后:WebSailor 12.0、GLM-4.5 26.4、DeepSeek-V3.1 30.0,而 OpenAI Deep Research 是 51.5。论文诊断的根本病因:通用基座模型缺少 agentic 归纳偏置,逼着 post-training 同时干两件互相打架的事——一边学 agent 能力(怎么用工具、多步推理),一边对齐专家示范(模仿正确轨迹),产生根本性的优化张力 (optimization tension)。SFT 依赖完整高质量轨迹(覆盖不全)、RL 监督信号稀疏,都把模型锁死在"复制特定行为模式"而非灵活决策。

解法:把"学能力"从 post-training 里剥出来,提前到一个新阶段单独做掉。在 pretrain 与 post-train 之间插入 Agentic 持续预训练 (Agentic CPT),先造出一个"天生就会 agent 行为"的 pre-aligned 基座 (AgentFounder-Base),让后续 SFT/RL 只需专心对齐。CPT 仍用标准 next-token prediction,只改数据不改 loss:

$\mathcal{L} = -\sum_{t=1}^{T} \log P(x_{t+1}\mid x_1, x_2, \dots, x_t),\quad P(x_{t+1}\mid x_{\le t}) = \mathrm{softmax}(W_o h_t)$

# 三阶段训练范式 (从 Qwen3-30B-A3B-Base)

# ===== 传统两段式 (缺 agent 偏置) =====
# Pretrain → Post-train(SFT+RL)
#   ↑ post-train 被迫同时:
#     学能力 + 对齐示范 → 优化张力

# ===== AgentFounder 三段式 =====
model = Qwen3_30B_A3B_Base()

# Agentic CPT Stage 1
model = CPT(model,
    data = FAS + short_HAS,  # ~200B tokens
    ctx  = 32_768)           # 习得工具调用+多步推理

# Agentic CPT Stage 2
model = CPT(model,
    data = high_quality_HAS, # 100B tokens
    ctx  = 131_072)          # 长程规划+复杂动作空间

# Post-train: 通用对话 + React 轨迹
agent = SFT(model)         # 只需专心对齐

关键 insight

优化张力是病根: 通用基座无 agentic 归纳偏置 → post-training 被迫"学能力 + 对齐"同时进行,两个目标冲突,模型只会模仿轨迹不会决策。
Agentic CPT 是中间 scaling 层: 第一个把 CPT 引入 agent 训练流水线,产出"pre-aligned 的 agent 基座",天然支持 agent 行为,让下游 fine-tune 更高效。
两条数据原则: ① seed 数据来源要广、不限单一领域(agent 能力是跨域的);② 训练数据要覆盖各种 agent 行为类型,防止模型记忆固定模式、压缩探索能力。
不改 loss: CPT 用和 pretrain 完全一样的 next-token prediction + 交叉熵,创新全在数据合成上 → 见下两节 FAS / HAS。

2. FAS — First-order Action Synthesis (零监督全离线合成)

FAS 解决"没有专家轨迹也要造 agent 数据"的问题:不依赖任何 API / 工具调用 / 人工标注,纯粹从开放世界知识里"想象"出 agent 的一阶动作。核心是把模型已有的知识重新组织成"实体锚定的稠密知识陈述",再围绕它合成各种风格的问题,然后让模型产出两类一阶动作——planning action(首步规划)和 reasoning action(纯推理,禁用工具)。关键发现:首步推理质量与任务完成率强正相关,所以把"练好第一步"单独拎出来做。

# FAS: 零监督一阶动作合成 (全离线)

# ① 实体锚定知识记忆
entity   = sample_entity(open_world)
mem      = LLM(f"围绕 {entity} 写稠密知识陈述")
#   entity → 多条事实/关系/属性

# ② 多风格问题合成
Q = [LLM(mem, style=s)
     for s in STYLES]   # K 个不同风格

# ③ planning action (首步规划)
for q in Q:                  # 每个问题独立
    a1 = LLM(f"分析 {q} 并给出第一步动作")
    # 发现: 首步质量 ∝ 完成率

# ④ reasoning action (两步, 禁工具)
A1 = LLM(q, tools=None)   # 第一段推理
A2 = LLM(q, A1, tools=None) # 第二段推理

# ⑤ LLM-as-Judge 拒绝采样
data = [d for d in all_synth
        if judge(d) == "good"]

FAS 关键点

实体锚定: 不凭空造问题,先选一个 entity 把模型脑子里关于它的知识"倒"成稠密陈述,再以此为锚生成问题 → 保证问题有真实知识支撑。
首步即胜负手: planning action 专练"第一步怎么走"。论文实证首步推理质量与最终完成率强正相关,所以把首步当成独立训练目标。
reasoning 禁工具: 两步法 A1→A2 全程不调用任何工具,逼模型练"纯内部推理",把知识转成推理链而非依赖外部检索。
全离线零 API: 整条 FAS 流水线不触碰任何真实工具/搜索 API,纯靠 LLM 自合成 + judge 拒绝采样控质量 → 可无限放大、成本极低。

3. HAS — High-order Action Synthesis (有监督, 把轨迹榨成决策空间)

FAS 解决"从零造一阶动作",HAS 则解决"已有专家轨迹但用得太浪费"。传统 SFT 把一条轨迹的每一步当成"唯一正确答案"去模仿,只学到一条线。HAS 把轨迹的每一步都展开成一个多选决策空间:① step-level scaling——每一步 $S_k$ 额外生成 $N$ 个候选动作,混入原始那一步,把单步扩成 $(N{+}1)$ 选项,整条轨迹 $K$ 步就构成 $(N{+}1)\times K$ 的决策空间;② contrastive decision-action synthesis——把它改写成"我选第 $n_k$ 个选项"的决策表述,接真实响应 $R_k$,再附一个"我的决策是对/错"的判断 $J$,让模型在对比中学会为什么选这个、为什么不选那些。

# HAS: 把一条轨迹榨成决策空间 (有监督)

traj = [S1, S2, ..., SK]      # 专家轨迹 K 步

# ① step-level scaling
for Sk in traj:
    cands = LLM(ctx_k, n=N)   # N 个候选动作
    options = [Sk] + cands    # 原始步混入 → N+1 选项
    # 整条轨迹 → (N+1)×K 决策空间

# ② contrastive decision-action synthesis
nk = options.index(Sk)        # 正确选项下标
decision = f"我选第 {nk} 个选项"   # 决策表述
Rk       = env_or_expert(Sk)        # 真实响应
J        = "我的决策是正确的"        # or "错误" (对比)

sample = decision + Rk + J     # 拼成训练样本

# 模型学到: 在 N+1 选项里挑对的那个,
#          并能解释对/错 → 决策力 ≠ 模仿力

HAS 关键点

从"一条线"到"决策空间": 传统 SFT 每步只有唯一答案,模型只能背。HAS 给每步配 $N$ 个干扰候选,把轨迹扩成 $(N{+}1)\times K$ 的多选题 → 逼模型学"判别"而非"复制"。
对比式监督: 不只给正确动作,还把"选了哪个 + 真实响应 + 对/错判断"打包成样本。模型同时看到对的和错的,学到的是决策依据,泛化更强。
数据利用率 ×(N+1): 同一批专家轨迹,FAS 之外再被 HAS 榨出 $(N{+}1)$ 倍监督信号,且候选生成同样全离线,不调真实 API。
scaling law: 论文实测 HAS 数据量↑ 与模型尺寸↑ 都让下游指标稳定提升 → Agentic CPT 是个可 scale 的中间层,不是一次性 trick。
