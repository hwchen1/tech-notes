2. Five Models at a Glance

Qwen2.5-VL
Alibaba · 2025.01
675M params 14 patch Window+Full attn

从零训练的原生动态分辨率 ViT。引入 Window Attention（仅4层 Full Attn）大幅降低计算量。使用 mRoPE 统一时空位置编码，RMSNorm + SwiGLU 与 LLM 结构对齐。

Seed-ViT (Seed1.5-VL)
ByteDance · 2025.05
532M params 14 patch 2D RoPE 27 layers

三阶段预训练：MIM蒸馏 → 对比学习 → 全模态对齐。532M 参数即媲美 6B 的 InternVL-C。NaViT 式 packing，2×2 池化压缩 token。

MoonViT (Kimi-VL)
Moonshot AI · 2025.04
400M params CoCa loss 2D RoPE 128K ctx

SigLIP + Caption 双目标训练（2T token）。从 SigLIP-SO-400M 初始化，渐进分辨率采样。Pixel shuffle 2×2 压缩后接 MLP projector。支持 128K 上下文长视频。

Perception Encoder (Step3-VL)
StepFun · 2026.01
1.8B params 1D RoPE 16× compression Multi-crop

选用 language-optimized 变体确保与 LLM 更好对齐。728×728 global + 504×504 local crops。两层 stride-2 projector 实现 16× 空间压缩。全参数解冻联合训练。

CogViT (GLM-5V-Turbo)
Z.ai & Tsinghua · 2026.05
403M params QK-Norm Muon optim Dual-teacher

双教师蒸馏：SigLIP2（语义）+ DINOv3（纹理）。QK-Norm 防 logit 爆炸，Muon 优化器加速收敛。<|image|> placeholder 替代直接视觉 embedding，降低 MTP 复杂度。

8. Emerging Trends

9. Key Takeaways

  • 400-675M 是 ViT 的效率甜点 — Seed-ViT 532M 即可比肩 6B InternVL-C 的 zero-shot 分类；CogViT 403M 超越 632M 的 DFN-H
  • 原生分辨率已成共识 — 所有模型均支持任意长宽比输入，通过 2D RoPE / mRoPE / NaFlex 实现灵活位置编码
  • Window Attention 解决效率问题 — Qwen2.5-VL 率先引入（仅4层 Full），大幅降低高分辨率下 ViT 的计算负担
  • Token 压缩是必选项 — 从 4× (pooling/shuffle) 到 16× (stride-2 projector)，减少视觉 token 对 LLM 上下文的占用
  • 全参数解冻联合训练成主流 — 冻结 ViT 的时代已过，ViT + Projector + LLM 端到端联合优化建立更强的跨模态对齐
  • 后训练 RL 爆发式增长 — 从 SFT-only 到 1000+ 迭代 RL（RLVR/RLHF/PaCoRe），推理和 Agent 能力大幅跃升
  • 视频理解一体化 — Timestamp token + 动态帧率 + 时间维度 RoPE，让同一个 ViT 同时处理图像和视频
  • LLM 架构特征回流 ViT — RMSNorm、SwiGLU、QK-Norm 等 LLM 组件被引入 ViT，简化整体设计并提升训练稳定性
