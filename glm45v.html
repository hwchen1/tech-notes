GLM-4.5V Architecture

46 Layers (1 Dense + 45 MoE) | 128 Experts Top-8 | mRoPE [8,12,12] | Conv3d Vision Encoder + SwiGLU Merger

Dense + MoE Hybrid: The first layer uses a standard dense FFN (SwiGLU, ffn_dim=13696), while layers 1-45 replace the FFN with a Mixture-of-Experts block.

GQA: All layers use Grouped Query Attention with 96 query heads and 8 KV heads (12:1 ratio). head_dim = 4096/96 = 128 per head (but only 64 dims use RoPE).

QK-LayerNorm: LayerNorm is applied to Q and K projections before computing attention scores, stabilizing training at scale.

2. mRoPE [8, 12, 12]

# Multimodal RoPE
# head_dim = 128, rotary_percent = 0.5
# rotary_dim = 64, split into [8,12,12]

def mrope(q, k, pos_ids):
    # pos_ids: [B, 3, S]
    # 3 channels: temporal, height, width
    rotary_dim = head_dim * 0.5  # 64
    sections = [8, 12, 12]  # =32 pairs

    q_rot = q[..., :rotary_dim]
    q_pass = q[..., rotary_dim:]  # no RoPE

    # Split rotary part by section
    q_t = apply_rope(q_rot[..., :16], pos_ids[:,0])
    q_h = apply_rope(q_rot[..., 16:40], pos_ids[:,1])
    q_w = apply_rope(q_rot[..., 40:64], pos_ids[:,2])

    return cat([q_t, q_h, q_w, q_pass])
Multimodal RoPE: The rotary embedding is split into 3 sections [8,12,12] pairs (= [16,24,24] dims) for temporal, height, and width position encoding.

rotary_percent=0.5: Only the first 64 of 128 dims per head receive RoPE. The remaining 64 dims are position-agnostic (pass-through).

Text tokens: All 3 position channels share the same sequential position ID.

Image tokens: Temporal channel encodes frame index, height/width channels encode spatial grid position within the image.

3. MoE: 128 Experts, Top-8 Routing

class GLM45V_MoE:
    num_experts = 128
    top_k = 8
    routing_fn = "sigmoid"  # not softmax
    expert_bias = True  # load balance

    # Shared expert (always active)
    shared_expert = SwiGLU(
        dim=4096, ffn_dim=1408
    )

def forward(self, x):
    # Router: sigmoid + bias
    logits = self.gate(x)  # [B*S, 128]
    scores = sigmoid(logits) + self.bias
    top8_idx = topk(scores, 8)

    # Dispatch to experts
    expert_out = dispatch(x, top8_idx)

    # Add shared expert
    shared_out = self.shared_expert(x)
    return expert_out + shared_out
Sigmoid Routing: Unlike standard softmax gating, GLM-4.5V uses sigmoid activation for routing scores. This allows multiple experts to receive high scores independently.

Expert Bias: A learnable bias per expert is added to routing scores for load balancing, encouraging uniform expert utilization.

Shared Expert: A small SwiGLU FFN (ffn_dim=1408) runs on every token regardless of routing. This provides a baseline capacity and stabilizes training.

Scale: 128 experts x top-8 = each token activates 6.25% of expert parameters per layer.

4. Vision Encoder (Conv3d + ViT + Spatial Downsample)

class VisionEncoder:
    # Patch embedding
    patch_embed = Conv3d(
        in_channels=3,
        out_channels=1536,
        kernel_size=(2, 14, 14),
        # temporal=2, spatial=14x14
    )

    # 24 ViT blocks
    blocks = [ViTBlock(
        dim=1536,
        num_heads=12,  # head_dim=128
        mlp_ratio=4,
        rope=RotaryEmb2D()  # 2D RoPE
    ) for _ in range(24)]

    # Spatial downsample: 4x reduction
    merger = Conv2d(
        in_channels=1536 * 4,
        out_channels=1536,
        kernel_size=2, stride=2
        # merge_size=2 → 4x fewer tokens
    )
Conv3d Patch Embedding: Processes video/image input with temporal_patch_size=2 and spatial_patch_size=14. For images, temporal dim is padded to 2 frames.

2D RoPE in ViT: The vision transformer uses 2D rotary embeddings (height, width) for spatial position encoding within patches.

Spatial Downsample: After ViT processing, a Conv2d with merge_size=2 groups 2x2 spatial patches and reduces token count by 4x. This keeps the vision token budget manageable for the language model.

Output: 1536-dim vision tokens, ready for the SwiGLU merger projector.

5. SwiGLU Merger Projector

class SwiGLUMerger:
    # Projects vision dim -> language dim
    # vision_dim=1536, language_dim=4096

    ln = LayerNorm(1536)
    gate_proj = Linear(1536, 4096)
    up_proj = Linear(1536, 4096)
    down_proj = Linear(4096, 4096)

def forward(self, vision_tokens):
    # vision_tokens: [N, 1536]
    x = self.ln(vision_tokens)
    gate = silu(self.gate_proj(x))
    up = self.up_proj(x)
    x = gate * up  # SwiGLU
    x = self.down_proj(x)
    # output: [N, 4096] -> language space
    return x
SwiGLU Architecture: The merger uses the same SwiGLU pattern as the language model FFN: gate_proj with SiLU activation multiplied by up_proj, then projected down.

Dimension Bridge: Input is 1536-dim (vision encoder output), output is 4096-dim (language model hidden size). This is the only cross-modal projection layer.

Placement: Vision tokens pass through this merger before being inserted into the text sequence at positions marked by the image special token.
