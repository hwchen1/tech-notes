1. 核心理念 — 全模态 Duplex 交互

# VITA: Omni-Modal Duplex Architecture
# Key: 同时处理输入输出 (双工模式)

# Vision Encoder: InternViT-6B
class VITAVisionEncoder:
    # InternViT for image & video frames
    vision_encoder = InternViT_6B(
        img_size=448,
        patch_size=14,
        embed_dim=3200,
        layers=48,
    )
    # MLP projector → LLM hidden dim
    vision_proj = MLP(3200, 4096)

# Audio Encoder (Whisper-style)
class VITAAudioEncoder:
    audio_encoder = AudioEncoder(
        d_model=1280,
        n_layers=32,
        # mel spectrogram input
    )
    audio_proj = MLP(1280, 4096)

# LLM Backbone: Mixtral-8x7B (MoE)
class VITA(nn.Module):
    def __init__(self):
        self.llm = Mixtral8x7B(
            d_model=4096,
            n_experts=8,
            top_k=2,
            layers=32,
        )
        self.tts_head = TTSDecoder()

    def duplex_forward(self, visual, audio):
        # 关键: 模型生成输出的同时
        # 仍然监听新的音频输入!
        output_stream = self.llm.generate(
            visual, audio
        )
        # 同时运行 audio monitoring
        new_audio = self.listen_while_speaking()
        if new_audio.is_interrupt():
            output_stream.stop()
            return self.respond(new_audio)
        return output_stream

Duplex: Listen While Speaking

VITA supports non-awakening interaction — no wake word needed. The model continuously monitors the audio stream and autonomously decides when to respond.

The audio interrupt mechanism allows users to cut in while the model is speaking, immediately stopping generation and processing the new input.

First open-source omni-modal model with true duplex capabilities. Prior work (GPT-4o) is closed-source; prior open models require turn-taking (half-duplex).

The formula for attention in duplex mode spans both input and partially-generated output:

\(\text{Attn}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V\), where \(K\) includes both new input tokens and generated output tokens.

# Video Processing Pipeline
class VITAVideoProcessor:
    def __init__(self):
        self.vision_encoder = InternViT_6B()
        self.fps = 1  # 1-2 FPS sampling

    def process_video(self, video):
        # Step 1: Sample frames at low FPS
        frames = sample_frames(
            video, fps=self.fps
        )
        # e.g., 60s video → 60-120 frames

        # Step 2: Encode each frame
        frame_tokens = []
        for frame in frames:
            tokens = self.vision_encoder(frame)
            # each frame → 256 tokens
            frame_tokens.append(tokens)

        # Step 3: Temporal aggregation
        # Reduce redundancy across frames
        video_tokens = temporal_pool(
            frame_tokens,
            method="adaptive_avg"
        )
        return video_tokens


# Audio Processing Pipeline
class VITAAudioProcessor:
    def __init__(self):
        self.audio_encoder = WhisperEncoder()
        self.vad = VoiceActivityDetector()

    def process_streaming(self, audio_stream):
        # VAD: detect speech segments
        chunks = self.vad.segment(
            audio_stream,
            min_duration=0.3,  # 300ms
            max_duration=30.0, # 30s max
        )

        audio_tokens = []
        for chunk in chunks:
            # Mel spectrogram → encoder
            mel = compute_mel(chunk)
            tokens = self.audio_encoder(mel)
            audio_tokens.append(tokens)

            # Non-awakening: check if should respond
            if self.should_respond(tokens):
                yield audio_tokens
                audio_tokens = []

    def should_respond(self, tokens):
        # Learned classifier: is user addressing me?
        # No wake word needed!
        score = self.query_classifier(tokens)
        return score \gt 0.5

Parallel Modality Processing

Vision path: InternViT-6B encodes each sampled frame independently, then temporal aggregation reduces redundancy across consecutive frames.

Audio path: Streaming VAD-based chunking enables real-time response without waiting for the full utterance.

Model Scale: InternViT-6B (vision) + Mixtral-8x7B (LLM) = 46.7B total parameters, but only 12.9B active per token thanks to MoE routing (top-2 of 8 experts).

3. 训练策略 — 三阶段 + Duplex 训练

# VITA Training: 3 Stages

# ========== Stage 1 ==========
# Vision-Language Alignment
class Stage1_VisionAlignment:
    # Freeze: LLM, Vision Encoder
    # Train: Vision Projector (MLP)
    data = [
        ImageCaptionPairs(),   # ~5M
        VideoQA(),             # ~1M
    ]
    trainable = ["vision_proj"]
    lr = 1e-3

# ========== Stage 2 ==========
# Omni-Modal Integration
class Stage2_AudioIntegration:
    # Freeze: Vision Encoder
    # Train: Audio Encoder + LLM (LoRA)
    data = [
        SpeechRecognition(),   # ASR data
        SpeechQA(),            # speech → text answer
        AudioDescription(),    # audio event understanding
        ImageAudioPairs(),     # cross-modal pairs
    ]
    trainable = [
        "audio_encoder",
        "audio_proj",
        "llm_lora",  # rank=64
    ]
    lr = 2e-4

# ========== Stage 3 ==========
# Duplex Training (最关键!)
class Stage3_DuplexTraining:
    # Train: Full model fine-tune
    data = [
        DuplexConversations(),  # synthetic
        InterruptScenarios(),   # user cuts in
        NonAwakeningData(),     # filter noise
    ]

    # Special training format for duplex:
    # [SYS] [USR_AUDIO] [MODEL_GENERATING...]
    #                   [USR_INTERRUPT!]
    #       [MODEL_STOP] [NEW_RESPONSE...]
    #
    # Key: overlapping input/output in time

    def duplex_loss(self, batch):
        # Loss 1: generation quality
        l_gen = ntp_loss(batch.response)
        # Loss 2: interrupt detection
        l_int = bce_loss(batch.interrupt_flag)
        # Loss 3: response timing
        l_tim = timing_loss(batch.start_time)
        return l_gen + 0.5*l_int + 0.3*l_tim

    trainable = ["all"]
    lr = 5e-5

Progressive Complexity

Stage 1 aligns vision with language using standard image/video captioning data. Only the projection MLP is trained.

Stage 2 integrates audio understanding. The audio encoder and LLM (via LoRA rank=64) are trained on ASR and speech QA tasks.

Stage 3 is the critical duplex training. The full model is fine-tuned on synthetic conversations where input/output overlap in time.

Duplex training is the hardest part — requires synthetic data where conversations overlap. The model must learn:
1) When to start speaking (timing)
2) When to stop (interrupted by user)
3) How to process new input while generating output

The combined loss function:

\(\mathcal{L} = \mathcal{L}_{\text{gen}} + 0.5 \cdot \mathcal{L}_{\text{interrupt}} + 0.3 \cdot \mathcal{L}_{\text{timing}}\)

This multi-objective loss ensures the model balances response quality with interaction responsiveness. This is what makes VITA unique vs simpler omni models that only do half-duplex turn-taking.
