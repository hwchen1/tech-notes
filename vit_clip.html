CLIP ViT

OpenAI 2021 — 对比学习将图像和文本统一到同一嵌入空间，开启 VLM 时代

1. 对比学习框架

# CLIP 训练伪代码
# 图像-文本对 (I, T)

I_emb = vision_encoder(image)   # ViT
T_emb = text_encoder(text)      # Transformer

# L2 normalize
I_emb = I_emb / ||I_emb||
T_emb = T_emb / ||T_emb||

# cosine similarity matrix
logits = I_emb @ T_emb.T * exp(τ)

# symmetric cross-entropy
loss_i = CE(logits, labels)     # image→text
loss_t = CE(logits.T, labels)   # text→image
loss = (loss_i + loss_t) / 2

核心思想

图像和文本各有独立 encoder，通过对比学习将匹配的图文对拉近、不匹配的推远。

训练数据：400M 图文对（WebImageText），不需要人工标注。

$\mathcal{L} = -\frac{1}{N}\sum_{i=1}^{N}\log\frac{\exp(\text{sim}(I_i, T_i)/\tau)}{\sum_{j=1}^{N}\exp(\text{sim}(I_i, T_j)/\tau)}$

τ 是可学习的温度参数（初始 0.07），控制分布的锐度。

2. ViT 视觉编码器结构

# ViT-L/14 config (CLIP 最常用)
image_size    = 224
patch_size    = 14
num_patches   = (224/14)² = 256
hidden_dim    = 1024
num_heads     = 16
num_layers    = 24
mlp_ratio     = 4
output_dim    = 768  # projection

# 输入处理
patches = Conv2d(3, 1024,
    kernel=14, stride=14)
x = patches + pos_embed   # learned
x = [CLS] + x             # prepend

# N 层 Transformer Block
for block in blocks:
    x = x + MHSA(LN(x))
    x = x + MLP(LN(x))

# 输出 CLS token 的 projection
out = proj(LN(x[0]))      # → R^768

结构特点

标准 ViT 架构 + Pre-LayerNorm + QuickGELU（而非 GELU）。

位置编码：可学习的绝对位置编码（非 sinusoidal）。

输出：取 [CLS] token 经线性投射到共享嵌入空间。

$\text{QuickGELU}(x) = x \cdot \sigma(1.702x)$ $\text{Params (ViT-L/14): } 304\text{M}$

与原版 ViT 差异

1. 额外的 projection head 映射到对比学习空间

2. 使用 QuickGELU 而非 GELU

3. 训练时 text encoder 参与梯度

命名规则：ViT-{Size}/{Patch} — Size 决定 Transformer 宽度和深度，Patch 决定输入分辨率粒度。Patch 越小 → token 数越多 → 计算量越大但细节保留更好。
$\text{Tokens} = \left(\frac{H}{P}\right)^2 + 1_{\text{CLS}}, \quad \text{FLOPs} \propto \text{Tokens}^2 \cdot d$ $\text{ViT-L/14@224: } (224/14)^2 + 1 = 257 \text{ tokens}$ $\text{ViT-L/14@336: } (336/14)^2 + 1 = 577 \text{ tokens (2.2× 计算量)}$
下游使用：CLIP ViT 是 VLM 时代最广泛使用的视觉编码器。LLaVA、InternVL-1、Qwen-VL 初代、OpenFlamingo 等均使用 CLIP ViT-L/14 作为 backbone。其 zero-shot 能力来自对比学习预训练带来的强语义对齐。
