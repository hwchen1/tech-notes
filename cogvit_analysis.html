💡 为什么用双教师？

SigLIP2 擅长"这是一只猫"这种高层语义，但对纹理细节（如 GUI 按钮边框、文档排版）感知弱。DINOv3 擅长低层视觉特征（边缘、纹理、局部模式），两者互补让 CogViT 同时具备 "认识是什么" + "看清长什么样" 的能力。

💡 NaFlex 为什么重要？

传统 ViT 强制 resize 到正方形（如 384×384），一张 1920×1080 的截图被压扁后细节全丢。NaFlex 按原始宽高比切 patch，一张宽屏截图保留全部水平信息。这对 GUI Agent（需要精确定位按钮坐标）和文档理解（需要保持排版结构）至关重要。

Multimodal Multi-Token Prediction (MMTP)

MTP 是 DeepSeek 提出的训练技巧（预测后续多个 token 加速训练）。但扩展到多模态时遇到问题：图像 token 怎么传给 MTP head？

方案 1：传视觉嵌入

把视觉 embedding 原样传给 MTP head

❌ 需要跨 PP stage 传播大量视觉向量
❌ MTP head 太轻量，无法有效处理视觉分布
❌ 训练 loss 更高

方案 2：Mask 掉视觉 token

完全去掉视觉 token，退化为纯文本 MTP

❌ 丢失视觉位置信息
❌ 与 SP/CP 分区策略不兼容
❌ 需要额外 offset 映射逻辑

方案 3：<|image|> 占位符 ✓ 采用

保留视觉位置，用共享可学习 special token 替代

✅ 无需跨 PP stage 传视觉嵌入
✅ 与 SP/CP 天然兼容
✅ 训练 loss 更低、收敛更稳定
✅ 工程实现简单

💡 为什么 <|image|> token 反而效果更好？

论文假设：MTP head 很轻量（可能只有 1-2 层），面对与文本嵌入分布差异巨大的视觉向量时"消化不了"。而 <|image|> token 将输入统一为相同形式（类文本 embedding），降低了优化难度。效果上，这个设计在 0.5B 消融实验中 training loss 更低。

SigLIP2

强项：语义对齐、zero-shot 分类

弱项：纹理/空间感知弱，Agent 场景不足

被 CogViT 用作 Stage1 语义教师

DINOv3

训练：自监督（自蒸馏）

强项：低层视觉特征、边缘纹理

弱项：缺乏跨模态对齐能力

被 CogViT 用作 Stage1 纹理教师

InternViT-6B

训练：渐进式分辨率对比学习

强项：大规模、通用性强

弱项：6B 参数过重，推理成本高

CogViT 追求参数高效

CogViT ⭐

训练：双阶段（MIM蒸馏 → 对比对齐）

强项：语义+纹理+空间，Agent原生

独特：QK-Norm稳定性、NaFlex可变分辨率、Muon优化器

综合最优

CogViT 创新点总结

#创新传统做法CogViT 做法收益
1 双教师蒸馏 单教师（CLIP 或 DINO） SigLIP2 语义 + DINOv3 纹理 同时获取高低层视觉能力

4 QK-Norm 无/LayerNorm 注意力计算前 Q/K 归一化 大规模训练不崩
5 Muon 优化器 AdamW Muon + 模块独立 LR 更高效的参数更新
6 64K Batch 对比 32K 或更小 64K + SigLIP sigmoid loss 更好负样本覆盖+数值稳定
