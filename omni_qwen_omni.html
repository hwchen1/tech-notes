Qwen2.5-Omni — Thinker-Talker Architecture

Alibaba | Omni-Modal | Text + Image + Video + Audio Input | Text + Speech Output | Thinker-Talker Dual Module | 7B

1. 核心理念 — Thinker-Talker 双模块架构

# Qwen2.5-Omni: Thinker-Talker Architecture
# 核心: 分离推理(Thinker)与语音生成(Talker)

class Thinker(nn.Module):
    """大型 LLM — 负责理解 + 推理"""
    def __init__(self):
        self.vision_encoder = ViT()
        self.audio_encoder = AudioEncoder()
        self.llm = TransformerDecoder(
            layers=28,
            d_model=3584,  # 7B config
            heads=28,
        )

    def forward(self, multimodal_tokens):
        # 处理所有模态, 生成文本 + hidden states
        hidden = self.llm(multimodal_tokens)
        text_logits = self.text_head(hidden)
        return text_logits, hidden

class Talker(nn.Module):
    """小型流式模型 — 负责语音合成"""
    def __init__(self):
        self.speech_decoder = TransformerDecoder(
            layers=6,   # 远小于 Thinker
            d_model=1024,
        )
        self.speech_head = nn.Linear(
            1024, speech_codebook_size
        )

    def forward(self, thinker_hidden):
        # 接收 Thinker hidden states
        # 流式生成 speech tokens
        speech_codes = self.speech_decoder(
            thinker_hidden
        )
        return self.speech_head(speech_codes)

# 并行推理: Thinker 和 Talker 同步运行
def generate(inputs):
    for step in range(max_steps):
        text_out, hidden = thinker(inputs)
        # Talker 实时消费 hidden states
        speech = talker(hidden)  # 并行!
        yield text_out, speech

Thinker-Talker 分离

核心洞察: 将"思考"与"说话"分离，避免单一模型同时推理 + 生成语音时的质量下降。

Thinker: 大型 LLM (7B)，专注理解所有模态输入并进行推理，输出文本 + hidden states。
Talker: 小型流式模型，接收 Thinker 的 hidden states，实时生成语音 tokens。

并行执行

Thinker 和 Talker 并行运行 — Thinker 每生成一个 token 的 hidden state，Talker 立即将其转换为对应的语音编码。

优势

1. 推理质量不受损 — Thinker 无需分心生成语音

2. 低延迟流式输出 — Talker 独立优化语音生成

3. 灵活部署 — 只需文本时可跳过 Talker

2. 全模态输入编码

# Multi-Modal Encoding Pipeline
# 每种模态有专用 encoder → tokens → Thinker LLM

# === Vision: ViT + mRoPE ===
class VisionEncoder(nn.Module):
    """Same ViT as Qwen2.5-VL"""
    def __init__(self):
        self.vit = ViT(
            patch_size=14,
            d_model=1280,
            layers=32,
        )
        # mRoPE: 多维旋转位置编码
        # 分别编码 (h, w, t) 三个维度
        self.mrope = MultiDimRoPE(
            dims=[2, 2, 2]  # h,w,t
        )

    def forward(self, images_or_video):
        # Image: 动态分辨率
        # Video: 时间采样 + 空间编码
        patches = self.vit(images_or_video)
        visual_tokens = self.projector(patches)
        return visual_tokens

# === Audio: Whisper-style Encoder ===
class AudioEncoder(nn.Module):
    """Whisper-large-v3 style, streaming"""
    def __init__(self):
        self.encoder = WhisperEncoder(
            d_model=1280,
            layers=32,
            # 支持流式: 每 2s 一个 chunk
            chunk_size=2.0,  # seconds
        )
        self.projector = nn.Linear(
            1280, 3584  # → LLM dim
        )

    def forward(self, audio_mel):
        # 流式编码: chunk by chunk
        audio_feats = self.encoder(audio_mel)
        audio_tokens = self.projector(audio_feats)
        return audio_tokens

# === Token Interleaving ===
def build_input(text, images, audio):
    text_tokens = tokenizer(text)
    visual_tokens = vision_encoder(images)
    audio_tokens = audio_encoder(audio)

    # 按时间顺序/位置交错排列
    sequence = interleave(
        text_tokens,
        visual_tokens,
        audio_tokens,
    )
    return sequence  # → Thinker LLM

多模态编码

Vision (ViT + mRoPE): 与 Qwen2.5-VL 共享同一 ViT 架构，使用 mRoPE 编码图像的 (height, width) 和视频的时间维度。

Audio (Whisper-style): 基于 Whisper-large-v3 的编码器，支持流式处理 (每 2 秒一个 chunk)，将音频特征映射到 LLM 维度。

Token Interleaving

所有模态的 tokens 按时间/位置顺序交错排列，组成统一序列输入 Thinker LLM。模型可以同时接收任意组合的模态输入。

$\text{Input} = \text{Interleave}(V_{\text{visual}}, V_{\text{audio}}, V_{\text{text}})$

关键数据

组件规格

Audio EncoderWhisper-v3, 2s chunks
Thinker LLM7B, 28 layers, d=3584
Input ModalitiesText+Image+Video+Audio

3. 流式语音生成 + TMRoPE

# Talker: Streaming Speech Generation
# 接收 Thinker hidden states → 自回归生成 speech codes

class TalkerStreaming(nn.Module):
    def __init__(self):
        self.decoder = TransformerDecoder(
            layers=6, d_model=1024
        )
        # 语音 codebook (类似 SpeechTokenizer)
        self.codebook_head = nn.Linear(
            1024, codebook_size
        )
        # 多码本: 分层量化
        self.num_codebooks = 4

    def stream_generate(self, thinker_states):
        """每收到一个 hidden state 就生成语音"""
        for h in thinker_states:
            # 自回归生成 speech codes
            speech_code = self.decoder(h)
            codes = self.codebook_head(speech_code)
            # 解码为波形
            audio_chunk = vocoder(codes)
            yield audio_chunk
            # 延迟: ~300ms first token

# === TMRoPE: Time-aligned Multimodal RoPE ===
# 关键创新: 基于真实时间戳分配位置 ID

class TMRoPE(nn.Module):
    """时间对齐的多模态旋转位置编码"""
    def __init__(self, dim, base=10000):
        self.dim = dim

    def get_position_ids(self, tokens, timestamps):
        # 传统 RoPE: pos = [0, 1, 2, 3, ...]
        # TMRoPE: pos = timestamp / time_resolution
        #
        # 视频帧 @ t=0.0s → pos_id = 0
        # 音频帧 @ t=0.0s → pos_id = 0  (对齐!)
        # 视频帧 @ t=1.0s → pos_id = 25
        # 音频帧 @ t=1.0s → pos_id = 25 (对齐!)

        pos_ids = []
        for tok, ts in zip(tokens, timestamps):
            pos = int(ts / self.time_resolution)
            pos_ids.append(pos)
        return pos_ids

    # 效果: 同一时刻的视频帧和音频
    # 自动获得相同的位置编码
    # → 模型理解视听同步关系

流式语音生成

Talker 工作流: 每当 Thinker 生成一个 token 的 hidden state，Talker 立即接收并自回归生成对应的 speech codes，再通过 vocoder 解码为音频波形。

延迟: 首个 speech token 延迟约 300ms，之后流式持续输出。

TMRoPE 时间对齐

核心创新: 基于真实时间戳 (而非 token 位置) 分配 position IDs。同一时刻的视频帧和音频 chunk 自动获得相同的位置编码。

$\text{pos\_id} = \lfloor \frac{\text{timestamp}}{\text{time\_resolution}} \rfloor$

这使模型正确理解视听同步关系 — "看到的"和"听到的"在时间轴上自然对齐。

对比传统 RoPE

传统: pos = token_index，不同模态的 token 数量不同，导致时间错位。

TMRoPE: pos = f(timestamp)，无论每种模态有多少 tokens，同一时刻的内容共享位置编码。

语音 Codebook

Talker 使用分层量化的语音 codebook (类似 SpeechTokenizer)，支持 4 层 codes，实现高质量语音重建。
