Infini-attention

无限上下文 Transformer | Local Attention + Compressive Memory | 有界内存处理无限长度 | Google 2024

1. 核心理念：Local Attention + 压缩记忆 = 无限上下文

# 问题: 标准 Attention O(N²)
# 序列长 N=1M 时完全不可行
#
# 已有方案:
# - Sliding Window: 只看局部 → 丢失远程信息
# - Linear Attention: O(N) → 但质量下降
# - RAG: 外部检索 → 但不是端到端
#
# Infini-attention 的解法:
# 把序列切成固定大小的 segment
# 每个 segment 内做标准 attention (精确)
# 跨 segment 用压缩记忆 (无限)
# 两者用 learnable gate 融合

# 伪代码:
for segment in split(sequence, seg_len):
    Q, K, V = project(segment)

    # 1. 本地 attention (标准 softmax)
    A_local = softmax(Q @ K.T / sqrt(d)) @ V

    # 2. 从记忆中检索 (线性 attention)
    A_mem = retrieve(Q, S)

    # 3. Gate 融合
    output = gate * A_mem + (1-gate) * A_local

    # 4. 更新记忆 (写入当前 KV)
    S = update(S, K, V)

# 内存: O(d²) 固定 — 不随序列长度增长!
# 计算: O(N·d) 线性 — 每个 segment O(S²·d)

每个 segment: local attention (精确) + memory retrieve (压缩历史) → gate 融合

核心洞察

把 Linear Attention 当作"压缩记忆"，塞进标准 Attention 里。两全其美：

• Local Attention → 当前 segment 精确交互

• Memory (Linear Attn) → 所有历史的压缩

与 Linear Attention 的关系：Linear Attention 完全用核方法替代 softmax，全程 O(N)。Infini-attention 只在跨 segment 部分用 linear attn 当记忆，segment 内仍用标准 softmax → 质量更高。
与 Sliding Window 的关系：Sliding Window 丢弃窗口外信息。Infini-attention 把窗口外信息压缩到 M 中，永不丢弃。

复杂度

方案计算内存
StandardO(N²d)O(N²)
Linear AttnO(Nd²)O(d²)
Infini-attnO(N·S·d)O(d²)+O(S²)

S = segment length (固定，如 2048)

2. 压缩记忆：Linear Attention as Memory

# 记忆机制的数学
# S: d_key × d_value 记忆矩阵 (= Linear Attention 的状态 S)
# z: d_key 归一化向量
# φ: ELU+1 或 ReLU (特征映射, 同 Linear Attention 的 φ)

class InfiniAttention(nn.Module):
    def __init__(self, d, n_heads, seg_len):
        self.d = d
        self.n_heads = n_heads
        self.seg_len = seg_len
        self.Wq = nn.Linear(d, d)
        self.Wk = nn.Linear(d, d)
        self.Wv = nn.Linear(d, d)
        # Learnable gate per head
        self.beta = nn.Parameter(
            torch.zeros(n_heads, 1, 1))

    def forward(self, x, S, z):
        Q, K, V = self.Wq(x), self.Wk(x), self.Wv(x)

        # === 1. Memory Retrieve ===
        phi_Q = F.elu(Q) + 1  # 特征映射 φ
        # A_mem = φ(Q) @ S / (φ(Q) @ z)
        A_mem = (phi_Q @ S) / (
            phi_Q @ z.unsqueeze(-1) + 1e-6)

        # === 2. Local Attention ===
        A_local = F.scaled_dot_product_attention(
            Q, K, V, is_causal=True)

        # === 3. Gate Fusion ===
        gate = torch.sigmoid(self.beta)
        output = gate * A_mem + (1-gate) * A_local

        # === 4. Memory Update ===
        phi_K = F.elu(K) + 1
        # Delta rule (更稳定):
        old_V = (phi_K @ S) / (
            phi_K @ z.unsqueeze(-1) + 1e-6)
        S = S + phi_K.T @ (V - old_V)
        z = z + phi_K.sum(dim=-2)

        return output, S, z

Delta Rule: 只写入"预测误差"(新信息)，避免记忆膨胀

记忆 = 线性 Attention 的 KV 累积

回忆 Linear Attention 的递推形式：

$S_t = S_{t-1} + \varphi(k_t) v_t^T$ $o_t = \frac{\varphi(q_t) S_t}{\varphi(q_t) z_t}$

Infini-attention 的记忆就是 Linear Attention 的状态矩阵 S。区别是：按 segment 批量更新（不是逐 token），且用 Delta Rule 替代简单累加。

φ 的选择：ELU(x)+1 保证非负（线性 attention 要求 φ(x)≥0）。也可以用 ReLU、1+elu 等。这里 φ 就是 Linear Attention 中的核函数（特征映射）。

Gate 的含义

每个 attention head 有独立的 learnable gate β：

$O = \sigma(\beta) \cdot A_{\text{mem}} + (1 - \sigma(\beta)) \cdot A_{\text{local}}$

模型自动学习：哪些 head 更依赖历史记忆（大 β），哪些更依赖局部上下文（小 β）。

3. 与其他长上下文方案对比

# ====== 方案对比 ======

# 1. 标准 Attention
# 精确但 O(N²)，无法处理超长
# N=100K → 10B 次计算

# 2. Sliding Window (Mistral/Gemma)
# 只看最近 W 个 token
# 超出窗口的信息永远丢失
# 简单高效但"失忆"

# 3. Ring Attention / Context Parallel
# 把 KV 分布在多 GPU 上
# 精确但需要多 GPU + 通信开销

# 4. Linear Attention (完全替代)
# 全程用核方法，O(N)
# 但质量明显下降 (无 softmax 聚焦)

# 5. Infini-attention (混合)
# 局部精确 + 远程压缩
# 质量接近全 attention
# 内存 O(d²) 固定

# 6. Mamba / State Space
# 完全不同的架构 (非 attention)
# O(N) 但训练需要特殊 kernel
# 不能回溯 (不支持并行前缀和)

# Infini 的独特优势:
# - 可以直接改造现有 Transformer
# - 不需要从零训练新架构
# - Continue pretrain 即可获得长度能力

Infini-attention 位于"精确"与"压缩"之间的最佳平衡点

为什么选 Infini？

vs Sliding Window: 不丢失远程信息。Needle-in-haystack 测试中，Sliding Window 在远距离完全失败，Infini 能检索到。

vs Linear Attention: 局部用 softmax 保证质量。Linear Attn 在短序列上也比 softmax 差，Infini 在短序列上不受影响。

vs Ring Attention: 不需要多 GPU 通信。单卡即可处理无限长度。

最大卖点：可以直接把现有 LLM 改造成 Infini-attention 版本，只需 continue pretrain 少量步数。不需要从零训练新模型。

实验结果 (Google 论文)

1B 模型 + Infini-attention，处理 1M tokens 上下文：

• 书籍摘要：超越基线 114 tokens/s

• Passkey retrieval：1M 长度 100% 准确

• 内存：比全 attention 节省 114x

4. 实现细节与集成方式

# 完整的 Infini-attention 层
# 替换标准 MHA 即可

class InfiniAttentionLayer(nn.Module):
    def __init__(self, d, n_heads, seg_len=2048):
        self.seg_len = seg_len
        self.n_heads = n_heads
        self.d_head = d // n_heads

        self.qkv = nn.Linear(d, 3*d)
        self.out = nn.Linear(d, d)

        # Gate: per-head learnable
        self.beta = nn.Parameter(
            torch.zeros(n_heads, 1, 1))

    def forward(self, x, memory_state):
        B, N, D = x.shape
        S, z = memory_state
        # S: [B, H, d_k, d_v]  记忆状态矩阵
        # z: [B, H, d_k]  归一化向量

        outputs = []
        for i in range(0, N, self.seg_len):
            seg = x[:, i:i+self.seg_len]
            Q, K, V = self.qkv(seg).chunk(3, -1)
            # reshape to multi-head
            Q = Q.view(B, -1, self.n_heads, self.d_head)
            K = K.view(B, -1, self.n_heads, self.d_head)
            V = V.view(B, -1, self.n_heads, self.d_head)

            # Retrieve from memory
            phiQ = F.elu(Q) + 1  # φ(Q)
            A_mem = (phiQ @ S) / (
                (phiQ * z.unsqueeze(1)).sum(-1,
                keepdim=True) + 1e-6)

            # Local attention
            A_loc = scaled_dot_product(Q, K, V)

            # Gate
            g = torch.sigmoid(self.beta)
            out = g * A_mem + (1-g) * A_loc

            # Update memory (delta rule)
            phiK = F.elu(K) + 1  # φ(K)
            V_old = (phiK @ S) / (
                (phiK * z.unsqueeze(1)).sum(-1,
                keepdim=True) + 1e-6)
            S = S + phiK.transpose(-2,-1) @ (V-V_old)
            z = z + phiK.sum(dim=1)

            outputs.append(out)

        return torch.cat(outputs, 1), (S, z)

只需替换 Attention 层 + 初始化 gate=0 → 即可 continue pretrain

集成极简

改造现有 LLM 只需：

1. 替换 MHA → InfiniAttention

2. 添加 gate parameter β (初始化为 0)

3. 维护 memory state (S, z)

4. Continue pretrain 少量步数

β=0 初始化的妙处：sigmoid(0)=0.5，初始时 local 和 memory 各占一半。随着训练，模型自动学会平衡。如果 memory 没用，β 会趋向 -∞（完全忽略 memory）。

Segment Length 选择

短 segment (512-1K): 更频繁更新 memory，memory 利用率高，但 local attention 窗口小。

长 segment (2K-8K): local attention 覆盖更大范围，memory 只补充超远程。

实践建议：与模型原始训练长度对齐（如 LLaMA 用 4096）。

推理时的 KV Cache

Infini-attention 的推理 KV cache = 固定大小的 S + 当前 segment 的 KV。总内存有界，不随对话长度增长。
