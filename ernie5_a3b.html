ERNIE 5.1 A3B — 结构与 Shape 流

百度 ERNIE 5.1 MoE | 30B-A3B (≈3B 激活) | Megatron-Bridge 实现 | 门控 GQA (48Q/4KV, head_dim=128) + QK-Norm + 交错 RoPE | 细粒度 MoE 128 experts top-8 + 1 shared | sigmoid aux-free 路由 | MTP×1 | FP8-e4m3 + DeepEP | hidden=1536, 46 layers

1. 整体架构与 A3B 配置

ERNIE 5.1 A3B 是百度的细粒度 MoE 语言模型：≈30B 总参数、约 3B 激活。整体仍是标准 Decoder-only 堆叠（Pre-Norm + RMSNorm + 残差），但每一层把传统 dense FFN 换成 128 路由专家 (top-8) + 1 共享专家 的细粒度 MoE，并在注意力上叠加 门控 (gated attention)、QK-Norm 与 交错 RoPE。下面所有 shape 用 Megatron 的 [s, b, h]（seq, batch, hidden）布局。

# Decoder 主循环 (46 层)
x = embed(input_ids)        # [s,b,1536]
for layer in layers:        # ×46
    # --- Attention block ---
    h = RMSNorm(x)           # input_layernorm
    a = GatedGQA(h)          # §2
    x = x + a                  # 残差
    # --- MoE block ---
    h = RMSNorm(x)           # pre_mlp_layernorm
    m = MoE(h)               # §3, 128e top-8 + shared
    x = x + m
x = RMSNorm(x)               # final_layernorm
logits = output_layer(x)     # [s,b,102400]
# + MTP×1 (§4)

关键不变量

hidden 全程 1536： 注意力内部投影到 $6144=48\times128$，但 linear_proj 立刻投回 1536；MoE 内部到 1024 后也投回 1536。残差流维度恒定。
Pre-Norm 双残差： 每层两个子块（Attn / MoE）各自 RMSNorm → block → ⊕residual，norm 在残差分支外。
尾部 empty layers： 46 层里末尾 2 层是占位（num_empty_layers_add_in_tail=2），仅为 PP 切分对齐，不含真实参数。

激活量 ≈3B 来自：每 token 只走 8/128 路由专家 + 1 共享专家，而非全部 128 个。

2. 门控 GQA 注意力 — QK-Norm + 交错 RoPE + 交错 QKV 布局

A3B 注意力有三个非标准点：(1) GQA 12:1 — 48 query 头共享 4 个 KV 头；(2) 门控输出 — 一条独立的 linear_gate 投影产生 sigmoid 门控，乘到 attention 输出上；(3) QKV 权重以 GQA 交错布局离线排好，使 ColumnParallelLinear 沿 dim-0 的均匀切分恰好落在组边界。先看投影维度：

$q_{\text{size}} = n_q \cdot h_d = 48 \times 128 = 6144,\quad kv_{\text{size}} = n_{kv}\cdot h_d = 4 \times 128 = 512$

$\text{linear\_qkv}: 1536 \to q_{\text{size}} + 2\,kv_{\text{size}} = 6144 + 512 + 512 = 7168$

# Ernie5SelfAttention._get_qkv_interleaved
mixed_qkv,_ = linear_qkv(h)  # [s,b,7168]

# 每组: 48/4=12 个Q + 1K + 1V = 14 头
q_per_grp = np // ng        # 12
qkv_per_grp = q_per_grp + 2 # 14

# reshape 成按组交错布局
mixed = mixed_qkv.view(
    s, b, ng,               # ng=4
    qkv_per_grp * hd)       # 14*128=1792
# [s,b,4,1792]

split = [q_per_grp*hd, hd, hd]
                            # [1536,128,128]
q,k,v = split(mixed, split, dim=3)
q = q.reshape(s,b,-1,hd) # [s,b,48,128]
                            # k,v:  [s,b,4,128]

# QK-Norm (RMSNorm over head_dim)
q = q_layernorm(q)  # 归一化 128 维
k = k_layernorm(k)

# 交错 RoPE (even/odd 配对)
q = apply_rotary(q, theta=1e6)
k = apply_rotary(k, theta=1e6)

core = attn(q,k,v)      # [s,b,6144]

# === 门控输出 ===
g,_ = linear_gate(h)   # [s,b,6144]
out,bias = linear_proj(
    core * sigmoid(g))  # [s,b,1536]

为什么要交错布局？

组内打包： 不是 [全部Q | 全部K | 全部V]，而是按 4 个 KV 组打包成 [Q₁..₁₂,K,V, Q₁₃..₂₄,K,V, ...]。每组 14 头 × 128 = 1792。
对齐 TP 切分： ColumnParallelLinear 沿输出维均匀切分。交错后，按组切分天然把每个 KV 头和它的 12 个 Q 头分到同一 rank，避免跨 rank 通信。FusedQKVMapping 在 HF→Megatron 转换时离线完成重排。
QK-Norm： 对每个 head 的 128 维做 RMSNorm（不是对整个 hidden），稳定 attention logits 量纲。
门控注意力： $\text{out}=W_o\big(\text{Attn}(q,k,v)\odot\sigma(W_g h)\big)$。门控用原始 hidden_states 而非 attn 输出算，gate 投影到 query 维 6144。

3. 细粒度 MoE — sigmoid aux-free 路由 + 128 专家 top-8 + 共享专家

A3B 的 MoE 是"细粒度"设计：专家数多（128）、单专家小（中间维仅 1024），配合 top-8 路由，等效一个大 FFN 被切成更细的碎片以提升组合多样性。路由是 sigmoid 打分 + aux-free（专家偏置负载均衡），区别于 DeepSeek 的 softmax。

路由打分（每 token 对 128 专家）：$\;s_i = \sigma(W_r\,h)_i,\quad W_r \in \mathbb{R}^{128\times 1536}$（router 权重保持 fp32）

选择 + 缩放：$\;\mathcal{T}=\text{TopK}_8\big(s_i + b_i\big),\quad g_i = \dfrac{s_i}{\sum_{j\in\mathcal{T}} s_j}\cdot 2.5$

# MoE forward (per token, h:[1536])
# --- 路由 (fp32) ---
logits = h @ W_r.T          # [128]
s = sigmoid(logits)        # 非 softmax

# aux-free: 加专家偏置仅用于选择
idx = topk(s + expert_bias, 8)
g = s[idx] / s[idx].sum()
g = g * 2.5                 # routed_scaling

# --- 8 个路由专家 (SwiGLU) ---
y = 0
for e, w in zip(idx, g):
    # linear_fc1: 1536→2*1024 (gate,up)
    gate,up = fc1_e(h).chunk(2)
    act = silu(gate) * up  # [1024]
    # linear_fc2: 1024→1536
    y += w * fc2_e(act)   # [1536]

# --- 1 个共享专家 (始终激活) ---
gate,up = shared_fc1(h).chunk(2)
y += shared_fc2(silu(gate)*up)

# y: [1536]  (DeepEP a2a 调度 + FP8)

细粒度路由要点

sigmoid 而非 softmax： 每个专家独立打分 $\sigma(\cdot)\in(0,1)$，不跨专家归一化。选中后才在 top-8 内部归一化并 ×2.5 还原量纲。
aux-free 负载均衡： expert_bias（=moe_statics.e_score_correction_bias）只加在选择阶段（决定选哪 8 个），不进入最终权重 $g$。训练中按专家负载动态调整 bias 抹平热点，无需大的 aux loss（aux 系数仅 2.6e-7）。
共享专家： 1 个专家对所有 token 恒激活，承载通用计算，与 8 个路由专家输出相加。
DeepEP + FP8： flex dispatcher backend = DeepEP，专家计算用 FP8-e4m3 + GroupedGEMM；EP=8 时通过 all-to-all 把 token 发到专家所在 rank。router 权重与 expert_bias 强制 fp32（fp32_patterns）。

训练辅助机制

MTP×1： 在主干末尾接 1 个同构 transformer 层，预测下一个+1 token。把"未来 token embedding"经 enorm 与主干 hidden 经 hnorm 后拼接，eh_proj（2·1536→1536，带 bias）压回主维。MTP loss 以 0.1 缩放加入总 loss，提升训练信号密度。
Empty layers： num_empty_layers_add_in_tail=2 在层列表尾部插入空占位层，纯粹为让 46 层能被 PP=16 整齐切分、各 stage 计算量均衡，不含可学习参数。
选择性重计算： recompute_modules=[moe_act, layernorm, mlp] 只对这几个高显存模块重算前向，省激活显存而非全层重计算。
FP8 + DeepEP： 专家 GEMM 用 FP8-e4m3，token 调度走 DeepEP（moe_token_dispatcher_type=flex + backend=deepep），use_fp8_dispatch_a2a 让 all-to-all 通信也用 FP8，省带宽。
