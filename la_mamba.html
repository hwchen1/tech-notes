Mamba — Selective State Space Model

2023 | CMU / Princeton (Albert Gu, Tri Dao) | 输入依赖 SSM + 硬件感知 Selective Scan

1. 核心理念：选择性 SSM

# Traditional SSM (S4): 固定参数
# h_t = A · h_{t-1} + B · x_t
# y_t = C · h_t
# A, B, C 均为固定参数 → 无法选择性记忆

# Mamba (S6): 输入依赖参数
def mamba_selective_ssm(x, A, D):
    B, L, D_model = x.shape
    # 关键: B, C, Δ 全由输入 x 决定
    B_t = linear_B(x)       # [B, L, N]
    C_t = linear_C(x)       # [B, L, N]
    delta = softplus(linear_dt(x))  # [B, L, D]

    # 离散化: 连续 → 离散
    A_bar = torch.exp(delta.unsqueeze(-1) * A)
    B_bar = delta.unsqueeze(-1) * B_t.unsqueeze(2)

    # Selective scan (递推)
    h = torch.zeros(B, D_model, N)
    outputs = []
    for t in range(L):
        h = A_bar[:,t] * h + B_bar[:,t] * x[:,t].unsqueeze(-1)
        y_t = (h * C_t[:,t].unsqueeze(1)).sum(-1)
        outputs.append(y_t)

    y = torch.stack(outputs, dim=1)
    # Skip connection
    return y + x * D

选择性的本质

传统 SSM (S4) 的 A, B, C 是固定参数 — 对所有 token 做相同的状态转移。这限制了模型的表达力。

Mamba 的核心创新：让 B, C, Δ 全部依赖于输入 x_t：

$\bar{A}_t = \exp(\Delta_t \cdot A)$ $\bar{B}_t = \Delta_t \cdot B_t$ $h_t = \bar{A}_t \cdot h_{t-1} + \bar{B}_t \cdot x_t$ $y_t = C_t \cdot h_t$

Δ_t 扮演"门控"角色：大 Δ 强调当前输入（写入状态），小 Δ 保持历史（忽略输入）。

2. 硬件感知 Selective Scan

# 问题: 选择性参数化破坏了卷积可并行性
# S4 可以用 FFT 并行 (因为 A,B 固定)
# Mamba 的 A_bar, B_bar 每步不同 → 只能 scan

# 解决: 硬件感知的 parallel scan
# 核心技巧: 融合 kernel, 避免 materialize

# Mamba Block (完整)
def mamba_block(x):
    # 1. 扩展维度 (D → E·D)
    xz = linear_in(x)  # [B,L,2*E*D]
    x, z = xz.chunk(2, dim=-1)

    # 2. 1D 因果卷积 (局部特征)
    x = causal_conv1d(x, kernel_size=4)
    x = silu(x)

    # 3. SSM (核心)
    y = selective_scan(x)  # 硬件优化 kernel

    # 4. 门控输出
    y = y * silu(z)
    return linear_out(y)

# selective_scan kernel 特点:
# - 不 materialize 完整 [B,L,D,N] 中间态
# - 仅在 SRAM 中流式处理
# - IO 复杂度: O(B·L·D·N) → O(B·L·D)

为什么需要特殊 Kernel？

选择性打破了 S4 的卷积快捷方式（A, B 固定时可用 FFT）。Mamba 必须用sequential scan。

Tri Dao 的解决方案：

优化效果
Kernel FusionB, C, Δ 计算 + scan 合并为一个 kernel
不 Materialize中间态 [B,L,D,N] 不写入 HBM
Recomputation反向传播时重算而非存储
SRAM 流式scan 在 SRAM 中完成

结果：训练吞吐量是 FlashAttention-2 的 3-5×（序列越长优势越大）。

IO 复杂度：朴素实现 O(BLDN) 内存读写 → 融合 kernel 仅 O(BLD)，省去 N 倍中间态。

为什么这个等价重要？

Mamba-2 证明了 SSM 和 Linear Attention 是同一个数学对象的两种视角：

$\text{SSM: } h_t = A_t h_{t-1} + B_t x_t$ $\text{LinAttn: } S_t = \Lambda_t S_{t-1} + k_t \otimes v_t$ $\text{二者等价 (SSD 对偶)}$

实践意义：

场景用哪个视角
推理SSM (scan, O(1)/token)
训练LinAttn (chunk matmul)
理论分析两者互补
Mamba-2 的训练：约束 A 为标量后，可以用 chunk-wise matmul 替代 scan，训练速度比 Mamba-1 快 2-8 倍。这也是 GLA 采用的训练方式。
