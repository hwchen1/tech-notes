Griffin — RG-LRU + Local Attention

2024 | Google DeepMind | 门控线性递推 + 滑动窗口注意力 — 混合架构的代表

1. 核心理念：混合优于纯粹

# Griffin: 交替使用两种层
# - RG-LRU 层: 处理长距离依赖 (O(1)/token)
# - Local Attn 层: 精确短距离 recall

class GriffinBlock(nn.Module):
    def __init__(self, d, layer_idx):
        if layer_idx % 2 == 0:
            # 偶数层: RG-LRU (长距离)
            self.temporal = RGLRU(d)
        else:
            # 奇数层: Local Attention (精确)
            self.temporal = LocalAttention(
                d, window_size=1024
            )
        self.mlp = GatedMLP(d)

    def forward(self, x):
        x = x + self.temporal(rms_norm(x))
        x = x + self.mlp(rms_norm(x))
        return x

# RG-LRU: Real-Gated Linear Recurrence Unit
class RGLRU(nn.Module):
    def __init__(self, d):
        self.W_a = nn.Parameter(torch.randn(d))
        self.W_x = nn.Linear(d, d)
        self.input_gate = nn.Linear(d, d)
        self.forget_gate = nn.Linear(d, d)

    def forward(self, x):
        # Gates
        i = torch.sigmoid(self.input_gate(x))
        f = torch.sigmoid(self.forget_gate(x))
        # Decay (实数对角)
        a = -8 * f * F.softplus(self.W_a)
        a = torch.exp(a)  # ∈ (0, 1)
        # Recurrence
        h = a * h_prev + sqrt(1-a**2) * (i * self.W_x(x))
        return h

为什么要混合？

纯 linear attention/SSM 的致命弱点：recall 精度不足。需要在百万 token 中精确找到某个 fact 时，d×d 有限状态做不到。

纯 softmax attention 的致命弱点：O(N²) 不可扩展。

Griffin 的解决方案 — 各取所长：

层类型职责复杂度
RG-LRU全局长距离O(1)/token
Local Attn局部精确O(W)/token
关键洞察：模型大部分"理解"可以用线性递推完成（语义融合、位置编码传递），只有"精确检索"才需要 attention。所以 50% 就够了。

2. RG-LRU: Real-Gated Linear Recurrence Unit

# RG-LRU vs Mamba vs GRU
# RG-LRU 更简洁: 实数对角递推 + 双门控

# 递推公式:
# a_t = exp(-8·σ(W_f·x)·softplus(w_a))
# i_t = σ(W_i·x)
# h_t = a_t · h_{t-1} + √(1-a²) · i_t · (W_x·x)

# 关键设计选择:
# 1. 实数对角 (非复数) → 更简单
# 2. 双门控 (input + forget) → 更灵活
# 3. √(1-a²) 归一化 → 稳定训练
# 4. -8·softplus 参数化 → 保证 a∈(0,1)

# 对比:
# Mamba: 复杂 scan + 选择性 B,C,Δ
# RG-LRU: 简洁对角递推 + 两个 gate
# → RG-LRU 更易实现、更快训练
# → 质量差距 \lt 1% (靠 local attn 补)

# 训练: 使用 parallel scan
# (因为对角线性 → prefix sum)
# 推理: 逐步递推 O(d)/token

RG-LRU 的简洁之美

$a_t = \exp\bigl(-8 \cdot \sigma(W_f x_t) \cdot \text{softplus}(w_a)\bigr)$ $h_t = a_t \cdot h_{t-1} + \sqrt{1-a_t^2} \cdot \sigma(W_i x_t) \cdot W_x x_t$

关键设计决策：

选择理由
实数对角足够表达力，避免复数开销
√(1-a²) 归一化保持隐状态方差稳定
-8·softplus确保 a∈(0,1) 且初始接近 1
比 Mamba 简化：RG-LRU 没有选择性 B/C 投影，没有 1D Conv，没有复杂的 kernel fusion。用"简单递推 + local attention 补偿"的组合，达到接近 Mamba 的效果。

为什么 Griffin 成功？

Griffin 的实验证明了一个重要结论：

$\text{RG-LRU (长距离)} + \text{Local Attn (recall)}$ $\approx \text{Full Attention (质量)} + \text{RNN (效率)}$
