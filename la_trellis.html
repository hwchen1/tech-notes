1. 核心理念 — 固定记忆替代 KV Cache

# Transformer 的 KV Cache 问题:
# - 随序列长度线性增长: O(L × d)
# - 128K context → 数十 GB 显存
# - 推理延迟随长度增加
#
# 现有方案:
# - 量化 (4bit KV cache): 减 4x, 但仍线性增长
# - 窗口化: 丢失远距离信息
# - Linear Attention (GDN等): 固定 O(d²), 但质量有损
#
# Trellis 的方案:
# - 固定大小的 "压缩记忆" 替代 KV Cache
# - 用梯度下降动态写入 (同 Titans)
# - 加 forget gate (同 GDN)
# - 两遍机制: 先算梯度, 再非线性压缩
#
# 更新规则:
# g_t = ∇L(θ; k_t, v_t)  # 计算梯度
# θ = α_t · θ - η · σ(g_t)  # forget + 非线性写入
#
# 比 Titans 简单:
# - 记忆仍是矩阵 (不是网络)
# - 但写入方式比 GDN 更智能 (梯度+非线性)

固定大小 d×d 矩阵替代线性增长的 KV Cache — 以梯度写入实现高质量压缩

两遍 vs 一遍写入

GDN/DeltaNet 用一遍更新（直接 $S = \alpha S - \beta k(Sk)^\top + kv^\top$），Trellis 用两遍：第一遍计算"写入什么"（梯度），第二遍"非线性压缩后写入"。两遍更贵但更精确。

$\text{Pass 1: } g_t = \nabla_\theta L(\theta; k_t, v_t)$ $\text{Pass 2: } \theta \leftarrow \alpha_t \cdot \theta - \eta \cdot \sigma(g_t)$

其中 $\sigma$ 是非线性激活（如 tanh），$\alpha_t$ 是 forget gate，$\eta$ 是写入学习率。

核心定位：Trellis 是 Titans 和 GDN 的折中：用矩阵记忆（像 GDN），但用梯度写入（像 Titans），加非线性激活和 forget gate。在 NIAH recall 上 79.8 > GDN 75.8, PPL 20.28 < GDN 21.40。

超越 GDN 的原因：(1) 梯度写入比直接外积更精确, (2) 非线性激活增加了表达力, (3) forget gate 同样重要（去掉后 PPL +0.41）。但训练成本比 GDN 高（两遍计算）。

Ablation 分析

forget gate 贡献 PPL 0.41 的改进（10.87 vs 11.28），验证了遗忘机制对压缩记忆的重要性 — 不主动遗忘会导致旧信息干扰新写入。

非线性激活 $\sigma(g_t)$ 将梯度压缩到有界范围，防止写入过大导致记忆崩溃，同时增加函数逼近能力。

3. 定位与实践意义

# Trellis 在 "方法谱" 中的位置:
#
# 简单 ←────────────────→ 复杂
# GDN        Trellis      Titans/ATLAS
# (外积写入)  (梯度+非线性) (网络+BPTT)
#
# 快 ←────────────────→ 慢
# GDN/Mamba  Trellis      Titans
# (chunk-wise) (2-pass)   (BPTT)
#
# 弱 ←────────────────→ 强 (recall)
# GDN (75.8)  Trellis (79.8)  Moneta (93.5)
#
# Trellis 适合的场景:
# - 需要比 GDN 更好的 recall
# - 但不想承受 Titans/ATLAS 的训练成本
# - 需要固定显存推理 (无 KV cache)
# - 端到端可训练 (不需要 BPTT)
#
# 与 GDN-2 的互补:
# GDN-2 改进了 "擦写解耦"
# Trellis 改进了 "写入方式"
# 两者正交, 理论上可以结合

Trellis 位于 GDN 和 ATLAS 之间 — 比 GDN 更精确, 比 ATLAS 更简单高效

Sweet Spot 还是过渡品？

Trellis 处于 GDN 和 ATLAS 之间 — 比 GDN 更精确但比 ATLAS 更简单。如果 TNT 继续改进 ATLAS 的训练效率，Trellis 的 niche 可能会被压缩。但目前它是"想要比 GDN 更好但不想用 Titans"的最佳选择。

与 GDN-2 的关系：GDN-2 改进了"擦写解耦"（独立控制 erase/write 强度），Trellis 改进了"写入方式"（用梯度代替外积）。两者正交，理论上可以结合：用梯度计算写入内容 + 独立 erase/write 门控。

实践推荐

如果你的应用对 recall 敏感（多 key 检索、RAG 场景），Trellis 比 GDN 更值得考虑。如果只关心 PPL（语言建模），GDN-2 或 Mamba-3 可能更实用。

部署优势：推理时显存固定（不需要 KV cache），序列长度不影响显存占用。对于超长上下文推理（128K+）场景，这是 Trellis 相对 Transformer 的核心优势。训练时两遍计算带来约 1.5-2x 的 overhead（相对 GDN），但远低于 ATLAS 的 BPTT 成本。
