6. MoE 结构详解 (重点)

MoE Layer Forward 全流程

# MoE Layer Forward (per CP rank, per microbatch):
# Input: hidden [1, 32768, 4096] bf16
# Reshape: [32768, 4096] (flatten batch*seq)
#
# Step 1: Router
#   logits = hidden @ router_weight.T  → [32768, 128]
#   scores = softmax(logits, dim=-1)
#   top2_val, top2_idx = topk(scores, k=2)
#   → top2_idx: [32768, 2] (each token → 2 expert IDs)
#
# Step 2: AllToAll Dispatch (EP=16)
#   Each rank has 32768 tokens, each routed to 2 experts
#   Total token-copies: 32768 * 2 = 65536
#   Per dest rank: ~65536/16 = ~4096 token-copies
#   Dispatch tensor: [65536, 4096] → split across 16 ranks
#
# Step 3: Expert Compute (8 experts/rank)
#   Received tokens grouped by expert
#   GroupedGEMM: gate+up+down (SwiGLU)
#
# Step 4: AllToAll Combine
#   Send results back to source ranks
#
# Step 5: Weighted sum + shared expert
#   output = sum(routing_weight_i * expert_output_i)
#   output += shared_expert(hidden)

Token Permutation 具体示例

# Concrete permutation example (simplified, 8 tokens, 4 experts)
# Before permutation (on one rank):
#   token 0 → expert [1, 3]  (top-2)
#   token 1 → expert [0, 2]
#   token 2 → expert [1, 0]
#   token 3 → expert [3, 1]

# After permutation (sorted by expert):
#   Expert 0: [token1_copy0, token2_copy1]  → 2 tokens
#   Expert 1: [token0_copy0, token2_copy0, token3_copy1] → 3 tokens

# Permuted indices: [1,2, 0,2,3, 1, 0,3]
# After expert compute, unpermute restores original order
# Final: output[i] = w0*expert_out[i][0] + w1*expert_out[i][1]
#
# Real scale: 32768 tokens * 2 copies = 65536 entries
# Sorted into 128 expert buckets (avg 512 tokens/expert)

# Weight shapes (per rank, all 8 experts stacked):

# AllToAll Dispatch (per MoE layer, per microbatch):
#
# Each of 16 EP ranks has 32768 tokens * top_k=2 = 65536 copies
# Each copy destined for 1 of 128 experts across 16 ranks
# Uniform routing: 65536/16 = 4096 token-copies per dest rank

# Data sent per rank-pair:

7. RoPE 与 YaRN 位置编码 (重点)

7.1 基础 RoPE (Rotary Position Embedding)

# RoPE: Rotary Position Embedding
# Core idea: encode position by ROTATING Q,K vectors
#
# For head_dim=128, we have 64 frequency bands (pairs)
# Each pair (dim 2i, 2i+1) rotates at frequency theta_i:
#
#   theta_i = base^(-2i/d) where base=1000000, d=128
#
# Concrete values:
#   theta_0  = 1000000^(0/128)    = 1.0
#   theta_1  = 1000000^(-2/128)   = 0.794
#   theta_32 = 1000000^(-64/128)  = 0.001
#   theta_63 = 1000000^(-126/128) = 1.26e-6
#
# At position m, the rotation angle for band i:
#   angle_i(m) = m * theta_i
#
# Applied as 2D rotation to each (q_{2i}, q_{2i+1}):
#   q'_{2i}   = q_{2i}*cos(angle) - q_{2i+1}*sin(angle)
#   q'_{2i+1} = q_{2i}*sin(angle) + q_{2i+1}*cos(angle)
#
# Same rotation applied to K → dot(Q_rot, K_rot) depends
# only on RELATIVE position (m-n), not absolute position

7.2 RoPE 的长度限制问题

# Problem: RoPE trained at 40K cannot generalize to 128K
#
# At position m=128000, low-freq band i=63:
#   angle = 128000 * 1.26e-6 = 0.161 radians (fine)
#
# At position m=128000, high-freq band i=0:
#   angle = 128000 * 1.0 = 128000 radians
#   = 128000/(2*pi) = 20372 full rotations (fine, periodic)
#
# The REAL problem: mid-frequency bands
#   At i=32: angle = 128000 * 0.001 = 128 radians
#   During training (max 40K): max angle = 40 radians
#   At inference 128K: angle = 128 radians
#   → Model never saw angles > 40 for this band!
#   → Attention patterns break for distant tokens
#
# Solution: YaRN — different strategy per frequency band
#   High freq (short wavelength): extrapolate (keep original)
#   Low freq (long wavelength): interpolate (scale down)
#   Mid freq: smooth blend between the two

7.3 YaRN 三区域划分 (具体数值)

# YaRN parameters for this training:
#   base = 1000000
#   dim = 128 (head_dim)
#   factor = 3.2 (128K / 40K)
#   beta_fast = 32
#   beta_slow = 1
#   orig_max_pos = 40960
#
# Boundary wavelengths:
#   high_freq_wavelen = orig_max_pos / beta_fast
#                     = 40960 / 32 = 1280
#   low_freq_wavelen  = orig_max_pos / beta_slow
#                     = 40960 / 1 = 40960
#
# For each dim pair i (0..63):
#   base_freq = 1000000^(-2i/128)
#   wavelength = 2*pi / base_freq
#
# Concrete classification:
#   i=0:  wl = 2*pi/1.0 = 6.28        < 1280 → EXTRAPOLATE
#   i=8:  wl = 2*pi/0.398 = 15.8      < 1280 → EXTRAPOLATE
#   i=16: wl = 2*pi/0.158 = 39.7      < 1280 → EXTRAPOLATE
#   i=24: wl = 2*pi/0.063 = 99.7      < 1280 → EXTRAPOLATE
#   i=32: wl = 2*pi/0.001 = 6283      in [1280, 40960] → RAMP
#   i=40: wl = 2*pi/0.000398 = 15783  in [1280, 40960] → RAMP
#   i=48: wl = 2*pi/0.000158 = 39716  in [1280, 40960] → RAMP
#   i=56: wl = 2*pi/6.31e-5 = 99573   > 40960 → INTERPOLATE
#   i=63: wl = 2*pi/1.26e-6 = 5.0e6   > 40960 → INTERPOLATE

7.4 YaRN mscale 补偿

# Problem: interpolation reduces effective position differences
# → attention logits become smaller → softmax becomes more uniform
# → model "forgets" to attend sharply
#
# Solution: mscale (magnitude scale) correction
#   mscale = 0.1 * ln(factor) + 1.0
#   mscale = 0.1 * ln(3.2) + 1.0
#   mscale = 0.1 * 1.163 + 1.0
#   mscale = 1.116
#
# Applied to attention:
#   attn_score = (Q @ K.T) / sqrt(head_dim) * mscale^2
#   = (Q @ K.T) / sqrt(128) * 1.116^2
#   = (Q @ K.T) / 11.314 * 1.246
#   = (Q @ K.T) * 0.1101  (vs original 0.0884)
#
# Effect on attention scores:
#   Without mscale: scores shrink by ~1/factor → too uniform
#   With mscale: scores restored to training-time magnitude
#
# Before mscale (pos=100K, i=48 interpolated):
#   angle = 100000 * (theta/3.2) = reduced → small dot product
#   softmax → nearly uniform (bad!)
# After mscale (multiply by 1.246):
#   scores amplified → sharp attention peaks preserved

7.5 CP-aware Position (Zigzag)

# With CP=4 and non-packed sequences, positions use zigzag:
# 128K = 8 chunks of 16K each
#
# Zigzag assignment (ensures balanced KV cache):
#   CP rank 0: chunk[0] + chunk[7] = pos[0:16K] + pos[112K:128K]
#   CP rank 1: chunk[1] + chunk[6] = pos[16K:32K] + pos[96K:112K]
#   CP rank 2: chunk[2] + chunk[5] = pos[32K:48K] + pos[80K:96K]

# Each rank holds 32K tokens with non-contiguous position_ids
# RoPE must use the actual position_ids (not local indices!)
#
# In ring attention, each rank computes attention for its Q
# against ALL K,V (received via P2P ring exchange)
# Zigzag ensures each rank has both "early" and "late" positions

9. Backward Pass & 梯度同步 (重点)

Backward + Async Grad Reduce 重叠

# overlap_grad_reduce=True:
# Backward computes gradients layer by layer (48→1)
# Gradients are bucketed; when a bucket is full,
# ReduceScatter starts immediately on NCCL stream
#
# Timeline (compute stream vs NCCL stream):
# Compute: [L48 bwd] [L47 bwd] [L46 bwd] ... [L1 bwd]
# NCCL:         [RS bucket3] [RS bucket2] [RS bucket1] [RS bucket0]
#
# Each bucket: ~30.5B/16/4_buckets = ~476M params
# RS data per bucket: 476M * 2B = 952 MB
# RS time per bucket: 952MB / 50GB/s ~ 19ms
#
# Backward time per layer: ~60s/48/2 ~ 625ms
# → RS easily hidden behind backward compute
#
# register_grad_ready(param):
#   When all params in bucket have grads → start_grad_sync()
#   Async NCCL reduce_scatter on separate stream

Gradient Accumulation 机制

# DDP backward post-hook (registered per param):
def backward_post_hook(param):
    # Accumulate into contiguous buffer
    param.main_grad.add_(param.grad.data)
    param.grad = None  # free autograd grad

    # Only last microbatch triggers communication
    if is_last_microbatch and overlap_grad_reduce:
        bucket_group.register_grad_ready(param)

# no_sync() mechanism:
# MB0: is_last_microbatch=False → accumulate only
# MB1: is_last_microbatch=True  → accumulate + trigger RS
#
# register_grad_ready():
# When all params in bucket have grads →
# start_grad_sync() → async reduce_scatter on NCCL stream
#
# Memory: main_grad is pre-allocated contiguous buffer
# Size: 30.5B * 2B(bf16) / 16(DP) = ~3.8 GB per rank

Distributed Optimizer Step

# DistributedOptimizer (ZeRO-1 style):
# Each rank holds 1/16 of params + optimizer states
#
# After RS: each rank has reduced grad for its shard
# Adam update (only on local shard):
#   m = beta1*m + (1-beta1)*grad
#   v = beta2*v + (1-beta2)*grad^2
#   param -= lr * m / (sqrt(v) + eps)
#
# Memory per rank (optimizer states):
#   params:  30.5B/16 * 2B = 3.8 GB (bf16)
#   m:       30.5B/16 * 4B = 7.6 GB (fp32)
#   v:       30.5B/16 * 4B = 7.6 GB (fp32)
#   grad:    30.5B/16 * 2B = 3.8 GB (bf16)
#   Total optimizer memory: ~23 GB per rank
#
# After Adam: AllGather to restore full params
# overlap_param_gather=True: AG overlaps with next forward

10. 激活重计算 & 完整 Step 时间线

Full Activation Recomputation

# recompute_granularity="full"
# recompute_method="uniform"
# recompute_num_layers=1
#
# Forward: save only each layer's INPUT hidden states
#   Discard: attention scores, MoE intermediates, SwiGLU acts
#   Saved per layer: [1, 32768, 4096] * 2B = 256 MB
#   Total saved: 48 * 256 MB = 12.3 GB
#   (vs without recompute: ~42 GB activations)
#
# Backward: recompute each layer's forward before its backward
#   Layer 48: recompute fwd → compute grad → free
#   Layer 47: recompute fwd → compute grad → free
#   ...
#   Layer 1: recompute fwd → compute grad → free
