Linear Attention

从 O(N²) 到 O(N) — 用核技巧消除 softmax，实现线性复杂度注意力

1. 标准 Attention 的问题

核心洞察：去掉 softmax 后矩阵乘法满足结合律，可以先算 KᵀV (d×d) 再乘 Q，避免 N×N

$\text{Standard: } \mathbf{O} = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d}}\right) V \quad \Rightarrow \quad O(N^2)$ $\text{Linear: } \mathbf{O} = \varphi(Q) \cdot \bigl(\varphi(K)^\top V\bigr) \quad \Rightarrow \quad O(N \cdot d^2)$

2. 核心推导

Step 1 — Standard Attention (逐 token 输出)

$o_i = \sum_{j=1}^{N} \frac{\exp(q_i^\top k_j)}{\sum_{m=1}^{N} \exp(q_i^\top k_m)} \cdot v_j$

展开形式：对第 \(i\) 个 query，遍历所有 \(N\) 个 key 算 softmax 权重，再加权求和所有 value。

关键观察：分母 \(\sum_m \exp(q_i^\top k_m)\) 对求和变量 \(j\) 来说是常数（它独立遍历所有 \(m\)），可提到 \(\sum_j\) 外面：

$= \frac{\sum_{j=1}^{N} \exp(q_i^\top k_j) \cdot v_j}{\underbrace{\sum_{m=1}^{N} \exp(q_i^\top k_m)}_{= \sum_{j=1}^{N} \exp(q_i^\top k_j) \text{ (哑变量改名)}}}$ $= \frac{\sum_j \exp(q_i^\top k_j) \cdot v_j}{\sum_j \exp(q_i^\top k_j)}$

现在分子和分母是两个独立的求和，可以分别做 kernel trick 替换。这一步是从 "softmax 在求和内部" 变为 "分子/分母分离" 的关键。

Step 2 — Kernel Trick (去 softmax 化)

$\exp(q_i^\top k_j) \approx \varphi(q_i)^\top \varphi(k_j)$ $\varphi: \mathbb{R}^d \to \mathbb{R}^D, \quad \text{(非负特征映射)}$ $\Rightarrow o_i = \frac{\sum_{j=1}^{N} \varphi(q_i)^\top \varphi(k_j) \cdot v_j}{\sum_{j=1}^{N} \varphi(q_i)^\top \varphi(k_j)}$

Kernel trick：用特征映射 \(\varphi\) 将指数内积分解为两个映射后向量的内积。分子分母已经分离，直接逐项替换即可。此时公式结构不变，但为下一步的结合律创造条件。

Step 3 — Rearrange (交换求和顺序)

$\text{分子} = \sum_{j=1}^{N} \underbrace{\varphi(q_i)^\top \varphi(k_j)}_{\text{标量 (D 维内积)}} \cdot v_j$ $= \sum_{j=1}^{N} \left( \sum_{m=1}^{D} \varphi(q_i)_m \cdot \varphi(k_j)_m \right) \cdot v_j$ $\text{交换 } \sum_j \text{ 和 } \sum_m \text{ (有限求和交换律):}$ $= \sum_{m=1}^{D} \varphi(q_i)_m \cdot \left( \sum_{j=1}^{N} \varphi(k_j)_m \cdot v_j \right)$ $= \varphi(q_i)^\top \cdot \left[ \sum_{j=1}^{N} \varphi(k_j) \otimes v_j \right]$

同理分母：

$\text{分母} = \sum_{j=1}^{N} \varphi(q_i)^\top \varphi(k_j) = \varphi(q_i)^\top \left( \sum_{j=1}^{N} \varphi(k_j) \right)$

核心：\(\varphi(q_i)\) 对 \(j\) 是常数 → 可以提到 \(\sum_j\) 外面。结果变成 \(\varphi(q_i)^\top\) 乘以一个与 \(i\) 无关的累加矩阵。

Step 4 — State Matrix (定义全局状态)

$S = \sum_{j=1}^{N} \varphi(k_j) \otimes v_j \quad \in \mathbb{R}^{D \times d_v}$ $z = \sum_{j=1}^{N} \varphi(k_j) \quad \in \mathbb{R}^{D}$ $o_i = \frac{\varphi(q_i)^\top \cdot S}{\varphi(q_i)^\top \cdot z}$

\(S\) 和 \(z\) 只需算一次 (\(O(N \cdot D \cdot d_v)\))，之后每个 \(o_i\) 只需 \(O(D \cdot d_v)\) 查询。总复杂度 \(O(N \cdot D^2)\)，当 \(D \ll N\) 时为线性。

Step 3 为什么成立：
\(\varphi(q_i)^\top \varphi(k_j)\) 是标量，所以 \(\sum_j (\text{scalar}_j) \cdot v_j\) 可以把标量拆开，把跟 \(i\) 无关的部分 (\(\varphi(k_j), v_j\)) 预先累加成矩阵 \(S\)。本质是矩阵乘法结合律：先算 \(K^\top V\) 再乘 \(Q\)。

3. 递推形式（RNN 视角）

RNN 基础结构回顾

任何 RNN 都由三个核心概念组成：

$h_t = f(h_{t-1}, x_t) \quad \text{— 状态更新规则}$ $y_t = g(h_t) \quad \text{— 输出读取规则}$
RNN 概念物理意义类比
隐状态 \(h_t\)对历史信息的压缩记忆，有限容量的"记忆体"人脑的工作记忆
输入 \(x_t\)当前时刻新到达的信息当前读到的单词
更新规则 \(f\)如何把新信息写入记忆决定记住什么
输出规则 \(g\)如何从记忆中读取相关信息回忆相关内容
容量瓶颈\(h_t\) 的大小固定 → 越老的信息越模糊记忆遗忘

经典 RNN 的隐状态是一个 \(d\) 维向量，容量极小。Linear Attention 的突破在于：将隐状态升级为 \(d \times d\) 矩阵，容量从 \(O(d)\) 提升到 \(O(d^2)\)。

Linear Attention 的递推形式

$\text{写入 (Write): } S_t = S_{t-1} + \varphi(k_t) \otimes v_t$ $\text{归一化更新: } z_t = z_{t-1} + \varphi(k_t)$ $\text{读取 (Read): } o_t = \frac{\varphi(q_t)^\top \cdot S_t}{\varphi(q_t)^\top \cdot z_t}$

对比经典 RNN 的 \(h_t = \tanh(W_h h_{t-1} + W_x x_t)\)，Linear Attention 的更新是纯加法（无非线性激活、无矩阵乘），这使得：

  • 训练时可以并行展开（前缀和 / chunk-wise）
  • 推理时等价为 RNN（逐步更新 \(S_t\)）
  • 但缺点：纯加法 → 只能累加，不能覆盖旧记忆 (DeltaNet 解决此问题)

从 Step 4 到递推：因果约束

$\text{非因果 (全局): } S = \sum_{j=1}^{N} \varphi(k_j) \otimes v_j \quad \text{(一次性算完)}$ $\text{因果 (递推): } S_t = \sum_{j=1}^{t} \varphi(k_j) \otimes v_j = S_{t-1} + \varphi(k_t) \otimes v_t$

因果约束要求第 \(t\) 步只看前 \(t\) 个 token，所以 \(S\) 的求和上界从 \(N\) 变为 \(t\) → 自然得到递推更新。

各量的物理意义

量形状物理意义
\(S_t\)\(\mathbb{R}^{D \times d_v}\)记忆矩阵 — 存储截至 \(t\) 步的所有 key-value 关联。第 \(m\) 行 = "第 \(m\) 种特征对应的所有 value 加权和"
\(z_t\)\(\mathbb{R}^D\)访问计数器 — 记录每个特征维度被"写入"了多少次，用于归一化防止输出过大
\(\varphi(k_t)\)\(\mathbb{R}^D\)写地址 — 决定当前 value 写入记忆矩阵的哪些"槽位"（哪些行）
\(v_t\)\(\mathbb{R}^{d_v}\)写内容 — 要存入记忆的信息
\(\varphi(q_t)\)\(\mathbb{R}^D\)读地址 — 决定从记忆矩阵的哪些"槽位"读取信息
\(o_t\)\(\mathbb{R}^{d_v}\)读出结果 — 从记忆中提取出的 value 组合

记忆机制类比

写操作 \(\varphi(k_t) \otimes v_t\)：

= "在地址 \(\varphi(k_t)\) 处存入内容 \(v_t\)"
外积 = 将 \(v_t\) 广播写入到 \(\varphi(k_t)\) 指定的所有行

读操作 \(\varphi(q_t)^\top S_t\)：

= "用地址 \(\varphi(q_t)\) 查询记忆"
内积 = 按 \(\varphi(q_t)\) 的权重加权所有行

核心限制：

写操作只能 += (累加)，不能覆盖
→ 记忆只增不删 → 老信息永不消失但被稀释
→ GLA 加衰减门控, DeltaNet 加擦除

为什么是 "线性" RNN：
更新规则 \(S_t = S_{t-1} + \varphi(k_t) \otimes v_t\) 对 \(S_{t-1}\) 是线性变换（系数为 1 的恒等映射 + 加法）。这使得整条链可以用前缀和 (cumsum) 并行计算，不像经典 RNN 那样必须串行展开。

每个 token 到达时：(1) 输入产生 q,k,v → (2) 用 k,v 写入记忆 S → (3) 用 q 读取输出 o → (4) S 传递到下一步（循环）

4. 实现代码

import torch
import torch.nn.functional as F

def phi(x):
    """Feature map: 1 + elu ensures non-negative (causal-compatible)"""
    return 1 + F.elu(x)

def linear_attention_parallel(q, k, v):
    """Parallel mode — for training (processes full sequence at once)"""
    q, k = phi(q), phi(k)              # [B, H, N, d]
    kv = torch.einsum('bhnd,bhnm->bhdm', k, v)  # S = K^T V: [B,H,d,d]
    z = k.sum(dim=2)                    # normalizer: [B,H,d]
    out = torch.einsum('bhnd,bhdm->bhnm', q, kv)  # Q·S
    out = out / torch.einsum('bhnd,bhd->bhn', q, z).unsqueeze(-1)
    return out

def linear_attention_recurrent(q, k, v):
    """Recurrent mode — for inference (O(1) per token, no KV cache)"""
    B, H, N, d = q.shape
    q, k = phi(q), phi(k)
    S = torch.zeros(B, H, d, d, device=q.device)  # state matrix
    z = torch.zeros(B, H, d, device=q.device)      # normalizer
    outputs = []
    for t in range(N):
        S = S + torch.einsum('bhd,bhm->bhdm', k[:,:,t], v[:,:,t])
        z = z + k[:,:,t]
        o_t = torch.einsum('bhd,bhdm->bhm', q[:,:,t], S)
        o_t = o_t / torch.einsum('bhd,bhd->bh', q[:,:,t], z).unsqueeze(-1)
        outputs.append(o_t)
    return torch.stack(outputs, dim=2)
双模式：训练用 parallel 模式（矩阵乘，GPU 友好）；推理用 recurrent 模式（O(1) per token，无需 KV cache，恒定显存）。

5. Causal Linear Attention

Causal 场景下，chunk-wise 方法是实际标准：chunk 内用 quadratic attention 保持 GPU 效率，chunk 间用线性递推

$\text{Causal: } S_t = \sum_{j \leq t} \varphi(k_j) \otimes v_j \quad \text{(只看过去)}$ $\text{Chunk-wise: } S_{\text{chunk}} = S_{\text{prev}} + \sum_{j \in \text{chunk}} \varphi(k_j) \otimes v_j$ $\text{Chunk 内用 } O(C^2) \text{ 标准 attention}, \quad \text{Chunk 间传递 } O(d^2) \text{ 状态}$

7. 优劣对比

核心 tradeoff：softmax attention 用 O(N) 显存换精确 recall；linear attention 用 O(d²) 固定状态换线性复杂度

$\text{Softmax Attention — 训练: } O(N^2 d), \quad \text{推理 KV cache: } O(N \cdot d)$ $\text{Linear Attention — 训练: } O(N d^2), \quad \text{推理状态: } O(d^2) \text{ (恒定)}$ $\text{当 } d \ll N \text{ 时 (如 } d{=}128,\; N{=}128\text{K)}: \quad N d^2 \ll N^2 d$
2025-2026 趋势：混合架构成为主流 — 大部分层用 linear attention/SSM 处理长距离依赖，少数层（如每 4-8 层）插入 softmax attention 保证精确 recall。代表：Jamba (Mamba+Attention)、Zamba、Griffin、NVIDIA Hymba。
