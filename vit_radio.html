1. 多教师蒸馏框架

# AM-RADIO: Agglomerative Model
# 同时蒸馏多个教师

teachers = [
    CLIP_ViT_H,      # 语义对齐
    DINOv2_ViT_g,    # 局部特征
    SAM_ViT_H,       # 分割能力
]

student = ViT_H(632M)

for image in dataloader:
    s_feat = student(image)

    loss = 0
    for teacher, head in zip(teachers, heads):
        t_feat = teacher(image)  # frozen
        s_out = head(s_feat)
        loss += distill_loss(s_out, t_feat)

    loss.backward()

核心思想

不同 ViT 教师擅长不同能力：CLIP 擅长语义对齐、DINOv2 擅长局部特征、SAM 擅长分割。

RADIO 用一个 student 同时学习所有教师的能力，通过独立的 task-specific head 匹配各教师输出。

$\mathcal{L} = \sum_{t \in \text{teachers}} \lambda_t \cdot \|\text{head}_t(f_s) - f_t\|_2^2$

训练后去掉 heads，backbone 本身就具备多种能力。

2. 模型架构

RADIO 的目标：训练一次，用于所有视觉任务 — drop-in replacement for CLIP/DINO/SAM
