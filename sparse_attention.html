Sparse Attention

从 O(N²) 到 O(N√N) 或 O(N·k) — 通过稀疏化 attention mask 降低长序列计算量

1. 为什么需要 Sparse Attention

# Dense Attention 的问题
# 序列长度 N, 复杂度 O(N²)

N = 128000  # 128K context
d = 128     # head dim

# Dense: 每个 token attend 所有 token
attn_matrix = N × N = 16.4B elements
memory = 16.4B × 2 bytes = 32GB
# 单头就 32GB — 不可行!

# Sparse: 每个 token 只看部分 token
# 关键观察:
# 1. attention 天然稀疏
#    (大部分权重接近 0)
# 2. 局部性很强
#    (相邻 token 最相关)
# 3. 少量全局 token 够用
#    (不需要全连接)

# Sparse 目标:
# 保留最关键的连接
# 丢弃冗余的 attention 权重
# O(N²) → O(N·k) 或 O(N√N)

核心动机

长序列（128K+）下 dense attention 内存和计算都不可行。但 attention 权重分布天然稀疏，大部分接近零。

$\text{Dense: } O(N^2 \cdot d)$ $\text{Sparse: } O(N \cdot k \cdot d), \; k \ll N$ $k = w + g \quad (w\text{: 窗口}, g\text{: 全局})$

两种思路

Static sparse: 预定义 pattern（窗口、条纹、全局 token），无需数据依赖

Dynamic sparse: 根据 QK 相似度动态选 top-k，数据依赖但更精确

2. 主流 Sparse Pattern

四种主流 Sparse Attention Pattern：从静态到动态，从简单到灵活

3. 代表方法详解

# 1. Mistral Sliding Window Attention
window_size = 4096
# 每层只看 4K 窗口
# 但多层堆叠后感受野扩大:
# L 层后感受野 = L × window_size
# 32层 × 4K = 128K 有效上下文

# 2. Longformer (Window + Global)
window = 512
global_tokens = [CLS, SEP]  # 特定 token
# global token attend 全部
# 其他 token 只看 window + global

# 3. DeepSeek NSA (Native Sparse)
# 三路并行:
# a) Compression: 分块压缩 KV
blocks = chunk(K, block_size=64)
K_compressed = linear(mean(blocks))
# b) Selection: top-k 重要块
scores = Q @ K_compressed.T
top_blocks = topk(scores, k=16)
# c) Sliding window: 局部细节
K_local = K[pos-w:pos+w]

# 三路输出加权合并
out = gate_c * attn_compress \
    + gate_s * attn_select \
    + gate_w * attn_window

NSA 三路设计

Compression: 粗粒度全局 — 把整个序列压缩为 N/64 个 token，提供全局 context

Selection: 精确远程 — 基于压缩 score 选 top-k 块，展开做精细 attention

Window: 局部细节 — 滑动窗口保证近邻信息不丢

$O = g_c \cdot \text{Attn}(Q, K_c, V_c) + g_s \cdot \text{Attn}(Q, K_s, V_s) + g_w \cdot \text{Attn}(Q, K_w, V_w)$ $K_c = \text{Linear}(\text{MeanPool}(K, 64))$ $K_s = K[\text{top-}k\text{ blocks}], \; k=16$

可训练稀疏

NSA 的 gate 和 selection 都是可训练的，模型自己学习什么时候看全局、什么时候看局部。从预训练开始使用，不是后处理。

4. 方法对比

方法Pattern复杂度训练/推理代表模型
Sliding Window固定窗口O(N·w)训练+推理Mistral, Gemma2, Phi-3
Longformer窗口+全局 tokenO(N·(w+g))训练+推理Longformer, LED, BigBird
Sparse Transformer窗口+条纹O(N·√N)训练+推理GPT-3 (部分层)
NSA (DeepSeek)压缩+选择+窗口O(N·(N/b+k·b+w))训练+推理DeepSeek-V3
MInference动态 A-shape/BlockO(N·k)仅推理加速LLaMA/GPT-4 推理
H2OHeavy Hitter + RecentO(N·k)仅推理 (KV eviction)通用推理优化
StreamingLLMSink tokens + WindowO(N·(s+w))仅推理 (无限长)通用流式推理
$\text{Sliding Window 感受野: } L \times w \quad (L\text{层} \times w\text{窗口} = \text{有效上下文})$ $\text{NSA 有效 attend: } \frac{N}{b} + k \cdot b + w \approx 2K\text{-}3K \quad (N=128K)$ $\text{Longformer: } O(N \cdot w + N \cdot g + g \cdot N) = O(N \cdot (w + 2g))$
训练 vs 推理稀疏：Mistral/NSA 是训练时就使用稀疏 pattern，模型从头学习在稀疏约束下工作。MInference/H2O/StreamingLLM 是推理时对 dense 模型做稀疏加速（不改变训练）。前者效果更好（模型适应了稀疏），后者更通用（不需要重新训练）。
趋势：2024-2025 年的方向是将 sparse attention 融入预训练（NSA、Gemma2 的交替 sliding/global 层），而非作为推理 hack。DeepSeek-V3 的 NSA 证明了可训练动态稀疏在 128K 上下文下可以保持 dense 水平的质量，同时训练速度提升 2-4×。未来 MoA (Mixture of Attention) — 不同 head 用不同 sparse pattern — 可能成为主流。
