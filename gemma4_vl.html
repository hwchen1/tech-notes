1. Hybrid Sliding/Global Attention

Pattern (5,1): 5 sliding window layers (head_dim=256, 8 KV heads, window=1024) then 1 global layer (head_dim=512, 2 KV heads, K=V tying, RoPE base=1M). Repeating.
class HybridAttention(nn.Module):
    # Pattern: [S,S,S,S,S,G, S,S,S,S,S,G, ...]

    def forward(self, x, layer_idx):
        is_global = (layer_idx % 6) == 5

        if is_global:
            # head_dim=512, 2 KV heads
            # RoPE base=1M, partial_rotary=0.25
            q = self.q_proj(x)  # full seq
            k = self.kv_proj(x) # K=V tied!
            v = k               # same weights
            attn = full_attention(q, k, v)
        else:
            # head_dim=256, 8 KV heads
            # RoPE base=10000, window=1024
            q = self.q_proj(x)
            k = self.k_proj(x)
            v = self.v_proj(x)
            attn = sliding_window(
                q, k, v, window=1024
            )
        return self.o_proj(attn)
Why hybrid?
  • Sliding window layers are cheap — O(N*w) — and handle local patterns well
  • Global layers every 6th position allow long-range information flow
  • Global layers use larger head_dim (512 vs 256) but fewer KV heads (2 vs 8) — same KV cache budget
  • RoPE base 1M on global layers enables extrapolation to very long contexts
  • partial_rotary_factor=0.25 means only 25% of head_dim gets rotary encoding in global layers — rest is position-agnostic

2. All-MoE Architecture

Every layer is MoE: 128 experts, top-8 routing, expert ffn_dim=704, shared expert ffn_dim=2112. No dense FFN layers anywhere.
class Gemma4MoELayer(nn.Module):
    # 128 experts, top-8 gating
    # + 1 shared expert (always active)

    def forward(self, x):
        # Router: select top-8 of 128
        scores = self.gate(x)  # [B,N,128]
        topk_w, topk_idx = scores.topk(8)
        topk_w = softmax(topk_w)

        # Routed experts (ffn_dim=704)
        routed = 0
        for i, idx in enumerate(topk_idx):
            expert = self.experts[idx]
            routed += topk_w[i] * expert(x)

        # Shared expert (ffn_dim=2112)
        shared = self.shared_expert(x)

        return routed + shared
All-MoE design choices:
  • 128 experts, top-8: High capacity with sparse activation — only 8/128 = 6.25% of expert params active per token
  • Expert ffn_dim=704: Small per-expert — keeps active param count manageable
  • Shared expert (2112): Always-on expert captures common patterns, stabilizes training
  • No dense layers: Unlike Mixtral/DeepSeek which alternate dense+MoE, Gemma4 is pure MoE throughout
  • Effective active FFN: 8×704 + 2112 = 7744 active dim per token

3. Dual Pre-Norms + Logit Soft-Capping

Two RMSNorm before attention (fused at load time), per-layer scalar multiplier, post-FFN norm, and logit soft-capping at 30.
class Gemma4Block(nn.Module):
    def __init__(self, d_model):
        # Dual pre-norms (fused into one)
        self.norm1a = RMSNorm(d_model)
        self.norm1b = RMSNorm(d_model)
        # Per-layer scalar
        self.alpha = nn.Parameter(torch.ones(1))
        # Post-FFN norm
        self.post_ffn_norm = RMSNorm(d_model)

    def forward(self, x):
        # Dual pre-norm (fused: norm1a * norm1b)
        h = self.norm1a(x)
        h = self.norm1b(h)
        # Attention + scalar multiplier
        h = self.attn(h)
        x = x + self.alpha * h

        # FFN block
        h = self.ffn_norm(x)
        h = self.moe(h)
        h = self.post_ffn_norm(h)
        x = x + h
        return x

def soft_cap_logits(logits, cap=30.0):
    # Prevents logit explosion
    return cap * torch.tanh(logits / cap)
Normalization strategy:
  • Dual pre-norm: Two consecutive RMSNorm before attention — empirically improves training stability at scale
  • Fused at load: The two norms are fused into a single operation for inference efficiency (norm(norm(x)) = combined_norm(x))
  • Per-layer α: Learnable scalar multiplier on attention output — controls residual contribution per layer
  • Post-FFN norm: Additional RMSNorm after MoE output, before residual addition — prevents FFN output magnitude drift
  • Logit soft-cap (30): tanh(logits/30)*30 prevents extreme logit values without hard clipping — smooth gradient everywhere

4. Vision Encoder (SigLIP + Projector)

SigLIP-based vision encoder with parameter-free RMSNorm projector. Bidirectional attention within image token groups — images attend freely to each other, text remains causal.
class Gemma4Vision(nn.Module):
    # HF: Gemma4VisionModel (SigLIP)

    def forward(self, images, text_ids):
        # Encode images with SigLIP
        img_tokens = self.vision_encoder(images)
        # [B, num_patches, vision_dim]

        # Project: parameter-free RMSNorm
        img_tokens = rms_norm(img_tokens)
        img_tokens = self.linear_proj(img_tokens)
        # [B, num_patches, d_model]

        # Merge into text sequence
        hidden = interleave(text_ids, img_tokens)

        # Attention mask:
        # - text tokens: causal mask
        # - image tokens: bidirectional
        #   within same image group
        mask = build_hybrid_mask(hidden)
        return self.decoder(hidden, mask)
Vision design:
  • SigLIP encoder: Sigmoid-loss ViT — no [CLS] token, all patch tokens used
  • Parameter-free RMSNorm: Projector uses RMSNorm without learnable gain — just normalizes magnitude before linear projection
  • Bidirectional image attention: Image tokens within the same image attend to each other freely (no causal mask) — captures spatial relationships
  • Text stays causal: Text tokens still use standard causal masking — only image groups break causality
  • Cross-image isolation: Different images in the same sequence do NOT attend to each other

5. K=V Tying in Global Layers

Key and Value share the same weight matrix in global attention layers — reduces KV cache by 50% for long-context inference while maintaining model quality.
class GlobalAttentionKVTied(nn.Module):
    # Global layers: K and V share weights
    # head_dim=512, num_kv_heads=2

    def __init__(self, d_model):
        self.q_proj = Linear(
            d_model, num_q_heads * 512)
        # Single projection for both K and V
        self.kv_proj = Linear(
            d_model, 2 * 512)  # 2 heads
        # No separate v_proj!

    def forward(self, x):
        q = self.q_proj(x)
        kv = self.kv_proj(x)

        # K = V = same projection output
        k = kv
        v = kv  # tied!

        # RoPE only on 25% of head_dim
        q_rot = q[..., :128]  # 512*0.25
        k_rot = k[..., :128]
        q_rot = apply_rope(q_rot, base=1e6)
        k_rot = apply_rope(k_rot, base=1e6)

        return attention(q, k, v)
K=V Tying rationale:
  • 50% KV cache reduction: Only store one tensor instead of two for global layers — critical for long-context serving
  • Why it works: In practice, K and V learn similar representations in deep layers — tying them loses little expressivity
  • Only in global layers: Sliding window layers keep separate K,V since they handle fine-grained local patterns
  • Partial RoPE (25%): Only 128/512 dims get positional encoding — the rest are position-invariant, enabling better length generalization
  • RoPE base 1M: Very high base frequency allows smooth extrapolation to sequences far beyond training length
