1. Hybrid GDN + Gated Attention

64 layers total: repeating pattern of 3x GDN (Gated DeltaNet) + 1x Gated Attention per group of 4 layers. GDN provides linear-complexity recurrence; gated attention provides full expressiveness at regular intervals.

class GatedDeltaNet(nn.Module):
    """Gated DeltaNet: linear attention with delta rule"""
    def forward(self, x):
        q, k, v = self.qkv(x).chunk(3, dim=-1)
        beta = self.beta_proj(x).sigmoid()
        phi_k = self.feature_map(k)  # φ(k)

        # Delta rule update: erase then write
        # S_t = S_{t-1} - β·(S·φ(k))⊗φ(k) + φ(k)⊗v
        erase = beta * (S @ phi_k.unsqueeze(-1))
        S = S - erase @ phi_k.unsqueeze(-2) \
            + phi_k.unsqueeze(-1) @ v.unsqueeze(-2)

        o = (phi_q.unsqueeze(-2) @ S).squeeze(-2)
        return self.gate(x) * self.out_proj(o)

class GatedAttention(nn.Module):
    """Standard softmax attention with output gating"""
    def forward(self, x):
        q, k, v = self.qkv(x).chunk(3, dim=-1)
        attn = softmax(q @ k.T / sqrt(d)) @ v
        return self.gate(x) * self.out_proj(attn)

Why Hybrid?

GDN layers run in O(N) via recurrent state updates - ideal for long sequences. But pure linear attention loses expressiveness.

Every 4th layer uses full softmax attention to "correct" accumulated approximation errors and capture global dependencies.

$3 \times \text{GDN} + 1 \times \text{Attn} = 1 \text{ group}$ $16 \text{ groups} = 64 \text{ layers}$

The delta rule improves over vanilla linear attention: it can selectively erase outdated key-value associations via the learned beta gate.

Output gating on both GDN and Attention layers provides adaptive information flow control.

2. Vision Encoder

Conv3d patch embedding handles video (temporal_patch_size=2) and images. Learnable 2D positional embedding on a 48x48 grid with bicubic interpolation for arbitrary resolutions. 2D RoPE inside ViT blocks.

class VisionEncoder(nn.Module):
    def __init__(self):
        # Conv3d: handles temporal dim for video
        self.patch_embed = nn.Conv3d(
            in_channels=3,
            out_channels=hidden_dim,
            kernel_size=(2, 16, 16),  # t=2, h=16, w=16
            stride=(2, 16, 16)
        )
        # Learnable 2D pos embed (48x48 grid)
        self.pos_embed = nn.Parameter(
            torch.randn(1, 48*48, hidden_dim))

    def forward(self, pixels):
        # pixels: [B, C, T, H, W]
        x = self.patch_embed(pixels)  # [B, D, T/2, H/16, W/16]
        x = x.flatten(2).transpose(1,2)  # [B, N, D]

        # Interpolate pos embed to actual grid size
        pos = interpolate_2d(
            self.pos_embed, target_h, target_w)
        x = x + pos

        # ViT blocks with 2D RoPE
        for blk in self.blocks:
            x = blk(x, rope_2d=self.rope_2d)
        return x

Patch Embedding

Conv3d with temporal kernel=2 merges 2 consecutive frames into one temporal token, reducing video sequence length by 2x.

Spatial patch size 16x16 means a 224x224 image becomes 14x14 = 196 tokens.

Positional Encoding

Learnable 2D grid (48x48 = 2304 positions) supports up to 768x768 natively. Larger resolutions use bicubic interpolation.

2D RoPE inside ViT blocks encodes relative spatial position, enabling better generalization to unseen resolutions.

3. mRoPE (Multimodal RoPE)

Head dimension split into 3 parts for temporal/height/width positional encoding. Language tokens share the same temporal ID; vision tokens use spatial coordinates.

class MultimodalRoPE(nn.Module):
    """mRoPE: split head_dim into 3 parts for 3 axes"""
    # head_dim = 32 → [11, 11, 10] for t, h, w

    def forward(self, q, k, pos_ids):
        # pos_ids: [B, N, 3] → (t_id, h_id, w_id)
        # Split q,k along head_dim
        q_t, q_h, q_w = q.split([11,11,10], dim=-1)
        k_t, k_h, k_w = k.split([11,11,10], dim=-1)

        # Apply RoPE per axis
        q_t = apply_rope(q_t, pos_ids[...,0])
        q_h = apply_rope(q_h, pos_ids[...,1])
        q_w = apply_rope(q_w, pos_ids[...,2])

        k_t = apply_rope(k_t, pos_ids[...,0])
        k_h = apply_rope(k_h, pos_ids[...,1])
        k_w = apply_rope(k_w, pos_ids[...,2])

        return cat([q_t,q_h,q_w]), cat([k_t,k_h,k_w])

    # Language tokens: t=position, h=0, w=0
    # Vision tokens:  t=frame_id, h=row, w=col

Multimodal Position Encoding

Standard RoPE uses a single position ID per token. mRoPE splits the head dimension into 3 axes and applies independent rotary embeddings to each.

$\text{head\_dim} = 32 \rightarrow [11, 11, 10]$ $\text{axes: temporal, height, width}$

Language: Only temporal axis increments (h=0, w=0 always). This degenerates to standard 1D RoPE.

Vision: Temporal axis is the frame ID (same for all patches in one image). Height and width encode spatial grid position.

This unified scheme lets attention naturally compute spatial proximity for vision tokens and sequential proximity for language tokens.

4. MoE (Mixture of Experts)

512 experts with top-10 routing. Sigmoid gating with learned expert bias. Shared expert with gate for baseline capacity. First layer is dense, remaining layers are MoE.

class MoELayer(nn.Module):
    """512 experts, top-10 sigmoid routing"""
    def __init__(self, hidden, expert_dim, n_experts=512):
        self.gate = nn.Linear(hidden, n_experts)
        self.expert_bias = nn.Parameter(
            torch.zeros(n_experts))
        self.experts = nn.ModuleList([
            FFN(hidden, expert_dim)
            for _ in range(n_experts)])
        self.shared_expert = FFN(hidden, expert_dim)
        self.shared_gate = nn.Linear(hidden, 1)

    def forward(self, x):
        # Sigmoid routing with bias
        logits = self.gate(x) + self.expert_bias
        scores = logits.sigmoid()  # not softmax!

        # Top-10 selection
        topk_scores, topk_idx = scores.topk(10)
        topk_scores = topk_scores / topk_scores.sum(-1, keepdim=True)

        # Dispatch and combine
        out = dispatch_combine(x, topk_idx, topk_scores)

        # Shared expert (always active)
        shared = self.shared_expert(x)
        shared = shared * self.shared_gate(x).sigmoid()
        return out + shared

Sigmoid Routing

Unlike softmax routing (which forces competition), sigmoid gating allows independent expert activation scores. This avoids the "winner-take-all" problem.

$\text{score}_i = \sigma(W \cdot x + \text{bias}_i)$ $\text{top-10 selected, renormalized}$

Expert Bias

Learned per-expert bias helps balance load without auxiliary loss. Experts with low utilization get higher bias, encouraging routing diversity.

Shared Expert

A dedicated shared expert processes every token (gated by sigmoid). This ensures baseline capacity even when routed experts are suboptimal for a given input.

Layer 0 is dense (no routing) to establish stable representations before sparse routing begins.

5. Deepstack Visual Injection

Visual tokens are injected at intermediate layers rather than only prepended at the input. This enables deeper cross-modal fusion throughout the network.

class DeepstackVLM(nn.Module):
    """Inject visual tokens at multiple depths"""
    def __init__(self, n_layers=64, inject_layers=None):
        # inject_layers: e.g. [0, 8, 16, 24, 32]
        self.inject_layers = inject_layers or [
            0, 8, 16, 24, 32]
        self.vis_projectors = nn.ModuleDict({
            str(l): nn.Linear(vis_dim, hidden_dim)
            for l in self.inject_layers
        })

    def forward(self, text_tokens, vis_tokens):
        h = self.embed(text_tokens)

        for i, layer in enumerate(self.layers):
            if i in self.inject_layers:
                # Project and inject visual tokens
                v = self.vis_projectors[str(i)](vis_tokens)
                h = self.fuse(h, v)  # concat or cross-attn

            h = layer(h)

        return h

Why Deepstack?

Traditional VLMs prepend visual tokens only at layer 0. By the time information reaches deeper layers, visual signal is diluted.

Deepstack re-injects visual features at multiple intermediate layers, giving the LLM fresh visual context at different abstraction levels.

Benefits

- Lower layers get raw visual features for grounding

- Middle layers get refined features for reasoning

- Reduces the "forgetting" problem in deep networks

- Each injection point has its own projector, allowing layer-specific adaptation of visual representations.

$\text{Inject at layers: } [0,\, 8,\, 16,\, 24,\, 32]$ $\text{Each with independent Linear projector}$
