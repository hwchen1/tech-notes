四层亮点: 不追求单点创新，追求全栈系统化的持续改进能力

亮点深度解读

① NoPE (No Position Encoding)

Global attention 层不使用任何位置编码。引用 Kazemnejad et al. 2023 (arXiv 2305.19466)：NoPE 在 length generalization 上优于 RoPE/ALiBi，且更高效。实测在 scaling ladder 上与 RoPE 持平。

⑤ Adaptive Entropy Control

RL 爬坡最大风险是 entropy collapse（模型过早确定策略）。传统做法加 entropy bonus 超参敏感。MAI 用积分控制器：熵低→自动放宽 clip→鼓励探索；熵高→收紧→稳定训练。

⑥ Self-Distillation

解决三个问题：(1) 爬坡崩溃恢复 (2) 新基模型继承旧 RL 进度 (3) 过滤 reward hacking。~1M traces 足够。多 checkpoint 采样优于单一最终 checkpoint。

⑨ Rank Non-invariance

关键发现：小模型上最优的数据配比在大模型上不是最优的。STEM-heavy 在小规模赢，但 Code-heavy 在大规模赢（因为 STEM 数据多样性不足）。必须在目标规模验证。

⑪ Bitwise 确定性：完全相同配置 → 完全相同模型。需要控制：数据管道顺序、checkpoint 全状态、GPU kernel 累加顺序、NCCL 拓扑固定、禁用 NVLink SHARP。代价：MFU 有少量损失。收益：调试、回溯、科学复现。

3. 模型架构设计

# ====== MAI-Base-1 架构规格 ======
# Type: Decoder-only Transformer
# Active: 34.7B | Total: 962B
# Layers: 78
# Hidden: 6656
# FFN (dense): 13312
# Down-proj: 3072 (LatentMoE)
# Expert FFN: 10240
# Experts: 512, top-k=8
# KV heads: 8 (GQA), head_dim=128
# Q heads: 80
# Tokenizer: o200k_base (200K vocab)
# Embedding: tied input/output
# Norm: RMSNorm (pre + post residual)
# Bias: none
# Activation: SwiGLU

# ====== 注意力设计 ======
# 周期: 5 local + 1 global (per block)
# Local:
#   RoPE, base_freq=10000
#   Sliding window = 512 tokens
# Global:
#   NoPE (无位置编码)
#   Full context (无 window 限制)
# QK-RMSNorm (稳定大规模训练)
# FlashAttention-4 + Ulysses CP

# ====== FFN 设计 ======
# 交替排列:
# Layer 0: Local Attn + Dense FFN
# Layer 1: Local Attn + MoE
# Layer 2: Local Attn + Dense FFN
# Layer 3: Local Attn + MoE
# Layer 4: Local Attn + Dense FFN
# Layer 5: Global Attn + MoE
# ... repeat ×13 (= 78 layers)

# ====== LatentMoE ======
# token → shared_down_proj (6656→3072)
# → all-to-all dispatch (压缩空间)
# → expert FFN (3072→10240→3072)
# → all-to-all combine
# → shared_up_proj (3072→6656)
# 好处: 通信量降低 ~2× (6656→3072)

架构核心: 周期性 Local/Global 注意力 + 交替 Dense/MoE FFN + LatentMoE 压缩通信

关键架构决策

为什么 5:1 Local:Global？

Follow Gemma 3 设计。大幅减少 KV cache：只有 1/6 的层需要存全量 KV。训练时减少注意力计算量。推理时 KV cache 降低 ~5×。

为什么 NoPE 而非 RoPE？

Global 层做 full-context attention，NoPE 在 length generalization 上表现更好（causal mask 已隐式提供位置信息）。Scaling ladder 实验证实与 RoPE 持平，但更高效。

为什么交替 Dense+MoE？

Every-layer MoE 在 EG_FLOPs 上只高 3%，但 wall-clock (EG_Time) 上反而差 18%。原因：all-to-all 通信无法完全隐藏。交替设计让 dense 层 "缓冲" 通信延迟。

为什么 LatentMoE？

Expert parallelism 需要 all-to-all 通信。LatentMoE 在 dispatch 前将 token 从 hidden_dim(6656) 压缩到 down_dim(3072)，通信量减半。与 NVIDIA 联合设计。

Attention Output Zero-Init：解决初始化时注意力做 causal mean pooling 导致 token 表示坍缩→MoE routing 极度不平衡的问题。设 output RMSNorm gain=0，让模型初始行为等价于纯 FFN stack。

架构演进：5代 (v2→v5)，每代都是模型改进+基础设施优化的共同演进。最终 MFU 从 v2 的 18% 提升到 v5 的 20%+，同时 EG 增长 1.69×。

4. 训练设计：RL 爬坡方法论

# ====== RL 目标函数 (GRPO-based) ======
# J(θ) = E_q,y[1/Σ|yi| · Σ_i Σ_t
#   min(r_it·A_i, clip(r_it,1-ε,1+ε+k)·A_i)]
#
# r_it = π_θ(y_it|q,y_<t) / π_old(...)
# A_i = (R_i - mean(R_1:G)) / std(R_1:G)
# Token-level: 每个 token 贡献相等

# ====== Reward 分解 ======
# R(q,y) = R_task(q,y)
#        + w_lang · R_lang(y)
#        - w_len · R_len(y)
#
# R_task: 领域相关
#   STEM: 代码执行验证正确性
#   Agentic: 测试通过率 + AI judge
#   Helpfulness: AI judge + RM
#
# R_lang: 语言一致性 (防 CoT 混语言)
#   = max(1 - α·n_non_english, 0)
#   RL 后期 CoT 容易混入外语 token
#
# R_len: 长度惩罚
#   = ρ_q · |y| / ℓ_max
#   防止输出无限膨胀

# ====== 三个 Climb ======
# STEM Climb (最长):
#   数据: 自研 PDF→QA pipeline
#   Reward: 代码执行 / 数值比对
#   数千步, 多次 self-distillation
#
# Agentic Climb:
#   数据: SWE environments (Docker)
#   工具: bash + string_replace (仅2个)
#   多步 trajectory RL
#   混合 STEM 稳定训练
#
# Helpfulness Climb:
#   数据: 人类偏好 + 安全标注
#   Reward: AI judge + trained RM
#   Safety-Helpfulness 双目标

# ====== 合并策略 ======
# 三域专家 → Trace SFT → Final Climb
# Self-distillation SFT:
#   ~1M traces 足够
#   多 checkpoint 采样 > 单一最终
#   dropout=0.15 (高, 防止过拟合)
#   load_balance_coeff=1e-2 (高)
# Final Climb:
#   全域 RL, 统一模型

# ====== Hyperparams ======
# Optimizer: AdamW (β1=β2=0.95, ε=1e-15)
# LR: 1e-6 (constant, no warmup)
# Batch: 7040 packed (max 12000 unpacked)
# No weight decay

YOLO 分布式框架

自研框架，支持全训练阶段。关键特性：

• FP8 混合精度 + stochastic rounding

• Custom Triton/CUDA/CuteDSL kernels

• Dropless MoE (变长 all-to-all)

• ZeRO-2 optimizer sharding

• Activation offloading to host memory

• Bitwise 确定性 (所有累加固定顺序)

• 最终 MFU: ~20% on 8K GB200

Goodput 为生产 KPI：不只看 MFU，看"实际有用的训练时间 / 总 wall-clock"。故障不只丢时间，还触发 recompute、placement 扰动、MFU 降低。每个故障模式都有 owner。

为什么 RL 从零开始？

MAI-Thinking-1 是 Microsoft AI 的第一个推理模型。RL 爬坡从 mid-trained checkpoint 出发，没有任何推理 trace 的先验暴露。模型完全通过 RL 学会推理格式和策略——这验证了 RL 本身的充分性。

Agentic Climb 的关键洞察：只给 bash + string_replace 两个工具。RL 训练后模型自己发现了 grep→read→edit→test→revert→retry 的复杂策略。工具越简单→组合空间越大→RL 越能创新。这些策略不是 SFT 教的。

数据设计哲学

为什么不用合成数据？

"Capabilities should be learned, not inherited." 合成数据 = 蒸馏另一个模型的能力分布。MAI 认为这限制了模型的上界（只能模仿，不能超越数据源模型）。

为什么 Code 占 55%？

Rank non-invariance 实验表明：在大规模+长训练下，Code 数据因其高多样性（不同语言/框架/范式）对所有能力都有正迁移。包括 STEM 推理能力。

数据混合优化方法：

1. 定义目标函数：加权 NLL (Code 50% + STEM 17.5% + Math 17.5% + General 10% + Multi 5%)

2. 训练 183 个小模型在 61 种混合比例上

3. 找 Pareto frontier

4. 在目标规模验证 (rank 可能翻转!)

5. Cap 最大 epoch 为 8 (防过拟合)

Memorization-aware Epoch Capping：Mid-training 时，对预训练已高度记忆的数据源限制 epoch。检测方式：验证损失下降中"NLL<0.01 的 token 占比"高 = 记忆而非泛化。

评估去污染

公共 benchmark (MMLU, HumanEval 等) 在 GitHub 上被大量复制。MAI 的做法：

• 从训练数据中排除所有 ML repos

• 自建内部 NLL 评估集 (vendor + internal)

• 不在预训练阶段使用 accuracy-based eval

• NLL eval 更高效、更不受 confounding 影响

Wikipedia 的特殊处理：直接训练 raw wikitext (而非解析后的纯文本)。虽然 wikitext 体积大 3×，但保留了 infobox、markup 等结构信息，实测效果更好。对高价值小数据源"宁多勿少"。

对我们的启示

1. RL 爬坡是可预测的

给定足够的 compute 和稳定的 recipe，RL 性能呈 log-linear 增长。这意味着：更多 GPU 时间 = 更好的模型，且增长是可预测的。这使得模型开发变得像工程问题而非碰运气。

2. 从头训练不是浪费

虽然比蒸馏慢很多，但获得的是完全可控的能力。蒸馏的模型在 RL 爬坡中不稳定（缺乏可控性），MAI 的实验验证了这一点。

3. 简单工具 > 复杂工具

Agentic Climb 只给 bash + string_replace。RL 自己探索出复杂组合。如果给太多工具，搜索空间反而变小（每个工具都是 "提前给定的答案"）。
