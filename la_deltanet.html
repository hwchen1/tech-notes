DeltaNet — Delta Rule Linear Attention

2024 | MIT / Together AI | 先擦后写 — 解决 Linear Attention 不能覆盖旧记忆的根本缺陷

1. 核心问题：Linear Attention 不能覆盖旧记忆

# Linear Attention 的根本缺陷:
# S_t = S_{t-1} + k_t ⊗ v_t
# 只做 "累加"，无法 "更新" 或 "删除"

# 场景: 模型看到
# "The capital of France is Paris"
# → 写入 (France, Paris) 到状态 S

# 然后看到 (假设性):
# "Actually, the capital moved to Lyon"
# → 想写入 (France, Lyon)

# 标准 linear attention:
#   S += k_france ⊗ v_lyon
#   结果: S 中同时有 Paris 和 Lyon!
#   读取: q_france @ S → 混合结果 😱

# 即使加衰减 (RetNet/GLA):
#   S = γ·S + k⊗v
#   旧信息衰减太慢 → 仍然混合
#   衰减太快 → 有用信息也丢了

# DeltaNet 的解决方案:
# 先"擦除"旧的 (France, Paris)
# 再"写入"新的 (France, Lyon)
# → 精确覆盖，不混合!

为什么累加是致命的？

标准 linear attention 的状态 S 本质是 append-only — 只能往里加信息，不能修改或删除。

这导致两个问题：

问题后果
无法更新新旧信息混合，recall 不精确
状态污染噪声累积，长序列性能退化

衰减 (RetNet/GLA) 只是缓解而非解决 — 它对所有信息统一衰减，无法精确定位需要修改的内容。

DeltaNet 的洞察：借鉴 Hopfield 网络的 delta rule — 先"擦除"key 对应的旧 value，再写入新 value。这是联想记忆中的经典更新策略。

2. Delta Rule 更新

# DeltaNet 递推公式
def deltanet_recurrent(q, k, v, beta):
    """
    beta: 擦除强度 ∈ [0,1]
    beta=0 → 退化为标准 linear attention
    beta=1 → 完全覆盖旧值
    """
    B, H, N, d = q.shape
    S = torch.zeros(B, H, d, d)
    outputs = []

    for t in range(N):
        k_t = k[:,:,t]  # [B,H,d]
        v_t = v[:,:,t]
        q_t = q[:,:,t]
        b_t = beta[:,:,t]  # [B,H,1]

        # 1. 读取旧值
        old_v = S @ k_t.unsqueeze(-1)  # S·k
        old_v = old_v.squeeze(-1)

        # 2. 计算 delta (新-旧)
        delta_v = v_t - old_v

        # 3. 更新状态 (擦旧 + 写新)
        # 等价于: 先减 β·(旧kv), 再加 (kv)
        S = S + b_t.unsqueeze(-1) \
            * (k_t.unsqueeze(-1) * delta_v.unsqueeze(-2))

        # 4. 读取输出
        o_t = (q_t.unsqueeze(-2) @ S).squeeze(-2)
        outputs.append(o_t)

    return torch.stack(outputs, dim=2)

数学本质

$S_t = S_{t-1} + \beta_t \cdot k_t \otimes (v_t - S_{t-1} k_t)$ $= \underbrace{(I - \beta_t k_t k_t^\top)}_{\text{擦除旧关联}} S_{t-1} + \underbrace{\beta_t k_t v_t^\top}_{\text{写入新关联}}$

三个关键：

组件作用
S·k_t读出 k_t 对应的旧 value
v_t - S·k_t新旧差值 (delta)
β_t控制覆盖力度 (数据依赖)

当 β=1 且 k 正交时，DeltaNet 退化为精确的 key-value store — 每个 key 对应唯一 value。

来源：Delta Rule 源自神经科学的 Widrow-Hoff 学习规则 (1960)，也是 Hopfield 网络中 Hebbian learning 的改进版本。

3. Chunk-wise 并行实现

# DeltaNet 的挑战: 状态更新依赖 S 本身
# S_t = (I - β·kk^T)·S_{t-1} + β·kv^T
# → 不是简单的 S += kv (不能直接 cumsum)

# 解决: WY Representation
# 将 chunk 内的连续 delta 更新
# 压缩为一个矩阵乘

# 关键: 把 (I - β·kk^T) 的连乘
# 用 WY 分解表示为:
# Π_{i} (I - β_i k_i k_i^T)
# = I - W · Y^T
# 其中 W, Y ∈ R^{C×d} (C=chunk_size)

def deltanet_chunk_wise(q, k, v, beta, C=64):
    B, H, N, d = q.shape
    n_chunks = N // C
    S = torch.zeros(B, H, d, d)
    outputs = []

    for i in range(n_chunks):
        qc = q[:,:, i*C:(i+1)*C]
        kc = k[:,:, i*C:(i+1)*C]
        vc = v[:,:, i*C:(i+1)*C]
        bc = beta[:,:, i*C:(i+1)*C]

        # WY decomposition for chunk
        W, Y = compute_wy(kc, bc)

        # Intra-chunk output
        # (uses masked lower-triangular)
        intra = masked_delta_attn(qc, kc, vc, bc)

        # Inter-chunk: read from state
        inter = qc @ S

        outputs.append(intra + inter)

        # Update state using WY form
        S = S - W.T @ (Y.T @ S) + kc.T @ (bc * vc)

    return torch.cat(outputs, dim=2)

WY Representation

DeltaNet 的难点：每步更新 S_t = (I - βkk^T)·S，这个乘法让 chunk 内不能简单并行。

解决方案来自数值线性代数：

$\prod_{i=1}^{C} (I - \beta_i k_i k_i^\top) = I - W Y^\top$ $W, Y \in \mathbb{R}^{C \times d}$

这样 chunk 内的连续 delta 更新可以用一个矩阵乘表示，GPU 可以高效计算。

关键 insight：DeltaNet 牺牲了少量训练速度（比 GLA 慢 ~20%），但换来了显著更强的 recall 能力 — 因为它能精确更新记忆而非简单累加。在需要精确检索的任务上（MQAR benchmark），DeltaNet 远超 GLA 和 Mamba。
