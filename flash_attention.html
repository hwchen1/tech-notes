Flash Attention

从 V1 到 V3：IO-Aware → Parallelism → Asynchrony on Hopper

0. 标准 Attention 的瓶颈

标准 Attention 需要 4 次 HBM 读写，中间矩阵 S、P 占 O(N²) 显存

1. Flash Attention V1 — IO-Aware Tiling

核心思想：将 Q、K、V 分块（tiling），在 SRAM 中完成 softmax 和乘法，永远不把 N×N 矩阵写回 HBM。

V1 通过分块 + Online Softmax 将 IO 从 O(N²) 降至 O(N²d²/M)，M 为 SRAM 大小

Online Softmax 核心

# 增量更新 softmax，无需存储完整 N×N 矩阵
for j in range(num_kv_blocks):
    S_ij = Q_i @ K_j.T / sqrt(d)          # block attention score
    m_new = max(m_i, S_ij.max(dim=-1))    # running max
    P_ij = exp(S_ij - m_new)              # safe exp
    l_i = exp(m_i - m_new) * l_i + P_ij.sum(dim=-1)  # running sum
    O_i = exp(m_i - m_new) * O_i + P_ij @ V_j        # rescale + accumulate
    m_i = m_new
O_i = O_i / l_i                           # final normalize
V1 关键贡献：证明 exact attention（非近似）可以在 O(N) 显存下完成，速度提升 2-4×，训练长序列成为可能。

2. Flash Attention V2 — Better Parallelism

核心改进：调换内外循环顺序 + 减少非矩阵乘 FLOPs + 更好的 warp 并行。

V2 通过交换循环顺序让 Q blocks 跨 SM 并行，GPU 利用率翻倍

# V2 伪代码 — outer loop on Q enables parallelism
# Each thread block handles one Q_i block independently
parallel for i in range(num_q_blocks):   # ← parallel across SMs
    O_i, m_i, l_i = zeros, -inf, zeros
    for j in range(num_kv_blocks):       # ← sequential
        S_ij = Q_i @ K_j.T * scale
        m_new = max(m_i, rowmax(S_ij))
        P_ij = exp(S_ij - m_new)
        l_i = exp(m_i - m_new) * l_i + rowsum(P_ij)
        O_i = diag(exp(m_i - m_new)) @ O_i + P_ij @ V_j
        m_i = m_new
    O_i = O_i / l_i

3. Flash Attention V3 — Hopper Asynchrony

核心改进：利用 H100 Hopper 架构的 TMA + WGMMA + 异步流水线，实现计算与数据搬运完全重叠。

V3 利用 Hopper 的 TMA（异步搬运）和 WGMMA（Tensor Core），Producer-Consumer 流水线完全重叠 IO 与计算

4. Flash Attention V4 — Blackwell Co-Design

核心问题：Blackwell (B200) 的 Tensor Core 吞吐增长远快于 shared memory 带宽和 SFU（exp/div 特殊功能单元），非 matmul 操作成为新瓶颈。

V4 用 Pingpong 调度让两个 warpgroup 交替执行 QKᵀ 和 PV，同时用 MMA 软件模拟 exp 避开 SFU 瓶颈

V4 三大技术突破

① Pingpong Scheduling：两个 warpgroup 交替占用 Tensor Core — WG-A 做 S=QKᵀ 时，WG-B 做 O=PV。两个 GEMM 完全重叠，Tensor Core 利用率接近 100%。
② 软件模拟 exp2f：Blackwell 的 SFU 吞吐跟不上 Tensor Core。V4 用多项式近似 + MMA 指令实现 exp，将 softmax 计算从 SFU 卸载到 Tensor Core，消除功能单元争用。
③ 算法-Kernel 协同重排：重新安排 online softmax 的 rescale 时机，使其恰好落在 MMA 的 pipeline bubble 中被掩盖，而非阻塞主计算路径。

FlexAttention 集成

V4 与 PyTorch FlexAttention 深度集成：通过 CuTeDSL 自动生成自定义 score/mask 函数，JIT 编译为 FA4 kernel 变体。支持 causal、sliding window、document mask 等任意 attention pattern，无需手写 CUDA。
