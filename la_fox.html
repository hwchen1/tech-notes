Forgetting Transformer (FoX)

Softmax Attention + Forget Gate — 数据依赖遗忘,不需要位置编码 | Lin, Nikishin et al. | 2025.03 (ICLR 2025, arxiv 2503.02130)

1. 核心理念 — 给 Softmax Attention 加 Forget Gate

# 关键观察:
# Linear Attention 的 forget gate 已被证明关键
# (GLA, Mamba-2, GDN 都依赖门控遗忘)
# 但 Softmax Attention 从未有过显式遗忘!

# FoX 的做法: 在 attention logits 上加偏置
# 标准 Attention:
#   O = softmax(QK^T) · V
#
# Forgetting Attention:
#   O = softmax(QK^T + D) · V
#
# 其中 D 是数据依赖的下三角偏置矩阵:
#   f_t = sigmoid(w_f^T · x_t + b_f)  # 逐token
#   D_ij = Σ_{l=j+1}^{i} log(f_l)     # 累积
#
# 等价形式:
#   O_i = Σ_j F_ij · exp(q_i^T·k_j) · v_j
#         / Σ_j F_ij · exp(q_i^T·k_j)
#   where F_ij = Π_{l=j+1}^{i} f_l

# 与 Linear Attention 的统一:
# GLA:  o_i = Σ F_ij · φ(q)^T·φ(k) · v_j  (linear kernel)
# FoX:  o_i = Σ F_ij · exp(q^T·k) · v_j    (exp kernel)
# 唯一区别: kernel 函数!
# forget gate F_ij 的定义完全一致!

FoX 将 linear attention 的 forget gate 引入 softmax attention，建立两个方向的统一

统一 Linear Attention 与 Softmax Attention

FoX 证明了 forget gate 不是 linear attention 的专利。同样的机制可以直接应用于 softmax attention，且效果更好。这建立了两个看似不同的研究方向之间的桥梁。

核心洞察：FoX 的 forget gate 让模型自动学会: (1) 部分 head 做局部注意力 (\(f \approx 0\), 快速遗忘远处token), (2) 部分 head 做全局注意力 (\(f \approx 1\), 保持全上下文)。不需要手动设计 sliding window!
不需要位置编码：forget gate 隐式提供位置信息。移除 RoPE 后 FoX 仍然工作 (Transformer 不加 RoPE PPL 从 7.49 飙升到 29.30)。
$F_{ij} = \prod_{l=j+1}^{i} f_l, \quad f_t = \sigma(w_f^\top x_t + b_f)$ $D_{ij} = \sum_{l=j+1}^{i} \log f_l = c_i - c_j$

2. FlashAttention 兼容 + Pro Block

# FlashAttention 兼容实现:
# 预计算: c_i = Σ_{l=1}^{i} log(f_l)  (前缀和)
# 存储 c 向量到 HBM
# 在 FlashAttention tiling 中:
#   D_ij = c_i - c_j  (在 SRAM 中计算)
#   logits_ij = q_i^T·k_j + D_ij
# → D 矩阵永远不在 HBM 中实体化!
# → 开销可忽略 (f_t 是标量)

# "Pro" Block 设计:
# 借鉴 recurrent model 的好组件:
# 1. Output gate + output normalization
# 2. QK-norm (RMSNorm on Q,K)
# 3. Data-dependent KV-shift
#    (简化版 RWKV-6 的 token shift)

# 配置 (760M, 16K context):
config = {
    'n_layers': 24,
    'd_model': 1536,
    'n_heads': 12,
    'block_type': 'Pro',  # vs 'LLaMA'
    'pos_embed': None,  # 不需要!
    'forget_gate': 'per_head_scalar',
    'output_gate': True,
    'qk_norm': True,
    'kv_shift': True,
}

# FlashAttention overhead: <2% wall time

Pro Block 将 recurrent model 的成功组件 (output gate, QK-norm, KV-shift) 引入 Transformer

零开销遗忘

FoX 与 FlashAttention 完全兼容，额外计算开销 <2%。这是因为 forget gate 是标量，D 矩阵通过前缀和+差值在 SRAM 中 on-the-fly 计算。

Pro Block 是通用改进：不仅提升 FoX (+0.63 PPL)，也提升标准 Transformer (+0.67 PPL)。核心贡献是 output gate + QK-norm + KV-shift 三件套。

FlashAttention 实现细节：存储前缀和 \(c_i = \sum_{l=1}^{i} \log f_l\) 到 HBM，tiling 时在 SRAM 中计算 \(D_{ij} = c_i - c_j\)，无需实体化 \(N \times N\) 矩阵。

3. 实验 — 超越 Transformer，保持长上下文能力

# 760M, 48B tokens (LongCrawl64), context 16K

# Short-context downstream (12 benchmarks avg):
# DeltaNet         : 47.99
# HGRN2            : 49.45
# Transformer LLaMA: 49.09
# Mamba-2          : 50.21
# FoX LLaMA        : 49.82
# Transformer Pro  : 50.39
# FoX Pro          : 50.88  ← Best!

# Long-context: per-token loss
# FoX: 单调下降直到 16K (完全利用上下文)
# Mamba-2/DeltaNet: 5K 后 plateau
# → FoX 保持 Transformer 的长距离能力!

# Needle-in-Haystack:
# FoX: 训练长度内近乎完美
# Mamba-2/DeltaNet/HGRN2: 即使训练长度内也失败

# Length extrapolation:
# FoX: 超出训练长度仍能工作 (不需 RoPE!)
# Transformer: 超出训练长度急剧退化

# 核心结论:
# FoX = Transformer 的长距离能力
#      + Linear Attention 的遗忘优势
#      + 无需位置编码的外推能力

FoX 同时保持 Transformer 的长距离能力和超出训练长度的外推能力

三重优势

1. 超越 Transformer 质量 — Pro block + forget gate 在 12 个 benchmark 平均得分最高 (50.88)

2. 保持长距离精确检索 — 与 recurrent 模型的 plateau 不同，FoX 的 loss 持续下降到训练长度边界

3. 无需位置编码即可外推 — 移除 RoPE 后仍能在超出训练长度时工作

FoX 的启示：Linear Attention 研究中发现的 forget gate 机制是普适的。它不仅对 linear kernel 有效，对 exp kernel (softmax) 同样有效甚至更有效。这暗示 forget gate 可能成为所有 sequence model 的标配组件。
Scaling 特征：FoX 的优势随上下文长度增加而增大，但随模型增大而减小。暗示更大的模型可以通过 Q/K 模式隐式学习遗忘。
