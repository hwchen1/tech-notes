class DeepSeekV3Model(nn.Module):
    def __init__(self, config):
        self.embed_tokens = Embedding(
            129280, 7168)
        self.layers = ModuleList([...])
        self.norm = RMSNorm(7168)

    def forward(self, input_ids):
        x = self.embed_tokens(input_ids)
        # x: [B, N, 7168]
        for layer in self.layers:
            x = layer(x)
        x = self.norm(x)
        return x

Embedding 设计

不共享权重 (tie=False)：

输入 embedding 和输出 LM head 是两个独立矩阵。代价是多 0.93B 参数，但让输入和输出的表征空间独立优化。

$\text{Embed: } 129280 \times 7168 = 926\text{M}$
$\text{LM Head: } 7168 \times 129280 = 926\text{M}$
$\text{合计: } 1.85\text{B for IO}$

vs Qwen3 (tie=True):
Qwen3 共享 embed 和 head，节省参数但限制了输出层的独立表达。DeepSeek-V3 参数量够大，不需要省这点。

Vocab = 129280：
比 Qwen3 (152K) 小，但足够覆盖多语言 + 代码。更小 vocab 意味着 embedding 层更轻。

3. RMSNorm & Residual Stream

# Pre-Norm 结构 (每个 sub-layer 前)
# DeepSeek-V3 使用 RMSNorm, eps=1e-6

class RMSNorm(nn.Module):
    def __init__(self, dim=7168, eps=1e-6):
        self.weight = Parameter(torch.ones(dim))
        self.eps = eps

    def forward(self, x):
        # RMS = sqrt(mean(x²))
        rms = torch.sqrt(
            x.pow(2).mean(-1, keepdim=True)
            + self.eps)
        return x / rms * self.weight

# Transformer Layer 结构:
class DeepSeekLayer(nn.Module):
    def forward(self, x):
        # Sub-layer 1: Attention
        residual = x
        x = self.input_norm(x)   # RMSNorm
        x = self.attention(x)    # MLA
        x = residual + x         # Residual

        # Sub-layer 2: FFN/MoE
        residual = x
        x = self.post_norm(x)    # RMSNorm
        x = self.ffn(x)          # MoE or Dense
        x = residual + x         # Residual
        return x

# 61层，每层 2 个 RMSNorm
# + 1 个 final norm = 123 个 RMSNorm
# 每个: 7168 params (weight only)
# total: 123 × 7168 = 0.88M

Pre-Norm + Residual

为什么 Pre-Norm (不是 Post-Norm)：

Pre-Norm 让 residual stream 直通不变换，梯度可以直接跨层流动。这对 61 层深网络的训练稳定性至关重要。

RMSNorm vs LayerNorm：

$$\text{LayerNorm: } \frac{x - \mu}{\sigma}$$$$\text{RMSNorm: } \frac{x}{\text{RMS}(x)}$$$$\text{RMS} = \sqrt{\text{mean}(x^2) + \epsilon}$$

RMSNorm 去掉了 mean-shift (re-centering)，只保留 re-scaling。实践中效果相当，但计算更快（省一次 mean 计算 + 减法）。

Residual stream 是模型的"高速公路"：
每一层只往 stream 上"加"信息，不破坏已有信息。这是 Transformer 能堆深的根本原因。

4. MLA — Multi-head Latent Attention

class MLA(nn.Module):
    # hidden=7168, heads=128, head_dim=128
    # kv_lora_rank=512, q_lora_rank=1536
    # qk_nope_head_dim=128, qk_rope_head_dim=64

    def __init__(self):
        # === Q path: 低秩压缩 ===
        self.q_down = Linear(7168, 1536)
        self.q_norm = RMSNorm(1536)
        self.q_up = Linear(1536, 128*(128+64))
        # → 128 heads × (nope:128 + rope:64) = 24576

        # === KV path: 低秩压缩 ===
        self.kv_down = Linear(7168, 512+64)
        # → 512(latent) + 64(rope_key) = 576
        self.kv_norm = RMSNorm(512)
        self.kv_up = Linear(512, 128*(128+128))
        # → 128 heads × (k_nope:128 + v:128) = 32768

        # === Output ===
        self.o_proj = Linear(128*128, 7168)

    def forward(self, x, pos):
        # --- Q path ---
        q_latent = self.q_norm(self.q_down(x))
        q = self.q_up(q_latent)
        q_nope, q_rope = split(q, [128,64])
        q_rope = apply_rope(q_rope, pos)

        # --- KV path ---
        kv = self.kv_down(x)
        c_kv, k_rope = split(kv, [512,64])
        k_rope = apply_rope(k_rope, pos)

        # KV cache 只存: c_kv(512) + k_rope(64)

        # Expand KV from latent
        kv_expanded = self.kv_up(self.kv_norm(c_kv))
        k_nope, v = split(kv_expanded, [128,128])

        # Concat full Q and K
        q = cat([q_nope, q_rope])  # 192d
        k = cat([k_nope, k_rope])  # 192d

        # Attention
        out = sdpa(q, k, v)  # [B,N,128,128]
        return self.o_proj(out.flatten(-2))

MLA 关键设计

KV Cache 大小对比：

$\text{MHA: } 2 \times 128 \times 128 = 32768$
$\text{MLA: } 512 + 64 = 576$
$\text{压缩比: } \dfrac{32768}{576} \approx 57\times$

RoPE 解耦：
latent(512d) 编码内容，与位置无关。RoPE 单独走 64d 通道。这让 latent 可以在任何位置还原完整 KV。

Absorbed Attention (推理优化)：
decode 时，W_uk 可吸收进 Q 投影：
score = q_nope @ (W_uk @ c_kv).T
等价于 (q_nope @ W_uk) @ c_kv.T
→ 不需要还原完整 K，直接用 latent 计算 score！

Q 也做压缩：
7168→1536→24576
训练时减少激活内存，推理时不影响（Q 不缓存）。

5. DeepSeekMoE — 分组路由 + Shared Expert

class DeepSeekMoE(nn.Module):
    # 256 routed + 1 shared expert
    # 8 groups × 32 experts
    # pick top-4 groups, top-2 per group = 8

    def __init__(self):
        # Shared expert (always active)
        self.shared = SwiGLU(7168, 2048)

        # 256 routed experts
        self.experts = [
            SwiGLU(7168, 2048)
            for _ in range(256)
        ]

        # Sigmoid router (NOT softmax!)
        self.gate = Linear(7168, 256)

        # Bias for aux-loss-free balancing
        self.expert_bias = Parameter(
            torch.zeros(256))

    def forward(self, x):
        # 1. Shared expert (always runs)
        shared_out = self.shared(x)

        # 2. Sigmoid routing scores
        scores = (self.gate(x) +
                  self.expert_bias).sigmoid()

        # 3. Group selection
        # reshape: [256] → [8, 32]
        group_scores = scores.view(
            8, 32).mean(dim=-1)
        top4_groups = topk(group_scores, 4)

        # 4. Within-group top-2
        # → 4 groups × 2 = 8 experts
        selected = select_topk_in_groups(
            scores, top4_groups, k=2)

        # 5. Normalize & weighted sum
        weights = scores[selected]
        weights = weights / weights.sum()
        routed_out = sum(
            w * expert(x)
            for w, expert in
            zip(weights, selected_experts)
        )

        return shared_out + routed_out

MoE 路由创新

Sigmoid vs Softmax：
Softmax 强制归一化 → experts 互斥竞争。Sigmoid 独立打分 → 多 expert 可同时高分，表达更灵活。

分组路由的工程价值：
先选 group，再组内选 → expert 通信在 group 内局部化。EP (Expert Parallelism) 时减少跨节点通信。

$\text{shared: } 7168 \times 2048 \times 3 = 44\text{M}$
$\text{routed: } 8 \times 44\text{M} = 352\text{M}$
$\text{total FFN: } 396\text{M MACs/token}$

Aux-loss-free：
传统方法用辅助 loss 惩罚不平衡，但这会和主 loss 竞争。DeepSeek 改用 bias 项：负载高的 expert bias↓ → score↓ → 被选中概率↓。训练时动态调整，不影响梯度。

class DenseSwiGLU(nn.Module):
    # intermediate_size = 18432
    # ratio: 18432 / 7168 = 2.57x

    def __init__(self):
        self.gate_proj = Linear(7168, 18432)
        self.up_proj = Linear(7168, 18432)
        self.down_proj = Linear(18432, 7168)

    def forward(self, x):
        gate = F.silu(self.gate_proj(x))
        up = self.up_proj(x)
        return self.down_proj(gate * up)

# 参数量 per dense layer FFN:
# gate: 7168 × 18432 = 132M
# up:   7168 × 18432 = 132M
# down: 18432 × 7168 = 132M
# total: 396M per layer
# 3 layers: 1.19B

# 对比 MoE layer:
# 256 experts × 3 × 7168 × 2048 = 11.3B
# + shared: 44M
# → MoE层参数量 28.5x 于 Dense层
# 但激活计算量相当 (都是 ~400M)

为什么前3层用 Dense

底层需要全量表达：

前几层处理最基础的语言特征（词法、短语），所有 token 都需要相同的处理。MoE 的稀疏路由在底层可能导致某些基础特征被"漏掉"。

Dense vs MoE 对比：

$\text{Dense FFN: } 396\text{M params/layer}$
$\text{Dense compute: } 396\text{M MACs}$

$\text{MoE FFN: } 11.3\text{B params/layer}$
$\text{MoE compute: } 396\text{M MACs (same!)}$

关键洞察：MoE 不省计算，省的是"单位参数的计算成本"。前3层不需要大参数量（基础特征空间小），Dense 更合适。

另外，Dense 层没有路由开销，推理延迟更低。前3层作为"预热"，让表征进入稳定分布后再做 MoE 路由。

7. Multi-Token Prediction (MTP)

# num_nextn_predict_layers = 1
# 同时预测 token[t+1] 和 token[t+2]

class MTPModule(nn.Module):
    def __init__(self):
        # 将 hidden + next_emb 投影回 hidden
        self.proj = Linear(7168*2, 7168)
        self.norm = RMSNorm(7168)

        # 一个完整的 transformer layer
        self.attn = MLA(config)
        self.ffn = MoE(config)  # 或 Dense

        # 独立 LM head
        self.lm_head = Linear(7168, 129280)

    def forward(self, hidden, next_token_emb):
        # hidden: 主模型最后一层输出
        # next_token_emb: token[t+1] 的 embedding
        # (训练时 teacher forcing)

        h = cat([hidden, next_token_emb], dim=-1)
        h = self.proj(h)    # [B,N,14336]→[B,N,7168]
        h = self.norm(h)
        h = h + self.attn(h)
        h = h + self.ffn(h)
        logits = self.lm_head(h)
        return logits  # predict token[t+2]

# Training:
# L = L_main(token[t+1]) + λ·L_mtp(token[t+2])
# λ 通常 = 0.3~0.5

# Inference (speculative decoding):
# 1. Main model → token[t+1] (verified)
# 2. MTP head → token[t+2] (draft)
# 3. Next step: verify draft + get new draft
# → ~1.8x speedup

MTP 设计分析

训练增益：
预测 token[t+2] 迫使模型在位置 t 编码更多前瞻信息。hidden state 变得更"informative"，提升表征质量。

为什么只 1 层：
边际收益递减。论文实验显示 1 层 MTP 的 gain 已经接近 2 层的 90%，但参数开销只有一半。

推理加速 (Speculative Decoding)：

$\text{Standard: 1 token/step}$
$\text{MTP draft: 2 tokens/step}$
$\text{Verify + accept} \rightarrow \sim 1.8\times \text{ speedup}$

MTP 参数量：
proj(14336→7168) + 1 layer MLA + 1 layer MoE + LM head ≈ 12B
占总参数量 ~1.8%，但收益显著。

8. RoPE + YaRN 位置编码

# RoPE 基础配置
rope_theta = 10000

rope_scaling = {
    "type": "yarn",
    "factor": 40,

# 扩展后: 4096 × 40 = 163,840 tokens

# YaRN 三频段策略:
# freq_i = theta^(-2i/d)
# wavelength_i = 2π / freq_i
#
# 低频 (wavelength > original_len):
#   → 线性插值, 除以 factor
#   → 这些维度编码"全局位置"
#
# 高频 (wavelength < original_len/factor):
#   → 不修改 (保持原始)
#   → 这些维度编码"局部相对距离"
#
# 中频 (介于两者之间):
#   → NTK-aware 插值 (渐变)
#   → 平滑过渡区

# 对比 Qwen3 的方案:
# Qwen3: rope_theta = 1,000,000 (100x bigger)
# → 原生支持长位置, 不需要 YaRN
# DeepSeek: theta=10000 + YaRN(40x)
# → 训练时短序列, 推理时通过 YaRN 外推

位置编码策略

DeepSeek 方案：低 base + YaRN

$\theta_{\text{base}} = 10{,}000$
$\text{train length} = 4{,}096$
$\text{YaRN factor} = 40$
$\rightarrow \text{支持 } 4096 \times 40 = 163{,}840$

优势：训练时只需短序列（4K），节省大量计算。长序列能力通过 YaRN 外推获得。

vs Qwen3 方案（高 base）：

$\text{Qwen3: } \theta = 1{,}000{,}000$
$\rightarrow$ 原生长位置编码
$\rightarrow$ 不需要额外外推策略

Trade-off：
• DeepSeek: 训练便宜，但外推有衰减
• Qwen3: 训练贵（需见长序列），但位置编码更稳定

MLA 的 RoPE 只用 64 维：
MLA 的 qk_rope_head_dim=64，只有 64 维参与 RoPE。比 GQA 的 128 维全 RoPE 更省计算。

9. FP8 混合精度训练

# DeepSeek-V3 开创性地使用 FP8 训练
# 前向: FP8 (E4M3)
# 反向: FP8 (E5M2) + BF16 mixed
# 优化器: FP32 master weights

# Block-wise quantization:
# 不是 per-tensor，而是 per-block
block_size = [128, 128]
# 每 128×128 block 有独立的 scale factor

# FP8 格式:
# E4M3: 1 sign + 4 exp + 3 mantissa
#   range: ±448, precision: ~1/16
#   用于前向 (需要精度)
#
# E5M2: 1 sign + 5 exp + 2 mantissa
#   range: ±57344, precision: ~1/4
#   用于反向 (需要范围)

# 混合精度策略:
# - Linear matmul: FP8
# - Attention scores: BF16
# - Softmax/Sigmoid: FP32
# - Norm: FP32
# - Router: FP32
# - Residual add: BF16

# 训练效率提升:
# FP8 vs BF16:
# - GEMM 吞吐: ~2x
# - 内存带宽: ~2x
# - 总训练加速: ~1.5-1.7x
# (不是纯 2x, 因为有 BF16 混合部分)

FP8 训练创新

DeepSeek-V3 是首个大规模 FP8 训练的开源模型。

Block-wise vs Per-tensor：

$$\text{Per-tensor: 1 scale / 整个矩阵}$$$$\rightarrow \text{outlier 拉高 scale，其余精度损失}$$$$\text{Block-wise } [128\times128]\text{: 每块独立}$$$$\rightarrow \text{outlier 只影响局部，整体} \approx \text{BF16}$$

为什么 E4M3 + E5M2：
• 前向需要更高精度 (E4M3, 3位尾数)
• 梯度需要更大动态范围 (E5M2, 5位指数)
各取所长。

成本影响：
FP8 让 GEMM 吞吐翻倍。结合 MoE 的稀疏计算，DeepSeek-V3 的训练成本仅 \$5.5M — 比同量级模型低一个数量级。

11. 推理优化

# === Absorbed Attention ===
# 标准 MLA decode:
# k_nope = kv_up(c_kv)[:, :128]  ← 大矩阵乘
# score = q_nope @ k_nope.T

# Absorbed (吸收):
# score = q_nope @ W_uk @ c_kv.T
# 等价于: (q_nope @ W_uk) @ c_kv.T
# 令 q_absorbed = q_nope @ W_uk
#   shape: [128,128] @ [128,512] → [128,512]
# score = q_absorbed @ c_kv.T
#   shape: [1,512] @ [N,512].T → [1,N]

# 好处:
# 1. 不需要存展开后的 K (32768/token)
# 2. 直接用 c_kv (512/token) 计算
# 3. W_uk 的吸收可预计算

# === MTP Speculative Decoding ===
# Main model: 1 forward → token A
# MTP head: → draft token B
# Next step: verify B + predict C + draft D
# → 每步可确认 1.x tokens (avg ~1.8)

# === KV Cache 总结 ===
# per token per layer: 576 × 2 bytes = 1.15KB
# 61 layers: 576 × 61 = 35,136 per token
# 128K context: 35136 × 128K = 4.3GB
# (vs MHA: 32768×61×128K = 248GB !)

推理三大优化

1. Absorbed Attention：
将 W_uk 吸收进 Q，decode 时跳过 KV 展开。把 memory-bound 变成 compute-bound。

2. MTP Speculative：
MTP head 做 draft token，main model 验证。平均每步生成 1.8 tokens，加速 ~1.8x。

3. KV Cache 压缩：

$\text{MLA cache: } 576 \text{ per token/layer}$
$61 \times 576 \times 2\text{B} = 70\text{KB/token}$

$\text{128K context: } 70\text{KB} \times 128\text{K}$
$= 8.6\text{ GB total KV cache}$
$\rightarrow$ 一张 A100 80G 可以!

这三个优化叠加，让 671B MoE 模型可以在合理成本下服务长序列推理。
