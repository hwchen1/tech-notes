1. 核心理念：Visual Causal Flow

# DeepSeek-OCR 2 核心创新:
# "Visual Causal Flow" (视觉因果流)
#
# 关键洞察: 用 Decoder (causal) 做 Encoder
# 但给 image tokens 用 non-causal attention
# 给 query tokens 用 causal attention
#
# 这样 query tokens 可以"因果地"
# 从 image tokens 中提取信息
# = 类似 Omni-Encoder 的 Causal Query!

# 架构三件套:
# 1. SAM ViT-B → 高分辨率特征提取
# 2. Qwen2-0.5B (混合attn) → 视觉压缩
# 3. DeepSeek-V2 MoE → 文本生成

# token_type_ids 控制 attention mask:
# type=0 (image): 双向 attention (互相看)
# type=1 (query): 因果 attention (只看前面)

token_type_ids = torch.cat([
    torch.zeros(bs, n_img),   # image: 双向
    torch.ones(bs, n_query),  # query: 因果
], dim=1)

# Query 看所有 image + 前面的 query
# Image 互相都看得到
# → "因果流"从 image 流向 query

混合 Attention Mask: image tokens 双向 + query tokens 因果

"Visual Causal Flow" 含义

信息像"流水"一样从 image tokens "因果地流向" query tokens。Image 内部是双向的（保留全局视觉信息），但 query 对 image 的访问是单向的（query 看 image，image 看不到 query）。

vs Omni-Encoder: Omni-Encoder 全体 causal → image tokens 也只能看前面的 → 限制视觉交互。DeepSeek-OCR 2 让 image 保持双向 → 更好的视觉表征。
核心创新: 用一个 Qwen2 Decoder 同时实现两种注意力模式（通过 token_type_ids 控制），既做 encoder（image 双向）又做 compressor（query 因果）。

为什么叫 "Decoder-as-Encoder"？

Qwen2 本身是 decoder 架构，但通过修改 attention mask，让 image 部分变成 encoder 行为（双向）。一个模型同时扮演两个角色。

为什么用 SAM ViT-B？

SAM 的 ViT 为高分辨率设计（1024px），有 Window Attention 控制计算量。普通 CLIP ViT 设计分辨率只有 224-336px，不适合 OCR。

3. Causal Query 压缩机制

# Qwen2Decoder2Encoder 核心代码

class Qwen2Decoder2Encoder(nn.Module):
    def __init__(self, ...):
        self.model = CustomQwen2Decoder(...)
        # 两种分辨率的 queries
        self.query_768 = nn.Embedding(
            144, hidden_dim)  # 12×12
        self.query_1024 = nn.Embedding(
            256, hidden_dim)  # 16×16

    def forward(self, x):
        # x: SAM ViT 输出 [B, H, W, C]
        x = x.flatten(2).transpose(1, 2)
        # → [B, 144 or 256, 896]

        bs, n_img, _ = x.shape

        # 选择对应分辨率的 queries
        if n_img == 144:
            queries = self.query_768.weight
        elif n_img == 256:
            queries = self.query_1024.weight

        # 拼接: [image | queries]
        x_combined = torch.cat(
            [x, queries.expand(bs, -1, -1)],
            dim=1
        )  # [B, 2N, 896]

        # token_type_ids:
        # 0 = image (bidirectional)
        # 1 = query (causal)
        token_type_ids = torch.cat([
            torch.zeros(bs, n_img),
            torch.ones(bs, n_img),
        ], dim=1)

        # Forward through Qwen2
        y = self.model(x_combined, token_type_ids)

        # 只取 query 位置的输出!
        y = y[:, n_img:, :]
        # → [B, 144/256, 896]

        return y  # causal flow output

SAM 4096 patches → Qwen2 causal query → 144/256 compressed tokens

关键设计

Query 数量 = Image token 数量：这不是巧合——每个 query 对应一个空间位置，保留空间结构。144 = 12×12 grid。

为什么不直接用 SAM 输出？SAM ViT-B 只有 12 层，且是通用分割模型。Qwen2 的 24 层提供更深的特征变换 + 通过 causal query 实现信息压缩和重新编码。

# 1. OCR 文本内容
# 2. 每个区域的 bounding box

# 两种 prompt 模式:

动态切片 + Grounding 输出（坐标 + 区域类型 + 文本内容）

Grounding = OCR + Layout

DeepSeek-OCR 2 不仅输出文本，还输出每个元素的坐标和类型。这意味着它同时完成了传统 pipeline 中的：版面分析 + 文本检测 + 文本识别。

区域类型支持：title (标题), text (正文), image (图片区域), table (表格), formula (公式), code (代码块) 等。

与 Omni-Encoder 的关系

DeepSeek-OCR 2 的 "Visual Causal Flow" 本质上是 Omni-Encoder "Causal Query" 思路在 OCR 领域的应用。但保留了 SAM ViT 做初始特征提取，属于 ViT + LM(encoder) + MoE(decoder) 的三阶段架构。
