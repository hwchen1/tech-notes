LoRA

Low-Rank Adaptation — 用 0.1% 参数实现全参微调效果

1. 核心思想

问题：全参微调一个 7B 模型需要 ~28GB 梯度 + 优化器状态。LoRA 的洞察：微调时权重的变化量 ΔW 是低秩的，可以用两个小矩阵的乘积近似。

LoRA 将 ΔW 分解为 B(r×d) · A(d×r)，r≪d 时可训练参数从 d² 降至 2dr

关键性质：推理时可将 ΔW = BA 合并回原始权重 W' = W + BA，零额外推理延迟。训练时只需存储 A、B 的梯度和优化器状态。

2. 前向计算

前向传播：原始路径冻结，LoRA 分支 x→A(降维)→B(升维) 后加回主路径。α/r 控制 LoRA 贡献强度

3. 实现代码

import torch
import torch.nn as nn

class LoRALinear(nn.Module):
    def __init__(self, linear: nn.Linear, r=8, alpha=16):
        super().__init__()
        self.linear = linear
        self.linear.weight.requires_grad_(False)  # freeze W
        d_out, d_in = linear.weight.shape
        self.A = nn.Parameter(torch.randn(d_in, r) * 0.01)   # Gaussian init
        self.B = nn.Parameter(torch.zeros(r, d_out))            # zero init → ΔW=0 at start
        self.scale = alpha / r

    def forward(self, x):
        base = self.linear(x)                     # W·x (frozen)
        lora = (x @ self.A) @ self.B * self.scale # (α/r)·B·A·x
        return base + lora

    def merge(self):
        # 推理时合并回原权重 → 零额外开销
        self.linear.weight.data += (self.A @ self.B).T * self.scale
初始化策略：A 用小高斯初始化，B 初始化为零 → 训练开始时 ΔW = BA = 0，模型行为与预训练完全一致，保证稳定起步。

4. 应用位置

LoRA 最常加在 Attention 的 Q/K/V 投影上；追求更高质量可扩展到 FFN

5. 超参数选择

参数含义推荐值说明
r (rank)低秩维度8 ~ 64越大表达力越强，但参数越多。通常 r=16 是性价比最优
α (alpha)缩放因子16 ~ 32实际缩放为 α/r。α=2r 是常见选择
target modules应用层q,k,v,o至少 Q,V；全加 Q,K,V,O 效果更好
dropoutLoRA 分支 dropout0.0 ~ 0.1小数据集可加 0.05 防过拟合

6. LoRA 变体

变体改进点适用场景
LoRA (原始)固定 rank，A 高斯 / B 零初始化通用微调
QLoRA基础模型 4-bit 量化 + LoRA (FP16)单卡微调大模型 (70B on 48GB)
DoRA分解为 magnitude + direction，LoRA 只调方向更接近全参微调效果
rsLoRA缩放因子改为 α/√r（而非 α/r）大 rank 时更稳定
LoRA+A 和 B 用不同学习率（B 更大）加速收敛
实践建议：先用 LoRA r=16, α=32, target=q,k,v,o 作为 baseline。显存不够用 QLoRA。追求极致效果用 DoRA。多任务场景可训练多组 LoRA adapter 热切换。
