Q-Former

Salesforce 2023 — BLIP-2: 用可学习 Query 桥接冻结 ViT 和冻结 LLM，极致参数高效

1. Q-Former 核心结构

# Q-Former 配置 (BLIP-2)

num_query_tokens = 32     # 可学习 queries
hidden_dim       = 768
num_heads        = 12
num_layers       = 12     # BERT-base 架构
cross_attn_freq  = 2      # 每隔一层插入 CA

# 核心: 双路 attention
class QFormerLayer:
    # 1. Self-Attention
    # queries 之间互相 attend
    q = self_attn(queries)

    # 2. Cross-Attention (每隔一层)
    # queries attend to frozen ViT features
    q = cross_attn(
        query=queries,       # [32, 768]
        key=image_features,  # [257, 1024]
        value=image_features
    )

    # 3. FFN
    q = ffn(q)

# 输出: 32 个 query embeddings
# 作为视觉 tokens 送入 LLM

信息瓶颈设计

Q-Former 用 32 个可学习 queries 从冻结 ViT 的 257 tokens 中"提问"最相关的视觉信息。

核心 insight: 不需要把所有视觉 token 都送给 LLM，32 个 query 足以提取关键语义。

$\text{Cross-Attn: } \text{softmax}\!\left(\frac{Q_{\text{learn}} K_{\text{img}}^T}{\sqrt{d}}\right) V_{\text{img}}$ $Q_{\text{learn}} \in \mathbb{R}^{32 \times 768}, \quad K_{\text{img}} \in \mathbb{R}^{257 \times 768}$

极致高效

只训练 Q-Former (188M)，ViT 和 LLM 都冻结。总可训练参数不到全模型的 2%。

2. 两阶段预训练

Stage 1 让 Q-Former 学会从 ViT 提取有用视觉信息；Stage 2 让这些信息能被 LLM 理解和利用

$\text{Stage 1: } \mathcal{L} = \mathcal{L}_{\text{ITC}} + \mathcal{L}_{\text{ITM}} + \mathcal{L}_{\text{ITG}}$ $\text{Stage 2: } \mathcal{L} = \mathcal{L}_{\text{LM}} \quad (\text{Q-Former} \to \text{FC} \to \text{LLM prefix})$ $\text{Trainable: } 188\text{M} + \text{FC} \ll \text{Total params}$

3. Attention Mask 切换策略

# Q-Former 共享参数, 靠 mask 切换
# 同一个网络, 三种行为:

# 1. ITC: Bi-directional
# Queries: 互相可见 (full self-attn)
# Text: 互相可见 (bi-directional)
# Query ✗ Text (不互相看)
mask_itc = [
  [1,1,1, 0,0,0],  # Q sees Q only
  [0,0,0, 1,1,1],  # T sees T only
]

# 2. ITM: Bi-directional
# Query ↔ Text 双向可见
# Query 同时通过 cross-attn 看 image
mask_itm = [
  [1,1,1, 1,1,1],  # Q sees Q+T+Image(CA)
  [1,1,1, 1,1,1],  # T sees Q+T
]
# + hard negative mining 选难样本
# 输出: 每个 query token → binary cls

# 3. ITG: Causal on text
# Query: 可见 query + image
# Text: causal (只看前面的 text)
# Text 可看 Query (作为 prefix)
mask_itg = [
  [1,1,1, 0,0,0],  # Q: self only
  [1,1,1, 1,0,0],  # T: causal+Q
]

mask = task switch

Q-Former 的精妙之处在于用 attention mask 控制信息流，使同一网络在不同预训练目标下有不同行为：

ITC: Query 和 Text 隔离 → 学独立的视觉/语言表征

ITM: Query 和 Text 双向交互，同时 Query 通过 cross-attn 看 image → 学跨模态细粒度匹配（使用 hard negative mining 增加难度）

ITG: Text 侧 causal + 可看 Query → 学条件文本生成

$\text{ITC: } \text{sim}(q, t) = \max_i \frac{q_i \cdot t}{\|q_i\|\|t\|}$ $\text{ITM: } p_{\text{match}} = \text{FC}(q_i), \; \text{hard neg from ITC}$ $\text{ITG: } P(w_t \mid w_{<t}, q) = \text{causal LM}$

历史地位：Q-Former 是第一个系统性解决"如何高效连接冻结视觉编码器和冻结 LLM"的方案。虽然后续 LLaVA 证明简单 MLP 在全参微调场景下效果更好（因为 Q-Former 的信息瓶颈在全训练时反而成为限制），但 Q-Former 的 query-based 压缩思想深刻影响了 Perceiver、Flamingo、mPLUG-Owl 等后续工作。其核心 insight — 用少量 learnable queries 做信息瓶颈 — 至今仍是 token 压缩的主流范式之一。
为什么 LLaVA 胜出？ Q-Former 的 32 tokens 瓶颈在冻结 LLM 场景下是优势（高效压缩），但当 LLM 也参与训练时，信息损失成为瓶颈。LLaVA 的 576 tokens 保留了更多空间细节，配合 LLM 端训练可以自行学习哪些信息重要。趋势：高效推理用压缩（Q-Former），追求性能用全量 tokens（MLP）。
