关键设计决策

MLA-256: 将 head_dim 从 DeepSeek 的 192 提升到 256。为保持训练 FLOPs 不变，heads 数量减少 1/3。更大 head_dim 让 KV cache 压缩率更高，推理速度更快。

Muon Split: 将 up-projection 矩阵按 head 切分为独立子矩阵，每个子矩阵独立做 Muon 正交化 (Newton-Schulz)。避免全局矩阵正交化的通信开销。

MoE 设计: 256 路由专家 + 1 共享专家。每 token 激活 Top-8 = 40B active params。Sigmoid + noaux_tc 替代 softmax + 辅助 loss，负载均衡更稳定。

MTP: 3 层共享参数 (训练时)，推理时展开 speculative decoding。GLM-5.1 accept length 2.76 vs DeepSeek-V3.2 的 2.55。

NOT GDN: GDN (Gated DeltaNet) 仅在 GLM-9B 上做了 ablation 实验 (§2.1.2)。最终 GLM-5 选择 DSA 而非 GDN。

2. DSA — DeepSeek Sparse Attention (Continued Pre-Training)

// DSA 不是原生架构组件
// 而是通过 Continued PT 注入的稀疏 attention

// Phase 1: Warmup (1000 steps)
// 冻结 index head, 只训练 backbone
for step in range(1000):
    loss = model(x, use_full_attn=True)
    loss.backward()  // index_head frozen

// Phase 2: Sparse Adaptation (20B tokens)
// 解冻 index head, 全模型联合训练
for step in sparse_training:
    // index head 选择 top-2048 KV
    indices = index_head(q, k)  // [B,H,S,2048]
    // 只对选中的 KV 做 attention
    out = sparse_attn(q, k[indices], v[indices])

// DSA config:
// index_head_dim = 128
// index_n_heads = 32
// index_topk = 2048
// 每个 head 选 2048 个 KV positions
//
// 90% attention entries are redundant
// → 1.5-2× speedup on long sequences

DSA 核心机制

Index Head: 32 个轻量 attention head (dim=128)，每个 head 为 query 选出 top-2048 个最相关的 KV position。选中的 KV 子集送入标准 MLA 计算。

为什么不是原生架构: GLM-5 先用 Full Attention 完成 28.5T 预训练 + 中训，之后通过 Continued PT 注入 DSA。好处是预训练质量不受影响。

稀疏度: 论文发现 90% attention entries 是冗余的。DSA 仅保留 ~10% 的 KV，长序列推理加速 1.5-2×。

vs GDN ablation (§2.1.2): GDN (Gated DeltaNet) 仅在 GLM-9B 上测试。SimpleGDN 去除 Conv1d/gating，复用预训练 QKV 权重。但最终 DSA 在 744B 模型上效果更好且训练更简单。
训练效率: DSA warmup 1000 步 + 20B tokens adaptation。对比 DeepSeek-V3.2 需要 943.7B tokens 训练 NSA，GLM-5 的 DSA 方案训练成本降低 47×。

数据工程原则

Web 三级过滤: DCLM classifier (粗筛) → Sentence Embedding classifier (语义质量) → World Knowledge classifier (事实密度)。层层提纯。

Code 去重增量: 相比上代，unique code tokens 增加 28%。通过文件扩展名 + AST 分析的语言分类 pipeline，精确识别语言和去重。

Math/STEM 无合成: 显式设计选择 — 不使用任何 synthetic、AI-generated 或 template 数据。全部来自人类生成的数学/科学文档，用 LLM 进行质量评分。

Mid-Training 策略

渐进扩展: 32K (基础) → 128K (1T→500B, 更少 tokens 因为长文本更贵) → 200K (50B, 最终微调)。

合成长文本: Interleaved packing (将短文档交错排列，模拟长距离依赖) + MRCR-like variants (多轮上下文回忆任务，强化 needle-in-haystack 能力)。

总量: 28.5T pre-training + 1.55T mid-training tokens。Mid-training 中 DSA 通过额外 20B tokens continued PT 注入。

4. Post-Training — SFT → Reasoning RL → Agentic RL → General RL

// Post-Training Pipeline (Sequential)

// Stage 1: SFT (三种 Thinking 模式)
class ThinkingMode:
    INTERLEAVED = "interleaved"
    // 思考与回答交错
    PRESERVED   = "preserved"
    // 保留完整思考链
    TURN_LEVEL  = "turn_level"
    // 每轮独立思考

// Stage 2: Reasoning RL
// 数学/代码推理强化
// Outcome-based reward

// Stage 3: Agentic RL (Slime)
class SlimeRL:
    // 异步架构: 生成与训练解耦
    rollout_engines: List[Engine]
    // 大规模 agent 轨迹探索
    // SWE-Bench Pro #1

// Stage 4: General RL
// 通用能力对齐 (RLHF)
// 避免 reasoning 退化

// Stage 5: Cross-Stage Distillation
// On-policy 跨阶段蒸馏:
// 从 Stage 2/3/4 best model
// → 蒸馏到统一 final model
// 保留各阶段最优能力

SFT 三种思考模式

Interleaved: 思考和回答交错出现，适合复杂多步推理。

Preserved: 保留完整思考链输出，用户可见推理过程。

Turn-level: 每轮对话独立思考，适合多轮交互场景。

Slime 异步 RL

核心创新: Rollout engine 与 training 完全解耦。生成 agent 轨迹 (代码编写、工具调用、环境交互) 的计算与 RL 梯度更新分离，支持大规模并行探索。

成果: SWE-Bench Pro 和 SWE-Bench Verified 均排名第一。从 "Vibe Coding" 到真正的 Agentic Engineering。

Cross-Stage Distillation

问题: Sequential RL 各阶段优化不同能力，后续阶段可能覆盖前期成果。

解法: On-Policy 蒸馏 — 从每阶段最佳 checkpoint 生成 trajectory，蒸馏到统一模型。保留推理 + Agent + 通用三重能力。

INT4 QAT: 在 SFT 阶段即引入 INT4 量化感知训练，推理时直接部署量化模型，无需 post-hoc PTQ。

5. Training Infrastructure & 国产芯片适配

// Key Infra Optimizations

// 1. Pipeline ZeRO2 Gradient Sharding
// 在 pipeline stages 间分片梯度
// 减少显存峰值

// 2. MTP Flexible Placement
// MTP layers 可放在任意 PP stage
// 平衡各 stage 计算量

//   inter-node: RDMA all-to-all
// 减少跨节点通信量

// 6. INT4 QAT in SFT
// 量化感知训练从 SFT 阶段开始
// 推理直接用 INT4, 无需 PTQ

    "Enflame (燧原)",

显存优化四板斧

Pipeline ZeRO2: 梯度在 pipeline stages 间分片存储，每个 stage 只保留自己负责的梯度切片。

Activation Offloading: Pipeline 中空闲 stage 的 activation 转存至 CPU，仅在 backward 时搬回。

MTP Flexible Placement: MTP 层不固定绑在最后一个 stage，可以灵活放置以平衡各 stage 的计算/显存负载。

Muon Zero-Redundant: Newton-Schulz 正交化需要全局梯度统计。通过去重通信避免相同梯度在多个 rank 间重复传输。

Context Parallelism

Hierarchical All-to-All: 先做节点内 NVLink all-to-all (高带宽)，再做节点间 RDMA all-to-all (最小化跨机数据量)。支持 200K 序列长度的高效训练。

国产芯片

GLM-5 是首个完成 7 种国产 AI 芯片全栈适配的 744B 参数模型。覆盖华为昇腾、摩尔线程、海光、寒武纪、昆仑芯、沐曦、燧原。

INT4 QAT: 从 SFT 阶段开始量化感知训练。推理时直接以 INT4 部署，避免传统 PTQ 的精度损失。

同架构不同 Post-Training

GLM-5 和 GLM-5.1 的 config.json 完全一致 — 同样的 78 层、MLA-256、256+1 MoE Top-8、DSA。所有差异来自 post-training 阶段的改进。

5.1 的核心突破: 长时效 Agent

GLM-5 的瓶颈: 模型倾向于快速应用熟悉的技巧获得初始 gain，然后迅速 plateau。给更多计算时间不会改善结果。

GLM-5.1 的解法: 通过更深入的 Agentic RL 训练，模型能在长 session 中持续有效 — 分解问题、运行实验、读取结果、精确定位 blocker、反复修正策略。数百轮迭代和数千次 tool call 后仍在改善。

Benchmark 提升幅度

Agentic/coding 任务提升最大: CyberGym +42%, NL2Repo +19%, Terminal-Bench +13%。纯推理任务 (AIME, HMMT) 基本持平 — 符合 "post-training only" 的特征。

GLM-5.2?

截至 2026 年 6 月，GLM-5.2 不存在。HuggingFace zai-org 组织下仅有 GLM-5 和 GLM-5.1 两个版本。
