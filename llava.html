LLaVA (Large Language and Vision Assistant)

Simple & Elegant VLM: CLIP ViT Encoder + MLP Projector + LLM Decoder -- LLaVA-1.5 / LLaVA-NeXT

1. Overall Architecture (CLIP + MLP + LLM)

# LLaVA-1.5 architecture
# Elegantly simple: 3 components

class LLaVA(nn.Module):
    """Visual Instruction Tuning
    - Vision: CLIP ViT-L/14 @336px
    - Projector: 2-layer MLP
    - LLM: Vicuna/LLaMA 7B/13B
    """
    def __init__(self):
        self.vision_encoder = CLIPViT(
            image_size=336,
            patch_size=14,
            hidden_size=1024,
            num_layers=24,
            num_heads=16)
        self.projector = MLP(
            in_features=1024,
            hidden=4096,
            out_features=4096)
        self.llm = LlamaForCausalLM(
            hidden_size=4096,
            num_layers=32,
            num_heads=32)

    def forward(self, images, tokens):
        vis_feats = self.vision_encoder(images)
        vis_tokens = self.projector(vis_feats)
        # Concatenate at image position
        embeds = merge(vis_tokens, tokens)
        return self.llm(embeds)

Design Philosophy

Simplicity is key. LLaVA connects a pre-trained CLIP vision encoder to a pre-trained LLM with just a lightweight MLP projector. No complex cross-attention, no special vision tokens.

Image features are projected into the LLM's embedding space and simply concatenated into the token sequence at the <image> placeholder position.

LLaVA-1.0 vs 1.5: v1.0 used a single linear layer as projector. v1.5 upgraded to a 2-layer MLP with GELU activation, significantly improving performance.

No special tokens: Unlike Flamingo or BLIP-2, LLaVA does not use perceiver/Q-Former. Vision features directly become part of the input sequence.

2. CLIP ViT-L/14 Vision Encoder

# CLIP ViT-L/14 @ 336px
# Pre-trained, frozen in Stage 1

class CLIPVisionEncoder(nn.Module):
    """CLIP ViT-L/14
    Input: 336x336 image
    Patches: 14x14 -> 24x24 = 576 patches
    Output: 576 tokens x 1024-dim
    """
    def __init__(self):
        self.patch_embed = Conv2d(
            3, 1024,
            kernel_size=14, stride=14)
        self.pos_embed = Parameter(
            zeros(577, 1024))  # +1 CLS
        self.layers = nn.ModuleList([
            TransformerBlock(
                dim=1024,
                heads=16,
                mlp_dim=4096)
            for _ in range(24)])
        self.ln_post = LayerNorm(1024)

    def forward(self, img):
        # img: [B, 3, 336, 336]
        x = self.patch_embed(img)
        # x: [B, 576, 1024]
        x = x + self.pos_embed[1:]
        for layer in self.layers:
            x = layer(x)
        return self.ln_post(x)

Vision Encoder Details

CLIP ViT-L/14 @ 336px is the standard vision encoder for LLaVA-1.5. The higher resolution (336 vs 224) provides finer-grained visual features.

The image is split into 14x14 patches: 336/14 = 24 patches per side, yielding 576 visual tokens per image.

Pre-trained & frozen: In Stage 1 (projector pre-training), the vision encoder is completely frozen. In Stage 2 (instruction tuning), it can optionally be fine-tuned.

CLS token dropped: LLaVA uses all 576 patch tokens (not the CLS token) as visual features, preserving spatial information for the LLM.

3. MLP Projector (Vision-Language Connector)

# LLaVA-1.5: 2-layer MLP projector
# LLaVA-1.0: single Linear layer

class MultimodalProjector(nn.Module):
    """Projects vision features into
    the LLM embedding space.

    LLaVA-1.0: Linear(1024, 4096)
    LLaVA-1.5: MLP with GELU
    """
    def __init__(self,
                 input_size=1024,
                 hidden_size=4096,
                 output_size=4096):
        self.linear_1 = Linear(
            input_size, hidden_size)
        self.act = GELU()
        self.linear_2 = Linear(
            hidden_size, output_size)

    def forward(self, vis_features):
        # vis_features: [B, 576, 1024]
        x = self.linear_1(vis_features)
        x = self.act(x)
        x = self.linear_2(x)
        # output: [B, 576, 4096]
        return x

The Critical Connector

Minimal but effective. The projector is the only trainable component in Stage 1. It learns to align CLIP's vision space with the LLM's language space.

LLaVA-1.0 vs 1.5: Upgrading from a single linear layer to a 2-layer MLP with GELU was one of the key improvements in LLaVA-1.5, yielding significant accuracy gains.

Parameter count: Only ~21M parameters total -- negligible compared to the 7B LLM. Yet this tiny bridge enables powerful multimodal understanding.

In Megatron-Bridge: Implemented as MultimodalProjector with projector_type="mlp", configured via ModuleSpec in the LLaVA provider.

Training Strategy

Stage 1 -- Feature Alignment. Only the projector is trained on 595K image-caption pairs. This teaches the MLP to map CLIP features into the LLM's word embedding space. Fast: ~5 hours on 8 A100s.

Stage 2 -- Instruction Tuning. The projector and LLM are jointly fine-tuned on 665K multimodal instruction-following data (conversations, reasoning, detail descriptions).

Why two stages? Stage 1 bootstraps the cross-modal alignment cheaply. Stage 2 then teaches the model to follow complex visual instructions without catastrophic forgetting.

Megatron-Bridge support: The freeze_language_model and freeze_modality_encoders flags in MegatronMIMOProvider directly map to this two-stage paradigm.

5. LLaVA-NeXT: Dynamic Resolution (AnyRes)

# LLaVA-NeXT: AnyRes strategy
# Split high-res images into tiles
# Encode each tile + base image

class AnyResProcessor:
    """Dynamic resolution for LLaVA-NeXT.
    Supports arbitrary aspect ratios.
    Max grids: e.g. 2x2, 1x3, 3x1, etc.
    """
    def __init__(self,
                 base_size=336,
                 max_tiles=4):
        self.base_size = base_size
        self.max_tiles = max_tiles

    def process(self, image):
        # 1. Resize to base for global view
        base_img = resize(image, 336)

        # 2. Select best grid (e.g. 2x2)
        grid = select_grid(
            image.size, self.max_tiles)

        # 3. Split into tiles
        tiles = split_to_tiles(
            image, grid)  # e.g. 4 tiles

        # 4. Encode each independently
        base_feats = clip(base_img)
        # [576, 1024]
        tile_feats = [clip(t) for t in tiles]
        # 4 x [576, 1024]

        # 5. Concatenate all features
        return cat([base_feats] + tile_feats)
        # [2880, 1024] for 2x2 grid

AnyRes Innovation

Problem: Fixed 336px resolution loses fine details in high-res images (text, small objects). LLaVA-NeXT solves this with dynamic tiling.

Base + Tiles: A low-res base image captures global context. High-res tiles preserve local details. Each is encoded independently by the same CLIP encoder.

Grid selection: The optimal grid (1x2, 2x1, 2x2, 1x3, etc.) is chosen based on the image's aspect ratio, up to a maximum tile budget.

Token count: With a 2x2 grid: 576 (base) + 4*576 (tiles) = 2880 visual tokens. This increases compute but dramatically improves OCR, document understanding, and fine-grained recognition.
