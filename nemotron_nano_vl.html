Nemotron-Nano-12B-v2-VL

Hybrid Mamba+MoE Vision-Language Model — Zero Attention Layers, Pure State-Space Architecture

1. Hybrid Mamba+MoE (Zero Attention Layers)

# Layer pattern: repeating blocks
# NO attention layers anywhere

class NemotronNanoBlock:
    """Repeating pattern:
    Block 2k:   Mamba + Dense MLP
    Block 2k+1: Mamba + MoE
    """
    def __init__(self, layer_idx):
        self.mamba = MambaLayer(
            d_model=5120,
            d_inner=10240,
            n_heads=128,
        )
        if layer_idx % 2 == 0:
            self.ffn = DenseMLP(
                d_model=5120,
                d_ff=20480)
        else:
            self.ffn = MoELayer(
                n_experts=64,
                top_k=4)

    def forward(self, x):
        x = x + self.mamba(x)
        x = x + self.ffn(x)
        return x

Key Insight

Zero attention layers. Unlike typical hybrid models that mix attention with Mamba, Nemotron-Nano-v2-VL uses no attention at all in the language backbone.

The pattern alternates: Mamba + Dense MLP followed by Mamba + MoE. This gives the model both efficient sequence modeling (Mamba) and sparse capacity scaling (MoE).

MoE config: 64 experts with top-4 routing. Only 4 experts activate per token, keeping FLOPs manageable while providing massive parameter count.

Implications: No KV cache needed at inference. Memory is constant regardless of sequence length. Throughput scales linearly with sequence length.

2. Mamba Layer (Selective State Space)

class MambaLayer(nn.Module):
    """Selective State Space Model
    128 heads, head_dim=80
    d_inner=10240, d_state=128, d_conv=4
    """
    def __init__(self):
        self.d_inner = 10240
        self.d_state = 128
        self.d_conv = 4
        self.n_heads = 128
        self.head_dim = 80

        # Input projection
        self.in_proj = Linear(
            5120, 2*self.d_inner)
        # Conv for local context
        self.conv1d = Conv1d(
            self.d_inner, self.d_inner,
            kernel_size=4, groups=self.d_inner)
        # SSM parameters (input-dependent)
        self.dt_proj = Linear(self.d_inner, self.n_heads)
        self.B_proj = Linear(self.d_inner, self.d_state)
        self.C_proj = Linear(self.d_inner, self.d_state)

    def forward(self, x):
        # x: [B, L, 5120]
        xz = self.in_proj(x)
        x, z = xz.chunk(2, dim=-1)
        x = self.conv1d(x)
        x = silu(x)
        # Input-dependent SSM params
        dt = F.softplus(self.dt_proj(x))
        B = self.B_proj(x)
        C = self.C_proj(x)
        # Selective scan
        y = selective_scan(x, dt, B, C)
        return y * silu(z)

Selective State Space

Core idea: Unlike transformers that attend to all positions, Mamba uses a recurrent state that selectively remembers or forgets information based on the current input.

Parameters:

- 128 heads, head_dim=80
- d_inner=10240 (2x model dim)
- d_state=128 (recurrent state size)
- d_conv=4 (local context window)

Input-dependent: dt, B, C are all computed from the input. This is what makes it "selective" — the model learns what to remember based on content, not position.

Inference: O(1) per token. The recurrent state is fixed-size regardless of sequence length. No positional embeddings needed — position is encoded implicitly in the evolving state.

3. Vision Encoder: RADIO ViT

class RADIOViT(nn.Module):
    """RADIO: Robust And Diverse
    Image-text Object detection encoder

    Multi-teacher distillation:
      CLIP + DINOv2 + SAM
    """
    def __init__(self):
        self.num_layers = 32
        self.hidden_size = 1280
        self.patch_size = 16
        self.image_size = 512
        # 512/16 = 32 patches per side
        # 32*32 = 1024 tokens
        self.num_patches = 1024

        self.patch_embed = Conv2d(
            3, 1280,
            kernel_size=16, stride=16)
        self.layers = nn.ModuleList([
            ViTBlock(1280, heads=16)
            for _ in range(32)
        ])

    def forward(self, img):
        # img: [B, 3, 512, 512]
        x = self.patch_embed(img)
        # x: [B, 1024, 1280]
        for layer in self.layers:
            x = layer(x)
        return x  # [B, 1024, 1280]

RADIO Encoder

RADIO = Robust And Diverse Image-text Object detection encoder. A ViT trained via multi-teacher distillation from three complementary teachers:

- CLIP: semantic/text-aligned features
- DINOv2: spatial/structural features
- SAM: segmentation-quality features

Architecture: 32 layers, hidden_size=1280, 16 attention heads. Patch size 16 on 512x512 images produces 1024 visual tokens.

The result is a single encoder that captures semantics, spatial structure, and fine-grained segmentation — all properties needed for a strong VLM vision backbone.

4. Pixel Shuffle Downsampling

class PixelShuffleDown(nn.Module):
    """Reduce spatial tokens by
    rearranging into channel dim.

    scale_factor=0.5:
      32x32 spatial -> 16x16 spatial
      1024 tokens -> 256 tokens (4x reduction)
      1280 channels -> 5120 channels
    """
    def __init__(self, scale_factor=0.5):
        self.factor = int(1 / scale_factor)
        # factor = 2

    def forward(self, x):
        # x: [B, 1024, 1280]
        B, N, C = x.shape
        h = w = int(N ** 0.5)  # 32
        x = x.view(B, h, w, C)
        # Rearrange 2x2 spatial blocks
        # into channel dimension
        x = x.view(B,
            h//2, 2,
            w//2, 2, C)
        x = x.permute(0,1,3,2,4,5)
        x = x.reshape(B,
            h//2 * w//2,
            4 * C)
        # [B, 256, 5120]
        return x

Spatial Compression

Purpose: Reduce the number of visual tokens fed to the language model from 1024 to 256 — a 4x reduction that dramatically cuts compute in the language backbone.

Mechanism: Each 2x2 block of adjacent spatial patches is merged into a single token by concatenating their features along the channel dimension:

[B, 32, 32, 1280]
-> [B, 16, 16, 4*1280]
-> [B, 256, 5120]

Lossless: No information is discarded. Spatial relationships are preserved in the channel ordering. The projector MLP then maps 5120 back to the language model dimension.

5. MLP Projector (Squared ReLU)

class VisionProjector(nn.Module):
    """2-layer MLP with squared_relu.
    Maps vision features to LM dim.

    5120 -> 20480 -> 5120
    squared_relu: relu(x)^2
      - Sparsity-inducing activation
      - Most values become exactly 0
      - More selective than standard relu
    """
    def __init__(self):
        self.fc1 = Linear(5120, 20480)
        self.fc2 = Linear(20480, 5120)

    def squared_relu(self, x):
        return F.relu(x) ** 2

    def forward(self, x):
        # x: [B, 256, 5120]
        x = self.fc1(x)
        # [B, 256, 20480]
        x = self.squared_relu(x)
        # ~90% sparsity
        x = self.fc2(x)
        # [B, 256, 5120]
        return x

Squared ReLU Projector

Role: Bridge between vision encoder output (after pixel shuffle) and the Mamba language model input space.

Squared ReLU: f(x) = max(0, x)^2. Unlike standard ReLU, squaring creates a much sparser activation pattern. Approximately 90% of intermediate activations become zero.

Why sparsity matters:

- Forces the projector to be highly selective about which visual features to pass through
- Reduces effective compute in the second linear layer
- Acts as an information bottleneck that filters noise

Dimensions: 5120 -> 20480 (4x expansion) -> 5120. The output directly matches the Mamba model dimension, so vision tokens can be concatenated with text embeddings.

6. No Positional Embeddings in Language

class NemotronNanoVL(nn.Module):
    """Full model assembly.
    No positional embeddings in LM.
    Position encoded in Mamba state.
    """
    def __init__(self):
        # Vision
        self.vit = RADIOViT()
        self.pixel_shuffle = PixelShuffleDown(
            scale_factor=0.5)
        self.projector = VisionProjector()

        # Language (NO pos_embed!)
        self.embed = Embedding(
            vocab_size=131072,
            d_model=5120)
        # No self.pos_embed
        # No RoPE, no ALiBi, no learned PE

        self.layers = nn.ModuleList([
            NemotronNanoBlock(i)
            for i in range(80)
        ])
        self.head = Linear(5120, 131072)

    def forward(self, tokens, images):
        # Text embeddings
        txt = self.embed(tokens)
        # Vision pipeline
        vis = self.vit(images)
        vis = self.pixel_shuffle(vis)
        vis = self.projector(vis)
        # Merge (no pos needed)
        x = merge(txt, vis)
        for layer in self.layers:
            x = layer(x)
        return self.head(x)

Position-Free Design

Fundamental difference from transformers: This model has NO positional embeddings of any kind — no RoPE, no ALiBi, no learned positional encoding.

How position is encoded: Mamba's recurrent state naturally encodes position. Since h_t = f(h_{t-1}, x_t), the state at position t is a function of all previous inputs in order. The model implicitly knows "where" it is through its accumulated state.

Implications for VLM:

- Vision tokens and text tokens are simply concatenated
- No need for special position handling at the vision-text boundary
- Sequence length is theoretically unlimited (no position embedding table to overflow)

Inference: 80 layers, each with constant-memory Mamba + either dense MLP or MoE. Total inference cost is O(L) in sequence length with constant memory — fundamentally more efficient than transformer VLMs.
